# 星辰迁移工具说明（精斗云V3产品迁移）

# **一、特别说明**

- 本说明仅适用于金蝶云·星辰标准版/专业版，精斗云V3不支持迁移业务单据到金蝶云·星辰旗舰版。
- 目前已支持了精斗云V3迁移到金蝶云·星辰标准版、金蝶云·星辰专业版，升级方案有两种：结转业务余额、业务单据。升级方案为结转业务余额时将V3云进销存升级当日的余额作为金蝶云·星辰的期初数据，期间为当前月的下一期。结转业务余额升级建议选在月底进行升级。另升级方案为业务单据时，V3有使用空辅助属性商品、成本调整单含批次保质期/辅助属性商品的不支持迁移，可以删单后再升级。升级方案为业务单据时，V3有启用序列号的账套整体不支持升级，建议使用序列号的客户选择结转业务余额方式升级。
- V3云财贸不支持直接将业务和财务的数据全部升级，可以单独升级进销存或者单据升级财务。针对云财贸的客户可通过以下步骤将所有数据升级到星辰中：1）先单独升级财务，升级成功后将会计科目、财务凭证等财务数据按引入模板引出（注意自定义一个所有字段的引入模板）；2）备份星辰账套再重新初始化星辰账套后，再单独升级进销存；3）进销存升级成功后，引入步骤1中引出的财务数据。注意：①变通方案可以将业务、财务数据全部迁移，但历史业务单据与凭证不能再关联。②V3云会计迁移支持的会计制度包括以下四个：小企业会计准则（2013年颁）、新会计准则（2006年颁）、新会计准则（2019年未执行新收入、新金融准则）、新会计准则(2019年执行新金融、新收入和新租赁准则），其他会计制度暂不支持迁移。![上传图片](https://vip.kingdee.com/download/0100f0492d7211e9490cb73858d75da16709.png)
- **特别提醒**：因历史原因可能导致某些V3账套中存在一些脏数据，不能一次性升级成功。系统在升级前会进行一些数据检查，帮助找出不符合升级的数据；同时建议伙伴在给客户升级前可以先在自己的测试账套/省区测试账套（关于省区测试账套具体咨询对应省区分销经理）中模拟升级。**强烈建议：提前做好升级准备**，避免因V3过期导致无法调整数据、无法升级。**特别提醒**：升级过程中若因V3的异常数据导致要修改V3数据的，请自行反结账修改；若需要研发后台修复数据的，需要购买“数据修复服务”，详情请见产品发文。![上传图片](https://vip.kingdee.com/download/0100bec6f52a421e415f940f777465f9ccfa.png)
- 特别提醒：1）升级前，请将进销存结账到升级当天；升级业务单据也结账后再升级。2）升级过程中，原V3账套需要停用，否则升级会报错。升级请避开周二周四的例行发布日，升级过程中遇星辰发布会中断升级。3）升级过程中，关掉浏览器界面或关闭电脑，不影响升级服务后台升级，只是看不到升级进度而已。若因关闭浏览器导致无法得知是否升级成功，想要初始化重新升级的，请在初始化全部数据后暂停3分钟再重新操作升级，避免后台升级服务与新的升级请求冲突。4）升级成功后，不要对历史单据重算成本。重算成本大概率会导致重算后的历史单据上成本与原账套中不一致。星辰对于负库存、组装拆卸等地方的成本计算处理与V3有差异。5）升级成功后，建议将历史数据做结账处理。长时间不结账会影响星辰的使用性能，且容易对历史单据进行了重算成本。6）确认升级成功，核对数据无误后再开始使用星辰。
- 关于账号登录，为能更好读取数据，请使用手机号+密码登录。

# **二、精斗云V3迁移金蝶云·星辰标准版/专业版操作说明**

**以下为精斗云V3 迁移到金蝶云·星辰标准版/专业版不同升级方案下升级的具体数据：**

![上传图片](https://vip.kingdee.com/download/0100f3a487ecd86d4517a9efe6f0f266972c.png)

## **特别提醒**

       1）升级前，请将进销存结账到升级当天；升级业务单据也结账后再升级。

       2）升级过程中，原V3账套需要停用，否则升级会报错。升级请避开周二周四的例行发布日，升级过程中遇星辰发布会中断升级。

       3）升级过程中，关掉浏览器界面或关闭电脑，不影响升级服务后台升级，只是看不到升级进度而已。若因关闭浏览器导致无法得知是否升级成功，想要初始化重新升级的，请在初始化全部数据后暂停3分钟再重新操作升级，避免后台升级服务与新的升级请求冲突。

       4）升级业务单据时，若升级进度已经到销售订单，在后续升级出现了异常，可通过F5刷新或重新登录进入到断点续升页面，支持延续报错的单据类型继续升级。

       5）升级成功后，不要对历史单据重算成本。重算成本大概率会导致重算后的历史单据上成本与原账套中不一致。星辰对于负库存、组装拆卸等地方的成本计算处理与V3有差异。

       6）升级成功后，建议将历史数据做结账处理。长时间不结账会影响星辰的使用性能，且容易对历史单据进行了重算成本。

       7）确认升级成功，核对数据无误后再开始使用星辰。

## **进销存****结转业务余额迁移说明**

- 辅助属性超过五类、单个商品使用的计量单位超过四个时不允许升级；
- 精斗云V3账套需要结账至当前日期或当前月份月底；升级之后启用期间为当前日期所在期间的下一期；
- 支持升级商品、客户、供应商等业务基础资料；
- 结转商品库存余额、账户余额、往来余额为金蝶云·星辰期初数据；
- 往来余额以单据为维度进行结转，一笔未核销完的单据对应一笔期初数据；
- 不含会员、门店等零售基础资料。

## **进销存****业务单据迁移说明**

- 因为历史数据原因，升级业务单据一般不能一次性升级成功，强烈建议提前做好升级准备，避免因为V3过期导致无法调整数据、无法升级。升级过程中若因V3的异常数据导致要修改V3数据的，请自行反结账修改；若需要研发后台修复数据的，需要购买“数据修复服务”，详情请见产品发文。![上传图片](https://vip.kingdee.com/download/0100e406cde4abb944d7b40c091a951ec668.png)
- 辅助属性超过五类、单个商品使用的计量单位超过四个时不允许迁移；
- 精斗云V3账套启用序列号的整体不支持迁移；V3有空辅助属性商品、应付转应付（应付含其他支出单）、应收转应收（应收含其他收入单）、成本调整单含批次保质期/辅助属性商品的暂不支持迁移；
- 支持迁移商品、客户、供应商等业务基础资料；支持迁移进销存采购、销售、仓库、资金单据；
- 迁移成功后的单据，单据来源全部为“迁移工具”。

**升级完成****数据核对****：**

       升级完成界面会有升级成功的提示，若要核对数据，建议核对截止到升级当天的以下数据：库存查询表中库存/成本结存、应收账款汇总表、应付账款汇总表、账户余额表；

       **特别提醒**：查询库存查询表核对成本时，**不要重算成本**；勾选上“显示零库存商品”“显示已禁用商品”，否则可能成本会对不上（当历史单据被重算成本、存在商品零库存但有成本的情况，或禁用商品有库存或成本的情况）。

# **三、操作流程**

1.新建账套界面，点击“第三方导入”

![上传图片](https://vip.kingdee.com/download/0100db4f42fb49a94313af5a50521886a48a.png)

2.根据实际情况选择导入方案。具体升级方案支持升级的数据请往上查看“一、特别说明”部分，然后选择从精斗云3.0版本导入。

![上传图片](https://vip.kingdee.com/download/0100ba9170124e3a4b148fe21061ecf6d48d.png)

3.输入V3账套对应的精斗云账号、密码，点击登录：建议使用手机号+密码登录。

![选3.0升级.png](https://vip.kingdee.com/download/0100c41c973053c84bb5a09c48899ff8523a.png)

4.选择其中一个账套，选择具体的迁移方案，即可进行账套数据升级：

选择V3导入：

![V3选方案.png](https://vip.kingdee.com/download/0100079c18f154ad41ed8d5a031585830d72.png)

![V3升级中.png](https://vip.kingdee.com/download/0100a001864e669048939f4c8175bae25a4c.png)

5.如遇导入失败的情况，点击“详情”或者“查看日志”，可查看详细错误日志。

若错误信息较多，可用鼠标滑过选中内容，然后右键【复制】将所有报错信息全部复制出来，描述中有写建议的，可直接按修改建议调整数据后再进行升级。

![升级失败.png](https://vip.kingdee.com/download/0100120e67ac002c410286e176ca70caaba5.png)

# **四、金蝶云·星辰基础资料引入功能操作说明**

基础资料以商品为例：

1.  打开基础资料-商品信息，点击引入；在打开的引入数据弹窗中下载引入默认模板。

![上传图片](https://vip.kingdee.com/download/010076643a07f99440bea7e2636c5782c830.png)

![上传图片](https://vip.kingdee.com/download/01006a88463107004269818b7793c4ce308c.png)

2.  将商品信息录入到引入模板中，注意红色列为必录项。请不要删除引入模板的前两行，否则系统将不能识别引入文件导致引入失败。

![image.png](https://vip.kingdee.com/download/0100fab9b90b333c4368a9540b7afa9d49ed.png)

3.  将填写好的引入模板直接拖至引入数据弹框中，支持一次性上传多个文件，点击开始引入。

![上传图片](https://vip.kingdee.com/download/0100285994bd60494e95a24e4df560b4d4fd.png)

4.  引入完成会有对应的引入进度反馈。如果引入失败，点击查看详情会自动下载错误文件信息，错误原因在错误文件的第一列，可根据错误信息修改引入文件后再次引入。错误文件信息也可在引入-引入结果查询中下载。

5.  补充说明：如果需要自定义模板可在引入数据点击模板管理后新增模板。自定义模板支持设置字段是否引入、是否必录；如果是系统中如单位、客户等基础资料，支持设置引入时按编码还是名称引入。

![上传图片](https://vip.kingdee.com/download/010022b8984a5b794d6ebc5dbaa1874bada4.png)
