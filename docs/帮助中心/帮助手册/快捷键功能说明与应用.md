# 1 简介

## 1.1 功能介绍

在开单时，可以通过键盘的快捷键来完成一些高频操作，诸如保存、审核、新增、反审核、打印等等，从而减少鼠标的操作，提高开单的效率。

## 1.2 适用版本

星辰3.1以上所有版本

## 1.3 操作路径

单据详情内（具体什么单据支持，请以产品内实际为准）

## **2.2 常用快捷键功能说明**

| 快捷键 | 功能描述 |
| --- | --- |
| F7 | 打开基础资料选择器 |
| F6 | 批量填充 |
| F9 | 自动填充收退款金额，并携带默认收款账户。支持单据范围：专业版：自动填充采购入库单/退货单、销售出库单/退货单、委托结算单/结算退货单的收退款金额旗舰版：自动填充采购发票、销售发票、委托结算单/结算退货单的收退款金额 |
| Ctrl+S | 保存（在弹窗中执行“确认按钮”功能） |
| CTRL+V | 粘贴 |
| shift+delete | 删除当前行，鼠标右键可批量删除选中行tips文案：支持鼠标滑动选中多行 |
| CTRL+D | 复制选中行首行内容，向下填充使用方法：复制选中行首行内容，在选中区域内向下填充（具体应用流程见下文。） |
| CTRL+shift+← | 调出或关闭分屏 |

# 3 应用流程

直接录入快捷键即可生效。

## 3.1 快捷键F7——打开基础资料选择器

示例：开单时，鼠标单击商品编码单元格，**按F7，可以快速弹出商品选择器**：

![](https://vip.kingdee.com/download/010058c8c525769e4f8ab76dd3912275e018.gif)

## 3.2 快捷键F6——批量填充

使用的方法是，鼠标单据要填充的单元格，光标是蓝色（表示全部选中），然后按键盘F6键，即可向下填充。

- **示例一：批量填充单据体上的商品行备注**

![](https://vip.kingdee.com/download/01004292ac069a8444589e5e1aa996287912.gif)

- **示例二****：批量填充单据体上的自定义文本、数值字段**

![](https://vip.kingdee.com/download/010061a75edf15c14a2ab9c56f496b19d4a7.gif)

- **示例三****：批量填充日期**

如果填充的是日期字段，直接选中会弹出日期下拉控件，可以不用管下拉弹窗，**但需要手工选中整个日期字段，然后按F6即可一样向下填充。**

![上传图片](https://vip.kingdee.com/download/01004efdc2dc4b04428eb5ba80200e412c58.gif)

## 3.3 快捷键ctrl+S---保存

- **功能点1：直接键盘ctrl+s，可以保存单据**

![](https://vip.kingdee.com/download/010002b2104d0c474215aa10bc91a8ae1b64.gif)

- **功能点2：详情页二级弹窗点击ctrl+s支持携带选中的信息返填到单据**

操作说明：如果单据/基础资料的详情页中，打开了任意的二级弹窗，例如商品、二维录入弹窗、单据体客户/供应商等等，那么当在二级弹窗中点击键盘ctrl+s时，会直接携带二级弹窗中选中的信息返回到单据上（如下GIF图）

![](https://vip.kingdee.com/download/01009aaa2fbbfecb4ad8af3ec0be67af7e84.gif)

## 3.4 批量删除单据分录行

1）鼠标在单据体分录中，点击右键，选择“删除所有行”则删除单据体中所有有序号的行

![上传图片](https://vip.kingdee.com/download/01001cfb09b4cad74b44b0be29252df3541f.gif)

2）鼠标批量选中单据分录，点击右键，选择“删除选中行”，则只删除选中的行，不删除没有选中的行

![上传图片](https://vip.kingdee.com/download/0100dc12cee580344fcdbae4f201a6653445.gif)

## 3.6 Ctrl+D的使用方式及效果

假设一张单有五行分录，我们选中分录行中的第二三四行的“单价”字段，然后点击Ctrl+D，那么系统就会把第二行的单价2，批量向下填充到第三四行的“单价”，但是不会填充到第五行，第五行单价还是5。

# 4 常见问题

## 1、Ctrl+D与F6的区别？

- Ctrl+D是按指定区域批量填充，F6批量填充是以选中的行向下填充，是不区分选择哪几行的，统一按列来向下生效，因此用户可以根据实际业务需要进行选择和应用。
- 如果需要向下批量填充的数据太多时，用F6可能会比较慢，更建议用Ctrl+D
