## 2、产品概述

### 2.1产品介绍

采购业务是企业供应链活动中重要的一环。采购业务主要包括采购申请、发起采购需求；制定采购合同，与供应商按约定好的商品、价格、数量、支付方式及订金比例等，对于不做库存的商品或者库存不足的商品，还会根据订单需求进行补货；商品到货后进行入库，同时完成后续的付款动作。

采购业务还需处理因产品质量不合格或者需求变更，需要退货给供应商的采购退货场景。完成发货及退货后，接到供应商的发票，再完成发票的确认和登记，这部分属于单票分离的场景，又或者发票未到但需要先预估或核算财务成本或者发票与实际单据金额有差异，需要进行调整，这部分属于单票合一的场景。

### 2.2业务场景

目前星辰3.1支持以下采购场景：

• 采购申请（专业版/旗舰版）---用于采购正式下单前的申请环节，申请通过才转化成实际采购订单。

• 采购订单---用于记录合同信息，订单一旦确定不能随意修改，订单不影响实际库存和资金。

• 采购入库单---用于将货物转入仓库，增加商品库存，同时支出资金或增加应付账款。

• 采购退货单---用于将商品退回给供应商，并收回退款或减少应付账款。

• 采购发票（旗舰版）---用于企业收到结算供应商开出的发票或者收据，已经确定了实际的采购金额、税额后，需要在系统中录入，并通过发票确认支出的场景。

• 以销定购---用于企业为了实现零库存或避免库存积压，事先不采购商品放入仓库，当接收到客户订单后，根据采购订单的需求量进行采购的场景。

• 智能补货---用于企业为了避免出现断货的情况，需要提前补充库存的场景。

• 暂估调整（专业版）---用于处理在实际业务环节中，发票未到但需要先预估或核算财务成本、又或者发票与实际单据金额有差异需要调整的场景 。

• 采购核销（旗舰版）---用于：

1）采购退货后，原采购入库单的未开票数量无需继续开票，且采购退货单也无需开红冲发票，可将未开票数量相等的采购入库单与采购退货单进行对等核销。

2）采购发票与采购入库单分别录入，没有关联关系，通过匹配核销将采购发票与采购入库单进行关联，确认采购入库成本。（待开发）
