# 拣货验货整体介绍

# 1 功能介绍

对于出入库业务中，实际仓管员需要进行拣货验货，一次或多次核对出入库的商品和商品数量及外观，减少出错货导致客户经营损失的情况，通过优化流程、使用成本效益的技术和设备，降低拣货和仓储成本

# 2 适用版本

- 金蝶云星辰3.1以上所有版本
- **拣货验货模块为收费模块，2000/年**
- 不同版本支持的单据如下

- 采购类：所有版本均支持采购订单、采购入库单、采购退货单
- 库存类：所有版本均支持调拨出库单、调拨入库单
- 销售类：标准版 / 专业版支持销售订单、销售出库单、销售退货单；旗舰版在此基础上额外支持发货通知单

# 3 菜单说明

- **拣验货单：**用来生成拣验货任务，可以分配给对应仓管人员任务
- **拣验工作台：**实际执行拣验的工作台，PC端有拣验工作台的单独菜单，通过扫码、名称等方式核对商品，确保出货商品准确，且数量批次等其他基本信息无误；移动端无拣验工作台的单独菜单，当拣验货单为待拣验时，点击进去即为拣验工作台。PC端的拣验工作台支持单据列设置，暂不支持自定义字段；移动端从待拣验中进入的拣验工作台暂不支持列设置
- **拣验货配置：**用来配置拣验货业务流程的管控参数
- 备注：移动端APP无拣验工作台的单独菜单，当拣验货单为待拣验时，点击进去即为工作台，电脑端有单独的拣验工作台

# 5 拣货和验货的区别

**依据源单的不同，区分业务类型为拣货和验货，功能区分如下：**

| 业务类型 | 源单 | 前置条件 | 是否支持分批拣货/验货 | 是否支持部分拣货/验货 | 是否支持超额拣货/验货 |
| --- | --- | --- | --- | --- | --- |
| 拣货 | 销售订单采购订单发货通知单 | 源单已审核 | 是 | 是 | 是 |
| 验货 | 销售出库单销售退货单采购入库单采购退货单调拨出库单调拨入库单 | 源单未审核 | 否 | 否（目前支持整单验货，不支持部分验货） | 否 |

## 6.3 线路配置

通过提前配置拣验货线路，对应的商品将会按照仓库仓位的摆放顺序升序展示，方便仓管员准确高效的拣验货

**（1）设置仓库顺序**

按照仓库的摆放顺序吗，编辑仓库编码，设置完成之后，便于待拣商品按仓库汇总之后按仓库编码升序排序

**（2）设置仓位顺序**

按照仓库中仓位的摆放顺序吗，编辑仓位编码，设置完成之后，便于待拣商品按仓库汇总之后按仓位编码升序排序

**（3）设置商品默认仓库和仓位**

在商品资料中设置默认仓库仓位，开单时会自动携带出仓库仓位，源单下推到拣验货单中就携带过去，待拣商品可以按照仓库仓位摆放顺序做好线路规划，让拣货员快速拣货

## 7 其他说明

（1）APP的拣验货单，如果未拣验之前需要编辑，请点击见样货单列表的“编辑”按钮，选择单据后进行编辑（受拣验货单的“编辑”功能的权限控制）

![](https://vip.kingdee.com/download/01095a4303da9edb49df9272d6c0db91e102.png)

（2）支持全流程跟踪，在拣验货单详情界面支持关联单据信息联查

![上传图片](https://vip.kingdee.com/download/01006ccb04a41ab34eebb0d753702e064e5f.png)

（3）扫码支持的码：**商品条形码、SKU条形码、批次复合码、序列号码**

（4）针对拣货，拣验工作台会做商品的合并，合并规则：商品编码+仓库+仓位+是否赠品+批次+生产日期+保质期+到期日+套装编码，以上所有条件都一致时，则合并为一行

（5）拣验货单支持单据列设置

用户可依据单据的表头，表体，列表设置展示或隐藏，系统内置了关键必录字段不可设置，不系统内置必须展示的字段，用户可自行设置展示或隐藏。设置隐藏就在开单和详情界面不展示，移动端列设置只对移动端单据生效，与PC端列设置是分开的，互不影响，且移动端从待拣验中进入的拣验工作台暂不支持列设置，电脑端拣验工作台支持列设置

（6） 权限控制

• 拣验货单新增/修改/删除/拣验完成/反审核：通过库存管理-拣验货单-新增/修改/查询/审核/反审核的功能权限控制

• 拣验工作台：通过库存管理-拣验货单-修改的功能权限控制

• 拣验货配置查看/修改：通过库存管理-拣验货配置-查询/修改的功能权限控制

• 销售/采购订单列表/详情页、拣验货单列表/详情页的去拣验按钮：通过库存管理-拣验货单-新增+修改的功能权限控制，缺一不可

• 销售/采购订单列表/详情页的生成拣验货单按钮：过库存管理-拣验货单-新增的功能权限控制

（7）其他说明

拣验货单支持单据云打印、字段配置，单据操作控制，自定义字段（工作台暂不支持自定义字段）、消息设置等
