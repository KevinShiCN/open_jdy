# 成本计算表提示成本金额小于0,不进行补差是什么意思?

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题
> 作者：happy忆霖 | 阅读：99 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/724066686138931200)

【问题描述】

成本计算表提示成本金额小于0,不进行补差是什么意思?

【业务场景】

成本计算表的计算结果中提示有尾差不进行补差,一般原因是什么?

【解释说明】

**补差的主要作用为在成本计算时,可能存在四舍五入小数位除不尽的情况,需要将结存为0的数量、有金额的商品尾差部分结转到最后一张出库类的单据中。**

**无法补差的原因主要为:**

**1.补差的范围设置的较小,结存的金额较大,无法进行补差;**

**2.成本计算最后一张单据的成本不够结存金额进行补,比如最后一张单据的出库成本为100,结存金额为-400,补差后导致出库类单据的成本小于0了,也无法进行补差。**

【解决方案】

**情况一:补差的范围设置的较小,结存的金额较大,无法进行补差**

点击【系统设置】>【基础设置】>【系统参数】,进入到系统参数设置界面,选择“存货核算参数”

![](https://vip.kingdee.com/download/010994e799f8149a4c1db3ec10dec9e5850f.png)
在其他设置下,调整“尾差调整范围”,一般建议设置在【0,1】之间。若成本计算表中显示结存的尾差金额较大,建议结合成本计算表分析原因。

![](https://vip.kingdee.com/download/010919a79ff1551b4abaae0565255e7ddf31.png)

**情况二:成本计算最后一张单据的成本不够结存金额进行补**

分析成本计算表中出现0库存有金额的原因,将成本异常的根本问题进行解决。
