# 相同的查询日期，商品收发汇总表与库存查询表，数量一致，但成本不一致是为什么？

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：进阶技巧, 常见问题
> 作者：进销存赵嘉丽 | 阅读：3085 | 有用：18
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/178907590661809408)

# 【问题概述】

相同的查询日期，商品收发汇总表与库存查询表，数量一致，但成本不一致是为什么？

# 【排查方法】

当出现报表数据核对不上时，请按以下步骤和方法检查：

## **步骤一**

如果购买了财务云（只购买了进销存的可以跳到步骤二），请先检查不一致的期间的所有单据是否都生成凭证了，如有未生成凭证的，先生成凭证后重新检查，如确认都生成凭证后成本依然对不上，请继续往下看。

## **步骤二**

### **排查报表筛选条件的一致性，其中几个关键的筛选条件包括：**

#### 1、日期是否一致：

| 报表 | 日期 |
| --- | --- |
| 库存查询表 |  |
| 商品收发汇总表商品收发明细表 |  |

#### 2、单据审核状态是否一致：

| 库存更新方式 | 报表 | 默认审核状态 | 备注 |
| --- | --- | --- | --- |
| 保存时更新 | 库存查询表 | 所有已保存的单据（含未审核） | 由库存更新参数决定，不能筛选 |
| 商品收发汇总表商品收发明细表 |  | 可通过筛选器勾选审核状态，如果是保存时更新，请确保审核状态是“全部” |  |
| 审核时更新 | 库存查询表 | 所有已审核的单据 | 由库存更新参数决定，不能筛选 |
| 商品收发汇总表商品收发明细表 |  | 可通过筛选器勾选审核状态，如果是审核时更新，请确保审核状态是“已审核” |  |

#### 3、筛选参数是否一致：

| 可勾选的参数 | 参数含义 | 涉及的报表 | 备注 |
| --- | --- | --- | --- |
|  | 即“即时库存”为零的商品，默认不勾选， | 库存查询表 | 注意：即时库存为0，成本不一定为0，有可能由于历史业务原因，结余时有成本。 |
|  | 默认不勾选，即默认显示“启用”的商品。 | 库存查询表商品收发汇总表商品收发明细表 |  |
|  | 默认不勾选，勾选后可以计算库存商品的成本。 | 库存查询表 | 勾选后需先做“出库成本核算”，再刷新页面后，报表中的成本字段才有值 |

当条件1、2一致时，且其他筛选条件并无勾选，库存查询表成本与商品收发汇总表数量一致，但成本不一致，请在库存查询表上勾上“**显示零库存商品**”，再刷新看看是否数据对得上，如果对得上，那么造成这里差异的背后逻辑是：

只要商品历史发生过业务且有成本，库存查询表都会记录下来，而查询时是默认不显示零库存商品的库存的。**部分零库存的商品由于历史业务的原因，可能会出现即时库存为零，但是结余有成本的情况**，而商品收发明细表记录的是实际有发生业务的单据，像库存查询表里的零库存商品就不会出现在这里（因为没库存自然无法参与实际业务往来），所以这里的差异就是零库存商品的成本。

**如果不勾选“显示零库存商品”，而实际零库存商品有成本时，就会造成2个报表虽然数量相等，但是成本不相等的情况。这个时候只要把库存查询表勾上“显示零库存商品”就可以了。**

## **步骤三**

假设步骤二的一致性都没问题，勾选“显示零库存商品”后依然成本对不上，那么需要重新进行出库成本核算，如已经结账，需先反结账到最开始的期间，然后重新做出库成本核算，最后重新结账到原来已结账的期间。

举例：假设某公司是2021年1月启用第一期的，当前期间是第四期，已结账到第三期，但是发现成本对不上，那么就需要先反结账到第一期，然后重新做出库成本核算，最后重新结账到第三期，然后再刷新库存查询表与商品收发汇总表，检查成本是否一致。

>
注意：假设按以上步骤都操作和检查后，成本依然对不上，请直接联系技术支持提单处理，需对账套做进一步分析和排查才能定位到原因。
