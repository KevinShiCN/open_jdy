# 数据查询范围一样,不同的日期导出的数据不一样如何排查?

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题
> 作者：云社区用户0531183 | 阅读：5 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/793645822138836992)

【问题描述】

数据查询范围一样,不同的日期导出的数据不一样如何排查?

【情景描述】

在金蝶云星辰系统中查询某个期间的业务单据,导出了对应日期的单据数据;后续再次按相同的日期范围、相同的查询条件导出该时间段的单据时,发现两次导出的单据数量、明细信息(如数量、金额、状态)或数据条数存在不一致,无法确认哪份数据为准确数据,需排查差异原因。

【解释说明】

系统导出数据核心来源于系统中已保存的单据明细记录,导出功能仅为 “数据提取 + 格式转换”,不会主动修改原始数据。同一查询范围下不同日期导出数据不一致,可能原因是两次导出间隔期间,原始单据数据发生了变更,或查询时的隐含条件、权限存在差异。

【操作指引】

1.确认查询条件与权限完全一致

1)核对筛选条件:手动勾选日期范围,确认是否勾选 “已审核 / 未审核 / 全部状态”“商品分类” 等筛选项,与首次导出时的选择完全一致。

![](https://vip.kingdee.com/download/0109600e0b8ee7c143d49f29150d6f9af37d.png)

2)权限验证:若账号为子账号,登录管理员账号重复导出操作,对比数据是否一致,排除子账号的数据授权限制(如部分仓库 / 商品无查看权限)。

2. 自查导出间隔期的数据变更记录

1)查看操作日志:进入【系统设置】→【操作日志】,按以下条件筛选:

- 操作时间:设置为 “首次导出日期” 至 “本次导出日期” 之间;

- 操作对象:选择导出的单据,例如“其他出库单”;
- 查看结果:重点关注两次操作导出日期间的该单据是否有修改、新增、删除等操作记录。
- ![](https://vip.kingdee.com/download/0109bc2f741dff674fe18ad5c12e50fb0b79.png)![](https://vip.kingdee.com/download/0109e7cf2b8dde2f402b897ce1936c36dd78.png)
- 2)比对差异单据:

- 找出首次导出和本次导出的 Excel 文件,可以用 Excel 的 “VLOOKUP” 或 “COUNTIF” 函数,定位缺失 / 新增 / 数据变更的单据号(例如:首次导出有单据 001,本次无,或单据 001 的数量从 10 改为 8);

- 针对差异单据,在系统中查看该单据的操作日志,确认是否在导出间隔期内被修改。
