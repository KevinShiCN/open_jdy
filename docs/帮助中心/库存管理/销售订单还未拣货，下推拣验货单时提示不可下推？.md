# 销售订单还未拣货，下推拣验货单时提示不可下推？

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：移动生产白建凡 | 阅读：64 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/787365874570321152)

# **1 问题描述**

销售订单显示“未拣验”，但下推拣验货单时却提示“已下推完成，不可下推”？

![上传图片](https://vip.kingdee.com/download/0100e23ca29cf4f44d5f80d8c6bbac27156b.png)

# 2 适用版本

星辰3.0以上所有版本，已购买拣货验货模块

# **3 原因解析**

销售订单已全部下推销售出库，常见与多人协作场景（如仓管负责拣货，内勤提前下推出库了）

- 仓管仅通过 “行拣验货状态” 判断 “未拣验”，未注意订单整体已完成出库
- 系统判定逻辑：订单可下推拣验货单数量 = 订单总数量 - 已执行数量（已出库） - 已拣验数量 + 已拣验已执行数量，当结果为 0 时，即提示 “不可下推”

# **4 解决方案**

（1）先排查订单状态

进入销售订单列表，查看目标订单的“关闭状态”或“行关闭状态”（多上品订单需看对应商品行），若“关闭状态”为“已关闭”，则无法下推拣验货单

![上传图片](https://vip.kingdee.com/download/010060ca48b5cb8f4082814eeee42d78245e.png)

（2）从流程上规避，设置销售出库单必选拣验货单

- 进入“系统设置>个性化设施>业务流程设置”，找到 “销售流程” 的销售出库单
- 凯开启 “拣验货单”为“必选”
- 设置后，销售出库单只能通过拣验货单下推生成，避免 “先出库、后拣货” 的顺序错乱，从根源解决该问题

![上传图片](https://vip.kingdee.com/download/010001eec8d089974636ad72d2693bbb89c5.png)
