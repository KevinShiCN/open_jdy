# 为什么不能用单据日期来控制负库存？

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：欣168 | 阅读：1790 | 有用：4
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/371323785376196352)

**1、系统逻辑：**

    星辰产品中，是根据截止到当前时间点的**可用库存**来校验负库存时，查询的是系统中**所有单据**统计出来的库存余量。



**2、 用户问题：**

    仓库设置了不允许负库存，补录单据的时候为什么会提示负库存呢？为什么不能用单据日期来判断是否有负库存呢？

**3、以下通过具体案例来说明：**

①当01仓未开启负库存时，商品A的期初库存为0，所有出入库记录如下：

| 单据日期 | 业务类型 | 单位 | 入库数量 | 出库数量 | 期末库存 |
| --- | --- | --- | --- | --- | --- |
| 2022-10-01 | 采购入库 | 个 | 100 |  | 100 |
| 2022-10-03 | 销售出库 | 个 |  | 60 | 40 |
| 2022-10-05 | 采购入库 | 个 | 100 |  | 140 |
| 2022-10-08 | 销售出库 | 个 |  | 60 | 80 |
| 2022-10-11 | 销售出库 | 个 |  | 50 | 30 |
| 2022-10-15 | 销售出库 | 个 |  | 20 | 10 |

②此时若补录2022-10-10号销售出库50个，则会出现负库存，无法录入；原因是当前“A商品的可用库存只剩下10，不管录入哪一天的出库单据，最多都只能出库10个

③按照2022-10-10这个日期来看可用库存，累计入库200个，累计出库120个，期末库存为80个，此时允许补录2022-10-10号销售出库50个，若要允许保存成功，则录入后的出入库记录如下：

| 单据日期 | 业务类型 | 单位 | 入库数量 | 出库数量 | 期末库存 |
| --- | --- | --- | --- | --- | --- |
| 2022-10-01 | 采购入库 | 个 | 100 |  | 100 |
| 2022-10-03 | 销售出库 | 个 |  | 60 | 40 |
| 2022-10-05 | 采购入库 | 个 | 100 |  | 140 |
| 2022-10-08 | 销售出库 | 个 |  | 60 | 80 |
| 2022-10-10 | 销售出库 | 个 |  | 50 | 30 |
| 2022-10-11 | 销售出库 | 个 |  | 50 | -20 |
| 2022-10-15 | 销售出库 | 个 |  | 20 | -40 |

  这样就会导致10-11、10-15号的订单出现负库存，也违背了“仓库设置了不允许负库存”的规则，并且对于已经审核通过的单据，系统无法再次做出负库存提示，导致对账时就出现数据混乱的情况

   若不允许保存成功，则实际是按截止当前时间点的可用库存控制负库存的（即目前的系统逻辑）

④综上所述，当删除或补录单据时，若系统提示负库存，则需人工核对下该商品的所有收发明细，确认下是销售出库单数量、单据日期录入错误，还是入库单据忘记录入系统了，将库存核对清楚后，再补录单据
