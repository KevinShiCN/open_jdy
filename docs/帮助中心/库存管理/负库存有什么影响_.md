# 负库存有什么影响?

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题
> 作者：winnie_wen | 阅读：3295 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/569326636236287232)

【问题描述】

负库存有什么影响?

【概述】

1、负库存的概念:

负库存指系统中的或系统上的商品或物料数量小于实际库存数量,简单来说销售的时候,**实际销售数量大于系统库存数量**。

如果严格遵循先入库后出库的方式,是不会产生负库存的。负库存产生的原因除人为差错外,大部分是由于系统出入库和实际出入库在时间顺序上的不一致造成的,这是进销存管理中十分常见现象,因为许多负库存并不是错误的,可能是由于企业销售运营等流程的需要导致的。

那么负库存主要影响就是库存成本异常。

2、产生负库存的常见成因、解决方案:

1)常见原因

- 原因一:先卖后买:

顾客要的商品缺货,从附近商家或是批发商处暂借过来给销售,事后再补上采购入库单,这种情况下先开销售出库单,之后才录入采购入库单,在开销售出库单后,该商品的库存量就变成了负值。

- 原因二:人为录单差错:

场景①:采购入库时填少了(或供应商有赠品,但是赠品没做入库):

当实际库存数量大于系统库存数量,这时候如果按实际库存数量进行销售就可能会出现负库存,如系统库存数量为10个,实际库存数量为15个(有5个是供应商赠品),这时销售出库12个,系统库存会变成-2个。

场景②:销售出库时填错了,实发数少于应发数:

假设系统库存是10个,实发8个,但出库单填了10个,等到盘点实物时会发现系统库存与实际库存对不上,实际库存多出了2个(10-8=2)。这时再对这2个实物继续做销售,系统库存会提示库存不足,显示-2。

2)解决方案:

针对上面的场景,对应的解决方案是:

场景①:入库单录入错误,需要反审核单据再修改,如入库单已经生成下游单据,则需先录入其他入库单,先把库存补足,然后再反审核修改错误的入库单。修改完毕后要删除补录的其他入库单。

场景②:出库单填写错误导致的实发与应发差异,造成系统与实际库存对不上的,出现了盘盈,需要通过盘点单来手动校正系统库存数。

3、负库存的影响:

负库存可能导致**成本异常**,在负库存的情况下出库,意味着当前出库的商品是将来采购入库的商品,由于是将来采购入库的商品,所以成本只能通过推测得出。

而导致成本异常的根本原因就是这个推测或参考的成本,如果它与将来实际采购入库商品的成本相差很大,就会严重影响库存商品的成本:

如果推测或参考成本大于实际成本,可能导致库存商品成本偏低,甚至为负值;

如果推测或参考成本小于实际成本,可能导致库存商品成本偏高。
