## 常见问题

### 1、 预留单的含义及应用场景

答：星辰支持自动预留和手工预留两种方式：

- **自动****预留**：可适用于业务员开单后，系统自动预留库存，避免被其他人开单占用，导致销售出库时库存不够的问题。还能够解决未录入仓位、或包含批次/保质期/序列号商品的销售订单库存需要占用的场景
- **手工预留**：适用于针对没有订单的商品，希望在出库前先做预留，把库存占用住，方便仓管或销售进行一些库存的处理和确认的场景

**详细知识请查看>>>**[预留单](https://vip.kingdee.com/link/s/lTL9o)

### 3、 销售订单上的批次/保质期/序列号商品想要自动占用库存该怎么办？

答：可以使用预留单的功能。

| 库存更新方式 | 负库存控制参数 | 说明 | 系统参数的设置 |
| --- | --- | --- | --- |
| 保存时更新 | 即时库存 | 需要通过手工新增预留单来预留。需要填写仓库仓位、批次、保质期、序列号等信息才能实现即时库存的占用。 |  |
| 可用库存 | 预计出库量勾选“已审核销售订单”时，开启“销售订单自动生成预留单” ，那么系统会自动处理（发货通知单同理） |  |  |
| 审核时更新 | 即时库存 | 需要通过手工新增预留单来预留。需要填写仓库仓位、批次、保质期、序列号等信息才能实现即时库存的占用。 |  |
| 可用库存 | 预计出库量勾选“已审核销售订单”时，开启“销售订单自动生成预留单” ，那么系统会自动处理（发货通知单同理） |  |  |

### 4、 按可用库存控制负库存时，库存查询表里即时库存（基本）旁边的小问号写的“不包含预留量”是什么意思？此时的即时库存应该怎么计算？

![](https://vip.kingdee.com/download/010932956774f0414d9aa80b5aee1def07bb.png)

答：首先要清楚预留量是怎么来的，如何计算的：

1、预留量包含：

- 进销存：已审核销售订单、已审核发货通知单、已审核手工预留单
- 零售：未审核发货单、未审核配货单、零售单
- 生产：已审核生产任务单，已审核委外加工单
- 订货商城：已审核销售订单
- 第三方：第三方占用

2）生成预留单后，则会增加“预留量”，减少可用库存数量（不再增加销售占用量）

划重点！只要开启了预留单的功能，库存里就会有单独的“预留量”，预留量不会扣减“即时库存”，扣减后的库存要查看“可出库数量”字段，这个才是单据实际可以出库的库存量。

下面通过例子说明：

假设系统参数如下：

![](https://vip.kingdee.com/download/0109344cd01e7ed74847a020f37b6ed32f67.png)

仓库不允许负库存，

![](https://vip-admin.kingdee.com/download/0109065da62981c74c229cd01618eb55e21f.png)

此时深圳仓有库存商品，即时库存为100个，可用库存也是100个，

![](https://vip.kingdee.com/download/01002d228ec850f147218173491d415ac72c.png)

我们先做一个销售订单，假设销售10个，并且审核

![](https://vip.kingdee.com/download/010097005d7708ee4e0da8286420f8e5734d.png)

此时系统会自动生成一张已审核的预留单，预留了10个

![](https://vip.kingdee.com/download/010017014d2af619498cab6a0f5ee3bac4c9.png)

库存查询表的即时库存还是100个，并没有扣减前面已经占用的预留量

![](https://vip.kingdee.com/download/0100c17b137a520f4be49a94e9db448736ef.png)

这个就是“即时库存”小问号里表达的含义“不包含预留量”。

其他的预留方式产生的预留量也是一样的逻辑，本文就不一一枚举。

### 5、 按即时库存控制负库存时，此时即时库存是怎么计算的？

答：下面通过例子说明：假设系统参数如下，

![](https://vip.kingdee.com/download/010984156968bf66430997af36db21cb4f00.png)

仓库不允许负库存，

![](https://vip.kingdee.com/download/0109065da62981c74c229cd01618eb55e21f.png)

此时有库存商品，即时库存为100个，可用库存也是100个，

![](https://vip.kingdee.com/download/0100e37ecaea7044458da7e826ec56ecb0be.png)

先做一个销售订单，假设销售10个，并且审核

![](https://vip.kingdee.com/download/0100f98870c902f448cfa3edc3a8a1c0141d.png)

此时库存查询表的即时库存还是100个，可用库存是90个

![](https://vip.kingdee.com/download/01001462725d09b5435e8b812b0f4636a892.png)

然后我们做个手工预留，对销售订单中的商品预留10个

![](https://vip.kingdee.com/download/0100dd129e1ad69c4d63b6fefa13bad2a23a.png)

此时库存查询表的即时库存还是100个，可用库存是80个（最开始的库存100-销售订单10-预留量10=80）

![](https://vip.kingdee.com/download/01007463b805a5014e91956629dfc7be65d0.png)

如果这个时候新增一张销售订单，下单数量是100

![](https://vip.kingdee.com/download/01094374cfd7a324477e97c854b5b99e4503.png)

然后订单下推出库时，审核时就会提示负库存

![](https://vip.kingdee.com/download/0109ad2e4fc74252401fb089d1abe235efd7.png)

因为**按即时库存控制负库存时，可出库数量=即时库存-预留量**（见下图），也就是100-10=90，而出库单的100＞90，所有提示负库存了。

![](https://vip.kingdee.com/download/0109e80cf98a823a40abb698c41b9b2bc30f.png)

上面的例子只枚举了手工新增预留单一种预留方式，实际上，预留量包含以下的单据：

- 进销存：已审核销售订单、已审核发货通知单、已审核手工预留单
- 零售：未审核发货单、未审核配货单、零售单
- 生产：已审核生产任务单，已审核委外加工单
- 订货商城：已审核销售订单
- 第三方：第三方占用

**划重点！**按即时库存控制负库存时，做出库类单据时，仓库不允许负库存的前提下，不能只看“即时库存”了，要基于这个基础再扣减预留量，系统计算出来的结果就是库存查询表中的“可出库数量”

![](https://vip.kingdee.com/download/0100a5f555be0c3b4f66b922934d7e17cb19.png)

其他的预留方式产生的预留量也是一样的逻辑，本文就不一一枚举。

### 6、 按即时库存控制负库存时，为什么明明数量=即时库存，但是出库还是提示负库存？

![](https://vip.kingdee.com/download/01095db37a2d4de84b02b6aaafa80f47edd0.png)

![](https://vip.kingdee.com/download/0109e514558d236a4e53b6038c5d47d7c543.png)

答：参考本文第五个问题，原因也是一样的，按即时库存控制负库存时，即时库存跟可出库数量可能不一样，可出库数量要在即时库存的基础上扣减掉预留量，所以这种情况下，具体出库时，要看负库存弹窗中的“可出库数量”，

![](https://vip.kingdee.com/download/0109f387ebf722f74a28b823ca894ea6f57b.png)

或者直接查看库存查询表的“可出库数量”（不显示字段时要先在列设置中放出来）

![](https://vip.kingdee.com/download/01096cd4f911866241738ac58cbaa090ba99.png)

### 7、 预留单可以删除吗？

答：系统自动生成的预留单不允许操作删除，而在销售订单反审核、关闭的情况下，自动生成的预留单都会自动删除；

手工新增的预留单可支持审核、反审核、关闭、删除。
