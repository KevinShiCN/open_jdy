# 拣货时发现库存不足，如何提前避免？

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：精华汇总, 常见问题, 进阶技巧
> 作者：移动生产白建凡 | 阅读：62 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/787360734199084800)

# **1 问题描述**

销售订单下推拣验货单时，系统未提示库存不足（如订单有100个商品，实际库存只有97个），拣货环节才发现库存不够，这种情况如何避免？

![上传图片](https://vip.kingdee.com/download/0100cc7facaa2a63414bb449e3e0cd294fb0.png)

# 2 适用版本

星辰3.0以上所有版本，已购买拣货验货模块

# **3 原因解析**

拣验货单本身不占用库存，销售订单下推拣验货单时系统不会自动校验库存数量。我们可以通过销售订单下推预留单锁定库存，然后再下推拣验货单，此时拣验货单的待拣数量就是预留单预留的数量，通过这种方式来处理这类问题

# **4 解决方案**

1）第一步，销售订单先下推预留单，锁定库存

![上传图片](https://vip.kingdee.com/download/0100f0ee30629c984c3081863cf3dd3b3061.png)

2）第二步，销售订单下推生成拣验货单去拣验，或点击“去拣验”跳转到拣验工作台拣验

![上传图片](https://vip.kingdee.com/download/0100fa3f35ecd0774755bf35f7cbb0faf99e.png)

![上传图片](https://vip.kingdee.com/download/0100bd75ed248cc540b394cf9aaf79b46188.png)

此时拣验货单的 “待拣验数量” 会自动匹配预留单的锁定数量（即 97 个，而非订单原 100 个），从源头避免拣货时库存不足
