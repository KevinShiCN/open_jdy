# 审核时更新库存，为什么相同的查询日期和相同的查询条件，商品收发汇总表期末库存跟商品收发明细表的期末库存不一样呢？

> 作者：进销存赵嘉丽 | 阅读：859 | 有用：0

# 原因分析

出现上面问题的最根本原因是，商品收发汇总表期末结存是按库存参数来显示的，而明细数据是按过滤条件显示的。

下面我们通过一个例子说明一下这里的逻辑。

假设有测试账套，系统库存更新方式是审核时更新：

![](https://vip.kingdee.com/download/010089a8feb1eeb645d9b35ff08d4d8e198c.png)

做了一张销售出库单商品100，已保存，未审核。

![](https://vip.kingdee.com/download/0100106e4a4bc6a748528abe2b0c4dc39424.png)

在相同的查询日期和相同的查询条件（审核状态是“全部”）下，商品收发汇总表期末库存为空，

![](https://vip.kingdee.com/download/01006b48771c1d6d4b5c80c8b94df19d791f.png)

但是商品收发明细表的期末库存是-100。

![](https://vip.kingdee.com/download/01004a11fe7b67444c828e0520007057ffc0.png)

# 逻辑说明

**商品收发汇总表：**

• 期末结存是按库存参数来显示的，也就是跟库存查询的结存库存是一致的。

• 保存时更新，期末结存数量和成本不管过滤什么审核状态，都是显示全部状态的结存；

• 审核时更新，期末结存数量和成本不管过滤什么审核状态，都是显示已审核状态的结存；

**商品收发明细表：**

明细行是按过滤条件显示的，期末期末结存是**按过滤出来的明细来计算的**。

因此，当商品收发明细表审核状态为“全部”时，系统更新方式是审核时，而系统又存在已保存未审核的出入库单据时，这部分的数据在明细表里已经查询出来并且参与计算了，而商品收发汇总表由于保持跟库存更新方式一致，没审核的单据就不会算入库存，因此期末结存数量就是空的。

# 解决办法

系统更新方式为审核时更新，查询商品收发汇总表期末库存跟商品收发明细表的期末库存，最好审核状态都勾选“已审核”，以保持两边的数据范围一致，以免产生理解上的歧义。
