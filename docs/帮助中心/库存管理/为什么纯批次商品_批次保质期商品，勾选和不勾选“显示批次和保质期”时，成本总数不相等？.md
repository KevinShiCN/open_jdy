# 为什么纯批次商品/批次保质期商品，勾选和不勾选“显示批次和保质期”时，成本总数不相等？

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：快速入门, 精华汇总, 常见问题
> 作者：进销存赵嘉丽 | 阅读：967 | 有用：7
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/402885267876603136)

# 问题现象

为什么纯批次商品/批次保质期商品，勾选和不勾选“显示批次和保质期”时，成本总数不相等？为什么勾选“显示批次和保质期”时，成本反而还少了呢？是跟分仓或总仓核算有关吗？

# 原因分析

这个属于应用问题，跟分仓或总仓核算没有关系。

一般出现这个原因都是因为存在零库存但是有成本的商品导致的。一般企业业务中，如果出现出库成本低于入库成本的情况（例如打折处理、损耗处理等等），就会导致结存数量已经为0了，但是结存成本不为0的情况。

# 解决办法

星辰的库存查询表默认是不显示0库存的商品的，因此在查看特性商品（启用了批次保质期、辅助属性）或者启用了仓位时，对应仓库的库存及成本时，如果发现勾选跟不勾选筛选项时，成本不一致，把“显示零库存商品”参数勾选上即可。

下面通过一个简单的案例来说明一下这个逻辑。

# 案例说明

假设有纯批次商品SE00080，不限仓库，库存查询表不勾选“显示批次和保质期”，查询出来的总成本是358.3636元：

![](https://vip.kingdee.com/download/0100cf04413d554f4782baba2bc7d24d267e.png)

勾选时总成本是342元，差异是16.3636元，差异是在哪呢？

![](https://vip.kingdee.com/download/0100dda9466fc8d64b1bb7b3576244ca291d.png)

针对上面的示例，把“显示零库存商品” 参数勾选上后，就可以看到有一条结存数量为0，但是成本为16.3636的记录，上述的差异就是在这，这样查出来的成本就对上了。

![上传图片](https://vip.kingdee.com/download/01006a4b62b558f44623b898f16753cc8375.png)
