# 按即时库存盘点时,也没有发生其他出入库业务,为什么盘点单的账存数合计跟库存查询表的即时库存(基本)对不上?

> 作者：进销存赵嘉丽 | 阅读：51 | 有用：0

【问题概述】

按即时库存盘点时,也没有发生其他出入库业务,为什么盘点单的账存数合计跟库存查询表的即时库存(基本)对不上?

![](https://vip.kingdee.com/download/0109d7aebe15335c45709283fc92b6c38857.png)

![](https://vip.kingdee.com/download/0109473193209a64456c804a0e9453df72c6.png)

【原因说明】

一般这种情况是因为盘点中存在多单位商品,且该商品的“库存单位”不是基本单位导致的。

因为盘点单的“单位”默认是取商品资料中的“库存单位”,而库存查询表的“即时库存(基本)”默认是取商品资料中的计量单位,也就是基本单位。

如果商品设置的“库存单位”不等于基本单位,那么由于换算的原因,就会导致盘点单的账存数合计跟库存查询表的即时库存(基本)对不上。

【解决方案】

1)先把库存查询表列设置打开,开启“即时库存(辅助)”显示列

![](https://vip.kingdee.com/download/010913a2c298ddb4435f93f54eebbc7841c2.png)

2)当盘点单分录较多,无法一眼找出哪个商品有差异时,请把盘点单、库存查询表分别引出到本地,再用Excel文件中的vlookup函数来匹配检查是哪个商品的数量不一致

3)最后根据实际业务需求做调整。

如果需要保持账存数合计与“即时库存(基本)”一致:需把商品的库存单位改成跟基本单位一致

如果不需要保持账存数合计与“即时库存(基本)”一致:商品无需修改,但是要查看该商品的盘点单账存数是否跟库存查询表的“即时库存(辅助)”一致。

【适用版本】

星辰3.1以上所有版本
