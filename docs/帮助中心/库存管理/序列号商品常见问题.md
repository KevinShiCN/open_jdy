# 开单问题

### 1、不同商品的序列号可以重复吗？

答：星辰3.1以上专业版/旗舰版已经支持，标准版不支持。请查看：[同一个序列号是否可以给不同的商品使用？系统允许相同序列号入库吗?](https://vip.kingdee.com/link/s/ZAVUt)

### 2、序列号可以只入库的时候录入，或者只出库的时候录入，不用全程出入库都要输入吗？

答：不可以。星辰的序列号为了可以实现整个流转过程的管理，要求出入库时都必须录入序列号，否则会影响库存的计算。

### 3、序列号有长度限制吗,最多可以录多少字段?序列号支持自定义编码规则吗？

答：序列号最长可录入255个字符。序列号支持自定义编码规则，具体设置请查看[https://vip.kingdee.com/link/s/ZAV56](https://vip.kingdee.com/link/s/ZAV56)

### 4、启用序列号+多单位管理时，为啥开单时，序列号的单位不能切换成辅助单位？

答：因为序列号是一个整体标识，不会出一部分、留一部份序列号在库存里，所以星辰不支持多单位的序列号商品在开单时切换单位。因此当您的商品是需要根据单位换算，并且入库和出库单位有可能不一致时，不建议使用序列号+多单位管理。

### 5、为什么单据详情中，一个商品存在多个序列号，直接复制的时候，只能粘贴一个序列号？

### ![上传图片](https://vip.kingdee.com/download/01000aabfb9846984ed3a518d2012b1032c9.png)

答：复制时，只能负责SN字段里展示的第一个序列号。例如下图中的第二第三个都不支持直接复制。

![上传图片](https://vip.kingdee.com/download/01002583014043db417aaa8ec9bde77773d8.png)

### **6、企业互联的商品发送支持**支持发送序列号商品吗？

答：支持，前提条件是该序列号在接收方账套中的库存状态不是“在库”

# 序列号调整/修改相关

### 1、已经入库的商品，不想反审核单据，怎么修改在库的序列号呢？"序列号调整功能"的应用说明？

答：通过序列号调整功能可以解决。

**应用场景**：针对序列号出入库错误需要修改单据中的序列号信息，但由于已有下游单据发生的场景，可通过序列号状态表中查询出对应的序列号做序列号调整，调整后同时更新原单据上的序列号信息。（此功能可以控制授权，可以查看本文第2点）

**应用流程**：

在序列号状态表点击右上角的工具栏“序列号调整”，输入序列号进行调整，或者点击具体的序列号后，直接进行调整，具体应用可见下方GIF动图：

![上传图片](https://vip.kingdee.com/download/01003c00a32055a84f1c9868b61b5228b589.gif)

也支持批量调整序列号

![上传图片](https://vip.kingdee.com/download/01008fb4afa2734540a186b5684187b461e0.png)

### 2、为了避免人为操作不当修改了序列号，可以控制不让调整序列号吗？

答：可以。星辰支持对序列号状态表中【序列号调整】按钮实现权限控制，如不允许业务员或仓管员随意修改商品序列号的信息，保证信息前后一致性。

**应用流程：**

在功能授权>库存管理>序列号状态表中，勾选【序列号调整】。勾选，则允许修改，不勾选，则不允许修改。

![上传图片](https://vip.kingdee.com/download/0100ed9966de8a834033b506d6807473fbfe.png)

### 3、序列号发生流转后，已经有下游单据了，想要修改中间环节的单据的单价/数量，当系统参数为审核时更新时，系统不允许反审核，这是为什么？

答：目前序列号流转有先后顺序，审核时更新库存，不能通过反审核单据来改变中间环节的序列号状态。此种场景下，如果是单据上数量单价错误要修改，要么反审核相关下游单据后再修改，或者做退货后再重新出库。

# 序列号查看问题

### 1、商品期初引入备注后为什么列表看不到?

答：引入模板里的“备注”字段其实并不是商品行里的备注,而是序列号单据体的备注。详情请查看：[https://vip.kingdee.com/link/s/ZAV8V](https://vip.kingdee.com/link/s/ZAV8V)

### 2、单据上如何录入与查看序列号的备注?

答：请查看：[序列号备注录入与查看方法](https://vip.kingdee.com/link/s/ZAVii)

### 3、单据列表为什么不支持显示序列号字段？单据列表支持按序列号进行模糊搜索吗？

答：因为考虑到单个商品的序列号数量可能会有很多，后台做模糊查询时会耗时很久，因此基于性能的限制，单据列表不支持展示序列号，也不支持序列号的模糊查询，如需查询，请通过库存管理下的"序列号状态表"查询。

### **4、单据详情中的序列号字段支持表头过滤吗？为什么序列号字段明明有值，但是过滤不出来？**

答：序列号商品，当一行里同时包含多个序列号时，漏斗的过滤只能选到SN字段里展示的第一个序列号（也就是单据SN上直接显示的那个序列号），如果录入的序列号顺序不是第一个，而是第二或更后，则无法筛选到。

# 数量不一致问题

### **1、库存更新参数为**审核时更新库存时，查询序列号商品，点击库存字段后，为什么序列号联查弹窗中的数量跟库存查询表中的数量对不上？

![上传图片](https://vip.kingdee.com/download/010067dcb7b9ef9941b6b9bceed4b29fde63.png)

答：库存查询表查询的序列号商品的“即时库存”，是包含了序列号状态为“在库”+“待出库”的，序列号联查弹窗调用的是序列号状态表的数据，联查的时候，只显示了在库状态为“在库”的，

![上传图片](https://vip.kingdee.com/download/010035d6d9d298974ed6a66f0a72719b51cc.png)

所以当存在“待出库”商品时（如存在已保存未审核的销售出库单），此时就会两边数字就会对不上。解决的办法就是在序列号状态表里把“在库”+“待出库”分别查询出来，才等于库存查询表里的即时库存。

![上传图片](https://vip.kingdee.com/download/0100a5b38c99345d414cbc0a6e788329794d.png)

也可以对“在库状态”选择“全部”，引出到本地后，再过滤“在库”+“待出库”的合计数量就等于库存查询表里的数据了。
