# 调拨差异处理表

## 1、简介

### 1.1功能介绍

处理调拨过程中可能存在忘发、错发、损坏等导致调拨出库的商品数量与调拨入库的商品数量不一致的问题；

### 1.2适用版本

金蝶云·星辰所有版本

### 1.3应用场景

由于调拨过程中可能存在忘发、错发、损坏等导致调拨出库的商品数量与调拨入库的商品数量出现不一致的情况，此时会产生调拨差异。调拨差异处理表通过列表的方式，展示调拨过程中出现的差异详情，并实现对差异的处理。

### 1.4系统路径

库存管理-仓存单据-调拨差异处理表

## 2、 主要操作流程

对于一次调拨，如果调拨出库单和调拨入库单的商品数量不一致，则会被展示在调拨差异处理表中

处理步骤如下：

①选择需要进行差异处理的调拨记录

②确认处理方式：生成其他入库单（差异为负时） 或 生成其他出库单（差异为正时）

③对生成的其他出/入库单进行保存、审核操作，差异处理完成

此时，差异处理完成，该此调拨差异记录的“差异处理状态”更新为“已处理”，且生成的其他出/入库单据编号会被自动填写到“差异处理结果”中。

## 3、常见问题

### 场景1：A仓库（门店）向B仓库（门店）调出商品100件，B仓库收到了99件，少收了1件，请问怎么处理？

答：少收的1件商品，需要双方沟通确认，是在路上丢失或者报废，还是A仓库少发了这1件商品。

• 若是A仓库少发了1件商品，A仓库会补发，那么，建议B仓库按照实际和99件商品录入调拨入库单，并等待下次补发的1件商品到货后，再录入一张调拨入库单完成本次调拨。

• 若是A仓库少发了1件商品，且A仓库不会补发，那么，建议A仓库反审核原调拨出库单，修改数量为实际发货的99件，审核后，B仓库再录入调拨入库单，录入实际到货的99件商品，完成入库。

• 若双方确认这件商品丢失，那么，需要判断是谁的责任，若是由A仓库负责处理，那么，建议A仓库反审核原调拨出库单，修改数量为实际发货的99件，审核后，B仓库再录入调拨入库单，录入实际到货的99件商品，完成入库；若是由B仓库负责处理，那么，建议B仓库在调拨入库单中录入数量100件，再做其他出库单，将丢失的1件商品进行出库。

### 场景2：A仓库（门店）向B仓库（门店）调出商品100件，B仓库收到了100件，但其中1件商品损坏，请问怎么处理？

答：双方需要确认损坏的商品由哪方承担。

• 若是由A仓库承担且损坏的商品不需要B仓库寄回，那么，建议B仓库在调拨入库单中录入实际可用的商品99件，形成调拨差异，A仓库通过调拨差异处理表，生成其他入库单，将1件损坏商品重新入库，再做一张其他出库单，将这件损坏商品报废出库。

• 若是由A仓库承担且损坏的商品需要B仓库寄回，那么，建议B仓库在调拨入库单中录入实际到货的商品100件，再通过调拨出库单或其他出库单将其中的1件损坏商品寄回A仓库，A仓库入库后，再做报废处理。

• 若是由B仓库承担损坏的商品，那么，建议B仓库在调拨入库单中录入实际到货的商品100件，然后，做一张其他出库单，将其中的一件损坏商品做报废处理。

### 场景3：A仓库（门店）向B仓库（门店）调出商品100件，B仓库收到了101件，多收了1件，请问怎么处理？

答：多收的1件商品，需要双方沟通确认，是否需要将商品还回调出方。

• 若商品不需要还回A仓库，那么，建议B仓库在调拨入库单中录入实际到货的101件商品，形成调拨差异，A仓库通过调拨差异处理表，生成其他出库单，将多发的1件商品出库。

• 若商品需要还回A仓库，那么，建议B仓库在调拨入库单中录入实际到货的101件商品，形成调拨差异，A仓库通过调拨差异处理表，生成其他出库单，将多发的1件商品出库，B仓库手工建立其他出库单，将多收的1件商品发还给A仓库，A仓库收到货后，通过其他入库单，将商品入库。
