# 盘点单盈亏处理时,提示“第x行分录:基本单位与商品中的计量单位不一致,请重新录入商品”,但是检查并没有问题,要如何解决?

**【问题描述】**

盘点单盈亏处理时,提示“第X行分录:基本单位与商品中的计量单位不一致,请重新录入商品”,但是检查并没有问题,要如何解决?

**  【原因说明】**

**盈亏处理时,校验的实际分别是盘点单【盘盈商品】和【盘亏商品】页签里的商品分录。**

上述示例中,实际有问题的分录是在**盘点单的【盘盈商品】页签**中的第二行商品:

![](https://vip.kingdee.com/download/0109be135de82d1c47f5b2443a50ac16b61e.png)商品的实际计量单位是“个”:

![](https://vip.kingdee.com/download/01091910d707082e479e8f026defe1184861.png)
那什么时候会出现这种提示呢?

一般新增一个商品,然后录盘点单,在盘点单还没保存审核时,又人为去修改了商品的计量单位,然后再审核盘点单,点击“盈亏处理”时就容易出现这个情况。

**  【解决方案】**

首先就是建议**开单后不要随意修改商品信息**。

其次,如果开单后发现有错误,**确实需要修改商品信息,建议删除单据上已经录入的商品分录行,然后重新录入商品**,保证商品带出的信息都是跟最新的基础资料保证一致即可。

针对上面示例的情况,就是要先把盘点单反审核,然后将【全部商品】中的商品SP00907删除,然后再重新录入该商品及盘点数,再重新保存审核盘点单,再点击盈亏处理即可。

![](https://vip.kingdee.com/download/0109218088f1b3b34be2a463c47ac46eedea.png)
**  【适用版本】**

星辰2.0及3.1所有版本
