【问题描述】

拆卸单子件/组件成本如何计算?

#### 【相关视频】

#### [拆卸单子件成本如何计算?](https://vip.kingdee.com/school/detail/662739051140797696?get_from=school-detail-id&productLineId=35&lang=zh-CN&cosmic_type=video)

【业务场景】

**拆卸单是将一个或多个组件拆卸成多个子件的单据:**

**1.组件的商品成本是经过成本核算计算出组件的出库成本;**

**2.子件商品成本根据参数,由“成本分摊比例”将组件出库成本合计分摊到每行子件的成本,或取系统参数中指定的成本来源,或手工录入。详细可参考下方操作流程。**

**若您想要了解组装单成本问题,请点击:[组装单成本如何计算](https://vip.kingdee.com/knowledge/395742735199694080?productLineId=35&isKnowledge=2&lang=zh-CN)**

【操作流程】

拆卸单子件成本来源可点击【系统设置】>【系统参数】>【存货核算参数】进行设置

![](https://vip.kingdee.com/download/010922a6dbf5adb944719e94e477e85a7689.png)

主要有三种取数方式:
![](https://vip.kingdee.com/download/01098365cd2340f84340af370858e5b01574.png)
**1)按成本来源取数:**

子件取商品本身的成本来源,分别有子件对应的“上次入库成本”、“期初平均成本”、“上次出库成本”和“商品参考成本”

**2)子件根据组装件成本进行比例分摊**

**在进销存参数选择子件分摊比例是按参考成本计算还是按数量计算**

**按参考成本计算:**在【基础资料】>【商品信息】,商品信息里面填好参考成本,详情请点击:**[商品的参考成本在哪里设置?](https://vip.kingdee.com/knowledge/607377468122556672?productLineId=35&isKnowledge=2&lang=zh-CN)**

**按数量计算:**根据拆分出来的子件的数据计算分摊比例,分摊比例计算出来之后再按比例分摊组件的总成本

**tips:两种取数方式互斥,只能设置其中一种**

![](https://vip.kingdee.com/download/0109842653462f994c3ba05bd5194d51552f.png)

拆卸单完成后,选择“根据组装件成本进行比例分摊”时,需优先做出库成本核算,核算出组装件的出库成本才能带出子件的成本,若核算完成后,未能带出子件成本,需检查以下数据:

①系统设置-系统参数-存货核算参数中,拆卸单的成本来源是否正确选择为“根据组装件成本进行比例分摊”

②出库成本核算结果,组装件成本计算表是否正常计算

③子件是否有拆卸费用未分摊,拆卸费用未正常分摊,例如:新增一个拆卸单,分摊费用为100元,需要做作为子件成本分摊,但是拆卸费用分摊没有数据,需补录进去
![](https://vip.kingdee.com/download/0109de62ca6766c44dc89bf2f0029a7c5fd7.png)

补录完成后,重新做**[出库成本核算](https://vip.kingdee.com/knowledge/391350577239897088?productLineId=35&isKnowledge=2&lang=zh-CN)**后打开拆卸单查看成本数据,发现成本只有拆卸分摊的数据,这时候需要检查拆卸费用的分摊规则
![](https://vip.kingdee.com/download/01098956c8c0229d4ee2af3cc9c065b784bb.png)

分摊规则若为“成本分摊比例”时,需检查子件分录是否有对应的分摊比例
![](https://vip.kingdee.com/download/01096fb6b10344804bdabb7b61b8e90e31ac.png)
![](https://vip.kingdee.com/download/010902065f43261342e59e01d129aeb06514.png)

**tips:1.若分摊比例为空或是合计不等于100%时, 先检查进销存参数拆卸单的分摊比例是选择的哪一种方式 ,**

**2.若选的是按参考成本就到商品信息检查有没有录参考成本;**

**3.如果改成按数量计算分摊比例则,保存参数之后需要反审核拆卸单,选择一行子件把数量删除,再重新录入正确的数量,这样操作之后就会按数量重新计算所有子件的分摊比例,有了分摊比例再做出库成本核算重新核算子件成本。**

**3)手工录入子件成本**

方案一:在拆卸单上“商品成本”字段手工录入子件成本,组件的成本经过出库成本核算后自动带出

![](https://vip.kingdee.com/download/0109e2648e5028b540c788346401c77aae1e.png)
由于子件成本是手工录入,会出现跟组件成本不一致的情况,成本差异部分会记入拆卸成本差异

![](https://vip.kingdee.com/download/0109affe22ca501141a798c92ebd1822b677.png)

方案二、在核算单据查询中录入

**核算单据查询录入成本的详情,请点击:[核算单据查询功能如何使用?](https://vip.kingdee.com/knowledge/391351046616069376?productLineId=35&isKnowledge=2&lang=zh-CN)**

**tips:1.支持在核算单据查询中录入子件和组件的成本,由于成本皆为手动录入,可能会出现子件成本之和不等于组件成本的情况,存在着拆卸成本差异的情况;**

**2.单据审核后可以在核算单据查询里录入;**

**3.录入完成后需要及时将拆卸单生成凭证或者直接结账以锁定录入的成本,否则当期再次核算成本时会被覆盖更新,详情请点击:[为什么通过核算单据查询手动录入成本后,重新进行出库成本核算,成本却消失了?](https://vip.kingdee.com/knowledge/391350578330145280?productLineId=35&isKnowledge=2&lang=zh-CN)**

![](https://vip.kingdee.com/download/01091310d807c68e40d490ce40d83262e033.png)

【温馨提示】

常见出库核算失败原因及解决办法如下:

| 失败原因 | 解决办法 |
| --- | --- |
| 拆卸单据未审核 | 审核对应单据 |
| 拆卸单据已生成暂存凭证 | 删除暂存凭证 |
| 拆卸单据日期不在核算范围内 | 业务结账或反结账到单据所在期间再重新核算 |
| 拆卸单子件无成本 | 检查存货核算参数中拆卸单的成本来源 |

**tips:①若拆卸单的成本来源选择按组装件成本进行比例分摊计算子件成本,还需要排查组件是否有成本。**

**②若选择按成本来源序号优先取值,则按勾选情况排查成本来源是否为空或为零。**
