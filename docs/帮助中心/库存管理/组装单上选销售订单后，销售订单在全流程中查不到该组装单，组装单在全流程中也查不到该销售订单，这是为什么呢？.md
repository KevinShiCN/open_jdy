# 组装单上选销售订单后，销售订单在全流程中查不到该组装单，组装单在全流程中也查不到该销售订单，这是为什么呢？

**【问题概述】**

组装单上选择销售订单后，销售订单在全流程中查不到该组装单，组装单在全流程中也查不到该销售订单，这是为什么呢？

**【原因说明】**

因为组装单上选择销售订单仅用于计算子件，实际并不产生关联关系；全流程跟踪的依据是单据间必须有关联关系，而星辰中的关联关系是指修改本单后对上游单据存在反写或影响的单据关系。由于组装单与销售订单不存在这种关联，仅作为计算参考，因此不会在全流程跟踪中显示。

类似的场景还有：销售出库单上生成的与费用相关的其他支出单，销售出库单与其他支出单之间不存在关联关系；删除销售出库单不会影响该其他支出单。

**【解决方案】**

若需要建立关联关系，该场景本质上属于以销定产的生产模式，建议使用星辰生产云模块处理；组装单不会扩展支持此需求。

**【适用版本】**

星辰3.1以上所有版本
