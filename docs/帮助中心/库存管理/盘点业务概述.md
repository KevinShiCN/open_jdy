# 1 简介

## **1.1 盘点功能介绍**

盘点是指定期或临时对库存商品实际数量进行清点，将结果与系统数据进行对比，调整并统一系统数据的业务。

## 1.2 适用版本

星辰3.1以上专业版/旗舰版——涵盖盘点业务的所有功能

星辰3.1以上标准版——不包含盘点业务中的“多人盘点流程”功能

## 1.3 盘点流程说明

星辰提供单人盘点及多人盘点两种方式。

多人盘点——星辰专业版/旗舰版支持，标准版不支持。

## 1.5 名词解释

| 名称 | 含义 | 特别说明 |
| --- | --- | --- |
| 任务名称 | 即盘点单的任务名称，企业可以根据需要把如日期、时间、地点等信息记录到名称中。 |  |
| 盘点仓库 | 即需要盘点的仓库，星辰不支持同时对多个仓库进行盘点，每个盘点任务只能盘一个仓库 |  |
| 锁盘 | 勾选“锁盘”时，该盘点仓库在结束盘点之前不允许进行各种出入库操作 | 勾选“锁盘”时会检查所选仓库是否有未审核的出入库单据，包括采购入库单、销售出库单等。 |
| 库存日期 | 即时库存：即创建时刻对应的仓库库存，跟创建时刻的库存查询表按即时库存查询的结果是一致的。指定日期：商品的账存数取指定日期的库存（指定日期盘点时，不支持序列号商品盘点） |  |
| 盘点模式 | 单人盘点：也就是一个人盘点，适用于盘点量不多的场景，或者需要指定专人跟进的盘点场景。多人盘点：适用于企业商品种类多、数量大，每月末盘点时都需要多人协同完成，每个人对自己的盘点数据负责，最终结果由盘点负责人进行汇总的场景。 | 单人盘点时，如果有漏盘，可以把漏盘商品直接添加到盘点单，多人盘点则不支持直接添加漏盘商品，因为多人盘点单必须通过分量盘点单来汇总，且分量盘点单是可以覆盖更新汇总到多人盘点单上。为了避免多人盘点单的数据被分类盘点数据覆盖，如果有漏盘的商品，只能通过分量盘点单来补录，不能直接添加到多人盘点单上。 |
| 指定盘点人 | 即指定盘点单由特定的业务员来录入，不设置则只要有权限的人都可以编辑这张单。 |  |
| 商品来源 | 在盘点仓库中有交易记录的商品：来源于《库存查询表》按仓库过滤出有交易记录的商品，全盘时为全部商品，抽盘时，用于指定按商品类别、商品品牌、商品、仓位盘点时的商品范围。商品资料中所有商品：来源于商品基础资料中的商品，全盘时为全部商品，抽盘时，用于指定按商品类别、商品品牌盘点时的商品范围。 |  |
| 盘点范围 | 全盘：即所有商品都盘抽盘：即抽取部分商品盘点。用于指定按商品类别、商品品牌、商品、仓位盘点时的商品范围。 | 支持自定义盘点单是否包含禁用商品、包含零库存商品 |
| 盘点规则 | 按批次和保质期盘点：勾选后，盘点清单中的批次和保质期商品会详细展示对应的批次和保质期信息；若不勾选，且盘点清单中存在批次和保质期商品，本次盘点将被视为“按总量盘点”。按序列号盘点：勾选后，盘点时商品需要录入序列号；不勾选，盘点时商品直接录入总数量，不必录入序列号。 |  |
| 商品清单展示 | 空白盘点单：在盘点单中不预先显示商品信息，盘点完成后，以用户录入或者引入的商品顺序为准。显示商品清单：将所有要盘点的商品显示在盘点单中，打开盘点单后按商品清单进行盘点。 |  |
| 账存数 | 系统中商品的即时库存数量。 |  |
| 漏盘 | 创建盘点任务时，设置了商品的盘点范围，将此商品范围与盘点单已盘点的商品进行比较，若盘点范围指定的商品未在盘点单中出现，表示漏盘。在盘点任务中，若选择抽盘，又未指定商品范围，则表示随机盘点，则漏盘查询为空。 |  |

## **1.6 盘点类型及其应用场景**

（1）**按空白单据盘点**：盘点时将商品名称、盘点数量记录在纸上或使用盘点机盘点，盘点结束后直接将数据录入或导入到空白的盘点单中，与系统数据对比得到盈亏结果。适用于商品较多、摆放随意的仓库盘点。

（2）**按商品清单盘点**：选择商品创建盘点清单，打印后按清单进行盘点，盘点完成后将数据录入或导入系统，与系统数据对比得到盈亏结果。适用于商品较少、摆放规律的仓库盘点。

（3）**按商品总量盘点**：对于批次和保质期、序列号商品，允许不按批次和保质期、序列号进行盘点，初步仅统计数量差异。若有盈亏数据产生，再选择按批次和保质期重新盘点或者在盈亏单中手工录入批次和保质期。

（4）**指定日期盘点**：当用户去仓库实际清点库存的日期不是当前日期，而是前几天时，盘点日期就要设置为清点库存的那个日期，这时盘点单提取的账存数就是指定日期那个时点的即时库存。

>
注意：**序列号商品不支持按指定日期盘点**！因为序列号的在库状态只有一个最新状态，不会记录历史日期的在库状态，也就不知道历史日期盘点时的在库状态了，只能对“在库”的商品进行盘点。

（5）**多人盘点：**记录每个人的数据，再汇总后形成最终盘点结果，核对库存，调整库存差异。

# 2 制单指南

## 2.1 多人盘点（标准版不支持）

1) **业务场景：**

  企业A的商品种类多、数量大，每月末盘点时都需要多人协同完成，每个人对自己的盘点数据负责，最终结果由盘点负责人进行汇总。

2) **多人盘点概述**

**业务流程：**

![](https://vip-admin.kingdee.com/download/0100bc4c5b745d97471c94bcc5249a3fa87d.png)

**操作步骤：**

（1）新增盘点单，进入盘点单向导。选择创建盘点任务。

![](https://vip-admin.kingdee.com/download/01006ae11387f4734a4eb2c7cf8e38156245.png)

（2）在盘点任务中选择“多人盘点”，并根据需要设置参数。

![](https://vip-admin.kingdee.com/download/010034e82934435d418883fb589961cd134a.png)

- **库存日期**：即时库存表示取将商品加入盘点单时的库存；指定日期表示取截止到所指定的日期的库存，在未锁盘时，考虑到出、入库单据还允许录入，因此，盘点单在审核时，还会再次刷新商品的账存数为指定日期的库存。
- **盘点范围/商品来源**：指定了本次盘点单的盘点范围，若选择全盘，表示商品来源中的所有商品都需要盘点，若盘点单中没有这个商品，在【漏盘查询】时，可以查到；若选择抽盘，并指定了商品类别、商品品牌、商品、仓位，那么将按照指定的商品范围及来源核查漏盘商品，若选择抽盘，但是并未指定商品范围，说明本次盘点对象是随机的或者目标明确，无需进行漏盘查询，漏盘查询数据为空。
- **盘点规则**：确定商品盘点时，是否必须录入批次、保质期、序列号，若都不勾选，表示这类商品只需要盘点总数量，无须盘点批次、保质期、序列号明细。但是若有盘点差异，还是需要补录差异数量 批次、保质期、序列号，以更新对应的库存。
- **商品清单展示**：设置是否根据商品来源和盘点范围将商品显示到盘点单中，提供显示和不显示两个选项。
- 注意：盘点单中若已录入盘点数据，这些参数值都不允许再修改了。

（3） 点击【创建任务】，生成多人总量盘点单，该盘点单不允许手工录入商品，只能通过【汇总分量盘点单】生成盘点数据。

（4）新增分量盘点单，方法有二种。

**方法一：**是在创建任务并保存成功时，点击【去盘点】，就会生成该多人盘点任务对应的分量盘点单，直接录入盘点数据。

![](https://vip-admin.kingdee.com/download/01002cf5f8b02de8431bb87107cec31c84c0.png)

**方法二：**在盘点单列表，点击【新增】，选择【已有任务，去盘点】，单击“进行盘点”，系统会弹出盘点任务选择界面，用户选择要进行盘点的任务，按照该任务进行分量盘点操作。

![](https://vip-admin.kingdee.com/download/010077bd6bac8714499180d8329cd31ca2d3.png)

**分量盘点单如下：**

![](https://vip-admin.kingdee.com/download/0100d0a70f0ac0684a87a93d8744ee050fd4.png)

（5）在盘点单列表中，进入多人总量盘点单，点击【汇总分量盘点单】，将各盘点人的盘点数据进行汇总（只能选择该任务关联的已审核的分量盘点单）。

![](https://vip-admin.kingdee.com/download/0100ca8a59db5cde46209b67ac2dcab7c3bf.png)

（6）审核总量盘点单，点击【盈亏处理】生成盘盈单、盘亏单。

![](https://vip-admin.kingdee.com/download/0100bf8926eb45d3415e9e1c2260e5342eed.png)

（7）审核盘盈单、盘亏单，更新商品库存。盘点完成。

## 2.2 批次商品盘点

1) **业务场景**

 食品、医疗器械等行业的客户，商品启用了批次、保质期管理，在盘点时，需要对这部份商品进行盘点。

2) **批次商品盘点概述**

  系统提供按批次和保质期盘点、按总量盘点两种模式，用户可以根据实际需求，通过参数控制盘点的精细度。

  在盘点时，若需要记录每个批次、保质期的数量，在创建盘点任务时，勾选“按批次保质期盘点”，在盘点单中录入盘点数量就会控制必须录入批次、保质期才能保存成功。

![](https://vip.kingdee.com/download/0100568331aa40454c4199c23b9ffe16953f.png)

  一般情况下，商品即使启用了批次和保质期管理，盘点时，也不会按照序列号来盘点，那么，在创建盘点任务时，不要勾选“按批次保质期盘点”，在盘点单中直接录入盘点数量，核对与账存数是否相符，若是相符，盘点单在【处理盈亏】时，自动标识盘点完成，若是不相符，则会提示：

![](https://vip.kingdee.com/download/010034c875ecaaee47feb8c278cd0a7110c5.png)

  由用户选择这部份有差异的商品是在新的盘点单中按照批次、保质期重新盘点还是直接生成盘盈单、盘亏单，在盘盈单、盘亏单中补录有差异的批次、保质期，完成盘点。

  注意：盘点单中账存数与盘点数相等、无盈亏数时，盘点单点击【盈亏处理】，不会生成盘盈单、盘亏单，盘点单标识“盘点完成”；若盘点单中账存数与盘点数不相等、有盈亏数时，盘点单点击【盈亏处理】，生成盘盈单、盘亏单，将盘盈单、盘亏单审核后，盘点单才会标识“盘点完成”

在锁盘的情况下，盘点单只有标识“盘点完成”后，才会自动解锁。

![](https://vip.kingdee.com/download/0100551c3961050b415e973498b4a8b69035.png)

## 2.3 序列号商品盘点

1) **业务场景**

  数码3C等行业的客户，商品启用了序列号管理，在盘点时，需要对这部份商品进行盘点，针对价值特别大的商品，还需要按照序列号进行盘点。

2) **序列号商品盘点概述**

  系统提供按序列号盘点、按总量盘点两种模式，用户可以根据实际需求，通过参数控制盘点的精细度。

  针对价值特别大的商品，需要盘点到序列号维度，在创建盘点任务时，勾选“按序列号盘点”，在盘点单中录入盘点数量就会控制必须录入序列号才能保存成功。

![](https://vip.kingdee.com/download/0100fbbef285b5764973993b130218c8baa1.png)

  注意：盘点单中账存数与盘点数相等、无盈亏数时，盘点单点击【盈亏处理】，不会生成盘盈单、盘亏单，盘点单标识“盘点完成”；若盘点单中账存数与盘点数不相等、有盈亏数时，盘点单点击【盈亏处理】，生成盘盈单、盘亏单，将盘盈单、盘亏单审核后，盘点单才会标识“盘点完成”
