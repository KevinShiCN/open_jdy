# 库存查询表分页查询说明

> 产品：金蝶云·星辰 | 分类：库存管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：99 | 有用：0

# 【问题概述】

- 库存查询表，勾选“仅显示小计”后，为什么明明一页可以显示所有的小计行，但实际却分了好多页？
- 库存查询表，勾选“仅显示小计”后，为什么只看到了一部分商品的合计行？
- 库存查询表，勾选“显示小计”后，为什么有的商品看不到小计行？

# 【原因说明】

以上三类问题都跟库存查询表分页查询有关。

## 分页的含义

分页就是把一份数据分成多个页面展示。

## 分页应用价值

不分页前，当库存查询表数据很多时，这种整页查询方式对性能的消耗较大，因此从性能优化角度出发，产品做了改造，改成了分页查询，每页显示的行数受右下角的页签数影响。

![上传图片](https://vip.kingdee.com/download/01000ffb0c90e68441e0919ff7af08022284.png)

## 分页后，“显示小计”、“仅显示小计”是基于分页结果进行查询

示例1：设置了每页显示50条，查询结果共163条，就会分成4个分页，而小计行就是跟着对应商品走的。

![上传图片](https://vip.kingdee.com/download/0100718b0e024d544e068d0625c5608cd384.png)

勾选“仅显示小计”后，商品行隐藏，小计行还在，就会出现每个分页都有小计行的情况。

![上传图片](https://vip.kingdee.com/download/01002c11e7639bcb487b8fd5203ff5950c98.png)

如果一个商品对应在多个仓库有库存或商品是特性商品，勾选了“显示辅助属性”、“显示批次和保质期”，那么分页后，可能就会存在商品行刚好跨页了

示例2：承接上一个例子，例如第50-55行都是同一个商品，但由于分页，商品分别展示在了第一页的最后一行以及第二页的前五行。

![上传图片](https://vip.kingdee.com/download/0100b7bcea8020124135a370382b051de7f1.png)

![上传图片](https://vip.kingdee.com/download/01002942d12e855c443c9fa4d28586cc183c.png)

此时勾选“显示小计”，由于该商品的最后一行在第二页，就会出现第一页的商品没有小计行的情况。

![上传图片](https://vip.kingdee.com/download/01007d6cedafd0184adfb17d1a9e029c03a6.png)

右下角页签切换到第二页就能看到了。

![上传图片](https://vip.kingdee.com/download/01009527e9e08632464d9e4edf090657474d.png)

# 【解决方案】

右下角的页签数调整到最大：1000条/页。同时通过上一页/下一页来查看分页的全部数据。

# 【适用版本】

星辰3.1以上所有版本
