| 实施前必读1. 申请小程序和开通支付通道前，要使用一致的公司主体名称。小程序的主体必须与支付通道的认证主体保持一致，否则小程序商城无法使用微信支付。例如小程序的主体是金蝶软件公司，则支付通道的主体也必须是金蝶软件公司。目前门店商城小程序第三方支付仅支持金蝶支付和通联支付。注册公司主题不满一年客户，使用微信直连方案或者走金蝶支付/通联支付小程序收银台方案。金蝶支付咨询及开户信息登记入口：https://docs.qq.com/form/page/DWWpLaVFDZkxxeW1v 金蝶妙想（通联）咨询及开会信息登记入口：https://docs.qq.com/form/page/DV01udGNQZ1NHRFJD2. 在发布小程序前，请根据操作手册自查一遍，确保你已经完成了以下必须步骤：a. 到零售后台至少创建 1 个线上门店并上架商品，商品建议至少1-2 个普通商品，避免商品太少影响审核通过率。等待小程序发布上线后，可以逐步完善门店和商品资料。如果你有批次/保质期商品，商品上架后到零售后台《门店管理 - 参数设置 - 门店管理参数》开启「自动匹配批次开单」参数或者提交小程序审核发布前先不要上架批次/保质期商品，等待发布成功后再上架。b. 到微信小程序后台《开发 - 开发管理》，开通相应的微信接口（非常重要，3个接口请务必开通，否则无法提交发布）。c. 到零售后台《门店营销 - 商城管理 - 商城配置 - 小程序授权》完成小程序授权绑定。3. 确认清楚你是否有开展快递或同城配送业务，若有请先开通微信物流助手，这一步不影响小程序发布，但是会影响业务运转，请关注：a. 第 1 步：通过每个物流公司支持的方式与发货地附近的线下网点合作签约，注册账号。b. 第 2 步：到《微信小程序后台 - 物流服务》绑定物流或同城配送公司。c. 第 3 步：到零售后台《门店营销 - 商城管理 - 配送管理》完成快递发货或同城配送配置及品类设置。 |

一、名词说明

说明：以下内容，线上商城简称为线上，线下门店简称为线下；客户/商户/商家均指商家；用户/消费者/顾客均指消费者。

二、功能概述

门店把店铺搬到线上，开展线上商城业务。购买商城服务后，需要在后台进行一系列配置，维护必须的资料和参数信息。维护好之后方可正常开展线上电商业务，支持消费者线上下单。标准版暂不支持门店小程序商城。

消费者可以进入商城完成以下事情：

| 1. 购物主线1. 基于用户LSB进入最近门店，并可切换其他门店；查看门店营业信息和推荐信息 2. 根据用户需要，选择不同的配送方式，支持配送到家与门店自取3. 浏览商品，查看商品详情4. 选品后确定规格，加入购物车或立即购买5. 下单结算，支持使用优惠券/积分抵扣，完成支付（储值余额抵扣、微信、云闪付）6. 查看商城订单和线下门店订单，根据不同状态的订单完成相应操作 | 1. 营销侧1. 支持用户到领券中心领券2. 支持用户在线充值 | 1. 基础设施1. 支持查看会员资产，包括会员储值余额、优惠券、积分2. 支持查看交易明细、积分明细3. 支持查看和修改会员资料（性别）4. 支持查看会员码5. 支持用户新增或编辑收货地址 |
| --- | --- | --- |

消费者下单成功后订单会推送至POS配送台，商家可以在配送台完成以下事情：

| 1. POS - 配送台 - 商城订单1. 待确认订单：接单或拒绝订单；打印订单；查看接单倒计时；查看订单详情2. 接单中：取消订单、拣货；完成拣货、完成拣货，去配送/去提货；打印订单；查看订单详情3. 待配送：取消订单、配送；打印订单；查看订单详情4. 待提货：取消订单、提货；打印订单；查看订单详情5. 配送中：完成配送；打印订单；查看订单详情6. 已完成：查询订单；退货；导出、打印 |  |

五、操作说明

第 1 步：金蝶官方渠道购买门店小程序商城服务

进入金蝶官方购买渠道，选择「门店商城」点击进入，点击【立即购买】进入购买页，按实际需求购买即可。（*注：官方购买渠道包括「kdadmin、云管家PC、云管家移动端、官网PC、官网移动端」，新购/续费/变更略有不同，若有疑问请联系官方客服或片区分销经理。）

第 2 步：申请微信小程序，完成微信认证

| 说明1. 若你已经有微信小程序，则可以直接使用该小程序开展线上商城业务。2. 若你暂时还没有微信小程序，则需要前往微信公众平台申请一个小程序。3. 若你暂时还没有微信小程序，但是已经有微信公众号，则可以复用该公众号主体资质申请一个小程序。 |
| --- |

关于小程序申请与认证，详见：[微信小程序申请与认证操作指南 (kingdee.com)](https://vip.kingdee.com/article/359013549760761344?productLineId=35&isKnowledge=2)

第 3 步：门店资料创建与完善

1. 完善门店基础资料

|  |  |  |
| --- | --- | --- |

进入零售后台，前往《门店管理 - 资料 - 门店资料》维护门店资料，有两种处理方式：

1. 若你之前已经创建过门店，且是在原有门店基础上开通线上商城，可进入门店资料页修改门店信息，门店类型选择“线上门店或线上+线下门店”，并维护门店经营信息等。

2. 若你之前没有创建过门店，或者需要新建线上门店商城，请点击【新增】进入新增门店资料，门店类型选择“线上门店或线上+线下门店”，并维护门店经营信息等。

便于用户在商城可以查看门店与用户之间的距离，并且用户下单时会根据用户收货地址与门店地址计算两者之间的距离，系统自动匹配相应的配送策略与运费规则，请务必维护好门店所在行政区划与详细地址。

| 切换门店类型时，有几点需要注意，以下分场景说明 |
| --- |

1. 线上→线下：直接修改，无需提示。切换后该线上门店商城不可用，商品跟随下架；线下门店需手动上架商品

2. 线上→线上+线下：切换时会提示是否“将线上商品资料同步上架到线下门店”，默认同步上架，可以取消勾选，若取消勾选，则需到《门店管理 - 资料 - 商品上下架》手动上架商品到线下门店（*注：切换门店类型后，请确保您购买的门店服务是否可正常使用）

3. 线下→线上：直接修改，无需提示。切换后该线下门店不可用，商品跟随下架；线上门店商城需手动上架商品

4. 线下→线上+线下：切换时会提示是否“将线下商品资料同步上架到线上商城” ，默认同步上架，可以取消勾选，若取消勾选，则需到《门店管理 - 资料 - 商品上下架》手动上架商品到线上商城（*注：切换门店类型后，请确保您已购买商城小程序服务，否则无法使用）

5. 线上+线下→线上：切换时会提示是否“将线下商品资料从线下门店同步下架” ，默认强制同步下架，不可取消勾选，商品从该线下门店下架后将无法正常展示（*注：切换门店类型后，请确保您购买的门店服务是否已过期或不再使用）

6. 线上+线下→线下：切换时会提示是否“将线上商品资料从线上商城同步下架”，默认强制同步下架，不可取消勾选，商品从该线上商城下架后将无法正常展示（*注：切换门店类型后，请确保您购买的商城小程序服务是否已过期或不再使用）

2. 完善商城经营信息

|  |  |
| --- | --- |

1. 营业时间：可以设置全天营业，或者自定义营业时间。设置后会展示在商城门店列表和分类页顶部门店信息处。

2. 门店推荐：可以设置门店推荐信息，建议填写门店的推荐活动或商品或特色服务，例如买一送一、免费停车和wifi等。设置后会展示在商城分类页顶部门店信息处。

3. 接单方式：可以设置门店对商城订单的接单方式，支持手动接单或自动接单。若选择手动接单：用户下单后，需要商家手动确认接单。若选择自动接单：用户下单后，无需商家手动确认接单，系统会自动确认接单。

4. 配送方式：可以设置门店商城的配送方式，支持门店自取和配送到家。勾选对应的配送方式，用户在商城下单时即可选择该方式。

5. 自提/配送点设置：若你一个线上商城对应多个自提/配送点，可以开启并设置。添加门店后可以根据业务需要设置是否支持自提/支持配送/接单权限/退款权限。设置自提点后，用户在商城下单时，可以切换提货门店。 设置配送点后，用户在商城下单时，系统会自动匹配距离收货人最近的门店。（*注：这一步非必须操作，仅当你一个门店有多个自提点或配送点时才需要配置。）

6. 发货仓配置：若你有发货仓模式，可以开启并设置。启用后，可通过发货仓处理当前门店商城的特定订单，进行备货、发货等操作。（*注：这一步非必须操作，仅当你有发货仓模式时才需要配置。）具体操作请参考[金蝶云·星辰零售 - 门店小程序商城 · 发货仓模式操作手册 (kingdee.com)](https://vip.kingdee.com/article/353479385430680064?productLineId=35&isKnowledge=2)

7. 提货码必填：可以设置用户在门店提货时是否需要录入提货码，开启表示必填。

8. 接单自动打印：可以设置接单自动打印单据类型，包括商城小票和拣货单，勾选后接单时会自动打印。

9. 手动打印时需打印：可以设置手动打印单据的类型，包括商城小票和拣货单，勾选表示会打印。

第 4 步：零售后台配置商城参数*（一定需要配置）*

|  |  |  |
| --- | --- | --- |

进入零售后台，前往《门店管理 - 设置 - 商城对接参数》，点击【新增】进入商城对接参数配置页面，点击「应用」下拉选择「自营小程序商城」，维护好“默认门店、默认客户、业务员、部门、收银员”等信息后保存即可。（*注：1.因为门店线下零售单必须关联收银员，这一步操作仅为了将商城零售单关联门店与收银员等信息，不影响商城订单实际归属。2.门店和收银员等信息只需要维护一个即可，无需根据实际门店数配置。）

第 5 步：商城信息配置

![](https://vip.kingdee.com/download/0100dac544bc9ab845a2a3ff90217db9f12f.png)

进入零售后台，前往《门店营销 - 商城管理 - 商城配置》维护商城基本信息：

| 1. 完善品牌信息填写品牌名称，上传品牌LOGO，将会在商城主要界面的页面底部与“金蝶云星辰技术服务”技术服务标识统一展示，增强用户品牌认知。 | 1. 完善商城负责人信息填写商城负责人姓名/联系方式。，方便用户在商城合适的场景下联系商家。 |

第 6 步：商品资料维护

1. 维护商城商品资料

|  |  |  |
| --- | --- | --- |

进入零售后台，前往《门店营销 - 商城管理 - 商品资料》维护商品资料，商城的商品资料直接取进销存的商品资料，点击商品资料进入编辑，维护“商品分享描述、商品分组、划线价”等信息。

1. 商品基础资料

a. 商品图片：无需单独维护，目前直接取商品资料的图片。小程序商品详情图取「除首图之外的其他图片」，后续会规划支持单独配置商城图片，敬请期待

| 图片建议尺寸：1. 商品列表图片（小图片）：宽度 250 像素，高度不限2. 商品详情图片（大图片）：• 图片尺寸（1:1比例）：710*710像素• 图片尺寸（3:2比例）：710*480像素 |
| --- |

1. 商城商品基础资料

a. 分享描述：维护后用户在商城分享商品给微信好友时，其好友可以看到该分享文案。为空时将用默认文案分享

b. 商品分组：维护商品分组后，将会在商城的全部商品页顶部商品分组展示该商品；也可以在商城首页配置时配置对应的分组在首页展示（商品分组默认展示在分类导航栏的顶部）

2. 价格

a. 划线价：维护划线价后，将会在商城相应页面展示。（手动划线价即为此处配置的划线价）

i. 商城各界面价格展示规则：

1. 首页：手动划线价、零售价

2. 搜索页：手动划线价、零售价

3. 分类页：手动划线价、零售价

4. 商详页：会员价、促销价、零售价、手动划线价

5. 购物车页：会员价、促销价、零售价、手动划线价

6. 确认订单页：会员价、促销价、零售价、手动划线价

7. 订单详情页：实际购买价格

ii. 划线价展示逻辑：

1. 仅当会员价或促销价小于零售价或手动划线价时，展示划线价

2. 若存在手动划线价，优先展示手动划线价

3. 若不存在手动划线，优先展示零售价

3. 高级参数

a. 商品类型：设置商品类型，选择现货商品或定制商品。现货商品无需生产加工，下单会校验和消耗库存；定制商品下单默认不校验库存，可通过时段限量功能配置时段供应量

b. 备货时间：按小时或天设置备货时间，下单选择期望提货/送达时间时，会加上该商品的备货时间

2. 创建运费服务商品

![](https://vip.kingdee.com/download/0100e21c7cad9274487381685f5bc02b7299.png)

| *注：小程序上线之初，商家须手工创建一个运费服务商品，商品名称必须为运费或者运费商品。 |
| --- |

进入零售后台，前往《基础资料 - 商品资料 - 商品信息》创建一个运费服务商品，这是为了用户在商城下单时选择的运费有服务商品承载，在订单中记录运费信息。商品名称必须为运费或者运费商品，不可以使用其他名称，便于系统将此商品与运费进行关联映射；商品类型选择服务。

3. 维护商品分组

|  |  |  |
| --- | --- | --- |

进入零售后台，前往《门店营销 - 商城管理 - 商品分组》维护商品分组，可以完成以下事情：

1. 新增商品分组：可以新增商品分组，维护分组名称、分组别名、图片、适用商品和适用门店等信息

2. 编辑商品分组：选择某个已创建的分组编号，可以二次编辑分组

3. 启用/禁用：选择若干分组，可以启用或禁用该分组，启用后该分组才能在商城分类页正常展示

4. 删除：选择若干分组，可以删除该分组，删除后该分组不再展示在后台，商城也无法正常展示

5. 排序：可以点击某个分组的上下排序控件调整分组在商城的展示顺序，越靠上在商城分类页展示越靠前

十、注意事项

1. 小程序提交审核和发布更新时，避免微信官方审核拒绝，提高审核通过率，请务必先完成以下工作：

| 必须操作登录零售后台给全部线上门店上架商品，无数量要求，但建议至少 3-5 个，避免商品太少影响审核通过率。完成商城主页装修，避免首页内容单调。维护好门店详细地址，否则可能会影响运费获取，导致用户无法下单。提交微信接口审核通过。 |
| --- |

2. 小程序的主体（即公司名称）必须与通联支付的认证主体保持一致，否则小程序商城无法使用微信支付。比如小程序的主体是金蝶软件公司，则通联支付的主体也必须是金蝶软件公司。

3. 开通支付通道后，请登录该支付通道后台开通小程序微信支付及退款功能，并到零售后台完成支付通道与门店收款方式配置，后者操作与线下门店一致。
