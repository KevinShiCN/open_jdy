2、 **功能概述**

**预订单订金支持用礼品卡支付，订金下推零售单的时候核销充值的预收款金额。**

使用场景：烘焙、家电行业，使用预订业务，会员先交部分订金，等蛋糕完成或者家电到货再补交剩余款项。订金用会员礼品卡的金额进行支付，减少支付摩擦，提升预订便利性。

2.1，提升会员粘性与资金沉淀：通过礼品卡支付订金，可增强用户忠诚度，同时提前锁定消费资金，优化门店现金流。

2.2，简化支付流程与促进转化：预订单与会员账户无缝衔接，减少支付环节摩擦，提升订单完成率，并刺激后续消费。

同时需要有礼品卡方案，并且售卖礼品卡到会员名下。

![](https://vip.kingdee.com/download/01002042bf67bc53497d98fd1a4f59511aad.png)

3.2 POS 支付订金用礼品卡的结算方式

![](https://vip.kingdee.com/download/01006fe2a340273e4a65a86bb8ca7055d787.png)

3.2.1支付成功之后在POS预订单记录中可以看到订单状态

下推零售单——核销订金，生成商品销售类型的零售单

编辑——对订单进行编辑修改

关闭——关闭预订单

退订金——关闭预订单之后可以退订金

![](https://vip.kingdee.com/download/0100c9bd962fd38c4c9badde6c007e72a48a.png)

3.3 星辰后台生成不同业务类型以及收款类型的零售单。核心是下推零售单才会核销转换为订金的礼品卡金额。

**说明：下推零售单之后的金额核销顺序：**

订金核销顺序是先核销现金，再核销礼品卡金额。

若是零售单里面只是用了礼品卡支付，退货可以用现金或者其他（原有逻辑不做改变）

3.4 零售单/结存单等零售相关报表可以记录商品以及结算明细信息，不做赘述。
