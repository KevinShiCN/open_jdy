1.1操作流程

1.2传秤

（1）. 首先，秤上的数据一般是由PC机或POS，将整理好的商品信息，通过局域网，发送到秤上的。（所以请先参考硬件使用手册检查条码秤的传输设置，IP设置，网络是否联通等）

（2）. 条码秤里面的储存空间是有限的。顶尖LS-6X支持999999个，但部分条码秤、如大华TM-30F，只支持5000个（或3000个、4000个）商品，此处以5000个为例，秤上以PLU码作为内码区别商品，PLU码是数值型的，从1到5000。由于商品编码通常涉及到商品分类和财务核算，编码规则较复杂，所以一般传输过程中是把商品条码作为PLU码传到秤上的。对于条码秤而言，生鲜商品的条码应当是纯数值类型的，PLU为1或01或001或0001或00001，都是同一个商品，没有区别。并且大小在1到5000之间（具体范围与使用的秤的型号有关）。在设置生鲜商品信息的过程中不应该出现条码数值相同的情况。条码为0000的生鲜商品，可能会导致数据传输失败，因为条码秤无法识别无法记录PLU为0的商品。

（3）. 虽然顶尖LS-2X可以支持999999个商品，但由于条码类型格式的限制，条码最大只能到99999，具体可参考后面的条码类型格式的说明

1.3生鲜条码解析

1.3.1条码格式

| 生鲜条码格式 | 格式说明 |
| --- | --- |
| 13位重量码 | DD(2) IIIII(5) WWWWW(5) X |
| 13位金额码 | DD(2) IIIII(5) PPPPP(5) X |
| 18位重量金额码 | DD(2) IIIII(5) WWWWW(5) PPPPP(5) X |
| 18位金额重量码 | DD(2) IIIII(5) PPPPP(5) WWWWW(5) X |
| 18位单价重量码 | DD(2) IIIII(5) UUUUU(5) WWWWW(5) X |
| 18位重量单价码 | DD(2) IIIII(5) WWWWW(5) UUUUU(5) X |

其中：

DD(2)表示两位门店号，与WEB的两位称重条码商品前缀对应

IIIII(5)表示五位商品条码，用于识别商品，所以商品条码最大只能到99999，除非支持其他条码类型格式

WWWWW(5)表示五位重量，一般是以KG或个为单位，三位小数位。最后一位表示1g（貌似行业对重量的精度有要求，一般不随便调整）

UUUUU(5)表示五位单价

PPPPP(5)表示五位小计金额，可以通过秤的设置精确到角，或者直接不显示分位

X表示校验位，有效避免收银员通过手输条码时手误输错的情况。

备注：X校验位的计算方法：

13位的计算方法【（偶数位之和*3）+奇数位之和+Z=10的倍数】（从右往左第三位开始，所有奇位数之和）

18位的计算方法【（奇数位之和*3）+偶数位之和+Z=10的倍数】（从右往左第二位开始，所有奇位数之和）

![](https://vip.kingdee.com/download/01000ac22f9d530f4c9c89cc2cc206673cf3.jpeg)

以上图为例，生鲜条码为2921026002701，其中，29表示条码秤的部门，也就是系统里面的称重条码商品前缀；21026表示商品西红柿的条码；00270表示小计金额，两位小数精度（后台参数设置）；1表示校验位；所以上面这个条码是13位金额码。

校验位X计算 :（9+1+2+0+2+0）*3 +（2+2+0+6+0+7）+X =10的倍数，所以X=1

1.3.2解析逻辑

首先，如果输入的条码是纯数值，且长度以及前缀与条码秤参数设置的称重条码商品前缀和条码秤格式一致的话，则当做是生鲜条码进行解析，然后检测校验位是否合法，不合法则抛出提示，合法则根据商品条码进行数值检索（例如，条码上得出的五位商品条码是00017，检索的时候只检索17，检索出来的结果可以是17、017、0017、00017、000017，但不能包括10017，设置条码时应该注意条码不可以数值上重复，否则检索的结果不具有唯一性）

目前支持上述六中条码类型，根据条码检索出商品后，对于各种条码类型处理如下

13位重量码：通过条码获取数量，价格取系统价格（促销价会员价等），然后计算小计。13位金额码：通过条码获取小计金额，然后取系统的当前单位的零售价格，倒推出商品数量后，再取商品系统价格，计算小计。

18位重量金额码和18位金额重量码：通过条码获取数量以及金额，倒推出商品价格

18位单价重量码和18位重量单价码：通过条码获取数量以及单价，计算得出小计

注1：由于13位和18位生鲜条码中最多包含重量、单价、金额中的一个或者两个数据，并且有精度限制，所以计算出来其他数据可能会存在误差。

注2：目前软件不支持按分录四舍五入，所以建议在条码秤设置金额精度保留分位。如果使用金额码又没有保留分位，会导致计算出来的重量或者单价与实际不符。

条码秤格式

设置生鲜条码秤标签格式，详见1.3.1

如上图，根据标签信息可以看出是13位金额码

称重条码商品前缀：该参数用于区分普通商品以及生鲜条码秤商品的条码，对应上图的标签前面两位29，如果后台这个参数设置的不是13位的条码以及29的前缀，那就无法正确识别上面的标签，会当做普通商品条码进行检索；对该设置没有特殊要求，尽量不与客户的普通商品条码冲突即可；

如果打印出来的标签条码前面两位与系统设置不一致，可能存在以下几种情况

1. 后台参数设置不正确

2. 星辰POS未同步到该参数

3. 该商品是历史数据，并非是通过星辰POS传秤的商品数据

称重商品重量小数位、非称重商品重量小数位：这两个参数分别针对称重以及计件两种类型的条码秤商品，一般只用于解析条码；

生鲜条码如果是重量码（13位重量码或18位重量单价码等），条码中有五位是包含无精度的重量信息，比如01000，如果称重商品重量小数位设为3位数，解析得到的重量为1kg；如果非称重商品重量小数位设置为3位数，解析得到的数量为1（件）；

称重商品重量小数位一般设置为3位，非称重商品小数位一般设置为3位（顶尖）或者0位（大华）

称重商品金额小数位：这个参数用于传秤（部分秤）以及解析，生鲜条码如果是金额码（13位金额码或18位重量金额码等），条码中有五位是包含无精度的金额（小计）信息，如上图的00270；

如果此参数设置为1位精度，解析得到27.0元，如果设置为2位精度，解析得到2.70元；另外，从标签上可以看出单价5.16元/kg，重量0.522kg，5.16*0.522=2.69352，由此可见秤上设置了金额精度是1位，此时使用金额码会导致反算的重量与实际不符，2.70/5.16≈0.523kg，应该要将秤上的金额精度设置为2位小数；

注：修改参数后都应该使用星辰POS云同步后重新传秤，如果传秤后没有生效，请先尝试在网页端修改参数后，重启星辰POS，待云同步完成后再进行传秤操作；并在星辰POS传秤商品列表确认是否存在该商品以及商品的合法性，排除是历史数据；

2.2称重商品清单（网页端）

设置路径：[门店管理]-[资料]-[称重商品清单]

![](https://vip.kingdee.com/download/01009ae4a7d12e56489d96418303ee42ab20.png)![](https://vip.kingdee.com/download/0100aaa89b8905ea48e5be35a938751d8800.png)

需要传秤生鲜商品需要先添加到【称重商品清单】，另外，门店也需要上架商品。

添加到称重商品清单符合以下规则：

1. 条码为5位纯数字，不足5位则前面补零再添加（目前系统是使用商品条码作为PLU码（秤上的唯一码），后续会新增PLU码字段，区分条码以及PLU码）；

2. 商品的价格需大于0；（部分条码秤打印价格为0的商品标签会报错或者不显示生鲜码）

3. 称重以及计件的商品都需要添加到【称重商品清单】，目前是以单位名称进行区分；当单位名称是以下单位时认为是称重商品：千克、kg、克、g，其余均为计件商品；（目前不支持斤、500g，后续会支持）

三、生鲜价格策略

生鲜商品也符合系统取价策略，下面针对几种不一样的表现进行说明：

（1）13位重量码在秤上改价无效

（2）13位金额码在秤上改价会导致反算重量异常

（3）18位码扫码后按生鲜条码进行取价，但是选择会员后会按系统设置的会员价格进行取价。

四、常见问题

4.1传秤通讯异常

1. 条码秤需要接网线，和星辰POS在同一个局域网，并确保可以PING通

(1).windows端

[开始]-[Windows系统]-[命令提示符]

![](https://vip.kingdee.com/download/010088a4c87f26db420d926ea88b5a32ce58.png)

（或者按键【win】+【R】在运行界面打开cmd）

![](https://vip.kingdee.com/download/010077be8db88521462a99834f4b3b234d80.png)

在界面输入ping 条码秤IP地址（如ping 172.20.57.134），并回车，查看返回结果，判断是否PC与条码秤是否在同一个局域网内。

![](https://vip.kingdee.com/download/0100a3e8dc55745849e6970166d0faf76477.png)

(2).android端

可以安装下面这个应用，在Ping界面输入IP地址进行连接测试。

![](https://vip.kingdee.com/download/0100a2c6cfbbdb6942c3bc4888cf1d236ec1.png)

![](https://vip.kingdee.com/download/01006875727495834ad89afa1ab4b97a2235.png)![](https://vip.kingdee.com/download/01002c9bbc3abee442328550026fb1532a05.png)

2. 确认IP地址是否冲突

条码秤的IP地址是固定IP地址，但手机等设备一般是动态IP，可能存在IP冲突。

可以在上一个步骤测试连接成功后，关闭条码秤电源，确认当条码秤电源关闭后，PC或者安卓POS机无法连接IP地址即可。如果关闭条码秤后仍然可以PING成功，请重新设置秤的IP地址。

4.2生鲜标签在星辰POS无法识别

1.查看该商品是否在星辰POS的传秤商品列表中（见2.4）

传秤是覆盖更新数据，所以可能会存在历史数据。如需要清空数据一般使用条码秤厂家的秤工具（如顶尖的LINK66，大华的TMA4.0，容大得众的RLS1000）。

2.确认生鲜标签是否与网页端条码秤参数设置相符（见2.1）

如果不一致，请尝试重新云同步，待同步完成后再进行传秤操作；

如果仍然无法解决，修改网页端条码秤参数设置（见2.1）中的值并保存，重启星辰POS直到完成云同步后再进行传秤测试

4.3星辰POS解析出来的金额或者重量精度不正确

![](https://vip.kingdee.com/download/0100de39b54b72a64e4bbe8ccf9f5cae7859.png)![](https://vip.kingdee.com/download/0100e6efd484c6794d01bd379076c67f78e2.jpeg)

修改网页端条码秤参数设置（见2.1），云同步完成后重新扫码即可。（部分秤需要重新传秤重新打标签）

如果是重量码，则修改称重商品重量小数位和非称重商品重量小数位。如果是金额码，则修改称重商品金额小数位

称重商品重量小数位一般设置为3位，非称重商品小数位一般设置为3位（顶尖）或者0位（大华），金额小数位一般设置为2位

4.4区分称重商品和计件商品

目前根据单位名称进行判断，单位名称是千克、KG、克、G的商品判断为称重商品，其余判断为计件商品。（后续会增加斤、500G，暂不支持）

4.5修改标签模板

秤出厂后都会预置模板，如需要修改标签模板，请使用条码秤厂家的工具（如顶尖的LINK66，大华的TMA4.0，容大得众的RLS1000）进行修改，一般是windows端的工具，此处不做详细步骤说明。

需要将标签纸压在缺纸传感器下，直到亮绿灯再压下扳手。
