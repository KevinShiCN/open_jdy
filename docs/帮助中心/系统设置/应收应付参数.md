# **1 简介**

## **1.1 操作路径**

系统设置>系统参数>应收应付参数

## **1.2 适用版本**

星辰3.1以上所有版本

# 2 参数含义及应用场景

## 2.1 基础参数

1）按单据核销：选择此种核销方式时，录入收付款单、核销单时，选源单显示的是出入库单据的整单信息，按单据应收应付款总额进行核销。

2）按分录核销：选择此种核销方式时，录入收付款单、核销单时，选源单显示的是出入库单据的分录信息，按单据应收应付款分录金额进行核销。

3）按余额核销：选择此种核销方式时，录入收付款单、核销单时，在单据显示应收应付余额，不提供选择源单核销功能，直接录入收付款金额即可，系统自动按指定的方式进行应收应付款核销。

- 选择“按收付款时按单据结算日期先后顺序”时，系统自动将所有待核销的单据按结算日期先后顺序依次核销。
- 选择按“收付款时按单据类型 + 结算日期先后顺序 ”时，可以调整单据的核销顺序，会先按单据类型排序，同一种单据类型按结算日期先后顺序依次核销。

4）若您希望应收应付单据对账后，不能再反审核修改：勾选后，则不允许反审核已对账单据，不勾选则不控制。

## 2.2 信用管理

1）销售信用额度控制方式：用于销售类等客户往来相关单据时，当客户往来余额超过信用额度时的控制方式，支持保存时和审核时两种方式选择。

2）客户信用额度控制范围：支持对销售订单、销售出库单、其他收入单、委托代销结算单控制

3）采购信用额度控制方式：用于录入采购类等供应商往来相关单据时，当供应商往来余额超过信用额度时的控制方式，支持保存时和审核时两种方式。

4）供应商信用额度控制单据范围：支持对采购订单、采购入库单、暂估调整单、其他支出单、受托代销结算单、联营结算单、委外加工费控制

## 2.3 账龄管理参数

用于控制应收账款、应付账款超期预警管理，结合客户和供应商中设置的结算期限，在录入业务单据时系统自动计算结算日期，在结算日期到期前进行收付款提示，也可以在对账日前按指定的天数预警。
