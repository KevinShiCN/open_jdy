# 用户管理右上角的用户数要怎么看呢？数字分别代表什么意思？

> 作者：进销存赵嘉丽 | 阅读：2855 | 有用：44

# **问题描述**

用户管理右上角的用户数要怎么看呢？数字分别代表什么意思？

![](https://vip.kingdee.com/download/0109df8c5100f72e4ea6bc9e42d9bbc44f46.png)

答：用户管理列表的右上角会显示当前服务已授权用户数、PC剩余用户数、移动端剩余用户数、轻分析剩余用户数（需购买）、生态扩展产品PC剩余用户数、电商管理用户数（需购买）等等。

# 字段说明

**1、“服务”是指就是能从工作台进入的产品（如下图），一个服务内可以有多个账套**

![上传图片](https://vip.kingdee.com/download/01005b7ffd448639419eb2ee877111666c67.png)

**2、用户管理中包含了几种类型：**

| 用户类型 | 含义 | 说明 | 对应jdy.com购买页面（点击图片可放大查看） |
| --- | --- | --- | --- |
| 星辰PC | 购买了进销存云或财务云PC端的用户数 | 同一服务下的同一个用户在不同账套内都是启用状态，只算一个用户数，不会重复计算（详见下文第4点详情） |  |
| 移动端 | 单独购买移动端使用的用户数 | 单独购买的移动端用户数不会计入PC端用户数中（详见下文第5点详情） |  |
| 电商管理 | 购买了电商管理的用户 | 购买了电商管理的用户默认有20用户，但是这20用户只能在PC端使用电商管理与基础资料模块。 |  |
| 轻分析 | 购买了轻分析的用户 | 授权该用户在当前帐套下使用轻分析搭建数据看板及查看数据看板各账套用户数加总计数，若同一用户在不同帐套下均需使用，要分别在不同帐套下对此项进行勾选授权，计多个用户数（同一个用户不做去重）。举例：1个服务包含2账套（分别是A和B)，如果购买了一个1轻分析用户，那么轻分析只能在A里面用，不能在B里面用,如果A和B都要用轻分析，必须再买一个轻分析用户。 |  |
| 生态扩展产品 | 购买了生态扩展产品的用户 | 星辰用户是统计该租户（即工作台里同一个服务）下面所有账套里面的启用用户数的（会去重），属于租户级别的去重统计，而“生态扩展产品用户”是按账套级别，只统计当前账套的，不做去重。 | 具体请咨询ISV厂商。 |

**3、“当前服务已授权用户数”是怎么计算的？XX剩余数又是怎么算的？**

**答：**

| 字段名称 | 显示的逻辑及含义 | 特别说明 |
| --- | --- | --- |
| 当前服务已授权用户数 | “/”左边的数字表示当前服务已授权的用户数，“/”右边的数字表示当前服务可授权的用户总数 | 当前服务可授权的用户总数：是包含了星辰PC、仅移动端、生态扩展产品PC三种类型的用户，不含轻分析的 |
| 星辰PC剩余 | “/”左边的数字表示星辰PC还可以授权的用户数，“/”右边的数字表示星辰PC总的可授权的用户总数 | 都是按租户级别统计的，也就是：同一用户在同一个租户下不同账套都是启用状态，只算一个用户数，会去重计算。 |
| 仅移动端剩余 | “/”左边的数字表示仅移动端还可以授权的用户数，“/”右边的数字表示仅移动端总的可授权的用户总数 | 都是按租户级别统计的，也就是：同一用户在同一个租户下不同账套都是启用状态，只算一个用户数，会去重计算。 |
| 轻分析剩余 | “/”左边的数字表示仅轻分析还可以授权的用户数，“/”右边的数字表示轻分析总的可授权的用户总数 | 都是按账套级别统计的，也就是：同一用户在同一个租户下不同账套都是启用状态，会计多个用户数，不会去重计算。 |
| 生态扩展产品PC剩余 | “/”左边的数字表示仅生态扩展产品PC还可以授权的用户数，“/”右边的数字表示生态扩展产品PC总的可授权的用户总数 | 是按账套别统计的，也就是：同一用户在同一个租户下不同账套都是启用状态，每个账套里都会占用用户数，不会去重计算。 |

下面我们通过案例来更好的理解这里的逻辑：

**案例1**：

![](https://vip-admin.kingdee.com/download/0109df8c5100f72e4ea6bc9e42d9bbc44f46.png)

因此，上图就代表：

- 该服务共购买了60个用户（其中星辰PC50个，移动端10个），现在已经授权了24个用户；
- 星辰PC端总共可授权用户数50个，其中已经启用了24个用户，剩余还可以启用26个用户；
- 仅移动端总共可授权用户数60个，其中50个是PC端本身自带的用户数（也就是只要购买了一个PC端的用户，默认拥有一个“PC+移动端”的用户数）；而PC+移动端里，已经启用了24个，还能启用26个，仅移动端的还可以启用10个，所以总共是还有36个可以启用的。
- 轻分析共可授权用户数20个，其中已经启用了2个，还可以启用18个用户

---

**案例2：**

租户A公司，有一个星辰服务，共购买了10个星辰PC用户，5个仅移动端用户，且该服务内包含了3个账套（分别是A/B/C），那么假设目前，**只有1个用户张三，在ABC账套中都启用了，授权范围是PC和移动端**。

因此在ABC账套中显示结果都如下：

- 当前服务已购买的用户数=10(星辰PC)+5(仅移动端)=15，
- 当前服务已授权用户数：1/15（星辰已授权的用户数是按租户级别统计的，只会算1个用户（去重统计））
- 星辰PC剩余数：9/10
- 仅移动端剩余数：14/15

---

还是接着案例2的例子，如果该租户做了ISV，即二开，**且A账套里购买了扩展用户数5人，并且启用了李四1人，对他授权了【星辰+扩展用户****】****的范围**，B账套购买了扩展用户数3人，但是没有启用任何扩展用户，那么：

**A账套**里显示结果如下：

- 当前服务已购买的用户数=10(星辰PC)+5(仅移动端)+5(生态扩展产品PC)=20
- 当前服务已授权用户数：**3/20**（星辰已授权的用户数是按租户级别统计的，只会算1个用户（去重统计），张三和李四去重后，各占1个用户数，而生态扩展的是按账套维度统计的，此账套启用了1人）
- 星辰PC剩余数：8/10
- 仅移动端剩余数：13/15
- 生态扩展产品PC剩余：4/5

---

**B账套**里显示结果如下：

- 当前服务已购买的用户数=10(星辰PC)+5(仅移动端)+3(生态扩展产品PC)=18
- 当前服务已授权用户数：**2/18**（星辰已授权的用户数是按租户级别统计的，只会算1个用户（去重统计），张三和李四去重后，各占1个用户数，生态扩展的是按账套维度统计的，此账套内没启用）
- 星辰PC剩余数：8/10
- 仅移动端剩余数：13/15
- 生态扩展产品PC剩余：3/3

**4、移动端的用户数是怎么计算的？**

如果购买了PC端1用户，会自动新增1个移动端用户数，如果是**单独购买**的移动端1用户数，则单独算一个用户数并统计在 “移动端剩余”中，不会计入PC端总用户数中。

**举例：**

假设购买了2个PC端用户，2仅移动用户，这样在用户管理显示是：PC端可授权总数是2，移动端可授权总数是4（其中2个用户可以单独设置只使用移动）。
