# 为什么数据表格下方还有空白位置,但是表尾区无法打印,会打印到下一页?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：1273 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/680948859354138624)

【问题描述】

为什么数据表格下方还有空间,但是表尾区(图片中的蓝色边框)无法打印,会打印到下一页?
![](https://vip.kingdee.com/download/0109be883df0c518459aa75ad844d47426d2.png)
【原因说明】

**因为表尾区打印是作为一个整体的来打印的,而且表尾区没有行高自适应的属性,****表尾高度是确定的,**不会根据打印纸张剩余的空白位置来自适应的,如果打印单据时,底部空白区域不能完整打印下整个表尾区,就会到下一页。

前面问题中的打印模板,我们可以看到,如果设置每页打印4行数据的话,**表尾区已经超出了单据的可打印范围了,**因此即使单据只有4行分录,但是表尾区也只能打印到下一页中。
![](https://vip.kingdee.com/download/010918f2821842d6471cb5a455b0d555a26d.png)

【解决方案】

| 方案 | 适用场景 | 设置方法 | 打印模板设置效果 | 调整后打印预览 |
| --- | --- | --- | --- | --- |
| 方案1:不使用表尾区 | 适用于表尾内容比较多的场景。例如:合同打印 | 1、取消数据表格的“表尾区域”;2、直接把表尾需要打印的字段拆成一个个单独的文本控件;3、对文本控件设置顺序打印。 |  |  |
| 方案2:使用表尾区 | 适用于表尾内容比较少的场景。例如只打印收货人、收款信息等内容 | 表尾区不要有多余的留白,不要预留那么多高度,高度要跟内容相匹配。选中表尾区,然后在控件属性中,设置高度即可。 |  |  |

【适用版本】

星辰3.0以上所有版本
