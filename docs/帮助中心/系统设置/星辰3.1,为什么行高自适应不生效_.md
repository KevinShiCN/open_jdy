# 星辰3.1,为什么行高自适应不生效?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：1082 | 有用：6
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/515330811529085440)

【问题描述】

    星辰3.1中,数据行或者合计行为什么设置了行高自适应,但是打印时不生效呢?

![](https://vip.kingdee.com/download/010992d88a2da2cf4e1a8308c28ca3e656a3.png)

![](https://vip.kingdee.com/download/01094e5e29376b3f48a4b19d7c69b028a4f1.png)

  【原因解析】

    在星辰3.1中,行高自适应不生效有几种情况,对应的解决方案也可以查看下表说明:

| 不生效的情形 | 具体设置 | 解决方案 |
| --- | --- | --- |
| 原因1:打印设置或者打印模板里设置了固定行打印,即每页打印分录数 |  | 如果要使用固定行打印,则不能使用行高自适应,建议在打印模板中对单元格设置“自动缩小” |
| 原因2:合计行。数据表格中合计行不支持行高自适应,标题行及数据行支持行高自适应。 |  | 若为合计行,请将合计行在模板设计时,预留足够的行高。 |
| 原因3:行高自适应与自动缩小的冲突(行或者单元格都要注意检查),不能同时设置 |  | 行高自适应与自动缩小只能二选一,不要同时设置 |

   【适用版本】

    适用星辰3.1以上全版本
