# 商品编码规则设置了商品类别和商品品牌,并且开启流水号依据及断号补偿,为什么同一个商品类别下,不同的品牌流水号共用了?

> 作者：进销存赵嘉丽 | 阅读：133 | 有用：1

【问题概述】

商品编码规则设置了商品类别和商品品牌,并且开启流水号依据及断号补偿,为什么新建商品时,同一个商品类别下,不同的品牌流水号共用了?

![](https://vip.kingdee.com/download/0109f1d81dd112724774a3ce05352e1699db.png)
商品品牌编码不一样
![](https://vip.kingdee.com/download/01099df1c017db4146bea4bcf72210adf120.png)
同一个商品类别,商品品牌不一样,但流水号共用了
![](https://vip.kingdee.com/download/01090cef5240fcc84a1ab0c03d7f1262e8a8.png)
【原因说明】

出现这个现象通常都是因为商品编码规则中没有开启分隔符导致的。

**数据库对于设置了多个业务字段,并且都开启了流水号依据的编码,需要通过分隔符来判断编码中间的业务字段的变化**,从而再更新流水号。如果没有分隔符,当第一个业务字段不变,只变化了中间的业务字段时(对于前面问题描述的示例来说,“商品品牌”就是一个中间的字段),由于数据库无法判断出中间业务字段的变化,只能判断到第一个字段,就会认为业务字段没有变化,从而直接递增了前面的流水号,导致流水号共用了。

【解决方案】

商品编码规则中,当设置了两个基础资料字段作为业务字段,并且都开启了流水号依据的,无论“使用模式”是“完全取值”还是“属性截取”,**如果希望任意一段业务字段变化都要更新流水号,一定要设置分隔符,不能设置“无”,这是必要条件。**

星辰3.1以上所有版本
