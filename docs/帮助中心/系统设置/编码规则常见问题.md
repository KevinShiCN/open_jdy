# **2 常见问题**

### 2.1 应用场景相关

**1、商品编码可以用商品类别的编码自动生成吗？**

答：可以，详情请查看：[https://vip.kingdee.com/link/s/ZMeBa](https://vip.kingdee.com/link/s/ZMeBa)

**2、商品引入可以自动编码吗？**

答：不可以，需要自己填写好再引入。

**3、单据引入的时候可以根据系统编码规则对单据编号自动编码吗？**

答：可以，详情请查看：[https://vip.kingdee.com/link/s/ZMeVh](https://vip.kingdee.com/link/s/ZMeVh)

**4、如何设置单据编码每天从1开始更新?**

答：编码规则中要有单据日期，并且格式为年月日，同时要开启该字段的流水号依据。详情请查看：[https://vip.kingdee.com/link/s/ZMewN](https://vip.kingdee.com/link/s/ZMewN)

**5、如何设置单据编码每月从1开始更新?单据编码如何按月设置流水号？**

答：编码规则中要有单据日期，并且格式为年月，同时要开启该字段的流水号依据。详情请查看：[https://vip.kingdee.com/link/s/ZMeFF](https://vip.kingdee.com/link/s/ZMeFF)

**6、如何设置单据编码每年从1开始编号？****单据编码如何按年设置流水号？**

答：编码规则中要有单据日期，并且格式为年，同时要开启该字段的流水号依据。详情请查看：[https://vip.kingdee.com/link/s/ZMeFJ](https://vip.kingdee.com/link/s/ZMeFJ)

**7、如何修改单据编号?**

答：单据编码规则是否允许修改:①需要检查单据编码规则设置②检查单据操作控制的参数是否开启了“单据审核前是否允许修改单据编号”,如果开启了,只要单据没有审核都允许修改单据编号。详情请查看：[https://vip.kingdee.com/link/s/ZMeTJ](https://vip.kingdee.com/link/s/ZMeTJ)

### 2.2 规则设置相关

**1、编码规则中的“新增显示且允许修改/不允许修改”的应用场景是什么？如果保存后还想修改编码，该怎么处理？**

答：

- **新增显示且允许修改**

一般适用于允许子账号手动修改系统新增的编码，子账号有一定的操作余地，或是编码规则没有严格要求一致，需要根据实际业务情况进行修改的场景。

- **新增显示且不允许修改**

一般适用于不允许子账号手动修改系统新增的编码，对系统编码有严格要求，需要保证业务资料的连贯性的场景。

详情请查看：[https://vip.kingdee.com/link/s/ZMego](https://vip.kingdee.com/link/s/ZMego)

**2、单据编码规则中流水号依据和编码依据变化时重新编码有什么区别?**

答：**流水号依据:**是指单据编码中的流水号,依据哪一个字段数据变化时从起始值重新开始编码。**编码依据变化时重新编码:**指当编码使用的日期或业务字段在修改单据时发生变化将重新编码。详情请查看：[https://vip.kingdee.com/link/s/ZMewJ](https://vip.kingdee.com/link/s/ZMewJ)

**3、单据编码中间的横杠是否可以不显示?**

答：可以,需要通过单据编码规则来进行设置。详情请查看：[https://vip.kingdee.com/link/s/ZMeT3](https://vip.kingdee.com/link/s/ZMeT3)

### 2.3 流水号相关

**1、断号回收功能说明**

答：断号回收功能主要是通过设置断号检查起始值(一个整数的流水号值)，实现从设定的起始值开始递进检查单据的断号并且回收，从而在下次保存新单据时，从优先取得间断的流水号。详情请查看：[https://vip.kingdee.com/link/s/ZMeTO](https://vip.kingdee.com/link/s/ZMeTO)

**2、单据/基础资料编号的流水号不够用,该如何设置**?

答：修改对应的编码规则,把流水号长度改大,再重新保存单据或者商品即可。详情请查看：[https://vip.kingdee.com/link/s/ZMeBT](https://vip.kingdee.com/link/s/ZMeBT)

**3、新增入库单/盘盈单等入库类单据时，录入商品时，提示【当前规则bd_batchno的流水号已经用完，请重新定义规则，当前获取的流水号X，当前规则最大允许长度:X】需要如何处理**？

答：出现上述问题是由于批次的流水号已经被用完了，系统自动获取流水号时发现当前的流水号长度已经无法满足，导致无法新增批次。解决办法是修改批次的编码规则，把流水号长度改成大些，再重现选择商品即可。详情请查看：[https://vip.kingdee.com/link/s/ZfJez](https://vip.kingdee.com/link/s/ZfJez)

**4、删除单据后流水号没有回收怎么处理**?

答：方法一:开启断号补偿,断号自动回收。方法二:手动回收断号。详情请查看：[https://vip.kingdee.com/link/s/ZMe3J](https://vip.kingdee.com/link/s/ZMe3J)

**5、单据为什么会出现断号?**

答：若编码规则未启用“允许断号补偿”,删除单据,或修改单据编号后就会出现断号情况。如果想回收重新使用这部分单号,单据编码规则可以开启“允许断号补偿”。详情请查看：[https://vip.kingdee.com/link/s/ZMDmi](https://vip.kingdee.com/link/s/ZMDmi)

**5、流水号断号可以不回收吗？编码规则如何设置流水号不补空/不补号?**

答：可以， 关闭“允许断号补偿”后,删除单据后就不会再自动回收断号,新增下一张单据以后按下一个编码正常新增。

>
注意：需要升级到星辰3.1才支持关闭“允许断号补偿”的按钮,旧版2.0星辰不支持。

详情请查看：[https://vip.kingdee.com/link/s/ZMDpE](https://vip.kingdee.com/link/s/ZMDpE)

**6、商品编码规则设置了商品类别和商品品牌,并且开启流水号依据及断号补偿,为什么同一个商品类别下,不同的品牌流水号共用了?**

答： 出现这个现象通常都是因为商品编码规则中没有开启分隔符导致的。

**数据库对于设置了多个业务字段,并且都开启了流水号依据的编码,需要通过分隔符来判断编码中间的业务字段的变化**,从而再更新流水号。如果没有分隔符,当第一个业务字段不变,只变化了中间的业务字段时(对于前面问题描述的示例来说,“商品品牌”就是一个中间的字段),由于数据库无法判断出中间业务字段的变化,只能判断到第一个字段,就会认为业务字段没有变化,从而直接递增了前面的流水号,导致流水号共用了。详情请查看：

[https://vip.kingdee.com/link/s/ZZg1o](https://vip.kingdee.com/link/s/ZZg1o)

### **2.4 不生效的问题**

**1、为什么商品辅助属性(sku)的编码规则中的属性编码带不出来？**

答：一般都是因为编码规则禁用了导致的。详情请查看：[https://vip.kingdee.com/link/s/ZMePE](https://vip.kingdee.com/link/s/ZMePE)

**2、星辰3.1,批次编码规则中设置了客户,为什么采购入库单不生效?**

答：批次规则中设置的客户字段,**只对入库类单据生效,当入库类单据的单据头上有该字段才会生效,**如:其他入库单。详情请查看：[https://vip.kingdee.com/link/s/ZfJCS](https://vip.kingdee.com/link/s/ZfJCS)

**3、批次编码规则是pc+单据日期+流水号,修改单据日期后批次编码为什么没有更新?**

答：因为批次号不会回收断号，因此修改单据日期后,需要手工更新已生成的批次编码。详情查看：[https://vip.kingdee.com/link/s/ZfHAQ](https://vip.kingdee.com/link/s/ZfHAQ)

**4、为什么编码规则里开启了新增显示且不允许修改,但是新增的单据编码还能修改?**

答：单据编码审核前是否允许修改,**除了受编码规则的设置影响外,还跟“【系统设置】-【单据操作控制】”里的参数“单据审核前是否允许修改单据编号”有关**,如果这里开启了,那么只要单据没有审核,都允许修改。详情查看：[https://vip.kingdee.com/link/s/ZMejY](https://vip.kingdee.com/link/s/ZMejY)
