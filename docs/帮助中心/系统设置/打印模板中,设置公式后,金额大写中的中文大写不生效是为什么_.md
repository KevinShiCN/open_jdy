# 打印模板中,设置公式后,金额大写中的中文大写不生效是为什么?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：1084 | 有用：1

【问题描述】

打印模板中,设置公式后,金额大写中的中文大写不生效是为什么?

【业务场景】

打印模板中,单元格格式是“金额”,设置公式后,小数位数也是≤2,但是金额大写中的中文大写不生效/异常是为什么?

![](https://vip.kingdee.com/download/010924b4a046c940420899d7d9de33da7f04.png)

例如下图,270元,显示成了贰佰柒拾点零零:

![](https://vip.kingdee.com/download/010922a8053c47094be9a90303efaa6bc024.png)

【原因说明】

当打印模板设置公式时,要实现金额中文大写,需要满足几个条件:

| 序号 | 条件 | 模板设计说明 |
| --- | --- | --- |
| 1 | 公式计算出来的结果必须是纯数值,不能有文本等其他非数值字段,也不能为空 |  |
| 2 | 币别类型必须指定一个值,不能录入“不使用” |  |
| 3 | 当币别类型选择了“从单据中获取币别”时,还需要检查币别编码的正确性。如果没有选择“从单据中获取币别”,则直接查看下一个条件。 | 若单据上币别是人民币,币别编码必须是CNY或RMB若单据上币别是美元,币别编码必须是USD若单据上币别是港币,币别编码必须是HKD若用户自行修改了上述编码,就会导致金额中文大写异常,请务必保证币别编码的规范与正确性。 |
| 4 | 小数位数≤2(因为人民币最小只到角分,所有只能设置2位) |  |
