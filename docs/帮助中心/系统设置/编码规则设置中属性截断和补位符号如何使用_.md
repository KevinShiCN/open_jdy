# 编码规则设置中属性截断和补位符号如何使用?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题

【问题描述】

编码规则设置中属性截断和补位符号如何使用?

【解释说明】

编码规则取“业务字段”-“属性截断”时会出现补位符号

①“属性截断”是指取截取“编码来源”的右侧或左侧几位

②“补位符号”主要用于单据或基础资料等没有选择相关属性(编码来源)时,或所选对象的总长度不够,流水号前用什么符号来补位

如商品的编码规则第一段使用模式为“属性截断”,截取的长度是4,编码来源取“商品类别编码”,补位符号设置的0,第二段编码是长度为5的流水号

![](https://vip.kingdee.com/download/010925b076670a7c43368317f03bd22ddc71.png)
新增商品时,商品没有选择商品类别,商品编码的前4为就被0补位了
![](https://vip.kingdee.com/download/0109c84e49104dd9495bb0a7e9570f8e2ecc.png)
选择商品类别后,商品编码按编码规则设置,第一段取商品类别编码前4位

![](https://vip.kingdee.com/download/0109b51b2fa9aa2144fe937fed5354228fef.png)
若所选商品类别的编码不够4位,则剩余空位被设置的0补上

![](https://vip.kingdee.com/download/0109942964ceb46e4375a618ca14c069712c.png)
