# 工作流审核日志中的“撤销挂起”和“强制终止”有什么区别?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题
> 作者：ylim | 阅读：1129 | 有用：3

【问题描述】

工作流审核日志中的“撤销挂起”和“强制终止”有什么区别?

【解释说明】

**审核日志中的撤销挂起和强制终止仅支持在网页端操作,不支持在手机端操作。**

**撤销挂起:**当流程中有条件不能满足时,比如有审批人禁用离职了,比如库存不足不能出库等,流程就会被系统自动挂起,被挂起的流程异常信息上会显示出挂起的原因,查看具体原因调整后到审核日志中撤销挂起,流程会继续往下走。比如审批流是3级审批,流程在2级审批时被挂起了,处理完后撤销挂起流程审批节点还会在2级等待被审批。

关于被挂起的流程如何处理,可点击链接查看:[单据审批在审批环节中提示异常挂起怎么处理?](https://vip.kingdee.com/knowledge/534899751742087680?productLineId=35&isKnowledge=2&lang=zh-CN)

**强制终止:**强制终止是流程走一半了,发现有需要修改等原因需要重新调整后提交,提交人无法撤回,也不想继续往下走完了再反审核,就可以用强制终止的功能,强制终止后单据流程直接是回到的提交前的状态,可以修改后重新提交不需要了也可以删除。
