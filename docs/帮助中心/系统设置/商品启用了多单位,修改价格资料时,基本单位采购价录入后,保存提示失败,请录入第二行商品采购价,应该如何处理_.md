# 商品启用了多单位,修改价格资料时,基本单位采购价录入后,保存提示失败,请录入第二行商品采购价,应该如何处理?

> 作者：winnie_wen | 阅读：125 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/566427462276740608)

【问题描述】

商品启用了多单位,修改价格资料时,基本单位采购价录入后,保存提示失败,请录入第二行商品采购价,应该如何处理?

如图:

![](https://vip.kingdee.com/download/0109c3b3c8f294df44f2bc1769b1c0082b51.png)

【解决方案】

查询【系统参数】-【单据字段配置】-【基础资料】-【商品信息】

![](https://vip.kingdee.com/download/01094afd1e1460664f418d624e1656878e3d.png)

查询是否设置了采购价为必录项

![](https://vip.kingdee.com/download/0109d09c5baa619342d39e544a3c73b0de63.png)

关闭采购价必录项开关

![](https://vip.kingdee.com/download/01096321887c408e4fa7802b378d013f146f.png)

此时返回商品价格资料,即可保存成功。
