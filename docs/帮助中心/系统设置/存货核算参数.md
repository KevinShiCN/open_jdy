# **1 简介**

## **1.1 应用场景**

可以对系统库存核算和业务期末结账处理规则和处理惯例进行统一的设置和管理。存货核算参数一般由财务负责人或会计，根据企业商品成本核算精细度进行相关设置。

## **1.2 操作路径**

系统设置>系统参数>存货核算参数

## **1.3 适用版本**

星辰3.1以上所有版本

# 2 参数含义及应用场景

## 2.1 核算维度

用于设置系统商品出库成本核算的维度，可选择：

1） 按参数设置核算成本：选择此参数，则系统所有商品均按系统参数中设置的计价方法进行出库成本核算，不再区分商品。需要在系统参数中选择计价方法和是否需要按辅助属性、按批次进行成本核算。是一种比较松散的商品成本管理方式，适用于企业经营品类比较单一或对商品成本不进行严格管控的企业。

2）按商品设置核算成本：选择此参数，则取商品中设置的计价方法进行出库成本核算。每种商品均可以设置不同的计价方法。

## 2.2 核算方式

含义：当企业的商品存放在不同仓库时，是否需要分别核算成本。

- 分仓核算：如果企业的各仓库中相同商品的成本是不一样的，且企业希望精确管理每个不同仓库商品的成本，则建议选择此参数。使用分仓核算，在查询与库存成本相关的报表时，建议按仓库分别查询。
- 总仓核算：如果企业的各仓库中相同商品的成本均一致，或者企业商品成本差异不大，企业不需要精确核算各仓库成本，则建议选择此参数。使用总仓核算，在查询与库存成本相关的报表时，建议使用【库存横向查询表】查询所有仓库数据。
- 调拨业务复杂程度：在分仓核算时影响异价调拨时的成本核算，如商品在不同仓库被循环调拨则需要选择调拨频繁（循环调拨举例：从仓库A调拨到仓库B，仓库B调拨到仓库C，仓库C调拨到仓库A）

相关知识：[总仓核算与分仓核算的说明](https://vip.kingdee.com/link/s/MEpJZ)

## 2.3 成本来源

应用场景：用于设置当企业业务出现负结存出库、发生无源单的销售退货业务、有拆卸单时子件入库成本等的取数规则。

>
说明：点击成本来源后方的小问号，可以查看相关的取数说明。下方成本来源勾选后，按顺序由系统在进行存货出库成本核算时选取并写入单据的成本对应字段。

![](https://vip.kingdee.com/download/0100ae362e90cec64bdda2d6f62977e6bdf2.png)

参数“上次出入库成本支持跨月取数”，勾选则支持，不勾选则不支持。

## 2.4 参数设置

包括如下影响业务期末结账和出库成本核算的参数：

![上传图片](https://vip.kingdee.com/download/0100925cb11bf7a8405498aad918c65b6c58.png)

1）业务系统单独结账，且总账无关联：在业务和财务期末结账时，不进行对方业务期间的检查和判断，均可以独立结账。如果企业不进行业务财务强关联应用，建议选择此参数。

2）业务系统单独结账，且必须早于总账结账：在财务进行期末结账时，会检查业务系统的当前期间，如果业务系统当前期间未结账，则财务不允许进行结账，必须先进行业务系统结账。

3）财务结账前检查未生成凭证的业务单据：用于财务结账时，检查对应期间业务系统的业务单据是否已生成凭证，如果勾选此参数，则在财务期末结账时会提示对应未生成凭证的单据信息。

4）出库核算自动进行期末余额尾差调整：在出库成本核算时，如果商品库存结存数量为0，而结存金额还有且在设置的范围内时，系统会自动剩余的金额计入到最后一笔出库业务成本中。

5）先进先出法同商品同单价合并为一条记录（成本计算表结果显示）

>
说明：用于控制先进先出法成本计算表结果显示：待发结存序列，按时间排序，依次汇总成本单价相同的序列，当中间出现不一致单价时，再行汇总。

6）采购入库、调拨入库、组装拆卸单与费用分摊单在同一核算时间范围内时，按分摊后总成本计算

>
说明：用于勾选该参数后，存货出库核算只支持核算本期成本，不支持设置核算以后期间范围；满足条件的费用分摊单，不参与成本计算，直接按分摊后总成本计算。
