# 工作流操作手册

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：进阶技巧, 功能介绍

## 1 简介

### 1.1应用场景

作为用户，想要对业务进行多级审批确认，以确保决策科学规范，例如公司的采购人员发起了采购申请单，需要主管和财务进行确认后才能下推采购订单。

### 1.2适用版本

星辰专业版/旗舰版

### 1.3 操作路径

系统设置>工作流

### 1.4功能特性

• 支持库存/采购/销售/应收应付/生产/委外单据

• 支持的单据及审批条件，请点击：[https://vip.kingdee.com/link/s/Mne8M](https://vip.kingdee.com/link/s/Mne8M)

• 支持按条件审批

• 支持提交人与审批人相同时自动审批

• 支持节点跳过功能

• 支持在审批时上传流程附件

• 支持手工指定下一级审批人

• 支持抄送、支持按分录字段设置审批条件，请点击：[https://vip.kingdee.com/link/s/l87GX](https://vip.kingdee.com/link/s/l87GX)

## 2 应用流程

第一步：在系统设置>工作流>流程设计中“新增”流程，成功发布后即生效

第二步：若需要修改已发布流程，可到系统设置>工作流>流程管理中修改，修改后保存即可

第三步：若需要处理待办任务或者查看在办申请等进度，可到系统设置>工作流>任务中心处理

第四步：若单据审批过程中出现异常（如挂起），可到系统设置>工作流>审核日志查看异常信息进行处理。

## 3 操作指南

### 3.1 流程设计

#### 3.1.1 基本介绍

• 含义：即新增工作流流程方案的地方。

• 列表按钮说明：

| 按钮名称 | 作用 |
| --- | --- |
| 删除 | 未发布过且保存状态下的流程允许删除，流程一经发布则无法删除，可以废弃 |
| 废弃 | 已经发布的流程不再使用 |
| 恢复 | 已经废弃的流程恢复正常可使用状态 |
| 发布 | 发布成功后，完成设计的工作流可以正常使用，没有发布的则不可使用。• 新发布的流程默认启用；• 一个单据类型可以发布多个流程，但只有一个流程是启用状态（是否启用可到流程管理中查看），• 新流程发布时，若【流程管理】已存在启用状态的流程，会将处于启用状态的流程默认禁用掉，因为一个单据类型只能默认一个启用状态的流程； |

• 发布前提条件：

发布工作流的单据类型，需保证当前单据都已审核，存在未审核的单据则流程发布不成功，可先保存工作流，待单据都已审核后再发布即可；

#### 3.1.2 新增页面介绍

![](https://vip.kingdee.com/download/0100e43b3e2731cc4e19bce4e49912462224.png)

| 按钮名称 | 填写说明 |
| --- | --- |
| 单据 | 支持库存/采购/销售/应收应付/生产/委外单据 |
| 名称 | 一个单据类型可以发布多个流程，但注意名称不能相同 |
| 编码 | 系统会自动编码，允许手工修改。流程编码可以使用字母、数字、下划线的组合方式，且必须以字母开头。新建向导完成后，流程编码不能再修改。 |
| 审核级数 | 最少1级，最多10级。 |
| 会审设置 | 支持会审和普通审批两种模式：• 会审，流程中某一节点上的审批人若为多人审批时，必须全部同意当前节点才审核通过，否则不通过；• 普通审批，若某一审批节点有需多人审核，任一人同意即可审核通过； |

#### 3.1.3 工作流设计器

**A.**画布注意事项

（1）开始，提交人，结束三个节点必须有

（2）节点之间必须用箭头线有效连接

（3）鼠标点击节点选中时，支持在页面右侧设置节点属性和名称，参与人等

（4）可通过删除节点重新选择审批和会审节点修改审批模式

温馨提示：在流程设计列表新增流程户后，工作流设计器内会自动生成对应的流程图，这里一般不建议用户再手动修改流程图的内容，例如删除流程线路、增删改节点等等，因为目前底层对工作流的校验很严格，通常手工修改的流程图容易导致异常问题。

**B.**全局设置

![](https://vip.kingdee.com/download/01007d8fef1162cd47948ce351073d00d65a.png)

（1） 展示了当前单据的类型、工作流名称

（2） 驳回设置：默认驳回到提交人节点，支持两种不同的驳回场景：

- 场景一：驳回修改后重新按流程走，即单据若被参与人驳回，重新提交后仍需按原流程一级一级审批通过；

       举例：若设置了四级审批，第四级审批人驳回后参与人重新提交，该单据需要重新走流程，一二三四级审批人要再次一级一级重新审批；

- 场景二：若选择直接回到原审批驳回的节点，则被驳回的单据修改后重新提交，审批节点会跳过中间节点，直接回到操作驳回的参与人节点；

       举例：若设置了四级审批，第四级审批人驳回，参与人重新提交后则直接回到第四级审批人处理，一二三节点审批人无需审批。

（3） 流程控制

• 提交人与审批人相同时自动审批：开启后，审批人对自己提交的流程自动审批通过。

• 流程挂起时通知提交人

### 3.2 流程管理

• 含义：即用来管理所有发布过的流程。只可以对流程节点属性即流程名称，参与人等信息进行修改，不能增删节点，保存即生效。

• 启用：表示当前单据类型所匹配到正在使用的工作流方案，同一个单据类型只能有一个“启用”的流程。

• 禁用：即不再使用“启用”状态的流程。

①禁用前提：该单据类型下，没有单据还没走完流程的，如果有流程实例正在运行中，那么请处理完成后再禁用流程。

②如何关闭单据的工作流？

答：当同一个单据类型下，所有的流程都禁用后,该单据类型上将不会再出现“提交”、“提交并新增”这些字段；

③已禁用流程对单据的影响：若有单据命中该已被禁用的流程，则继续按照该流程走，新提交的单据按新流程走，所以，流程管理的一部分作用可以理解为是用来保证已经在禁用流程中的业务单据继续走完；

• 版本号：流程在【流程设计】中每发布一次，版本号+1；

比如，流程：采购订单审批流程，第一次发布版本号默认为001，第二次则默认为002，第三次为003，方便在流程异常时按照流程名称和版本号查看并修改该单据命中的流程；

![](https://vip.kingdee.com/download/010020cceb9ef1284b89a8b977bbe0a022b6.png)

• 节点的维护

注：不能增删节点，只能对已发布流程修改属性等，所以无左侧审批节点，如下图：

保存即可生效，不是新发布设计方案，所以也没有发布按钮（可参考流程设计中截图看差异，便于理解）

### 3.4 审核日志

注意：只展示流程中和挂起的任务

异常流程处理方法：

（1）鼠标放置于异常信息处，即会展示出异常提醒；

（2）若是因为流程原因导致异常，可根据流程编码和版本号去【流程管理】中查找该流程并修改，修改后回到当前页面点击左上角撤销挂起，之后，流程即可恢复正常，已禁用的流程仍支持修改；

### 3.5 工作流发布后单据处理流程

（1）流程发布成功后，单据会新增“提交”和“提交并新增”按钮，单据头也会有“审批进度”字段，如图：

![](https://vip.kingdee.com/download/0109d443a0a2e1f44615bd5eb28b46220490.png)

（2）录单后点击“保存”后“提交”或者直接点击“提交”，该单据即可命中【流程管理】中处于启用状态的流程，提交成功的话，“审批进度”会变成“处理中。

已经提交的单据，若下一级参与人未处理，则可撤回，若已处理则不可撤回。

![](https://vip.kingdee.com/download/01092f62ebad561f4d6bacf77b2275aca663.png)

### 3.6 其他说明

1）允许审核后修改的字段在流程审批完以后仍然可以修改，包括限价，负库存校验等都保持与原逻辑保持一致，在最后一个审批人处校验。可以理解为，工作流就是在提交和审核中间加几个节点的确认，不会改变原生效节点。

2）系统【重新初始化】不会清空流程设计和流程管理，会清空任务中心和审核日志
