# 编码规则中的“新增显示且允许修改/不允许修改”的应用场景是什么？如果保存后还想修改编码，该怎么处理？

## 1 应用场景

### • 新增显示且允许修改

一般适用于允许子账号手动修改系统新增的编码，子账号有一定的操作余地，或是编码规则没有严格要求一致，需要根据实际业务情况进行修改的场景。

### • 新增显示且不允许修改

一般适用于不允许子账号手动修改系统新增的编码，对系统编码有严格要求，需要保证业务资料的连贯性的场景。

## 2 使用说明

### 2.1 新增显示且允许修改

此规则下，单据保存前，允许手工修改单据编号，但是保存后则不允许修改了。

举例：销售出库单编码规则如下，

![](https://vip.kingdee.com/download/0109ffa1292493754f5f9bbde18ab96b753e.png)

那么新增单据后，没保存前，单据编码是允许编辑的：

![](https://vip.kingdee.com/download/01090637b78dbf7240dcb5ff41dbf92c865a.png)

保存后就不允许编辑了（因为后台的流水号已经被这个单据占用了，还没有释放）：

![](https://vip.kingdee.com/download/010933fdd0f9cd8740078200931664495dd6.png)

### 2.2 新增显示且不允许修改

举例：商品编码编码规则如下，

![](https://vip.kingdee.com/download/01094c9dbe6766a04be2b971d1705fd28f31.png)

新建商品时，商品编码显示出来但是不允许修改编码，鼠标会是不可编辑的状态（如下图）：

![](https://vip.kingdee.com/download/010963c3a7b781a84f2fa9f318c36c766ef9.png)

### 2.3 保存后如何修改（利用单据操作控制中“单据审核前允许修改单据编号”）

如果单据/基础资料已经保存了，但是存在需要修改单据编号/基础资料编码的情形，这个时候该怎么办呢？

我们可以通过单据操作控制来实现。下面进行说明：

在系统设置>单据操作控制中，针对具体的单据类型开启“单据审核前允许修改单据编号”并保存，

![](https://vip.kingdee.com/download/0109d13d34a0d09a470db2bec4379f99d86a.png)

那么：

- 编码规则中设置了“新增显示且允许修改”，即使单据/基础资料已经保存了，只要未审核，就能随时修改单据编号/基础资料编码
- 编码规则中设置了“新增显示且不允许修改”，即使是新增的单据/基础资料，也能随时修改单据编号/基础资料编码
