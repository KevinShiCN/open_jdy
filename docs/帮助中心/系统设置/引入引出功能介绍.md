# 引入引出功能介绍

# 1 简介

## 1.1 应用场景

在系统上线或批量开单时，通过填写引入模板，可快速批量生成基础资料或单据；也可指定引出模板样式，实现数据的批量导出。

引入引出模板是系统中所有支持引入引出功能的模板列表。可在此直接为单据新增引入/引出模板，再前往对应单据列表使用。引入引出模板支持复制、删除（预置模板不允许删除）。

![](https://vip.kingdee.com/download/0100169780ece58b4368b6e1abc8daddc510.png)

## 1.2 适用版本

星辰3.1以上所有版本

## 1.3 功能说明

**新增**：在引入引出模板列表选择具体单据，点击“新增”；在模板类型中选择“引入模板”或“引出模板”，填写编码、名称；对需引入的字段开启“是否引入”，对引入过程中必须填写的项开启“是否必录”。针对基础资料，可将引入属性设为“编码”或“名称”，决定引入时以“编码”还是“名称”唯一识别基础资料。

![](https://vip.kingdee.com/download/0100038790dae4b44d9d90f9a217bc996853.png)

**下载**：可选择具体引入/引出模板下载；将数据填入下载的引入模板后，即可在对应单据列表直接引入。

**设为默认**：选择模板并设为默认模板后，引入、引出时该模板将显示为默认模板，引入界面下载的默认模板即为该模板。

**一键下载所有默认模板**：支持一键下载所有默认模板，便于集中整理数据后，再到各基础资料/单据列表直接引入。

**权限控制**：引入引出模板默认所有人拥有查询权限；系统仅额外提供新增、修改、删除权限。

![](https://vip.kingdee.com/download/0100043bd20a9009463682fe08db073a2ea6.png)

# 2 **引入**应用流程

1）先1）在“引入>模板管理”中，选择要引入的预置模板

![](https://vip.kingdee.com/download/0100bc9835b3375e42988f6e6b5f2fc0bedc.png)

选中后，点击下载即可下载到本地

![](https://vip.kingdee.com/download/01000fc7f35bf1be4584a924a3d709a55f08.png)

如预置模板无法满足需求，用户也可新增自定义模板

![](https://vip.kingdee.com/download/01004bb1a4da66374f378e7b2e4c12e36eea.png)

新增后，按照想要引入的字段进行设置，填写编码、名称等必录字段后，点击保存即可。用户可以自定义是否要作为默认模板。

![](https://vip.kingdee.com/download/010058e6ea6cb9294fc1b5790391f7df8686.png)

注意：对于商品等字段，存在“编码”和“名称”两种属性；用户可在引入属性/引出属性中设置按名称或编码导入。**引入属性仅支持单选，引出属性支持多选**。

![](https://vip.kingdee.com/download/010013a0a9f0144c45f6b95b264adeb17655.png)

2）填写引入模板时，带星号的必录字段必须填写

![](https://vip.kingdee.com/download/01009bf5cb16e8fd49dc9e0f933c3e8a9244.png)

如不清楚如何填写，可下载引入弹窗中的“引入填写示例”查看

![](https://vip.kingdee.com/download/0100e0adc18de5e6484a99e9debc51f3f4c7.png)

3）填写完毕后，点击“引入”，上传或直接拖拽文件至引入弹窗；待文件上传成功后，点击“开始引入”即可。

![](https://vip.kingdee.com/download/0100141d359cc9d24bc196e343cb9717f343.png)

4）若引入成功，系统会提示成功；点击“我知道了”关闭窗口即可。

![](https://vip.kingdee.com/download/010086961d59825f4ccd8512528e1ad45652.png)

5）若引入失败，点击“查看详情”可下载包含失败原因的文件。

![](https://vip.kingdee.com/download/01009215c7ab9f6c4a618c499663fbc24d17.png)

根据报错原因修改后，重新引入即可。

![](https://vip.kingdee.com/download/01008273b87464ec45d5af1a8f4d3fe304c5.png)

6）注意：当引入文件过大时，系统将在后台继续上传，可在“引入结果查询”中查看后续引入结果。

![](https://vip.kingdee.com/download/0100381ad42dd82e4ab2a7240725cb7ff87c.png)

若引入失败，也可直接下载失败文件。

![](https://vip.kingdee.com/download/010049aee93b396547d3abf9f22a7a1eb588.png)

# 3 **引出应用流程**

## **3.1 引出方式**

**引出支持以下三种方式：**

![](https://vip.kingdee.com/download/0100f494e2a899f0446eae860cd6ff760bca.png)

• **按列表引出**：所见即所得，导出字段与列表当前列设置一致。

• **按引入模板引出**：按引入模板中的样式导出

• **引出分析数据**：与“按引入模板引出”类似，但会将**单据头字段**逐行填充，便于后续数据分析。

## **3.2 引出结果查询**

操作方式：

1） 先选择要引出的数据

2） 点击“引出”，选择引出方式

![](https://vip.kingdee.com/download/0100101ea7fc91fe4ab3add16d4877cb248e.png)

3） 当引出数据量较小时，浏览器将直接下载到本地（具体下载路径取决于用户浏览器的默认下载设置，星辰系统不做配置）。

![](https://vip.kingdee.com/download/01003e5581d533ad4d989d6753cb1040d7a8.png)

![](https://vip.kingdee.com/download/01005e6255d7e50d4f6abb332c7f68c49c6f.png)

4） 当引出数据较大时（当一次引出数据超过500行），系统会自动在后台继续下载，可以在“引出结果查询”中查看后续的引出结果。跟引入是一样的逻辑。

# 4 引入/引出注意事项

- 引入/引出结果查询仅保存最近7天的记录，请及时下载引出文件。
- 引入文件上传成功后，数据库考虑到存储成本及容量问题，不会保存引入文件，因此如需追溯历史过往的引入文件，请务必在本地做好存档
- 商品资料中的商品编码是必录的，不能通过引入自动生成
- 单据的单据编号可以不录入，可以通过设置编码规则后，引入自动生成
- 引入/引出未做权限控制，任何账号均可操作。
- 目前基础资料引入支持“覆盖新增”（即批量更新），单据引入暂不支持。
