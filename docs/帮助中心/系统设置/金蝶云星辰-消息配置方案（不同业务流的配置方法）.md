# 金蝶云星辰-消息配置方案（不同业务流的配置方法）

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：应用案例, 新特性, 进阶技巧
> 作者：移动生产白建凡 | 阅读：896 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/531176147233833984)

### 1. 简介

【消息中心+审批流】的灵活配置提升企业内部协同，提高团队协作和沟通效率。

### 2. 配置方案

不同公司的业务流程存在差异性，下面是总结了几种常见的业务提供了配置的建议，可在这基础上修改来配置一套最符合自己公司实际业务的流程。

配置路径：
审核：系统设置—>消息和预警—>消息设置—>单据消息：通知对象

工作流：系统设置—>工作流—>流程设计：参与人

### 2.1以销订购的销售模式的配置方案

主要的销售流程可以根据自己的业务来决定的，以下是作为参考：

![image.png](https://vip.kingdee.com/download/01093922697a98ef4176bc814c702d4778b3.png)

1) 销售订单（业务员—>主管—>采购）

保存消息：开启，通知对象为部门负责人/【主管账号】

审核消息：开启，通知对象为【采购账号】

消息内容根据业务需要自定义



2) 采购订单（采购—>财务—>老板—>采购）

保存消息：开启，通知对象【财务账号】（也可以用审批流，一级通知财务，二级通知老板，审批时指定下一节点审批人）

审核消息：开启，通知对象【采购账号】

消息内容根据业务需要自定义



3) 采购入库（采购—>仓管—>业务员）

保存消息：开启，通知对象【采购、仓管账号】（也可以用审批流，一级通知财务，二级通知老板，审批时指定下一节点审批人）

审核消息：开启，通知对象【业务员账号】

消息内容根据业务需要自定义



4) 销售出库（业务员—>仓管—>配送）

保存消息：开启，通知对象【仓管账号】（也可以用审批流，一级通知主管，二级通知经理，审批时指定下一节点审批人）

审核消息：开启，通知对象【业务员、配送账号】

消息内容根据业务需要自定义



5) 收款单（业务员—>财务—>业务员）

保存消息：开启，通知对象【财务账号】（也可以用审批流，一级通知主管，二级通知财务，审批时指定下一节点审批人）

审核消息：开启，通知对象【业务员账号】

消息内容根据业务需要自定义
