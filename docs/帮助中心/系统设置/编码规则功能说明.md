# 编码规则功能说明

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：快速入门, 精华汇总, 功能介绍
> 作者：进销存赵嘉丽 | 阅读：4282 | 有用：5
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/23604)

## 1 应用场景

企业在经营过程中可能有这样的需求，如业务单据/基础资料编号自动编码，业务单据需要从单据编号中体现出业务发生的部门、业务员、日期等。系统为基础资料和业务单据预置了一条编码规则。若预置的编码规则与实际使用的规则存在差异，需要调整编码规则时，可以点击基础资料-编码规则针对需要调整的规则进行修改。

## 2 应用流程

点击系统设置<个性化设置<编码规则，单击具体单据类型的编码规则进入到编码规则编辑界面，设置规则后，保存即可生效。如果不需要使用该编码规则，回到编码规则列表，对其禁用即可。

## 3 功能说明

### 3.1 编码示例

根据编码分段展示当前规则的示例。

### 3.2 基础设置

#### 3.2.1 单据类型

即显示当前编码规则适用的单据或者基础资料。

#### 3.2.2 编码分段

指编码由几部分组成，基础资料的编码分段可选1-3段，单据的编码分段可选2-4段。

#### 3.2.3 段间分隔符

默认短横线，可选无，#，短横线。

>
注意：如果编码规则中包含了2段启用了流水号依据的字段的话，分隔符必须设置，不选选择“无”。

详见知识：[商品编码规则设置了商品类别和商品品牌,并且开启流水号依据及断号补偿,为什么同一个商品类别下,不同的品牌流水号共用了?](https://vip.kingdee.com/link/s/ZZg1o)

指编码由几部分组成，基础资料的编码分段可选1-3段，单据的编码分段可选2-4段。

#### 3.2.4 应用规则

默认新增显示且允许修改，可选新增显示且允许修改、新增显示且不允许修改。

- 新增显示且允许修改：以商品编码为例，在新建商品时，商品编码显示出来且允许修改编码。修改后下次新增商品时扔按编码规则继续编码。
- 新增显示且不允许修改：以商品编码为例，在新建商品时，商品编码显示出来但是不允许修改编码；

### 3.3 编码依据变化时重新编码

若开启，当编码使用的日期、业务字段发生变化时将重新编码。

举例说明：如某单据使用的编码规则为“常量-日期-流水号”，新增单据时单据日期由4月1日修改为3月30日，则对应单据编码会由**20190401***自动重新编码为**20190330***

### 3.4 允许断号补偿

默认不开启。开启后，删除单据引起的断号会自动回收并重新参与编码。若关闭后重新开启，请先操作断号回收，否则不会对参数关闭期间产生的断号进行补偿。

使用说明：例如有单据001,002,003，然后删除002后：

- 不开启，再新增单据时：流水号会递增变成004
- 开启，再新增单据时：流水号会重新获取002

### 3.5 编码设置规则说明

| 内容 | 含义 | 举例说明 |
| --- | --- | --- |
| 常量 | 固定文本串，一般为字母，也可以录入数字、中文等 | 如：CGDD、XSDD、PC |
| 单据日期 | 仅适用于单据类编码规则。即单据头的单据日期字段 | 直接取单据上的值、支持设置日期格式假设今日日期为2019-04-30，yyyymmdd格式即为20190430，yyyymm格式为201904，mmdd格式为0430。 |
| 系统日期 | 指服务器日期，不允许修改，若使用系统日期参与编码，注意修改历史单据会更新系统日期 |  |
| 流水号 | 需设置流水起始值和长度 | 假设流水号长度是5，从1开始编码。那么可以编码的范围就是从00001至99999。 |
| 业务字段 | 依据不同的单据类型可选不同的内容，一般与单据本身有关，如单据上部门的编码、单据业务员的编码等，具体从编码来源中选择。完全取值：将编码来源中内容全部取用属性截断：将编码来源中内容截取部分（与设定的长度有关）取用补位字符：当编码来源中取用的内容不够长度时用补位字符补位补位：如左侧，指不够长度时用补位字符补到左侧截取：如左侧，指超出长度时截掉左侧的内容 | 假设有业务字段ABC：完全取值时，即取ABC作为业务字段这一编码段；属性截断时：若长度设为2（小于ABC的长度3时），截取左侧，则取AB作为业务字段这一编码段；若长度设为4（大于ABC的长度3时），补位左侧，补位字符0，则取0ABC作为业务字段这一编码段。 |
| 流水号依据 | 指流水号编号的依据。若日期设置为流水号依据，则日期不同时流水号重新编号。 | 假设某编码规则为“XSDD-日期yyyymmdd-流水号（3位）”，今天日期为2019-04-01，开启了日期的流水号依据，那么流水号从001开始编号，则01日的单据编号从XSDD-20190401-001、XSDD-20190401-002、…、XSDD-20190401-099这样编号,02日的单据编号从XSDD-20190402-001开始编号。若日期没开启流水号依据，且没有设置其他的流水号依据，则04/02的单据编号从04/01最后的流水号接着编号，假设04/01号最后的流水号是023，那么04/02第一张单就是即从XSDD-20190402-024开始编号。 |

### 3.5 断号回收

主要是通过设置断号检查起始值(一个整数的流水号值)，实现从设定的起始值开始递进检查单据的断号并且回收，从而在下次保存新单据时，从优先取得间断的流水号。知识详情：[https://vip.kingdee.com/link/s/MUORp](https://vip.kingdee.com/link/s/MUORp)
