# 流程设计中参与人条件“属于集合”“不属于集合”是什么意思?如何使用?

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：常见问题

【问题描述】

流程设计中参与人条件“属于集合”“不属于集合”是什么意思?如何使用?

【情况说明】

流程设计中参与人条件“属于集合”和“不属于集合”是用于某个人员属于或不属于指定的多个人员其中一个人时,则此条件成立。

例如:需要设置当领料人为A、B和C其中一员的情况下,由X审核。则条件设置为,<参数>选择“领料人”,<条件>选择“属于集合”,<值>选择“A”、“B”和“C”。此时如果单据上的领料人为A、B和C其中一人时则此条件成立。反之,需要设置当领料人不为A、B和C其中一员的情况下,由Y审核。则条件设置为,<参数>选择“领料人”,<条件>选择“不属于集合”,<值>选择“A”、“B”和“C”。此时如果单据上的领料人不为A、B和C其中一人时则此条件成立。
