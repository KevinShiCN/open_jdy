# 系统设置

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：功能介绍, 快速入门
> 作者：Cat_Chow | 阅读：7535 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/637657857575508736)

系统设置是整个系统的基础设置，设置会影响后续业务的流程。系统设置需要结合企业实际业务流程和管理需求进行设置。

![](https://vip.kingdee.com/download/01098a33e5b13ff64d3a970c0364d92a6465.png)

| 步骤 | 具体说明 | 学习链接 |

| 1.基础设置 |  | —— |
| 1.1系统参数 | 公共参数、财务参数、资产管理、出纳管理、进销存云等模块会有各自启用参数设置，可参考对应的新手攻略学习了解。本节主要介账进销存模块的启用参数。 | 点击学习 |
| 1.1.1基础资料参数 | 即商品参数，商品统一参数设置，如商品默认税率、特性商品维护等。 | 点击学习 |
| 1.1.2进销存参数 | 对进销存系统的业务处理规则和处理惯例进行统一的设置和管理。 | 点击学习 |
| 1.1.3存货核算参数 | 成本核算方式及核算维护设置，特殊业务成本取数规则，可用库存的计算公式等 | 点击学习 |
| 1.1.4应收应付参数 | 往来核销方式：按单据核销、按分录核销、按余额核销。信息额度设置及账龄管理参数。 | 点击学习 |
| 1.2用户管理 | (1)账套用户管理，如新增用户、同步工作台用户、用户禁用/删除、用户分组管理、管理员设置等 | 点击学习 |
| (2)账套权限管理：单独授权、用户组授权、复制权限；其中权限分为：功能授权、字段授权、数据授权。 | 点击学习 |  |
| 1.3操作日志 | 用于记录用户使用系统的操作行为日志记录。通过操作日志，可以查询到系统每个操作员使用软件的记录 | 点击学习 |
| 1.4重新初始化 | 企业在使用系统之后，可以根据需要对系统进行重新初始化。重新初始化有三种方式可以选择:(1)单据: 清空所有单据,保留基础资料、期初、设置;(2)单据+期初: 清空所有单据、期初,保留基础资料、设置;(3)全部数据: 清空所有单据、期初、基础资料、设置、并重建账套。注意：重新初始化操作，数据将被删除无法恢复,请慎用。建议操作前先备份或是导出EXCEL存档。 | 点击学习 |
| 1.5网络控制 | 系统为最大限度的保证网络并发控制和数据一致性而设的网络控制。在用户出现异常中断、死机等原因可能造成单据锁定，会导致其他用户不能执行互斥功能。另外，如果是浏览器关闭或是关机，产品是捕捉不到退出信息的无法解锁。所以如果关闭浏览器或者直接关机时正打开着单据，即使这个人关机了，别人再操作这个单据仍会提示有人编辑着。 | 点击学习 |
| 1.6帐套备份与恢复 | 系统独具有云备份功能,只要不主动删除数据,数据一般不会丢失,也无需定期备份数据。不过，系统也提供手动备份功能，支持管理员手动备份当前账套数据，也支持恢复备份数据至原账套下。说明：仅服务管理员和账套管理员有账套备份与恢复的权限。 | 点击学习 |
| 2.个性化设置 | 企业可以根据实际情况做一些自定义的配置，包括：编码规则、打印模板、单据自定义、基础资料自定义、统一列设置等等 | —— |
| 2.1编码规则 | 系统为基础资料和业务单据预置了一条编码规则。若预置的编码规则与实际使用的规则存在差异，可自定义编码规则。 | 点击学习 |
| 2.2打印模板设计 |  | —— |
| 2.2.1打印模板 | 系统已预置多样式的打印模版，也支持自定义单据、业务报表的打印模版 | 点击学习 |
| 2.2.2打印控制 | 单据需要审核后才能打印单据，避免保存状态打印了单据又修改单据后导致系统单据与打印出来的单据不一致的情况；单据打印需要控制打印次数，避免单据多次打印造成重复发货。 | 点击学习 |
| 2.3引入引出模板 | 系统预置的所有引入引出模板，可统一进行模板管理及自定义引入引出模板 | 点击学习 |
| 2.4自定义字段 |  | —— |
| 2.4.1基础资料新增字段 | 通过对基础资料进行自定义，满足企业业务需求。适用基础资料有供应商、客户、商品。可为引用文本、数值、日期或引用辅助资料。标准版：最多可自定义8个字段专业版、旗舰版：最多可自定义20个字段 | 点击学习 |
| 2.4.2单据新增字段 | 若系统中单据字段无法满足业务需求，可通过单据新增字段增加单据自定义字段，适用单据：销售、采购、仓存、应收应付四类单据。 | 点击学习 |
| 2.5单据字段配置 | 目前财务云支持对出纳管理的日记账和序时簿单据的部门、业务员、结算方式、备注字段进行设置是否为必录项 | 点击学习 |
| 2.6单据操作控制 | 业务场景：(1)进销存云单据的审核人与制单人为同一人设置(2)反审核人与审核人只能为同一人设置 (3)只能修改、删除本人录入的单据设置 (4)单据审核前允许修改单据编号设置 | 点击学习 |
| 2.7智能选仓 | 场景一：企业有多地仓库，比如上海仓、广州仓，上海仓负责向华东地区的客户发货，广州仓负责向华南地区的客户发货。通过智能选仓功能，在销售订单、销售出库单中选择客户自动携带出收货地址后，系统根据收货地址自动匹配不同的仓库进行出库。场景二：企业有多地仓库，且每个仓库按仓位管理，每个商品有固定存放的仓位，通过智能选仓，在匹配到发货仓库后，系统可以计算有库存的仓位进行自动匹配，减少人工选择，提高发货效率。说明：该功能专业版及旗舰版均支持，标准版不支持。 | 点击学习 |
| 2.8统一列设置 | 对系统业务单据的列设置进行统一管理，包括列显示、列隐藏、列表顺序调整（上移、下移）、冻结列、取消列冻结等，列设置统一调整后，所有子账号查看业务单据时均生效，无需子账号另外进行列设置。 | 点击学习 |
| 2.9业务流程设置 | 按公司内部业务流程设置系统单据的流转流程,让上下游部门更明确操作路径,部门协同更高效。说明：该功能专业版及旗舰版均支持，标准版不支持。 | 点击学习 |
| 3.企业互联 | 企业互联可以实现，企业上下游互联，实现人、数据、流程、和事物的联通，从而提高生产力，并推动企业可持续的差异化竞争。加入企业B是某品牌的总代，企业A是企业B的一级经销商，企业A需要向企业B采购一批商品，在其软件系统中下达采购订单，可直接发送到企业B的系统中，形成企业B的销售订单。   | 点击学习 |
| 3.1对接密钥 | 对接密钥是企业互联的基础。对接密钥:就是账套 id,用于系统识别账套唯一标识。系统自动生成,直接复制将本企业的对接密钥发给对方企业。参与互联的企业都必须操作。注意：企业所属的角色,将正确密钥提供对方。 | 点击学习 |
| 3.2互联客户 | 互联客户是企业互联的一部分。互联客户就是我公司销售的下游公司。 | 点击学习 |
| 3.3互联供应商 | 互联供应商是企业互联的一部分。互联供应商就是我公司采购的上游公司。互联供应商如何操作链接更新中 | 点击学习 |
| 3.4外部商品库 | 企业互联在发送单据时,需要根据客户商品与星辰商品的对应关系,转换商品生成单据 | 点击学习 |
| 4.工作流 | 单据审批流程设计， 想要对业务进行多级审批确认,以确保决策科学规范,例如公司的采购人员发起了采购申请单,需要主管和财务进行确认后才能下推采购订单。说明：该功能专业版及旗舰版均支持，标准版不支持。 | —— |
| 4.1流程设计 | 使用审批流程前，先设计流程节点，添加流程参与人及审批条件等。 | 点击学习 |
| 4.2流程管理 | 来管理所有发布过的流程。只可以对流程节点属性即流程名称，参与人等信息进行修改，不能增删节点，保存即生效。 | 点击学习 |
| 4.3任务中心 | 查看我提交的任务的进度和需要我处理的任务。   | 点击学习 |
| 4.5审核日志 | 展示处理在流程中的任务和被挂起的任务   | 点击学习 |
| 5.消息和预警 |  | —— |
| 5.1消息设置 | 金蝶云星辰的消息中心是为不同岗位的用户提供各类操作支持和业务提示,实现业务流程的畅通和协同,提升企业运营效率。功能包含：后台设置、后台消息中心列表、web端消息通知、app消息通知、钉钉消息通知、企微消息通知。业务涵盖：单据消息提醒（增删审核）、经营消息、预警通知等。 | 点击学习 |
| 5.2消息中心 | 查看历史消息记录 | 点击学习 |
| 5.3预警设置 | 预警设置有商品预警 、价格预警 、客户预警 、仓存预警 、交期预警 、信用预警、 账龄预警。提示管理者针对即将、正在、或者已经发生的情况采取相应的措施，进行相关的处理。通过预警可以保证企业计划执行的有效性，加强管理控制职能。   | 点击学习 |
