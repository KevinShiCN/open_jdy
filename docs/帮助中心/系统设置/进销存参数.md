# 进销存参数

> 产品：金蝶云·星辰 | 分类：系统设置 | 类型：快速入门, 精华汇总, 功能介绍
> 作者：进销存赵嘉丽 | 阅读：2959 | 有用：8
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/464146540966416640)

# **1 简介**

## **1.1 应用场景**

可以对进销存系统的业务处理规则和处理惯例进行统一的设置和管理。

## **1.2 操作路径**

系统设置>系统参数>进销存参数

## **1.3 适用版本**

不同版本的参数存在一些差异，请以具体版本内展示的参数为准。

# 2 参数含义及应用场景

## **2.1 全局参数**

1）**启用期间：**进销存模块的系统启用期间，默认与建账时设置的系统启用期间一致。刚启用的账套，如果没有录入任何数据，可以修改启用期间；录入数据但未结过账，如果需要再修改，系统会给出对应提示，需要将初始余额数据进行调整，这里请根据实际业务情况进行判断是删除还是调整到新的启用期间。

2）**当前期间**：当前业务所在期间，仅显示，不允许修改，由系统结账情况决定。

3）**启用外币**：若企业有外币业务，则需要勾选此参数，设置后业务系统所有业务单据和报表均显示币别、汇率以及外币和本位币所有字段。

4）**小数位精度控制**

- 系统单价、金额小数位：根据不同的币别分别控制，请在币别中设置对应币别的小位数信息。
- 数量小数位：根据不同的单位分别控制，请在计量单位进行设置。

5）**按币别维护供应商价格资料、客户价格资料**：控制客户价格资料、供应商价格资料是否按币别定价

注意：参数开启，系统存在非本位币的价格资料后，参数不可关闭

6）**单据以商品价格资料取价或限价时按汇率自动换算**：控制商品价格资料按本位币定价，单据取价和限价按汇率自动换算。

>
注意：参数5和6需要在开启“启用外币”参数后可见，如果没有启用外币，这两个参数不可见。

7）**启用自动填充结算金额**：用于控制录入业务单据时是否自动填充单据结算金额，主要用于企业全部为现收、现付业务处理的场景。勾选后，录入采购入库和销售出库单时，系统将自动完成结算金额相关字段内容填写，不需要手工录入。启用此参数建议在账户管理中设置默认收付款账户。

8）**新增单据时商品数量默认填充为**：默认开单商品数量为0，需要开单时根据实际情况填写商品数量；默认开单商品数量为1，用于开单时商品数量全部为1，或大部分为1的情况。

9）**不允许反审核已对账单据**：勾选后对于进销存单据中已经标记对账的单据，不能再反审核。

10）**启用第三方结算：**

• 含义：双方发生往来业务中，但最终的应收应付是跟第三方结算。例如，

在采购业务中，向A公司采购，应付账款记入B公司；

在销售业务中，销售给C公司，应收账款记入D公司。

• **启用第三方结算，可维护结算客户/供应商，应收应付账款将归集到结算客户/供应商**

![](https://vip.kingdee.com/download/01009acdf165af384fa886304519fc6fca99.png)

• **启用参数后不允许取消，请谨慎操作！**

11）**在采购、生产、委外业务中启用质检流程【仅旗舰版支持】**：启用后采购检验、委外检验流程需通过收料通知单下推质检单；生产检验流程需通过生产任务单下推质检单。

12）**强制分摊同单费用参数，控制同单费用的流程，包含三类参数：**

- 选择需要强制分摊同单费用的单据
- 选择需要默认费用为待支出的单据
- 删除单据时，必须同步删除同单费用生成的单据

13）**在业务结账后，不允许新增/修改/变更/审核/反审核采购订单和销售订单。**

## 2.2 采购参数

1）**采购单价为0时**：此参数用于企业对于新增采购类单据，采购单价为0时的处理，默认“提示，允许继续操作”，可根据企业实际需要调整为“不控制”、“提示，不允许继续操作”。

2）**支持修改税额本位币/价税合计本位币字段参数**

**适用场景：**外贸企业中，与供应商/客户用外币结算，但是税额是按人民币计算。采购/销售类单据中价税合计本位币，是根据每行的价税合计和汇率计算并截取精度后，在汇总相加得到本单的价税合计本位币；而单据表头的本次应收账款本位币，是按汇总的价税合计原币和汇率计算得出，两种计算结果存在精度差异。若需要调整为一致，可在“系统参数>进销存参数”中，设置允许修改单据的税额本位币，价税合计本位币字段。

参数设置路径如下：

| 路径 | 参数名称 | 适用单据 |
| --- | --- | --- |
| 采购参数 | 若需要手工修改XX单据（见右列“适用单据”）的本位币字段 | 标准版：采购入库单/采购退货单专业版：采购入库单/采购退货单/暂估调整单旗舰版：采购发票 |
| 销售参数 | 若需要手工修改XX单据（见右列“适用单据”）的本位币字段 | 标准版/专业版：销售出库单/采购退货单旗舰版：销售发票 |

3）采购类单据生成下游单据，下游单据数量大于源单

| 参数名称 | 应用场景 | 适用版本 |
| --- | --- | --- |
| 允许入库数量大于订单数量 | 此参数用于企业不严格控制入库与订单的关联数量的场景，如果不勾选此参数，通过采购订单关联录入采购入库单时，如果入库数量大于订单数量，则系统控制不允许保存采购入库单。参考知识：如何设置采购订单是否能超额入库? | 所有版本 |
| 允许入库数量及开票数量大于订单数量 | 在某些商品需要称重过磅的行业或者小体积商品，在实际出入库尤其是入库环节，入库数量会超订单数量，出于成本管控、资金流动性等考虑，需要控制入库数量不能超过订单一定比例，否则可能会引起库存积压、资金占用等问题。参知识：超收超发管控 | 专业版/旗舰版 |
| 允许入库数量大于收料通知单数量 | 旗舰版 |  |
| 按比例控制超收 | 专业版/旗舰版 |  |

4）**允许修改采购费用（已审核采购类单据）**：此参数用于控制采购类单据（采购订单、采购入库）在审核后，是否能修改单据上的“采购费用”。勾选此参数，在单据审核状态下，采购费用字段仍可以录入数据并进行费用分摊操作。用于企业采购业务发生并已审核确认后，又发生了采购费用需要补充录入的业务场景

5）**采购类单据合并下推时携带采购费用**：此参数用于控制采购类单据批量下推时，是否携带费用到下游单据。

>
说明：

1.单据整单执行状态为“未执行”才能携带预计采购费用；

2.包含以下几种下推类型：采购订单生成采购入库单，采购订单生成收料通知单，收料通知单生成采购入库单，采购订单生成采购发票，采购发票生成采购入库单。

6）**已审核的采购类单据允许重新取价**：此参数用于控制采购类单据已经审核但还没有关联下游单据（如关联入库、退货、付款/退款、生成凭证等）时可重新根据价格取数优先级进行取价。

>
注意：

1.采购类单据包括采购订单、收料通知单、采购入库单、退料通知单、采购退货单、采购发票

2.采购订单已关闭、已入库； 采购入库单已生成凭证、已付款、已开票、有退货； 采购退货单已生成凭证、已退款，则不允许重新取价。

7）**采购订单审核时，自动生成预付款**：此参数用于控制预付采购订金时的业务处理，勾选此参数后，当采购订单中录入了预付订金数据时，审核采购订单，系统就会自动生成对应的未审核的预付款单。

![](https://vip.kingdee.com/download/0100e9b59288ca80445f833abdd6e5da3724.png)

8）**旗舰版专用参数**

| 参数名称 | 应用场景 |
| --- | --- |
| 审核采购入库单、采购退货单时，系统自动生成对应的采购发票 | 审核后自动生成发票的场景 |
| 关联相同采购订单的采购入库单/采购发票审核后自动勾稽 | 自动勾稽的场景 |
| 采购入库单关联的采购退货单审核后自动对等核销，扣减采购入库单或暂估入库期初可开票数量 | 自动对等核销 |

9） **启用暂估调整**（仅适用于专业版）: 此参数适用于在采购入库时不确定采购价格，需要在以后通过暂估调整单进行调整的业务场景；（开启后，采购入库单上会有对应的按钮），支持设置是否默认开启。

10） **启用受托代销**（仅适用于专业版/旗舰版）：此参数适用于有受托代销的业务场景。设置保存成功后,须按"F5"键刷新页面或重新登录后生效。（开启后，刷新页面后，采购管理模块下会显示受托代销相关菜单）

## **2.3 销售参数**

销售参数与采购参数基本一致，应用场景和对应控制可参考采购参数说明。除与采购参数中对应的销售场景下的参数外，销售参数还提供了以下参数：

1）**设置订单订金比率至少为x%才可生成销售出库或其他单据（仅适用于专业版/旗舰版****）**：参考知识：[订金优化](https://vip.kingdee.com/link/s/ZTaqk)

2）**新增销售类单据时不允许修改单据日期**

| 参数名称 | 应用场景 |
| --- | --- |
| 不允许修改销售类单据日期（网页端） | 此参数用于是否需要严格控制当天单据只能当天录入，勾选后，录入销售类单据时，单据日期自动取系统日期，不允许修改。此参数控制的是网页端操作。 |
| 不允许修改销售类单据日期（移动端） | 此参数控制的是移动端操作时，是否能修改单据日期，勾选后，在移动端不能修改销售类单据日期，单据日期自动取系统日期。 |

3）**已审核销售类单据允许重新取价：**此参数用于**销售类单据**已经审核但还没有关联下游单据（如关联出库、退货、收款/退款、生成凭证等）时可重新根据价格取数优先级进行取价。

>
注意：

- 销售类单据包括：销售订单、销售出库单、销售退货单、销售换货申请单、销售退货申请单、发货通知单、委托代销单、委托结算单、销售发票
- 销售订单已关闭、已出库； 销售出库单已生成凭证、已收款、已开票、有退货；销售退货单已生成凭证、已退款，则不允许重新取价

4）**销售换货申请审核后，自动生成销售出库单、销售退货单**：此参数控制的是录入销售换货申请单并审核时，是否自动生成换入商品的销售退货单和换出商品的销售出库单，如果勾选，则审核时自动生成对应单据并审核，如果不勾选，则需要手工在已审核销售换货单上通过【换入】和【换出】生成对应单据。

>
勾选此参数还需要注意以下事项：

①销售换货申请单、销售出库单和退货单是否有必录的自定义字段，如有，可以在单据自定义中设置适用单据，则在生成单据时，可以自动携带对应的自定义字段内容，否则单据保存会不成功。

②对应换出的商品库存是否足够，如果仓库库存不足，且不允许负库存出库，则销售出库单保存会失败。

③是否对客户进行信用额度控制，如果进行信用额度控制，且换出生成销售出库单超出信用额度，则系统会给予对应提示。

5）**旗舰版专用参数**

| 参数名称 | 应用场景 |
| --- | --- |
| 审核销售出库单、销售退货单时，自动生成销售发票 | 审核后自动生成发票的场景 |
| 关联相同销售订单的销售出库单/销售发票审核后自动勾稽 | 自动勾稽的场景 |
| 销售出库单关联的销售退货单审核后自动对等核销，扣减销售出库单或销售出库期初可开票数量 | 自动对等核销 |

6）**客户承担费用处理方式**：此参数用于销售出库存在客户承担费用时，客户承担费用的对账问题。若客户承担费用与货款一起对账，则选择“计入应收账款”，但客户承担费用不能分摊进商品行进行按分录核销收款；若客户承担费用与货款分开对账，则选择“计入其他应收款”，此时会自动生成一张其他应收款。此参数可以随时修改，但修改后只影响后续新增的单据，对历史单据不改变客户承担费用的处理方式。

7）**退还客户承担费用处理方式**：此参数适用场景与“客户承担费用处理方式”对应，用于销售退货单存在要退换客户承担费用时，退还的客户承担费用的对账问题。若退还的客户承担费用与退货款一起对账，则选择“计入应退账款”，但退还的客户承担费用不能分摊进商品行进行按分录核销退款；若退还的客户承担费用与退货款单独对账，则选择“计入其他收入退款”，此时会自动生成一张其他收入退款。此参数可以随时修改，但修改后只影响后续新增的单据，对历史单据不改变退还客户承担费用的处理方式。

8）**直运业务**（仅适用于专业版/旗舰版）：参考知识：** **[直运销售](https://vip.kingdee.com/link/s/ZgS8H)

## **2.4 仓存参数**

1）**批次或保质期商品在出库时批次自动填充规则**：此参数设置用于当商品启用了批次或批次+保质期管理时，通过订单进行出库时，批次是否自动填充。控制规则如下：

| 参数名称 | 应用场景 |
| --- | --- |
| 手工选择 | 出库时，需要手工录入或选择批次相关信息 |
| 批号顺序出库 | 出库时，按批号顺序自动取批号，如果商品同时启用的保质期管理，则同时会取对应批号的保质期信息 |
| 近效期先出（同时启用批次和保质期商品） | 出库时，按保质期到期先后顺序取批号和保质期信息。注意：此参数既适用于同时启用批次和保质期商品，也适用于只启用保质期管理的商品。 |
| 批次保质期自动填充范围 | 勾选前：必须先选择仓库，才能自动带出批次、保质期勾选后：在所有仓库范围内按批次顺序出库、近效期先出，并根据批次保质期自动带出对应的仓库、仓位 |

2）**库存更新的方式：**

| 参数名称 | 应用场景 |
| --- | --- |
| 保存时更新 | 单据保存即更新系统的即时库存数量，如果不允许负库存出库，则在出现负库存时，单据无法保存，是单据审核前就扣减库存的管控方式。 |
| 审核时更新 | 单据审核时才更新系统的即时库存数量。如果不允许负库存出库，则在出现负库存时，单据无法审核。 |

3）**订单类单据负库存管控方式**：可选择不控制、按可用库存控制

4）**出入库类单据负库存管控方式**：依赖于上一个参数的设置。上一个参数的选项会影响此参数的选择。

5）**可用库存参数**：该参数用于设置预计可用库存表中可用库存和单据录入时可用库存字段取数计算规则。企业根据实际业务需要勾选对应参数设置即可

6）**预计可用库存参数：**统计在可用库存基础上增加的批次/保质期/序列号商品、或普通商品在销售订单和采购订单上没有录仓库/仓位时的预计出入库量（采购在途和销售占用）。用户可结合实际业务场景来设置。一般适用于销售及采购所参考的可用库存依据是不一样的场景，如销售时是根据可用库存判断，而采购补货或者库存预警时，需要剔除对应的已审核的销售订单，以实现更精准的补货。相关的知识：[预计可用库存优化](https://vip.kingdee.com/link/s/MU15D)

7）**库存预警与异常检查**

| 参数名称 | 应用场景 |
| --- | --- |
| 允许无期初和业务时提醒 | 勾选后，在库存预警表和首页预警提示中，会包括没有录入商品期初和本期没有业务发生的商品。是一种比较严格的预警控制，可以对所有商品进行最高、最低库存控制 |
| 库存控制策略：当基本单位库存量与默认浮动单位库存量一个为正一个为负 | 若选择不予控制，最终浮动计量的商品库存可能会存在基本单位库存量、默认浮动单位库存量一个为正，一个为负，即使仓库设置了不允许负库存，也会出现某个单位库存量为负。若选择严格控制，则在上述情况下系统会严格控制，不允许出库。需要调整本单出库数量、辅助数量或者其他调整库存的操作使最终库存不会出现上述情况为止才可以出库。参考知识：负库存设置及判断逻辑说明 |
| 库存控制策略：当基本单位库存量与默认浮动单位库存量一个为零一个不为零 | 若选择不予控制，最终浮动计量的商品库存可能会存在基本单位库存量、默认浮动单位库存量一个为零，一个不为零，这时需要进行盘点才可以将基本单位库存量、默认浮动单位库存量都盘成零。若选择严格控制，则在上述情况下系统会严格控制，不允许出库。需要调整本单出库数量、辅助数量或其他调整库存的操作使最终库存全部为零或全部为正才可以正常出库。参考知识：负库存设置及判断逻辑说明 |
| 单据负库存预警 | 勾选单据后，则单据保存、提交等操作节点就可以进行负库存预警，当前库存减去当前单据中的数量后，若出现负库存，则会进行预警提示，仓库允许负库存且出现库存为负时，也会进行预警提示。参考知识：库存功能优化（2025年） |

8）**组装单参数**

   ①允许修改组装费用（已审核组装单）：与“允许修改采购费用（已审核采购类单据）”类似，控制是否允许在组装单已审核的情况修改组装费用。

   ②多组件业务，组件成本分摊比例计算依据：控制组装单上多个组件时，子件的成本+组装费用以怎样的比例分摊到多个组件上。提供按商品资料上的参考成本、数量两种方式进行分摊。

9）**拆卸单参数**

   ①允许修改拆卸费用（已审核拆卸单）：与“允许修改组装费用（已审核组装单）”类似，控制是否允许在拆卸单已审核的情况修改拆卸费用。

   ②子件成本分摊比例计算依据：控制拆卸单上多个子件时，组件的成本+拆卸费用以怎样的比例分摊到多个子件上。提供按商品资料上的参考成本、数量两种方式进行分摊。

10）**调拨单参数**

   ①允许修改拆调拨费用（已审核调拨出库单/调拨入库单） ：控制是否允许在调拨单已审核的情况修改调拨费用。

   ②调拨费用分摊比例计算依据：提供按商品资料上的参考成本、数量两种方式进行分摊。
