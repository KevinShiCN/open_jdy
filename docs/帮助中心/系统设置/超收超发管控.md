# 超收超发管控

# **1 简介**

## **1.1 应用场景**

在某些商品需要称重过磅的行业或者小体积商品，在实际出入库尤其是入库环节，入库数量会超订单数量，出于成本管控、资金流动性等考虑，需要控制入库数量不能超过订单一定比例，否则可能会引起库存积压、资金占用等问题。

## **1.2 功能特性**

- 【专业版/旗舰版】销售出库控制不允许超过订单发货上限、采购入库控制不允许超过订单收货上限
- 【旗舰版】销售出库控制不允许超过发货通知单上限、采购入库控制不允许超过收料通知单上限
- 【旗舰版】销售发票控制不允许超过销售订单发货上限、采购发票控制不允许超过采购订单收货上限
- 【旗舰版】发货通知单下推销售出库单允许大于发货通知单数量、收料通知单下推采购入库允许大于收料通知单数量
- 同步支持移动端

## **1.3 适用版本**

星辰3.0及以上专业版/旗舰版

## **1.4 系统路径**

系统参数>采购参数、销售参数中分别新增“按比例控制超收”、“按比例控制超发”的参数

## 2.2 单据应用流程

本文以采购环节举例，销售同理，不再赘述。

### 1）采购入库控制不允许超过订单收货上限

采购订单单据体分录列设置开启“控制收货上限”、“超收比例（%）”、“收货上限”字段的显示

![上传图片](https://vip.kingdee.com/download/0100aa2b8186d5c643d0879811ae20730b68.png)

录入商品后，开启商品行的“控制收货上限”，维护“超收比例（%）”，系统会自动算出“收货上限”，其中：

>
收货上限=数量*（1+超收比例）

收货上限（基本）：表示基本单位对应的收货上限数量。

![](https://vip.kingdee.com/download/010020212137b8864b8eb6e42bf33ebf07c3.png)

当订单下推入库，入库数量大于订单上限时，单据保存失败，会做出对应的提示。

![](https://vip.kingdee.com/download/010047a0794c50754dc58b1fdd29c7e073f7.png)

### 2）收料通知单下推采购入库允许大于收料通知单数量

系统参数开启“允许入库数量大于收料通知单数量”

![](https://vip.kingdee.com/download/0100308bbc7d434143d1bb6bcc76c278f638.png)

采购入库时入库数量可以大于收料通知单数量。

![](https://vip.kingdee.com/download/01007c244311ed06434484a87db0eb0c31e7.png)

### 3）若同时系统参数勾选“按比例控制超收”，那么入库数量不能大于收料通知单的收货上限

单据上维护了超收信息：

![](https://vip.kingdee.com/download/0100fca95ea2372b4cb6ac358cca62066319.png)

如果入库数量大于收料通知单上限，那么保存时就会提示超出上限，不允许保存。

![](https://vip.kingdee.com/download/0100f43e2bd3f0024d41a8c7426a701d3457.png)

### 4）采购发票控制不允许超过采购订单收货上限

单据上维护了超收信息：

![](https://vip.kingdee.com/download/0100d03e8ac0a7c04220bab47303747acfbf.png)

如果采购发票数量大于采购订单上限，那么保存时就会提示超出上限，不允许保存。

![](https://vip.kingdee.com/download/0100513173b8fc514e6593821e494bd1548b.png)

### 5）一些特殊情况说明

- **一笔订单部分下推入库，同时部分下推收料通知单时，最终入库数量需要在订单上限范围内。**

示例：比如订单数量100，下推01入库单50，未审核；下推01收料通知但50，审核；修改01入库单数量为60，审核；01收料通知单下推入库单，最多只能入库50。

![上传图片](https://vip.kingdee.com/download/0100e5958e3982d54bc0a734c7116640e97d.png)

- **一笔订单部分通过入库单下推发票，部分通过订单直接下推发票，最终开票数量需要在订单上限范围内。**

示例：比如订单数量100，下推01入库单100，审核；订单直接下推01发票单50，审核；01入库单下推发票时，最多只能开票60。01入库单未下推发票的数量40只能与01发票单进行勾稽。

![上传图片](https://vip.kingdee.com/download/010013e0b2d294e3473d9bf70308d7ea1046.png)

- **一笔订单下推入库，同时订单下推发票，但入库数量与发票数量不一致时，超出部分只能从入库单下推发票或者从发票下推入库。**

示例1：比如订单数量100，订单下推入库100，订单下推发票110，那么发票多开的部分只能从发票下推入库；

![上传图片](https://vip.kingdee.com/download/010015ae731507a6485ca8574bc9cbffa4e8.png)

示例2：同理若订单数量100，订单下推入库110，订单下推发票100，那么入库多入库的部分只能从入库单下推发票。

![上传图片](https://vip.kingdee.com/download/010088d90ab80ad54bcd861ac85f172a3483.png)
