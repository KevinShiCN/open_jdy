# 用户没有修改过单据,操作日志里的,"单据主键:xxx, 编码从:xxxx 变更为:xxx"是什么意思?为什么有这个日志,这个提示有什么影响吗?

【问题概述】

用户没有修改过单据,操作日志里的,"单据主键:XXX, 编码从:XXXX 变更为:XXX"是什么意思?为什么有这个日志,这个提示有什么影响吗?

![](https://vip.kingdee.com/download/010928f74bbc4ddd40e88e13c5dc5eb668c7.png)

 【字段说明】

1、主键:就是数据库里的唯一索引值,例如单据、基础资料,在创建后数据库都有一个唯一的ID,是一个后台字段,**不在产品内直接显示,用户不必关注**。

2、**单据主键从A变更到B,一般是指单据的编码发生了转移**。

例如:后台有一张新增的订单的主键(后台ID)是123456,然后一开始保存时,创建的单据编码是001(对应主键是123456),然后因为一些场景,单据编码发生了变更,从001变成002,那么由于主键是唯一值,不会改变的,所以主键123456转移到了新的单据编号002上了,同时001这个单据编号会重新释放,意思没有被单据引用,可以继续被其他单据引用。

 【原因说明】

当您的单据编码开启了日期流水号依据,当出现主键变更这个日志一般是有以下几种场景:

| 场景序号 | 场景描述 |
| --- | --- |
| 1 | 仓库不允许负库存,单据批量下推时,出现负库存提示,然后更换成有库存的仓库或者补足库存后,再保存单据 |
| 2 | 系统参数设置了0单价控制,单据批量下推时,出现0单价提示,然后录入了单价后再保存单据 |
| 3 | 系统设置了采购销售限价控制,或者客户价格资料设置了限价,单据批量下推时,出现了限价提示,然后修改了单价后再保存单据 |
| 4 | 新增单据并且保存了,然后修改了单据日期,例如从3号改成了4号 |

  【影响及解决办法】

1、主要影响:当出现主键变更时,会出现断号,也就是导致单据编码出现不连贯的现象。

2、解决办法:

①如果您的系统是**允许断号**的,但是针对以上场景希望可以不要断号,这个要等后面优化处理,目前只能在新增单据时手工补上断号;

②如果您的系统是**不允许断号**的,但是针对以上场景希望可以**断号可以回收**,您可以在编码规则中,**开启“断号补充”**

![](https://vip.kingdee.com/download/010991ac95b204a94857add8b9c3cc413286.png)
③若需手工修改单据编号为变更前单据编号,可参考:[如何修改单据编号?](https://vip.kingdee.com/link/s/Zvtmh)
