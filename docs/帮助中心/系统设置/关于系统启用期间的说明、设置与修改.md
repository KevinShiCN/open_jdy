# 关于系统启用期间的说明、设置与修改

# 一、**名称解释**

- **系统起始期间**：即从哪一个时期开始记账，录入数据。
- **自然年度期间：**在我国，会计期间是按自然年度作为一个会计年度，自公历1月1日起至12月31日。
- **自定义期间：**有的地区，企业可以根据自己的需要自定义区间，比如中国香港一般每年的4月1日至次年的3月31日作为一个会计年度，这种期间在星辰的设置里就属于自定义期间。需在建账界面的【期间设置】里进行设置。（下图红箭头处）

![](https://vip.kingdee.com/download/0100c10cb58dfe464c95b18088d47e90544c.png)

>
**温馨提示：如业务是按自然年度（公历1月1日-12月31日）管理的，请不要选择【期间设置】。**

# **二、系统起始期间设置**

**例1：购买时有历史业务数据需要导入**

如果是2021年6月份开始购买，但2021年1-5月份都有相应的业务数据需要导入，那么系统起始期间建议从2021年的第一期开始启用，**即：建议按最早发生业务数据的时间来启用**。

![](https://vip.kingdee.com/download/0100cdac132a7e644f36bb342f384bf8788b.png)

**例2：新开公司或不需要导入历史业务数据**

如果是新开公司，或者不需要将以前的每一笔业务数据导入系统，只需要将商品库存、客户供应商的往来账款余额导入系统，**推荐按真实使用时间选择**。

举例：假设是2021年6月中旬购买，打算从2021年7月1日正式开始使用，那么系统起始期间可以从2021年的第七期开始。

>
**注意事项：****系统期间一旦在建账时选择了，后续不能更改，只能通过账套初始化进行重设，故在建账时请务必要慎重选择！**否则一旦建账产生了业务数据，重新初始化后需再次重新录入所有的数据，将耗费大量的人力成本。

# **三、期间修改**

**1、系统起始期间与各模块启用期间的区别与设置路径、修改条件**

| 模块 | 修改条件（图片可点击放大查看） |
| --- | --- |
| 公共参数 | 启用账套时已经设置好的，无法修改，只能通过账套初始化重新设置！ |
| 进销存参数 | 一、没有录入单据的情况下，可以修改，存在2种修改场景：1、修改成早于当前期间（例如2025年01期改到2024年12期），原往来初始初始、商品、账户初始余额可选择系统提供的修改方案，需仔细阅读，一经确认无法返回。注：关于“修改启用期间”弹窗中选项的含义及其适用场景，详见本文最后一条说明。2、修改成晚于当前期间（例如从第1期改成第2期），原启用期间所录客户初始数据与供应商初始数据不受影响，商品及账户余额可选择系统提供的修改方案，需仔细阅读，一经确认无法返回。注：关于“修改启用期间”弹窗中选项的含义及其适用场景，详见本文最后一条说明。二、已经录入单据，但是未结账的情况下：修改期间不能晚于业务单据最早的创建时间。系统会弹出以下提示，不允许操作：举例：假设系统最早的单据是2021年7月10日创建的，那么只能把期间改早于7月10日以前的，例如改成7月1日，而不能往后改，例如改成7月15日。 |
| 财务参数 | 财务云的启用期间在账套未录入凭证的情况下，可以在财务参数中修改。 |
| 资产参数 | 资产的启用期间在账套未录入任何单据、凭证的情况下可以修改启用期间。 |

**2、模块（不包括公共参数）的启用期间修改规则：可以往前修改5年，往后15年**

例：系统启用期间是2021年3期，则往前可以修改成2016年3期，往后可以修改成2036年3期。
