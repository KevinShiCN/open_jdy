# 业务员离职了，如何把离职业务员负责的客户信息和单据转移到现有的业务员手上？

> 作者：进销存赵嘉丽 | 阅读：1771 | 有用：22

# 应用场景

原有的业务员“李四”离职了，需要把业务资料和单据转移给现有的业务员“张三”（张三本身也在用系统）手里，该怎么处理呢？

# 解决方案

针对上述场景，需要处理2个问题，一个是客户资料的转移，一个是离职业务员历史操作了但是还没处理完的单据，因此我们的操作也是从这2个方面入手：

- ## 客户资料的转移

第一步：账套管理员，先在用户管理中，禁用业务员李四（目的是不让业务员李四可以登录产品，保证数据安全，其次是可以减少对用户数的占用，给后面新邀请进账套的用户预留用户数）；

![](https://vip.kingdee.com/download/01002555680ab3bb4314ae066985f5285430.png)

第二步：账套管理员，打开客户列表，批量修改离职业务员为“李四”所负责的客户资料，批量修改成张三的名字即可；

![](https://vip.kingdee.com/download/01004e9aa38ffcbb4fa1b4348bf9ef8e6a95.png)

![](https://vip.kingdee.com/download/01009542567ef7624df4b96c4682966dbedc.png)

## 历史单据的处理

因为李四名下可能还存在没有处理完的单据，需要张三接过他的单据继续完成。一般而言，业务员只能操作和修改自己的单据，因此如果需要让张三可以操作李四之前做的单，涉及到权限的问题：

注意：下面示例只适用于只对子账号做了用户授权（没有用户组授权）的情况下。

第一步，给张三数据授权“制单人”字段，制单人选“李四”，并保存权限（如下图）：

![](https://vip.kingdee.com/download/01001ed890d778f84bf2be59655d0be69cf3.png)

第二步，授权完毕后：

- 针对**已审核已执行部分出库**的单据，建议让张三继续操作完成后续的出库，直到单据关闭。
- 针对**已审核未执行、已保存未审核**的单据，建议让张三直接**反审核/修改**单据上的业务员，改成自己即可（如下图）。

![](https://vip.kingdee.com/download/0100412a0656f7b941208718c3c4493c0465.png)

至此，关于离职业务员负责的客户信息和单据转移就操作完毕了。
