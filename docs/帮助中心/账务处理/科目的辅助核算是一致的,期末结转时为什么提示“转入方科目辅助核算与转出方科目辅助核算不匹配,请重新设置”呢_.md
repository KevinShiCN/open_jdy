# 科目的辅助核算是一致的,期末结转时为什么提示“转入方科目辅助核算与转出方科目辅助核算不匹配,请重新设置”呢?

【问题描述】

科目的辅助核算是一致的,期末结转时为什么提示“转入方科目辅助核算与转出方科目辅助核算不匹配,请重新设置”呢?

![](https://vip.kingdee.com/download/010984948173cbf44454819db55d06a71f93.png)
![](https://vip.kingdee.com/download/0109046df8ee29e14877a11510a8ba6238b8.png)
【解释说明】

科目的辅助核算都是一致的,转入科目开启了辅助核算必录,转出科目没有开启辅助核算必录,且生成的凭证也没有录入相应的辅助核算,期末结转时就会提示“转入方科目辅助核算与转出方科目辅助核算不匹配,请重新设置”。

如,转入科目60910开启了部门的必录

![](https://vip.kingdee.com/download/0109aaac78373a5d48f18e7fde0450188645.png)
其中一个转出科目60911未开启部门的必录

![](https://vip.kingdee.com/download/01095b2285f63d1f4a64b7d4fe2953c44c46.png)
生成凭证时,转出科目60911也没有填写部门

![](https://vip.kingdee.com/download/0109cbb93f69e5334e449177b992e6aeaa80.png)
期末智能结转,按照辅助核算项目结转时,由于转入科目60910部门是必录的,没有填写部门的科目60911不能结转,导致失败。

![](https://vip.kingdee.com/download/01092a37f85ed81e4cc7b42ecb82beea5028.png)

取消转入科目60910部门必录,重新结转效果

![](https://vip.kingdee.com/download/0109a76660bf7b78429fa449c4e651fb7a19.png)
