# 金蝶云·星辰订货商城，商城下的订单明明已经付款，销售订单为什么显示未付款？应该怎么排查及处理

> 作者：Lizzy | 阅读：1597 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/243325453053088256)

问题描述：

商城下的订单明明已经付款，销售订单为什么显示未付款？应该怎么排查及处理

解决方案：

关于商城的付款流程及扭转，场景较多，请按照以下方案进行排查及对应业务处理。

### 第一步，查看付款的顺序：

商城订单列表，找到销售订单对应的商城订单，商城订单详情中操作日志。确认是先付款还是先审核商城订单。

![](https://vip.kingdee.com/download/0109d389bfaa45454bb59acf15373094d049.png)

### 一  若是先审核商城订单再付款

由于销售订单先于预收款单生成，无法形成销售订单上的定向收款。需要手动核销预收款单和出库单，完成收款操作。

先审核后付款建议设置自动核销方案”预收冲应收“勾选“仅订单关联的预收（付）单和其下推的销售出库（采购入库）核销”

![](https://vip.kingdee.com/download/0109270785af94c8480e8ac0b0d20e402409.png)

### 二  若是先微信或支付宝在线付款成功再审核商城订单

该种情况下正常应该将收款金额填写入销售订单下方的收款金额，并生成已审核的定向预收款单。那么我们需要以下排除：

销售订单下方的定向预收款是否已经生成且已审核？

![](https://vip.kingdee.com/download/01090216aaa635254ebabd258dbd9b8c46b3.png)

#### （1）若有填写收款信息但是没有生成预收款单。

检查系统参数-进销存参数-销售订单审核时，生成预收款单，该参数是否已经勾选。检查到未勾选，请选择上。

![](https://vip.kingdee.com/download/0109fe5e2f7ec5f9441588fdd177310ab4b0.png)

若有勾选但是没有预收款单，请查看系统的操作日志，时间为销售订单审核时间后看预收款单相关报错，例如审核销售订单账号没有预收款单权限等。请按照报错提醒进行设置。

#### （2）若有生成预收款单，但是预收款单是未审核状态。

看销售订单操作日志新建时间和审核时间是否近乎相同。只有由订货审核下推为已审核的销售订单时，程序才会自动找到关联的定向预收款单将其审核。

没有生成已审核的销售订单可先查看商城系统参数是否设置下推成已审核的销售订单；再查系统操作日志搜索销售订单编号查看审核失败的原因，例如库存不足，必填项设置，没有审核权限。

### 三  若是线下付款提交了凭证，再审核商城订单。

批量审核和自动审核商城订单默认预收款单需要人工确认，需手动审核预收款单。
