# 订货商城使用储值支付后，储值消费记录显示消费暂扣的情况，应该如何解决？

1 问题描述

订货商城使用储值支付后，储值消费记录显示消费暂扣的情况，应该如何解决？

![](https://vip.kingdee.com/download/010964b76433b3734556b30ea2f5e601989e.png)

## 2 适用版本

金蝶云·星辰v3.1专业版

## 3 原因解析

出现“消费暂扣”的原因有两种：

一、由于系统升级中断，或者由于核销单新增失败等异常原因，导致储值消费记录状态显示在储值消费状态为“消费暂扣”状态。

二、储值支付下推的销售出库单与之前未核销的预收款单进行核销，导致储值消费记录状态显示在储值消费状态为“消费暂扣”状态。

## 4 解决方案

第一种解决方案：储值记录列表新增“检查储值核销按钮”，勾选“消费暂扣”状态的储值消费记录，点击检查储值核销，系统将检查对应订单是否已经出库，并对应生成核销单。

第二种解决方案：找到储值支付对应的销售出库单进行反审核，删除对应的核销单，再进行审核，勾选“消费暂扣”状态的储值消费记录，点击检查储值核销。
