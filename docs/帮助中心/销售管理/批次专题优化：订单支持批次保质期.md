# 批次专题优化：订单支持批次保质期

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：功能介绍, 新特性, 快速入门
> 作者：进销存熊诗琴 | 阅读：960 | 有用：3
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/667780264776721408)

## 1 简介

### 1.1 功能介绍

不同批次保质期的商品，存在价格、颜色、规格等差异，下订单时就需要确认批次、保质期，仓库按照订单中批次保质期出库。采购订单中录入批次保质期后，在途的可用库存，允许业务员下销售订单，但是不允许出库。

### 1.2 适用版本

星辰3.1以上所有版本

## 2 功能特性

### 2.1 采购订单、销售订单支持录入批次、保质期信息

![](https://vip.kingdee.com/download/0109e1711fec89c240a09b9765d1209abc87.png)点击库存或者批次选择图标，可以打开批次库存弹窗，自动分配或者批量录入批次保质期及开单数量：

![](https://vip.kingdee.com/download/01092e0b68439520448488fa5775d0793906.png)

### 2.2 单据优化

1、订单中支持录入批次、保质期信息，非必录字段，如有需要，需手工指定录入批次、保质期，采购订单不会按编码规则自动生成批次，销售订单不会按系统参数自动带出批次保质期。

2、单据列表增加批次、生产日期、保质期、保质期天数、到期日字段，支持漏斗筛选和排序，支持引入引出、打印、统一列设置，支持快速粘贴

3、订单下推、选源单时，支持携带订单中的批次保质期信息（销售订单下推销售出库单时，当销售订单中有录入批次或者保质期信息，则下推时该商品行只会携带订单中的信息，不会再拆分批次保质期）

4、订单中的分录行已关联下游单据或销售订单中分录行已预留时，不允许变更批次保质期；未下推且未预留的分录行，允许变更批次保质期

5、单据复制，支持携带订单中的批次保质期：

- 采购订单复制：复制本单、销售订单、销售出库单、委托代销单、其他入库单
- 销售订单复制：复制本单、采购订单、采购入库单、其他入库单、其他出库单、移仓单、调拨出库单、产品入库单

2、单据生成设置中，支持参数配置，生成的采购订单可以携带销售订单中的批次和保质期信息：

### 2.4 系统参数优化

出入库类单据支持按可用库存+即时库存双重控制负库存

● 系统参数交互调整：

优化前负库存参数：

![](https://vip.kingdee.com/download/0109d712f447f35a438baf19c975fc479f07.png)

优化后负库存参数：

![](https://vip.kingdee.com/download/01096c066a1c4ef044dca948797b8da3007e.png)

对应关系：

| 优化前 | 优化后 | 优化后交互图 |
| --- | --- | --- |
| 负库存管控方式=即时库存 | 订单类单据负库存管控方式=不控制出入库类单据负库存管控方式=即时库存 |  |
| 负库存管控方式=可用库存 | 订单类单据负库存管控方式=可用库存出入库类单据负库存管控方式=可用库存 |  |
| 无 | 订单类单据负库存管控方式=可用库存出入库类单据负库存管控方式=即时库存与可用库存双重控制 |  |

- 本次新增了出入库类单据支持按“即时库存与可用库存双重控制”的参数，避免采购订单录入批次保质期后，此时商品库存还是采购在途，就能直接被出库的业务场景。启用该参数后，对于出入库类单据（如销售出库单、委托代销单、其他出库单等）保存/审核时，只要即时库存和可用库存中有一个库存不足，则会提示负库存：

![](https://vip.kingdee.com/download/0109b84571620bb1421dbf638f748d45011c.png)

- 负库存弹窗中，点击“按现有库存出库”时，则可以将开单数量修订为负库存弹窗中的“可出库数量”：

![](https://vip.kingdee.com/download/010914a4be1f169c4929a5269b5a4de235f4.png)

- 重新审核即可成功出库：

![](https://vip.kingdee.com/download/0109147bf2c9326546d8a711e5c7f152bb37.png)

- 对于出入库类单据，双重控制下，“可出库数量”取值说明：取的是“可用库存”和“即时库存-预留量”中的较小值

注意：双重控制下，【库存查询表】中的“可出库数量”目前还是取的可用库存，后续会优化，也取“可用库存”和“即时库存-预留量”中的较小值。
