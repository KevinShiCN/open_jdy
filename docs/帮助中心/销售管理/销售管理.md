# 销售管理

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：功能介绍, 快速入门
> 作者：ylim | 阅读：9861 | 有用：6
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/630441326370106624)

销售业务是企业经营活动中最重要的一环。销售业务主要包括报价、与客户签订合同,按合同约定的商品数量、价格、发货时间、收款方式进行发货并收回款项;销售业务还需处理因各种原因导致客户换货、退货时,收回退货商品和给客户退回货款。此外还有一部分销售业务是先出库,后面再跟客户做结算,这部分属于委托代销业务。还有一部分客户,是通过发票来确认收入的,这部分是销售发票业务。

| 1.销售单据 |  | —— |
| 1.1销售报价单 | 适用于销售正式开单前的询价环节，报价确认后可直接转化为销售订单，可快速无缝衔接报价、开单业务，后续也可以进行历史查询及分析，使得现有的销售流程的场景覆盖得以扩展及完善。 | 点击学习 |
| 1.2销售订单 | 销售订单是购销双方共同签署的、确认购销活动的标志，是详细记录企业物资的循环流动轨迹、累积企业管理决策所需要的经营运作信息的关键。   | 点击学习 |
| 1.3发货通知单 | 发货通知单作为销售订单到销售出库单的中间流程,用于通知仓库发货。适用版本：旗舰版   | 点击学习 |
| 1.4销售出库 | 销售出库主要应用于销售下单后发出货物的场景，同时还能完成收款/抵扣/抹零的动作。   | 点击学习 |
| 1.5销售退货 | 销售退货单主要应用于销售发货后需要发起退货的场景。 | 点击学习 |
| 1.6销售发票 | 企业发生了销售业务后，最后以收到客户的发票为准来确认应收，那么在旗舰版可以通过销售发票来对完成销售出库单收入的确认。适用版本：旗舰版   | 点击学习 |
| 1.7销售退货申请 | 适用于某些需要严格控制退货的企业,如食品生鲜行业,当客户提出退货时,业务员需要先向上级申请退货,记录退货商品、数量和退货原因,由上级决定是否同意退货。退货申请审核通过后,才接收退货。  适用版本：专业版、旗舰版 | 点击学习 |
| 1.8销售换货申请 | 企业在接到客户的换货申请时，若同意对方的换货申请，可以在《销售换货申请》单中，录入客户的退换货原因，退回（换入）的商品明细和换出的商品明细。 适用版本：专业版、旗舰版  | 点击学习 |
| 1.9销售对等核销 | 销售退货后，原销售出库单的未开票数量无需继续开票，且销售退货单也无需开红冲发票，可将未开票数量相等的销售出库单与销售退货单进行对等核销。适用版本：旗舰版    | 点击学习 |
| 1.10销售税票登记表 | 主要应用场景如下：场景一：出库后又发生了退货，实际开出的发票金额是扣减退货后的金额（出库与退货合并登记发票、合并匹配税务发票）；场景二：商品分录是赠品或者0金额的行无法登记发票信息，导致单据一直显示部分开票；场景三：手工登记的发票想要直接修改、批量删除；场景四：一张发票多次登记，无法按发票查询已开票信息。 | 点击学习 |
| 2.委托代销 | 委托代销是指‌受托方按照委托方的要求销售委托方的货物,并收取手续费的经营活动。在这种模式下,受托方作为一个代理商,不拥有货物的所有权,而是代表委托方进行销售。委托结算单，委托代销退货单，不能手工新增，只能由委托代销单或委托代销期初余额关联产生说明：该功能专业版及旗舰版均支持，标准版不支持。 | 点击学习 |
| 2.1委托代销单 | 通过委托代销单，将委托销售的商品从本公司仓库发货给经销商   | 点击学习 |
| 2.2委托代销结算 | 登记下游反馈的实际销售数据，用于跟下游经销商结算应收回的货款，及计算还剩余多少未销售完 | 点击学习 |
| 2.3委托代销退货 | 下游经销商将商品发还到公司仓库   | 点击学习 |
| 2.4委托结算退货 | 与经销商已结算完成后，若终端客户因质量问题退货，损失由本公司承担时，可对委托结算单进行退货，减少应收账款   | 点击学习 |
| 2.5委托商品跟踪表 | 查询本公司发给下游经销商的每种商品数量，已结算数量，已退货数量，未结算数量   | 点击学习 |
| 2.6委托库存余额表 | 查询本公司发给下游经销商代销的商品还有多少没有结算完   | 点击学习 |
| 3.销售报表 |  | —— |
| 3.1销售明细表 | 统计一段时期内商品的销售及销售退货的详细情况，可选择全部客户、商品、仓库进行统计，也可挑选部分客户、商品、仓库进行统计。   | 点击学习 |
| 3.2销售汇总表 | 统计一段时期内商品的销售及售退货的总体情况，汇总依据可选择“商品/客户/商品类别/仓库/业务员等 | 点击学习 |
| 3.3销售发票明细表 | 统计一段时期内商品的销售及退货的发票详细情况，可以选全部客户、商品、仓库进行统计，也可挑选部分客户、商品、仓库进行统计。适用版本：旗舰版 | 点击学习 |
| 3.4销售发票汇总表 | 统计一段时期内商品的销售及售退货的发票总体情况，汇总依据可选择“商品，客户，商品类别，仓库，业务员等等适用版本：旗舰版 | 点击学习 |
| 3.5销售订单跟踪表 | 实时跟踪一段时期内所有销售订单的详细信息,可不选择选择客户、商品和出库状态(默认是全部)进行统计,也可挑选部分客户、商品、出库状态进行统计。   | 点击学习 |
| 3.6销售订单汇总表 | 主要应用于统计一段时期内商品的销售订单总体情况，支持九种汇总维度进行统计。   | 点击学习 |
| 3.7采购销售汇总表 | 统计一段时期内商品的采购和销售的总体情况，可选择全部商品、仓库进行统计，也可挑选部分商品、仓库进行统计，数据来源包含已审核和未审核的采购入库单，采购退货单，销售出库单，销售退货单。   | 点击学习 |
| 3.8销售排行表 | 从多个维度（商品/商品类别/品牌/客户/客户分类/部门/业务员/仓库）对一段时间的销售进行汇总并排序   | 点击学习 |
| 3.9销售收款一览表 | 统计一段时期内每笔销售或退货业务的实际收款或退款状态。  按不同的排序方式查询时，报表有不同的展现形式，当选择排序方式为“单据日期+单据编号”时，会按日期顺序展示销售单记录和销售退货单记录；当选择排序方式为“客户+单据日期+单据编号”时，会把相同客户的查询时间段的单据排在一起。   | 点击学习 |
| 3.10销售利润表 | 用户在开展完销售业务以后，除了在销售明细表和销售汇总表查看销售毛利外，还可以在单独的利润表中查看当前销售的利润，便于统计和经营决策。按指定的日期查询统计，能很清楚的看到当天的营业额，成本，费用支出和利润，以及收款状态该等，做到心中有数。 | 点击学习 |
| 3.11销售价格分析表 | 实现按商品、按商品+客户查询近10次销售价格，以分析销售价格波动。 | 点击学习 |
| 3.12销售税票汇总表 | 汇总统计一段时期内税票登记情况， 多次登记的单据会汇总显示在相同发票号下   | 点击学习 |
| 4.经营分析 |  | —— |
| 4.1销售分析 | 用户在【销售管理】-【经营日报】-【销售分析】中，可直观地查看销售相关的指标及对应的图表，能便捷高效地了解其整体销售状况，并提供销售策略的信息依据，若需要查看更多明细内容，可以直接通过不同的面板跳转到对应的报表。   | 点击学习 |
| 4.2销售环比分析 | 通过对日、周、月的周期性销售情况进行分析，让企业及时掌握商品的销售概况及其趋势，以便企业针对下一阶段的销售计划及相关策略进行制定   | 点击学习 |
| 4.3经营日报 | 可查看当天的采购、销售和库存的整体进销存情况，及时把控当天经营状况。查看很久没交易的客户数，提前预警以调整客户管理策略查询每天最新的滞销商品数，了解其库存及资金占用情况，及时诊断并制定对应的库存清理计划 | 点击学习 |
| 4.4滞销商品表 | 【业务场景】(1)通过滞销商品表，可快速了解商品的周转速度和库存积压情况，包括无销售天数，最近销售日期、入库日期、库存数量、金额等等，以便及时调整商品的销售策略，如通过促销活动加速库存周转和消耗。(2)如食品批发零售、加工、餐饮等行业，还可以结合保质期预警查询表一起使用，以便了解原材料或者半成品的积压情况及效期剩余时间，以便合理安排先进先出以及库存清理的工作，避免库存积压过多，效期变短带来的库存货值损失。(3)通过滞销商品表，结合销售分析表，可以对商品的整体销售效果有预判，通过筛选了解各商品类别、品牌下滞销商品的分布情况。 | 点击学习 |
| 4.5销售日报 | 通过销售日报，可以查询指定日期或者月份的销售概况，例如：销售订单总数、金额、销售出库单总数、金额、销售退货单总数、金额，还可以查看销售毛利及毛利率。   | 点击学习 |
| 4.6商品动销率表 | 主要用于当天查询节点为止，当前账套各仓库的库存商品总数，及本年、本月整体动销品种数和动销率指标，帮助企业了解商品的动销情况，以评估销售的效果，即库存商品中实际产生销售的商品种类及占比，从而制定有针对性的销售策略或指导补货。 | 点击学习 |
| 4.7企业经营利润表 | 按照业务维度出具的统计企业主营业务、其他业务的收入、成本、利润情况的综合报表,不同于财务云系统的利润表,用于统计进销存云系统的业务数据。   | 点击学习 |
| 4.8销售目标管理 | 销售目标管理支持按公司、部门、业务员、商品及商品品牌分别设置对应的销售目标。目标设置完成后，支持在电脑端和移动端随时查看销售目标达成情况，提供数据支撑及时做出决策调整。   | 点击学习 |
