# 引入表格里面的价税合计,和导入系统后的不一样是怎么回事?

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：常见问题
> 作者：云社区用户55os622 | 阅读：118 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/660649061569115392)

【问题描述】
引入表格里面的价税合计,和导入系统后的不一样是怎么回事?

例如下图,表格内商品的价税合计为100,引入系统后价税合计显示未99.99,是什么原因?
![](https://vip.kingdee.com/download/01090b192965a0864aeeb2b28f189aebb131.png)![](https://vip.kingdee.com/download/0109b6968761134a44a3be2d8d1816b1a63f.png)

【解释说明】
这是由于引入单据参数选择的是:【根据数量和单价计算价税合计】 或 【根据数量和含税单价计算价税合计】![](https://vip.kingdee.com/download/0109bb0d0a4f1d8e4fd3b384d23cfee6eef2.png)
![](https://vip.kingdee.com/download/0109515d846271294ed1a5b761196aca20f2.png)
当选择这些参数时,系统会重新根据数量和单价/含税单价计算价税合计。然而,Excel表格内的单价可能是通过价税合计除以数量反算得出的,这可能存在尾差。因此,在重新根据数量和单价/含税单价计算价税合计时,结果可能会与原表中的价税合计不一致。

以上述例子来分析:

|  | 数量 | 单价 | 价税合计 |
| --- | --- | --- | --- |
| Excel | 3 | 33.3(100/3=33.3333333333) | 100 |
| 引入系统后 | 3 | 33.3 | 99.9(33.3*3=99.9) |

【解决方案】
如需引入后的单据价税合计与表格中保持一致,将引入参数修改为【单价、价税合计按模板填写如实引入】即可![](https://vip.kingdee.com/download/0109b41f3c15927c4ce1a94ee909e9d4d98e.png)
