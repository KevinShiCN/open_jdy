# 保存单据或是导入单据时,出现提示:请填写“仓库”

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：常见问题, 进阶技巧
> 作者：进销存赵嘉丽 | 阅读：910 | 有用：9
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/462059981764916736)

【故障现象】

**现象1**:单据分录上明明已经录了仓库,为什么保存时还是提示请填写“仓库”?

![](https://vip.kingdee.com/download/010963afa1b1508a4ed88a4d848fbd359fc7.png)

**现象2:**或者引入时,明明填写了仓库,但是引入就是失败

![](https://vip.kingdee.com/download/0109dec8e492be3b49d595c9d09c1ebe016d.png)

【原因】

上面2个现象都是一样的原因,这个提示并不是校验单据分录上的仓库,而是**校验的单据头上的字段**。系统设置-单据字段配置是适用于一些以防开单人员漏填信息所做的一种强控制,**而单据字段配置里的“仓库”控制的是单据头。**

单据头的仓库其实是为了方便快速录入分录上的仓库而设的(即开单时,如果先填了单据头的仓库,后面新增商品行,就会默认携带这个仓库,不需要每行都录入一遍仓库)。

我们以销售单为例,假设用户在【系统设置】-【单据字段配置】里,开启了“仓库”必录(下图)

![](https://vip.kingdee.com/download/01092f384dd9a496440b9ffaa96cf202de52.png)而当单据头上的仓库没录时:

![](https://vip.kingdee.com/download/0109bc6dea6e6ba0408aaabaed37078e1304.png)保存时就会出现前面现象中的提示语。

【解决方案】

**情形1**:如果您确实需要对单据头的仓库做必录控制,那么只需要把表头仓库给填上即可保存

![](https://vip.kingdee.com/download/0109f246457118b84bc3919cb2051cbb8b9c.png)**情形2**:如果您的单据上一般会包含多个仓库,那么**不建议您设置单据头的必录,只需要关闭参数后,保存即可。**![](https://vip.kingdee.com/download/010917180a21ca8641daa66e29918456ec51.png)
