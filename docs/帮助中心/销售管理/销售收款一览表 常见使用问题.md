# 2 常见问题

## 2.1 销售收款一览表，查询时，为什么勾选“已收款金额为0不显示”，但是商品已收款为0的明细还是显示出来？

答：因为报表中的参数“已收款金额为0不显示”针对的是整单的收款，当应收应付参数选择“按分录核销”时，是到商品分录的，对应的收款字段是“商品已收款”，不是“已收款”，所以此时勾选了是不生效的：

![](https://vip.kingdee.com/download/01003b753bec68df41b988adfc8153d5d569.png)

![](https://vip.kingdee.com/download/0100c3133b471fdc4e058cb85e9a06450ccb.png)

只有应收应付参数选择“按单据核销”或“按余额核销”时，也就是按整单的收款时，

![](https://vip.kingdee.com/download/0100ee75a5c3b7bc4e3ebd3c1499b55760f2.png)

这个筛选项才能用，且对应的字段是“已收款”：

![](https://vip.kingdee.com/download/01005ca14245520d4468acdb8319ad243269.png)

## 2.2 查询单据日期为1月的单据，收款时间查询的是2月份，为啥查询结果里，其他1月已收款的不显示收款了呢？

如下图，查询1月份已收款的，共5笔的单据：

![](https://vip.kingdee.com/download/0100ba2c324034984b73b9ee25d1c45cfed8.png)

再查“收款日期”为2月份的，就只剩1笔了：

![](https://vip.kingdee.com/download/0100486684661bb745408601ea10ac070816.png)

答：销售收款一览表中

- 当筛选条件只选择“单据日期”,“收款日期”为空时，则统计的是单据所有的收款金额，汇总为“已收款”金额；
- 当用“单据日期”和“收款日期”组合查询时，报表逻辑是：先根据“单据日期”查询出该时间范围内的单据，再统计在“收款日期”范围内，这些单据有收款的记录，汇总为“已收款”金额

综上所述，上述的示例就是先查出1月份所有已付款的单据，再从这些付款单据里去过滤出符合“收款日期”为2月的，也就是上面看到的结果。

# 3 应用讲解

下面通过示例，详细说明单据日期、收款日期、已收款金额为0不显示的应用场景：

## 参数类型一：账套参数设置的是“按单据核销”或“按余额核销”

![](https://vip.kingdee.com/download/0100a2c6bc4c661a42f09e900d4a8ff05631.png)

2023年1月份查询共出13笔对应的销售单据，有5笔已经收款：

![](https://vip.kingdee.com/download/01002c2d14bc52544f1eab4a87ac17d693b9.png)

1）若只想查看这些单据的已收款金额统计，则可勾选 “已收款金额为0不显示”，这样筛选后，就只会显示有收款的5笔单据：

![](https://vip.kingdee.com/download/0100018f5724c99043e0a90fd183c35ed985.png)

2）若想看有多少是在1月份收的钱，那么指定收款日期为1月，就会发现有3笔，

![](https://vip.kingdee.com/download/01006600141cd2894afe9dc3ac7d5d58964a.png)

如果指定收款日期为2月，就只有1笔：

![](https://vip.kingdee.com/download/01000ea9e31266454aff9b7435ccc716af6c.png)

如果指定收款日期为3月，也只有1笔：

![](https://vip.kingdee.com/download/0100c3c50be4d83f4859b354bc691a703242.png)

这个查询结果就是基于“单据日期”先查询出符合条件的所有业务单据，再在这个范围下根据收款时间做进一步的过滤，是一个递进关系，查询结果也是逐步缩小和精确。

## 参数类型二：账套参数设置的是“按分录核销”时：

![](https://vip.kingdee.com/download/0100509a4ab99b06426d9a988edc57b1d3dc.png)

还是以上面的1月份数据为例，此时查询报表，收款字段对应的是“商品已收款”， “商品未收款”，即最细维度的记录，保持跟系统参数一致

![](https://vip.kingdee.com/download/0100aa6ff463140d4a22b67de78b0d2cd2da.png)

- **已收款金额为0不显示**

当按分录核销时，报表筛选项里的“已收款金额为0不显示”就不生效了（如下图），因为“已收款”是整单的字段，只有系统参数里按单据或分录核销才可以用，跟“商品已收款”是两个完全不同的字段。

![](https://vip.kingdee.com/download/01001447756aaeb04a25b6830cb64c681333.png)

- **那筛选项不能使用的话，要怎么查看商品已收款不为空的呢？**

答：可以使用**表头的快捷排序**，鼠标点击“商品已收款”字段，出现↓时，即可降序排序（见如下GIF动图），可以把已收款的商品排在前面，为空的排在后面。

![](https://vip.kingdee.com/download/0100b8d795312fe04b63b1887b9a2e0ddb86.gif)

注：后续星辰会考虑迭代增加“商品已收款”漏斗过滤来方便查询

- **按分录核销时，筛选条件****【收款日期】不能使用**

按分录核销时，筛选条件【收款日期】筛选后也是不生效的，因为“收款日期”对应的也是单据头的字段，并不会记录到商品分录上，所以不支持按收款日期来查询。

![](https://vip.kingdee.com/download/010056f2b969bb0c4ec389c4d0c40ced3d67.png)
