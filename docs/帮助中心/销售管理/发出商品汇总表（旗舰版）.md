# 发出商品汇总表（旗舰版）

# 1  简介

## 1.1  功能介绍

发出商品汇总表是根据当前的过滤条件，展示筛选出的发出商品的汇总数据，包括期初余额、本期发出、本期结转、期末余额。

![](https://vip.kingdee.com/download/0109c3a8d65590164db79ed780764e38b754.png)

## 1.2  适用版本

星辰3.1以上旗舰版

## 1.3  系统路径

销售管理>销售报表>发出商品汇总表

功能按钮通用操作说明[https://vip.kingdee.com/link/s/lSRAs](https://vip.kingdee.com/link/s/lSRAs)

# 2  报表查询说明

## 2.1  统计范围

发出商品汇总表中的数量、成本来源于以下单据的相关字段：

| 单 据 | 数量字段 | 成本字段 |
| --- | --- | --- |
| 销售出库期初 | 基本数量 | 出库成本本位币 |
| 销售出库单 | 基本数量 | 出库成本本位币 |
| 销售退货单 | 基本数量 | 出库成本本位币 |
| 销售发票(关联销售出库期初/销售出库/退货单的) | 基本数量 | 出库成本本位币 |
| 销售勾稽单 | 基本数量(销售出库期初/销售出库/销售退货单行) | 勾稽成本本位币(销售出库期初/销售出库/销售退货单行) |

## 2.2  数据说明

**期初余额：**数量取值截止上一期期末的单据（销售出库期初+销售出库单-销售退货单-销售发票-勾稽单）的基本单位数量余额；成本取值截止上一期期末的单据的出库成本本位币金额余额；当查询系统启用第一期时，期初余额来自销售出库期初单据。

**本期发出：**数量取值本期发生的销售出库单减销售退货单的数量；成本取值本期发生的销售出库单减销售退货单的出库成本本位币金额。

**本期结转：**发票结转+勾稽结转。

**发票结转：**数量取值本期与出库单关联的销售发票加与退货单关联的（负数）销售发票的基本单位数量；成本取值本期与出库单关联的销售发票加与退货单关联的（负数）销售发票的出库成本本位币。

**勾稽结转：**数量取值本期的勾稽单中的（销售出库单所在行的）勾稽基本数量；成本取值本期的勾稽单中的（销售出库单所在行的）勾稽成本本位币。

**期末余额：**期初余额+本期发出-本期结转。

## 2.3  查询条件

1）**查询期间**：选择一个业务期间,查询这一期的发出商品的汇总数据。

2）**汇总依据**：支持按商品、结算客户和仓库三个维度进行汇总统计。

按商品维度统计时，支持汇总统计商品的数量及成本；按结算客户维度统计时，仅支持汇总统计结算客户的成本；按仓库维度统计时，仅支持汇总统计仓库的成本。

3）**商品资料条件筛选**：如商品、商品类别、商品品牌、商品标签、规格型号，根据表头的筛选条件得到需要统计的商品信息，如果没有设置筛选条件则等同于选择基础资料中的所有商品。

4）**结算客户资料条件筛选**：如结算客户、结算客户分类，根据表头的筛选条件得到需要统计的结算客户，如果没有设置筛选条件则等同于选择基础资料中的所有客户。

5）**仓库资料条件筛选**：如仓库、仓库分类，根据表头的筛选条件得到需要统计的仓库，如果没有设置筛选条件则等同于选择基础资料中的所有仓库。

6）**审核状态**：可根据审核状态来筛选表体中需要参与统计的单据。审核状态仅影响查询期间的发生额（本期发出、本期结转），期初余额始终取截止到上期末的所有数据（含未审核数据）。

7) **无发生额不显示**：勾选时，将不统计所选时间范围内没有发生业务的商品/结算客户/仓库。

## 2.4  其他

1）在筛选设置中，将不常用的查询条件隐藏。

2）可将常用的查询条件保存为默认方案。

3）报表支持引出、打印、列设置。
