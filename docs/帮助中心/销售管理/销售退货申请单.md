# 销售退货申请单

# 1 简介

## 1.1 功能介绍

销售退货申请主要应用于销售发货后，如需发起退货要先发起申请，申请通过才允许退货的场景。

## 1.2 适用版本

星辰3.1以上专业版/旗舰版

## 1.3 应用场景

某些企业处理退货业务比较谨慎，需要先申请，审批通过后才能进行退货，当客户提出退货时，业务员可以通过销售退货申请单记录客户的退货需求，审批通过后，通知客户进行退货，仓库收到退货，下推生成正式的销售退货单，验收入库。

## 1.4 系统路径

销售管理>销售单据>销售退货申请

## 1.5 特殊按钮说明

- 作废：作废后将不能进行修改、删除、审核以及后续的退货操作，仅保留单据，允许查看。
- 反作废：即撤销“作废”。

# 2 操作指南

销售退货申请单是销售退货单的前置业务，也是销售出库单的后置业务，本系统提供三种销售退货申请单的操作方式：

- 方式1：打开销售管理>销售单据>销售退货申请，手工录入客户，退货商品及数量、金额。
- 方式2：在销售出库单列表中，找到对应的已审核的销售出库单，勾选后，点击【退货】下方的“退货申请”按钮，生成退货申请单。

![上传图片](https://vip.kingdee.com/download/010049cd10fdb1c54e6785dedfe1e884bfda.png)

- 方式3：打开销售退货申请单，选择客户后点“选源单”，打开销售出库单选择窗口，选择销售出库单，生成退货申请单。

# 3 常见问题

**1）销售退货单、销售退货申请单、销售换货申请单的应用场景，在什么情况下使用这三张单据？**

答：

- 在销售退货业务中，当仓库收到退货商品，可录入销售退货单，记录退货商品的详情，退货单保存或审核后，根据库存更新方式（单据保存时更新、单据审核时更新）会增加商品的库存；
- 某些企业进行精细化管理，或者一些特殊的商品，不允许随意退货，需要经过审批通过后才能退货，当客户提出退货要求时，业务员可以录入销售退货申请单，记录客户的退货需求，退货申请单审核通过后，通过客户退货，仓库收到退货商品，通过该销售退货申请单下推生成销售退货单，退货入库。
- 销售换货申请单则适用于换货的场景，即商品退货后，还要补发。销售换货单可以通过销售出库单下推生成，并在换货申请单中录入需要补发的商品，记录换入、换出的商品明细。审核后，可以通过销售换货申请单下推生成销售出库单和销售退货单。

>
注：销售退货申请单和销售换货申请单都不会影响库存。
