# 引入单据提示未录入数据,但实际已经录入了,是什么原因?

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：常见问题
> 作者：云社区用户55os622 | 阅读：256 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/660287104156477440)

【问题描述】
引入单据提示未录入数据,但实际已经录入了,是什么原因?![](https://vip.kingdee.com/download/01098b07c1b319f54e9c9ad87ccc18e619a2.png)

【解释说明】
出现该提示的原因:手动修改或删除了系统自带的表头字段,导致引入单据时系统无法正确识别到对应列的数据,从而判定为未录入。

例如,原客户名称表头为【*客户.名称 # customerid.nam**e*】*,如果手动删除了英文部分改为【*客户.名称 #*】*,此时系统将无法识别客户列,即使表格中已经录入了客户名称,系统也无法检测到。

【解决方案】
重新下载系统引入模板,保持表头字段不变,将内容对应复制到完整模板内再重新引入
