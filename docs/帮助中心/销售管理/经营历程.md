# 经营历程

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：快速入门, 精华汇总, 功能介绍
> 作者：进销存赵嘉丽 | 阅读：813 | 有用：1

## **1 简介  **

### **1.1 功能介绍**

经营历程支持汇总查询系统的销售管理、采购管理、库存管理、应收应收模块的所有单据（销售预测单、盘点单、资金转账单、调汇单除外），并显示不同类型单据的通用表头字段如单据日期、客户、供应商、部门、业务员、制单人、审核人、金额等。

经营历程可适用于以下场景：

- 老板希望查看业务员在一定时间范围内如当天做的所有单据，并检查单据的审核状态；
- 希望在一个界面查看与客户、供应商发生往来的所有业务单据和资金单据，如订单和出入库单一起查看，出入库单和退货单一起查看，出库单与收款单一起查看，方便连贯查询业务与核对资金；
- 财务希望在一个界面检查是否有涉及往来的单据未对账。

### **1.2 适用版本**

星辰3.1及以上所有版本

### **1.3 系统路径**

销售管理>经营分析>经营历程

## 2 功能特性

### 2.1 列表字段显示

#### 2.1.1 字段显示说明

单据的字段可见性受列设置、经营历程字段授权以及单据树列表（即下图中红框位置）选中节点控制。

![](https://vip.kingdee.com/download/0109bab9c9a042ce47b9aa9ac405734e59fe.png)

- 列设置关闭显示列，字段不可见
- 字段授权设置用户禁止查看字段，字段不可见

![](https://vip-admin.kingdee.com/download/0100f4f9e51505934567881ecc560eb7b2d8.png)

- 单据树列表选中特定的或汇总的单据类型无此字段时，字段不显示

如选中销售管理或者销售管理任一具体单据类型，不显示供应商、结算供应商、入库状态、付款状态等与销售类单据无关的字段；

选中全部或者通过表头单据类型过滤任意单据时，显示列表全部字段。

#### 2.1.2 字段取值说明

• **金额**

- 取自原单据列表的相应整单原币金额，退货退款类的单据金额取原币金额的负数；
- 金额支持总合计与已选行合计（ 列表列设置>列表控制>开启已选行合计）![](https://vip-admin.kingdee.com/download/0100ba86178676bf46eaa1e91689429f1680.png)
- 部分单据的金额取值特殊说明：

| 单据类型 | “金额”取值来源 |
| --- | --- |
| 暂估调整单 | 总金额差额 |
| 核销单 | 本次核销金额合计 |
| 销售换货申请单 | 本次应收账款 |
| 采购申请单 | 无 |
| 受托入库单 |  |
| 库存管理类 |  |

• **客户/结算客户、供应商/结算供应商**

- 在系统中未启用第三方结算时，经营历程中业务单据（销售管理、采购管理）与4张资金单据（其他收入单、其他收入退款单、其他支出单、其他支出退款单）增加显示结算客户、结算供应商字段，分别对应单据已有的客户、供应商；![](https://vip-admin.kingdee.com/download/0100c93dc5f204f74e089052e649f0a552e6.png)
- 资金单据（除其他收入单、其他收入退款单、其他支出单、其他支出退款单外）列表中增加显示客户、供应商字段，分别对应单据已有的结算客户、结算供应商；

以上2点均只影响经营历程列表显示，不影响原单据以及单据列表。

>
注意：核销单（手动核销）中“应收转应收”“应付转应付”两种核销类型存在转出方与转入方，暂不支持在经营历程列表查看结算客户与结算供应商。

- **部门、业务员**

| 单据类型 | 部门 | 业务员 |
| --- | --- | --- |
| 移仓单 | 调入部门 | 无 |
| 调拨出库单 | 调出部门 | 无 |
| 调拨入库单 | 调入部门 | 无 |
| 核销单 | 客户部门 | 客户业务员 |

### 2.2 单据汇总查询

**• **默认显示查询范围内的全部单据，单据按单据日期、单据编号、制单时间降序排列；

可以通过①表头筛选单据类型②单据树列表选中单据类型或者单据所属应用，实现过滤所需汇总查看的不同类型单据；

![](https://vip.kingdee.com/download/0100846a179f4e444e8f91d5c6992c41660c.png)

>
注意：若同时采用方式①与方式②，需两者查询的单据范围存在交集，否则查询无意义。

**• **提供单据列表跳转功能，便于查看单据相关分录行信息

![](https://vip.kingdee.com/download/0100d28e368d4a7f44a781b8309fc9ffb563.png)

**• **提供按列表引出功能，汇总引出查询的单据，或者仅引出已勾选单据

![](https://vip.kingdee.com/download/0100bdce27fe0ee9401c8112b9a39592ce95.png)

### 2.3 单据关系图

经营历程汇总查询的单据支持在页面全流程跟踪，且新增单据关系图功能。

![](https://vip.kingdee.com/download/0100c38bb2268af449fcbe139bbc7b924a75.png)

- 单据关系图不同于全流程跟踪/上查/下查以单据列表形式展示的是，一次选中一张单据后，以卡片形式展示单据的上游与下游
- 流程中同一类型单据数为3张及以上的单据卡片自动堆叠显示
- 对于启用工作流的单据，可以通过点击卡片，侧边栏查看单据的审批进度与审批详情

![](https://vip.kingdee.com/download/0100e867b747571a4b6290c5883266b5b364.png)

• 单据关系图支持权限控制，若用户追踪的单据本身有权限，但关联的单据无权限，单据关系图中的卡片同样无查看权限

![](https://vip.kingdee.com/download/01006cda240c190d45419bdd458043feb0ac.png)

>
注意：单据关系图支持追踪单据的上游与单据的下游，不支持追踪单据的下游单据的上游单据，或者单据的上游单据的下游单据。

**关于单据关系图VS全流程跟踪的使用建议：**

- 对于只想追踪单据的上游和单据的下游，**推荐使用单据关系图**，业务场景如销售出库单同单录入物流费用生成其他支出单，后续与物流供应商多笔支出单一起结算付款，查询销售出库单的单据关系图，只会显示同单生成的其他支出单，不会跟踪显示一起付款的其他支出单。
- 但对于以下业务场景，如销售订单同单生成预收款单YSKD-01，下推销售出库单并同单抵扣预收款YSKD-02，则查看销售订单的单据关系图，不会追踪到YSKD-02（属于订单的下游单据销售出库单的上游）；查看销售出库单的单据关系图，不会追踪到YSKD-01（属于销售出库单的上游单据销售订单的下游）；**此时建议结合全流程跟踪/上查/下查使用**。
