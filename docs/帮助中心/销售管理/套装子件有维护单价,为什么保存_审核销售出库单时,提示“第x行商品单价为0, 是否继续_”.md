# 套装子件有维护单价,为什么保存/审核销售出库单时,提示“第x行商品单价为0, 是否继续?”

> 产品：金蝶云·星辰 | 分类：销售管理 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：246 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/670433139910331136)

【问题描述】

套装子件商品明明有维护单价,为什么保存/审核销售出库单时,提示“第X行商品单价为0, 是否继续?”

![](https://vip.kingdee.com/download/0109a598c4913469436fa7c2cc4471f19457.png)
【原因说明】

本质在于“套装单价”为0,而并非“单价”为0。我们在列设置中,开启“套装单价”后,就会发现该字段为空。

![](https://vip.kingdee.com/download/010948f5b0c4889248059a3ceecf019628d6.png)

【解决方案】

1)要维护好套装商品内的价格资料

![](https://vip.kingdee.com/download/01094c91121dc55c42aebccae540f50311d1.png)

2)开启“销售类单据”价格取数优先级,要保证销售出库单可以取到价

![](https://vip.kingdee.com/download/010947aa686438f045a78105338d92647a67.png)
3)按以上步骤维护好后,可以选择在单据上重新录入该套装商品,也可以对销售出库单操作“重新取价”,这样可以保证“套装单价”有值,然后再保存/审核出库单就不会再提示单价为0了。

![](https://vip.kingdee.com/download/0109f4611e9e34c5452ea5c969b528602691.png)

【适用版本】

星辰3.0以上所有版本
