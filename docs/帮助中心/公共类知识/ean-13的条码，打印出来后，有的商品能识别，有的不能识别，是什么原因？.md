# ean-13的条码，打印出来后，有的商品能识别，有的不能识别，是什么原因？

> 产品：金蝶云·星辰 | 分类：公共类知识 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：393 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/685846459182285824)

## 1 问题描述

ean-13的条码，打印出来后，有的商品能识别，有的不能识别，是什么原因？

## 2 适用版本

星辰所有版本

## 3 原因解析及解决方案

### 原因一：条码规则没设对，使用了错误的编码规则/方式

正常的ean-13条码应该是下面表格第一行的样式。

| 编码规则 | 条码样式 |
| --- | --- |
| ean-13 |  |
| code 128 |  |

### 解决方案：

无论是通过星辰条码生成规则还是通过其他软件生成的，条码规则一定要是ean-13条码标签打印模板里的“编码方式”要选择ean-13，然后再重新打印，扫码验证。

关于ean-13编码的详细说明参考帖子：[条码设计基础](https://vip.kingdee.com/link/s/ZznHF)

---

### 原因二：条码规则是对的，但是校验码设置有误

如果编码规则本身选择的就是ean-13，但是依然不能识别，通常是由于条码最后一位的校验码设置有误导致的。

注意：EAN-13条码的最后一位是校验码，用于验证条码的正确性，第13位是根据前12位数值通过一定的计算规则算出来的，是不能手工随便录入的，如果手工录入的数值不等于校验码，就会导致扫码异常，无法识别。

### 解决方案：

怎么确定自己条码有问题呢？首先，用户也可以拿自己无法识别的条码和网上条码工具生成的条码进行对比，会发现最后一位的校验码应该是不一致的。如果发现确实不一致，那么我们按以下步骤操作。

1）通过网上第三方条码生成平台，规则选择ean-13，输入已经注册的条码的前12位，系统会自动生成最后一位校验码，从而输出一个完整的13位的条形码。

2）把正确的条形码重新维护到星辰的商品当中，同时重新打印正确的条码标签

3）扫码验证即可。
