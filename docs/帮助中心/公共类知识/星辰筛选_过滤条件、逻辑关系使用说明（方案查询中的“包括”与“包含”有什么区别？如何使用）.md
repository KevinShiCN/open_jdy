# **1 简介  **

## **1.1 功能介绍**

在进行查询时，我们需要通过一定的逻辑条件，来达到我们预期的搜索效果，这些限制了查询范围的字段被称作筛选条件或者过滤条件、条件。

## **1.2 适用版本**

星辰3.1以上所有版本

## **1.3 操作路径**

适用于各类单据列表、报表、单据详情中可以过滤查询的地方。

# 2 过滤条件

## 2.1 文本字段

### 2.1.1 含义

| 条件 | 含义 |
| --- | --- |
| 等于 | 筛选出与指定值完全相同的记录，是完全匹配 |
| 不等于 | 筛选出与指定值不相同的记录，是完全匹配 |
| 包括 | 筛选出包含指定文本的记录。精准匹配，相当于是多选，字段与字段的关系是“或” |
| 不包括 | 筛选出不包含指定文本的记录。精准匹配，相当于是多选，字段与字段的关系是“或” |
| 为空 | 筛选出字段值为空的记录 |
| 不为空 | 筛选出字段值非空的记录 |
| 包含 | 筛选出字段值中包含指定文本的记录。模糊匹配 |
| 不包含 | 筛选出字段值中不包含指定文本的记录，模糊匹配 |
| 以……开始 | 筛选出字段值以指定文本开始的记录，相当于左匹配 |
| 以……结束 | 筛选出字段值以指定文本结束的记录，相当于右匹配 |

## 2.2 日期字段

|  | 含义 | 说明 |
| --- | --- | --- |
| N天前 | 筛选从今天日期往前数N-1天之前的所有日期，过滤结果是一个时间段。 | 假设今天是1月5日，那么1天前，就是1月4日之前更早的所有日期（不含1月4日当天） |
| 前N天 | 筛选从今天日期往前数N天的日期，过滤结果是一个固定的日期值。 | 例如：今天是1月5日，当设置“前N天=1”，显示的都是1月4日的数据 |
| 从...到... | 用于过滤时间段，也就是从某个时间点作为起始点，到某个时间点结束，这个结果包含起始日和结束日当天。 | 例如：过滤从2025年1月1日到2025年1月31日的数据。过滤结果就是1月完整月份的所有数据。 |

# 3 逻辑关系

## 3.1 含义

目前在方案查询、条件设置等页面，有时候需要设置多个条件，我们会用到“并且”、“或者”两种常见的逻辑来组合查询。

| 逻辑 | 含义 | 例子 |
| --- | --- | --- |
| 并且（and） | 表示所有条件都必须满足，才能得到结果就像说：“既要……，又要……” | 从出库单列表中找出“单据日期=今天”并且“价税合计超过5000元”的单据。只有同时满足这两个条件的单据才会被选出来。 |
| 或者（or） | 表示只要满足其中一个条件，就可以得到结果就像说：“要么……，要么……” | 从商品表中找出“价格低于100元”或者“品牌是A”的商品。只要满足其中一个条件的商品都会被选出来 |

## 3.2 应用场景

- 方案查询
- 打印模板的过滤条件设置
- 用户管理的数据权限设置
- 工作流条件设置...

## 3.3 注意事项

- 方案查询中不支持使用“或者”，因为这样逻辑查询量会比较大，容易对数据库的性能有影响，故星辰的标准产品不支持
- 当同一个字段需要同时满足多个否定条件（例如“不等于”,“不包含”）时，必须使用“并且”

>
如：公司有财务部、采购部、销售部、物流部，管理员不允许张三看到财务部和采购部的单据数据。

- **用【或者】**

条件1：部门不等于财务部  或者  

条件2：部门不等于采购部

结果：张三可以看到采购部（条件1的结果），财务部（条件2的结果），最后还是看到了所有部门，这是错误的设置。

- **用【并且】**

条件1：部门不等于财务部  并且  

条件2：部门不等于采购部

结果：张三只能看到销售部、物流部，实现了目的。
