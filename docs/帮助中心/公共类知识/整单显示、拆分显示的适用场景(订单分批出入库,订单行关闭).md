# 整单显示、拆分显示的适用场景(订单分批出入库,订单行关闭)

【问题描述】

整单显示、拆分显示的操作方法和适用场景

【解决方案】

一、整单显示:

作用及场景:主要适用于在汇总表查询单据时,根据不同的查看需求:到单据明细(整单)或者分录明细(每一行记录),而选择不同的显示方式。

作用:可以将同一单据的分录进行拆分,方便对**分录行**进行相关操作(**入库/出库/关闭**)

     入库:对选中的商品进行入库。

     合并入库:对同一个供应商多张不同订单的商品同时入库。(如果是不同供应商的商品进行合并入库,则会按供应商生成多张入库单。)

     关闭:即终止选中的分录行或单据的当前业务流,不再执行下一环节,手工将其闭环。

应用场景:如适用于商品分批入库、出库,而不必等商品全部到仓后再入库,确保了库存管理的实时性及准确性,方便企业更合理有效安排业务。下面将举个例子说明此功能的使用。

场景一:商品分批入库

九州商贸公司从阿里巴巴购买了5种商品共计10件(如下图),先到货其中3种,需要先把已到货的商品进行部分入库,剩余的商品等到货后再入库。

在【采购订单列表】,找到要做部分入库的订单,在列设置(齿轮按钮)中,找到【选项】,开启“同一单据拆分显示”,点击【确定】:

刷新界面,可以看到同一单据已经拆分:

再根据商品到货顺序进行入库或合并入库操作:

场景二:订单行关闭

当采购订单或销售订单存在多分录的情况下,如果其中某一商品已经完成交易,不再执行下一节点操作时,可以在单据拆分的前提下对商品分录行进行手工关闭。

操作方法:

第一步,在单据列表界面右侧列设置(齿轮按钮)中,找到【选项】,开启“同一单据拆分显示”,点击【确定】:

第二步:勾选要关闭的行分录,点击【关闭】即可完成,关闭完成后,行关闭状态会显示“手工关闭”。

【温馨提示】

整单显示及单据拆分显示适用于:

各类单据汇总表,如销售订单列表/销售出库单列表/采购订单列表/采购入库单列表/调拨出库单列表/调拨入库单列表/盘亏单列表/盘盈单列表/组装单列表/拆卸单列表等。
