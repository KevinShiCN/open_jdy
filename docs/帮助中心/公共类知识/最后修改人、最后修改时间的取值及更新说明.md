# 最后修改人、最后修改时间的取值及更新说明

> 产品：金蝶云·星辰 | 分类：公共类知识 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：238 | 有用：1

【问题描述】

单据详情>操作日志里的最后修改人、最后修改时间是怎么取数和更新的?

![](https://vip.kingdee.com/download/0109c3c860b703304a1095a7af5439f72cf9.png)

【逻辑说明】

**1、含义**:最后修改人、最后修改时间记录了最后对这个单据操作了具体功能的用户及其操作时间。

**2、会更新这两个字段的常规操作:**

正常的单据操作,如:保存、审核、反审核、订单变更、金额变更等等(注意,打印不会更新最后修改人、最后修改时间)

**3、特别说明的场景:**

| 场景序号 | 应用操作 | 页面说明 | 是否会更新最后修改人、最后修改时间 | 操作前操作日志 | 操作后操作日志 |
| --- | --- | --- | --- | --- | --- |
| 1 | 提取库存 |  | 都更新 |  |  |
| 2 | 重新取价 |  | 都更新 |  |  |
| 3 | 单据下推下游单据,下游单据保存后反写源单 | 例如:销售订单下推销售出库单后,系统自动反写销售订单的相关字段(执行状态、出库状态、关闭状态等等) | 都不更新 |  |  |

【适用版本】

星辰3.1以上所有版本
