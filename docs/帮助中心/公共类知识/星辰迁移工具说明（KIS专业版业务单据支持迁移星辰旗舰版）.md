## 二、KIS云专业版不支持迁移到金蝶云·星辰的情况（请务必仔细阅读~！）

KIS云专业版存在下列情况的账账套，整体不支持迁移到星辰

| 会计期间设置为 13 期的账套 |
| --- |
| 财务和业务均未结束初始化的账套 |
| 科目级次超过 9 级，或增加了表外科目的账套科目体系为小企业会计制度或小企业会计准则，但增加了共同类科目的账套 |
| 会计制度是民间非盈利会计制度的账套 |
| 账套中辅助属性超过五个的账套辅助属性的账套 |
| 暂估差额生成方式是【差额调整】 |
| 录入启用期间前的采购发票/销售发票 |
| 其他收款单单张单表体同时存在正负数金额的分录 |
| 其他付款单单张单表体同时存在正负数金额的分录 |

## 三、其他注意事项

1）业务模块**需要结账至最新期间**，例库存单据中如果在2024年7期有业务数据，需要在KIS专业版中的【存货核算】——【业务期末结账】结账至2024年8期

2）目前仅支持账套版本号是**3.0.03.04**的KIS专业版账套进行迁移，如版本不匹配，建议按照前面的链接下载最新版本并升级账套

![image.png](https://vip.kingdee.com/download/010905a4a36a9b9846f6b4bcd21f7e8b4515.png)

3）作废状态的单据及凭证暂**不支持**迁移---【后续迭代优化】

4）现购业务发票迁移后将新增一个账户“现购业务账户”，并进行整单付款

5）现销业务发票迁移后将新增一个账户“现销业务账户”，并进行整单收款

6）单据自定义字段**不支持**迁移

7）组装拆卸单、盘点单、盘点方案**不支持**迁移

8）暂**不支持**表头金额是负数的预收单、预付单数据迁移---【后续迭代优化】

9）因KIS专业版上收付款单上的对应账户信息来源于会计科目，迁移后针对当前账户新增相应出纳账户

10）KIS 云专业版中的**用户名无法迁移**。业务单据、凭证、卡片、日记账迁移后，制单人、审核人会自动获取当前操作迁移的用户名---【后续迭代优化】

11）如果账套过大无法上传，请联系金蝶总部协助处理。此外如原有KIS专业版账套实体文件过大（mdf+ldf文件超过4G），我们建议您先通过金蝶KIS云专业版【系统工具】-【辅助工具】-【账套结转工具】，先结转新账套后再迁移星辰，原KIS云专业版数据可以继续在专业版中查询。

12）根据公司发文，升级前原账套的数据问题，需通过客户成功部提交“数据修复服务”申请

13)  **KIS云私有云恢复账套常见问题：**[https://vip.kingdee.com/article/283922085179815936](https://vip.kingdee.com/article/283922085179815936)

![image.png](https://vip.kingdee.com/download/0109e9d317bc07b84a9fb08ede4af29be75e.png)

**KIS专业版【业务余额+财务数据】迁移操作手册**: [https://vip.kingdee.com/link/s/lSjUl](https://vip.kingdee.com/link/s/lSjUl)

## 四、迁移操作步骤说明

1、账套备份

1）KIS云私有部署：通过KIS云专业版【系统管理】-【手动备份账套】，得到后缀为bak、dbb的两个文件

2）KIS云专业版公有云：

通过【账套列表】，操作中选择“手动备份”，

![image.png](https://vip.kingdee.com/download/0109b0378e2d2f0344eb8f0e13ff3a7a46ba.png)

再通过账套备份列表，操作中“下载”备份文件。

![image.png](https://vip.kingdee.com/download/0109281bc3fcfe324d449ecc22c9a847c3e8.png)

下载下来是ADF后缀的备份。![image.png](https://vip.kingdee.com/download/01094ed2aac133c34296aa405ed931e65968.png)

[需要把.ADF后缀改成.zip](https://vip.kingdee.com/tolink?target=http%3A%2F%2F%E9%9C%80%E8%A6%81%E6%8A%8A.ADF%E5%90%8E%E7%BC%80%E6%94%B9%E6%88%90.zip)：![image.png](https://vip.kingdee.com/download/01094b378d345ac546d696d827fbe1c1738b.png)

解压zip后，得到后缀为bak、dbb的两个文件![image.png](https://vip.kingdee.com/download/0109bc80ce756b3c461d8255b49003643a97.png)

2、请使用压缩工具将bak文件压缩成zip格式文件![上传图片](https://vip.kingdee.com/download/010057206beeddd1401196c53e434555c50b.png)

这里需要注意：

1）不要直接改bak的后缀为zip，需要使用压缩工具压缩成格式文件；

2）bak 的文件压缩前文件名不要使用特殊符号或空格，升级可能会识别不到文件。

3、进入金蝶云星辰旗舰版新建账套界面，切换到“第三方导入”页签，选择【从KIS云专业版导入】的迁移方案

![image.png](https://vip.kingdee.com/download/01099e8a2e8e0bee4d12a6735da4b0f8b6fd.png)

4、进入上传压缩文件界面，选择由bak直接压缩成的zip文件上传（**注，这里要取消勾选【仅业务余额+财务数据】**）

![image.png](https://vip.kingdee.com/download/010947881800077f482d92b5a7e18553d300.png)

5、账套上传后，自动预检并恢复数据到迁移服务器，检测数据是否属于BAK的压缩文件，是否满足迁移条件。

![image.png](https://vip.kingdee.com/download/01093e81ea4f189744438e2559825f180451.png)

6、预检成功后，我们建议您点击【检查日志】查看是否有不能迁移的数据。如果有，请在其他数据迁移完成后，再进行数据调整。

![image.png](https://vip.kingdee.com/download/010977ed6e7e3354480082eb11d5e05121ba.png)

7、预检成功后，点击“开始升级”，自动匹配当前初始化状态的账套进行升级

![image.png](https://vip.kingdee.com/download/0109edf090d33f3d4a9b961fbf66c65f5d0f.png)

8、迁移完成后，我们建议您点击【导入日志】查看迁移数据明细。如果有数据未迁移成功，会在日志列表显示，请根据日志提示进行数据修复后再迁移

![image.png](https://vip.kingdee.com/download/010941e837ae133a4221babadf52d3e536aa.png)

9、迁移成功后，可直接进入账套。如升级成功后，未出现进入账套按钮，请先退出星辰，重新登录账套即可。

![image.png](https://vip.kingdee.com/download/0109f7512d25ec2a4545a74c1956533caed7.png)
