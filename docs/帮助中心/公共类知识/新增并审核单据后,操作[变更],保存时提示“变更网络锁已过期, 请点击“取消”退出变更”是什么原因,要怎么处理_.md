# 新增并审核单据后,操作[变更],保存时提示“变更网络锁已过期, 请点击“取消”退出变更”是什么原因,要怎么处理?

【问题概述】

新增并审核单据后,操作【变更】,保存时提示“变更网络锁已过期, 请点击“取消”退出变更”是什么原因,要怎么处理?

![](https://vip.kingdee.com/download/0109dac011072d7e48939123ccfa060ebe77.jpeg)

【原因说明】

出现以上提示是因为有其他用户手工删除了业务锁。

重现路径:以上图的销售订单为例,当用户A新增并审核完销售订单后,此时点击了销售订单上的“变更”,进入了变更编辑状态,此时系统会自动产生一个业务锁,其目的是避免其他人对此单据进行修改。接着,在用户A进行变更的期间,用户B到[系统设置>网络控制]中删除了此业务锁。最后,当用户A完成变更后,操作“保存”就会提示“变更网络锁已过期, 请点击“取消”退出变更”。

【解决方案】

系统自动产生的业务锁是为了避免并发操作的场景,以免产生业务数据冲突。通常在业务完成后会自动解锁,无需人工干预。因此当产生了业务锁时,应尽量避免人为主动删除业务锁,以免影响其他用户的正常操作,增加不必要的解释。

【适用版本】

星辰3.1以上所有版本
