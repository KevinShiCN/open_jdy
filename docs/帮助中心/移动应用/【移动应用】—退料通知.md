# 【移动应用】—退料通知

> 产品：金蝶云·星辰 | 分类：移动应用 | 类型：快速入门, 整体介绍, 新特性
> 作者：Freya_peng | 阅读：170 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/664419321753482496)

# 1、 功能介绍

 星辰旗舰版增加《退料通知单》功能菜单，解决采购/委外检验后对于不合格的物料需要退回给供应商的场景

# 2、 应用场景

**场景1：**物料检验不合格时，收料通知需要下推退料通知，一方面通知仓管退料给供应商，另一方面通知供应商商品不合格退料情况

**场景2：**对于免检的物料，收料入库后如需退货，此时需要先做退料通知通知仓管人员（收料通知下推退料通知），仓管人员根据退料通知进行退货（由退料通知下推采购退货）

**场景3：**对于免检的物料，手动新增退料通知单通知仓管退货，退料通知单下推采购退货单

![](https://vip.kingdee.com/download/010984d0ea823636413096bceb8d9071a620.png)

# 3、 适用版本

• 星辰3.1旗舰版

# 4、 操作步骤

首先：需要系统参数-进销存参数开启了质检流程，退料通知单菜单才会显示

![](https://vip.kingdee.com/download/01004292d2f5d3b6425a920e93cec54f0b02.png)

系统路径：采购管理-退料通知/委外管理-退料通知（如看不到应用，可在应用管理处添加应用）

![](https://vip.kingdee.com/download/010911f1a06284924197aed1e87d644928d3.jpg)

1） 新增退料通知单，录入供应商后，用户去选源单

2） 跳转选源单页面，选择收料通知单，支持按整单选择源单，也支持按商品行选择源单，只能选到收料通知单判退数量大于0的是的数据

备注：仓库基础资料改动说明：基础资料-业务资料-仓库-增加了【仓库类型】，仓库类型增加了【待检仓】。【待检仓】仅收料通知单、退料通知单可选择，且收料通知单中需要检验的商品行才能选择【待检仓】

3） 已审核、未关闭的退料通知单，上查收料通知单

注意：

①退料通知单承接【采购】和【委外】两种业务，根据源单的类型自动写入收料通知单的【业务类型】。

②当收料通知单上判退数量大于0才能退料。

4） 退料通知单支持单据列设置，用户可依据单据的表头，表体，列表设置展示或隐藏，系统内置了关键必录字段不可设置，不系统内置必须展示的字段，用户可自行设置展示或隐藏。设置隐藏就在开单和详情界面不展示，移动端列设置只对移动端单据生效，与PC端列设置是分开的，互不影响。

5）其他说明

• 退料通知单还支持单据字段配置，单据操作控制，自定义字段、工作流、消息通知单等。

• 当在星辰旗舰版的当前账套购买了云蝶轻智造管理系统的“质量管理”模块“或”为谷的“质检管理”，在系统参数-进销存参数可自行配置显示生态的单据还是星辰自有的退料通知单，当已购买生态对应模块，默认不显示星辰的退料通知单
