# 移动应用-开单选商品/拣验货优化（食品行业）

> 作者：移动生产白建凡 | 阅读：688 | 有用：0

# **1 简介**

## **1.1 应用场景**

- 食品行业中，老客户复购时既要选之前常买的商品，又要加几种新品。开单时想快速调出该客户近期的交易记录，快捷开单
- 食品出货时，通常需要扫码快速拣验货，由于不同拣验状态的提示音完全一样，即使系统会有弹窗提示，也容易被忽略，进而出现扫到仓库里相邻的非本次出货商品，没及时发现就误拣出去，导致错发货；以及部分商品扫码后数量超出订单需求的情况
- 食品行业中，商品常存在多个单位（比如可乐有“瓶”和“箱”两种单位），开单时会显示所有单位，容易出现做销售单时选成采购单位，做采购单时选成销售单位
- 食品行业中，序列号/批次商品扫码开单时希望可以优先选择在库数据

## **1.2 适用版本**

- 星辰3.0以上所有版本

## **1.3 功能特性**

- 移动端开单选商品界面显示“近期交易”标签。选中后显示该客户一段时间内交易的商品，默认显示上月，也能自定义时间范围
- 移动端扫码拣验货时，已拣页签和当前扫码商品页面增加“已拣验整件散包”字段显示；PC端拣验工作台待拣页签，“待拣数量”改为“整件散包数”显示。扫码拣验货时开启播放声音，拣验失败或超出拣验时增加异常音提示
- 移动端开单时，在选商品或购物车界面的“商品单位”模式中，增加“方案5：仅显示单据开单默认单位”。选择该方案后，多单位商品只会展示在基础资料>商品中设置好的各类单据的开单单位。即开采购单时仅展示该商品的 “采购单位”；开销售单时仅展示该商品的 “销售单位”。不再显示无关单位，单位选择更精准
- 移动端序列号/批次商品扫码开单时，支持设置优先选择在库数据

# **2 特性亮点**

## **2.1 开单选商品显示近期交易标签**

开单选商品增加“近期交易”标签。选中后显示该客户一段时间内交易的商品，默认为上个月，也可以切换为本月、近三个月（包含当月），以及自定义时间区域

![上传图片](https://vip.kingdee.com/download/010052293cdc33244a97a686e6884bce0cd9.png)

适用单据：

销售类单据：销售订单、销售出库单、销售图退货单

采购类单据：采购申请单、采购订单、采购入库单、采购退货单

## **2.2 扫码拣验货体验优化**

- 移动端扫码拣验货时，已拣页签和当前扫码商品页面增加“已拣验整件散包”字段显示；PC端拣验工作台待拣页签，“待拣数量”改为“整件散包数”显示
- 扫码拣验货时开启播放声音，拣验失败或超出拣验时增加异常音提示

## **2.3 开单支持按开单单位作为默认单位开单**

移动端开单时，在选商品或购物车界面的“商品单位”模式中，增加“方案5：仅显示单据开单默认单位”。选择该方案后，多单位商品只会展示在基础资料>商品中设置好的各类单据的开单单位。即开采购单时仅展示该商品的 “采购单位”；开销售单时仅展示该商品的 “销售单位”。不再显示无关单位，单位选择更精准。功能展示：

（1）以 “可乐”“雪碧”“芬达” 三种商品为例（计量单位为 “瓶”、辅助单位为 “箱”，采购单位设为 “箱”、销售单位设为 “瓶”）

（2）移动端开单选商品页面，把“商品单位模式”设置为“方案5：仅显示单据开单默认单位”并保存

![上传图片](https://vip.kingdee.com/download/010084db4d97918744019d41751a404890d1.png)

- 开采购类单据时：商品列表中 “可乐”“雪碧”“芬达” 的单位仅显示 “箱”（下图左）
- 开销售类单据时：商品列表中 “可乐”“雪碧”“芬达” 的单位仅显示 “瓶”（下图右）

## **2.4 批次/序列号商品开单**支持优先选择在库数据

- 出入库类单据，批次 / 序列号商品开单时支持设置扫码模式。默认为 “快速模式（直接扫码）”，可修改为 “普通模式（手工选择在库数据）”，系统会记忆设置，下次开单默认沿用上次模式
