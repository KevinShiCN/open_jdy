# 移动直运销售

# 1 简介

## **1.1 应用场景**

- 直运销售功能适用于电商、批发、家具、数码家电等行业，适配 “客户订货后厂家直发货至客户地址” 场景。可解决销售公司直运业务中无实际出入库却需手工多次录单的低效问题，以及供应商收采购订单后不知客户联系人、送货地址的痛点

## **1.2 功能特性**

- 在PC端系统参数>销售参数下，增加“启用直运”参数，默认不开启。若开启，在销售订单中增加显示“业务模式”字段，用户可选择是“普通销售”还是“直运销售”。同时，销售出库单/退货单、采购订单、采购入库单/退货单也会增加显示业务模式，以记录该笔单据是否为直运业务
- 在单据流程上，直运销售订单可下推生成直运采购订单，直运采购入库单可下推生成直运销售出库单，直运销售退货单可下推生成直运采购退货单，支持全流程跟踪

![](https://vip.kingdee.com/download/0100bd3630504f65475183611775c57b0627.png)

## **1.3 适用版本**

星辰3.1及以上专业版和旗舰版

# **2 功能特性**

## **2.1 基础配置**

若要使用直运功能，需先在PC端系统参数>销售参数下，开启“启用直运”参数

![](https://vip.kingdee.com/download/0100a089739b70064401b1013cb2331ab119.png)

开启“启用直运”参数后，系统会自动在仓库基础资料增加显示“直运仓”，该资料不可编辑，目前仅支持唯一的直运仓，不支持新增，以便开单时可以根据业务模式快速携带仓库

![](https://vip.kingdee.com/download/01000123d23eff9b445792d2f8ab53e612b0.png)

## 2.2 直运销售相关单据功能说明

### 2.2.1 销售订单

**（1）新增销售订单时，可选择业务模式**

- 若为“普通销售”，则按系统原流程执行，如下推生成销售出库等，普通销售也支持部分行开启直运
- 若为“直运销售”，商品行会自动携带出直运仓且不可修改，用户可录入商品、数量、金额等信息，单据审核后，不可下推销售出库单，只能下推采购订单

![](https://vip.kingdee.com/download/0100689514325fdc4b0eb0a13687c257394e.png)

**（2）****在销售订单单据详情或单据列表**

在销售订单单据详情或单据列表中，提供了“直运采购”功能按钮，单据列表支持一对一下推和合并下推，下推时支持设置生成规则（PC端设置规则，移动端同步生效），用于设置采购订单携带的字段：单据日期、单据备注、业务员、部门、商品行备注、预计交货日期。对于商品、客户、客户收货地址则是默认携带到采购订单

![](https://vip.kingdee.com/download/01002ddf3419dccb4a6a965cf32541e939f8.jpeg)

**（3）销售订单生成采购订单**

用户可在销售订单的商品行中录入每行商品的供应商，未控制必录性，若未录入，可在生成采购订单时补录，若已录入，则采购订单会按销售订单商品行中的供应商拆单，相同供应商合并为一张采购订单，不同供应商生成不同的采购订单

### 2.2.2 采购订单

- **采购订单若是由销售订单下推生成**，业务模式、商品和数量不可修改
- **若是手工新增采购订单**，则允许修改业务模式，如选择“直运采购”，商品行中的仓库为直运仓，不可修改

![](https://vip.kingdee.com/download/01001de37264fbb84147aef7e5172e3d58be.png) ![](https://vip.kingdee.com/download/010019f225c57c7b4b45b17482b2864d26bf.png)

- **采购订单下推生成采购入库单**：直运采购业务模式与普通采购模式逻辑一致，允许多次入库。对于星辰旗舰版，支持先货后票，也支持先票后货

### 2.2.3 采购入库单

- **采购入库单若是从采购订单下推生成**，业务模式、客户不可修改
- **若是手工新增采购入库单**，则允许修改业务模式，支持新增直运采购的采购入库单，以满足未启用订单管理流程的客户需求。同样的，若选择直运采购，商品行中的仓库为直运仓，不可修改
- **采购入库单生成销售出库单**：在采购入库单单据详情或单据列表中，提供了“直运出库”功能按钮，单据列表支持一对一下推和合并下推，下推时支持设置生成规则（PC端设置规则，移动端同步生效），用于设置采购订单携带的字段：单据日期、单据备注、业务员、部门、商品行备注。对于商品、客户、客户收货地址则是默认携带到销售出库单

![](https://vip.kingdee.com/download/0100f55c14933fbc4a84946838d49d9a7be1.png)

合并下推时，系统自动区分不同客户的采购入库单，将相同客户合并为一张销售出库单，不同客户拆分不同的销售出库单

### 2.2.4 销售出库单

- **销售出库单不允许新增直运模式的单据，只能通过采购入库单下推生成**。生成时，销售出库单不允许修改业务模式、供应商、商品及商品数量，若该采购入库单的源头有关联的销售订单，则该销售出库单会自动关联销售订单，反写销售订单的状态及出库数量。销售出库单保存后，采购入库单会记录直运销售出库状态为“已销售出库”
- **销售出库单生成销售退货单**：直运销售业务模式与普通销售模式逻辑一致，允许部分退货
- **销售出库单生成销售退货申请单**：直运销售业务模式与普通销售模式逻辑一致，允许部分退货申请

### 2.2.5 销售退货单

- **允许手工新增直运销售退货单**，以便选源单时通过业务模式过滤出直运销售出库单，若为直运销售模式，商品行为直运仓，不可修改，表体的供应商在单据审核时要求必录
- **销售退货单生成采购退货单**：在销售退货单单据详情或单据列表中，提供了“直运退货”功能按钮，单据列表支持一对一下推和合并下推，下推时支持设置生成规则，用于设置采购退货单携带的字段：单据日期、单据备注、业务员、部门、商品行备注。对于商品、供应商则是默认携带到采购退货单。批量下推时会根据销售退货单商品行中的供应商拆分生成采购退货单。

![](https://vip.kingdee.com/download/01005e090397b3c24e1eb4bed8e6864516c1.png)

### 2.2.6 采购退货单

- **采购退货单不允许新增直运模式的单据，只能通过销售退货单下推生成**，生成时，采购退货单不允许修改业务模式、商品及商品数量，若该销售退货单的源头有关联的采购订单、采购入库单，则该采购退货单会自动关联采购订单、采购入库单，反写采购订单、采购入库单的退货数量。采购退货单保存后，销售退货单会记录直运采购退货状态为“已采购退货”

## 2.3 单据联查

以上流程，都可以在单据详情页的“联查信息”中展示关联单据

![](https://vip.kingdee.com/download/0100707c05eb4d964a2e8f13a78746579ef1.png)

## 2.4 待发布功能预告

- 退货支持退给供应商、退回企业两种方式，退给供应商表示需要操作销售退货单生成采购退货单的流程；退回企业表示不需要退给供应商，只收回自己的仓库，可以选择非直运仓存放库存
- 自动生单，系统参数，启用直运参数下方增加销售订单审核时自动生成采购订单、采购入库单审核时自动生成销售出库单、销售退货单审核时自动生成采购退货单，由用户根据需要勾选
