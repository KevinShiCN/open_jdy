# 如何录入账户初始数据?

【问题描述】

账户初始数据操作指南

【相关视频】

[账户初始数据录入](https://vip.kingdee.com/school/167570444780679936?productLineId=35&lang=zh-CN&cosmic_type=video)

【解释说明】

账户初始数据为账套启用期间之前的账户余额,作为期初数录入系统,以便系统后续业务计算出正确的账户余额。

**tips:1.账户初始数据录入/修改仅支持在启用期间录入,如果已经结账需要反结账到启用期间;**

**2.出纳日记账的期初余额也在账户初始数据处录入/修改。**

【操作流程】

**1、期初启用时账户初始余额录入**

单击【基础资料】-【账户初始数据】,即可进入账户初始数据录入界面,系统默认携带当前基础资料中设置好的账户。

![](https://vip.kingdee.com/download/0109895f85e6937842a6aff5d5bc46ee6ada.png)**
**

**2、使用中的账套中途增加的账户可以做一张其他收入单增加该账户的余额作为初始余额。**

**3、根据账套启用期间不同,需要录入不同的账户初始数据**

3.1、启用期间为当前年度第1期时,只显示年初余额;

3.2、对应启用期间为当前年度非第1期时,以第3期为例,需要录入3月期初余额,1-2月收入合计,1-2月支出合计,2020年初余额由系统根据录入的数据计算填列,年初余额=期初余额-收入合计+支出合计;

**4、账户初始数据启用期间默认规则**

4.1、如果进销存中的启用期间小于出纳启用期间,以业务期间为初始期间,并在出纳日记账中通过**【重算初始余额】**调整出纳账户的期初余额

如:进销存启用期间2023年2期,出纳启用期间2023年3期,就把截止到23年1月底的余额作为账户初始余额录入账户初始余额

![](https://vip.kingdee.com/download/0109df02f67b54a44f03b754e7e2df76f1e9.png)
若进销存2月发生了收支业务并录入了相关收支单据,出纳管理因为启用期间是3月,无法录入2月的日记账,所以需要重算初始余额,保证两边账户余额一致。

![](https://vip.kingdee.com/download/010984d7c61011a4453fb02a37d64a9b88c5.png)

4.2、如果进销存中的启用期间大于出纳启用期间,以出纳启用期间为初始期间,**同时在账户余额表中通过【同步出纳余额】调整账户期初余额**

如:进销存启用期间2023年2期,出纳启用期间2023年1期,就把截止到22年12月底的余额作为账户初始余额录入

![](https://vip.kingdee.com/download/0109ed53275d383a4936a127bfa45d88281b.png)
若出纳录入了1月的日记账,因为进销存的启用期间是2月,无法录入1月的收支单据,所以进销存需要在【应收应付】-【账户余额表】操作<同步出纳余额>更新账户的期初余额,保持两边账户余额一致

![](https://vip.kingdee.com/download/010950d460d9b9e045b49cecd9b0c5461f91.png)
