# 销售出库期初（仅星辰旗舰版适用）

> 作者：进销存赵嘉丽 | 阅读：1792 | 有用：3

# **1 简介**

## **1.1 **应用场景

业务系统处于初始化阶段时，对于启用期间之前发生的已销售出库、未确认应收账款的业务，需在系统中继续跟进，可以通过录入销售出库期初单来记录。假设系统的启用期间为2023年1月，对于启用期间之前的比如2022年发生的销售出库还未确认应收账款的部分就可以录入销售出库期初单，后续可对其进行下推销售发票。

《销售出库期初》不会减少库存，也不会增加应收账款，只是作为期初单据记录启用期间前的未完成的业务。确认应收账款时，可通过《销售出库期初》单下推销售发票，形成应收账款。

## **1.2 操作路径**

基础资料>期初录入>销售出库期初

## **1.3 适用版本**

星辰3.1以上旗舰版

# 2 操作指南

## **2.1 手工新增**

启用账套后，可通过菜单路径【基础资料->销售出库期初】新增销售出库期初。

单据新增时，注意事项：

**• 业务类型**

区分“出库”、“退货”，在下推销售发票时，会根据业务类型生成相应的业务类型的销售发票，出库对应销售出库，出库类型的销售发票会增加应收账款，退货对应销售退货，退货类型的销售发票减少应收账款。

**• 发票类型**

并不会限制销售发票的类型，相当于一个备注字段，说明前期确认的这笔业务的发票类型，在下推销售发票时，销售发票中可以修改为正式确认的发票类型。

**• 结算客户**

若系统参数“启用第三方结算”开启，则业务单据中会增加显示“结算客户”，区别于“客户”。客户用来记录收货方，结算客户用来记录结算应收账款的单位。客户与结算客户可以相同，也可以不相同，都为单据的必录项。通过销售出库期初下推生成销售发票时，销售发票中的客户、结算客户不允许变更。

• 录入单据时，提供了"选项"功能，控制修改价税合计、税额时的对其他字段的影响。

• 当系统的第一期即启用期间进行了业务期末结账，《销售出库期初》不允许再录入。

## **2.2 批量引入新增**

可根据业务需求设置期初引入的模板，下载模板后按模板要求维护好对应的信息后再进行引入，系统提供了若干的预置模板，用户可按需下载，也可以自定义引入模板。
