# 供应商价格资料支持限价

# **1 业务场景**

同一商品向不同供应商采购，采购价不同，且不同供应商的限价要求也不同。比如，家具五金行业，公司的商品有常用的采购供应商，但是当常用供应商没有货时，就会去找其他供应商采购，而常用供应商和临时供应商的采购价要求是不同的，所以在限价的时候，需要能按供应商+商品的维度做采购限价。系统支持按照供应商价格资料设置限价。

# **2 适用版本**

星辰3.0以上所有版本

# **3 功能介绍**

## 3.1 供应商价格资料支持设置限价

1、供应商价格资料，“供应商组+商品”、“商品+供应商”两种方案类型支持设置限价。

![](https://vip.kingdee.com/download/0100693b9bf3174a49b7b5b878129cfd5e76.png)

2、 以“商品+供应商”方案类型为例，在价格方案详情中，可按商品+供应商，分别设置不同供应商的采购限价。

![](https://vip.kingdee.com/download/010005f93faa4e5b441d939927353e9dffae.png)

3、 供应商价格资料设置限价完成后，在开单时，可根据采购销售限价中设置的限价单据范围，控制采购限价。

![](https://vip.kingdee.com/download/010044be99ed2f2e4944808c559e9324bc7a.png)

采购限价时，若按供应商+商品匹配到供应商价格资料的方案，优先按供应商价格资料控制限价；若无匹配的供应商价格资料方案，再根据采购限价方案的优先级，读取下一优先级的限价，最终仅会执行其中一个限价。

![](https://vip.kingdee.com/download/01007b65ca5b259e4e17a1ae903fd9e8b178.png)

## 3.2 采购、销售限价逻辑优化

本次还优化了销售限价的逻辑，以解决用户反馈的关于销售限价未按客户价格资料中设置的限价控制的问题；同时，采购类单据限价也会按最新的逻辑来执行，即不依赖于单据取价，在单据保存或审核时，优先按供应商价格资料中的方案进行限价控制。
