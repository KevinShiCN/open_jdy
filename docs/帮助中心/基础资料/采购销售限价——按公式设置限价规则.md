# 采购销售限价——按公式设置限价规则

# **1 简介  **

## **1.1 功能介绍**

- 采购销售限价支持按公式设置限价规则
- 采购销售限价支持设置方案优先级

## **1.2 应用场景**

- 电子行业，经营电子元器件，按照最小单位来管理商品，单价相对较低，比如某款商品正常销售价为0.013元，但开单时输错成0.13元或0.0013元，就会导致单价过高或过低。希望在商品销售毛利过高或过低时弹出警告，例如将商品的销售限价设为“销售单价不能低于最近采购成本的120%”。
- 企不同业的商品限价规则不同，例如：商品A需设置“销售单价不能低于最低销售价”，商品B需设置“销售单价不能低于参考成本”。那么可以通过销售限价的优先级处理此业务诉求。

## **1.3 适用版本**

星辰3.1以上所有版本

## **1.4 系统路径**

基础资料>价格设置>采购销售限价

# 2 操作指南

打开基础资料>价格设置>采购销售限价，选择采购/销售类型进行新增。支持同时设置多个限价规则，并设置公式和优先级。

![上传图片](https://vip.kingdee.com/download/0100854f420545a24b07a279c34bdc564ceb.png)

**注意事项：**

1） 新增限价方案时，一行为一个规则，需要设置一个优先级，优先级不能重复。

2） 限价对象中的实际销售单价，对应单据上的“单价/含税单价”字段，具体是哪个字段取决于系统参数是否设置了价格为含税价

![上传图片](https://vip.kingdee.com/download/0100e7cea165482745fd86906454790116f9.png)

3） 关于限价方案优先级，说明如下：

         A、数值越小，优先级越高；

         B、商品的限价依据有值(大于0)的方案才会参与限价；

         C、单据限价取到高优先级限价方案后，就不再执行低优先级的限价，即商品最终执行的限价只有一个方案。

4） 维护了限价方案，还需要设置【限价单据设置】，制定限价的单据范围和控制规则，否则，单据未启用限价，无法执行限价方案。

# 3 **应用说明**

下面通过销售出库单的一个例子进行说明。

第一步，商品价格资料中，商品A的参考成本为0或为空，最低销售价有值，商品B的参考成本有值，最低销售价为0或为空。

第一步，销售限价里，设置实际销售单价不能低于最低销售价，优先级为1；同时设置销售限价不能低于参考成本，优先级为2；

第三步，设置限价单据为“销售出库单”，保存时预警提示。

实现结果为：

新增销售出库单，系统参数设置价格为不含税价，录有商品A和B，并且都录入了单价，读取到优先级为1的限价方案。

此时商品A按照最低销售价进行限价控制，不再执行其他方案的限价；商品B由于最低销售价为0，不参与此方案的限价，继续获取优先级为2的限价方案，按照参考成本进行限价。
