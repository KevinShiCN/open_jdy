# 星辰2.0进销存单据条形码打印常见问题

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：585 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/408191356306318592)

# 本文可解答以下问题

1、 打印模板里有2个条形码字段，有啥区别吗？应该绑哪个？

2、 为啥商品资料里的条形码有值，但是单据上的条形码还是为空？

3、 当商品资料里的条码更新后，历史单据打印时，是否可以取到最新的条码？

4、 条码备注支持打印吗？

5、 为啥商品资料里的条码备注有值，但是打印出来还是为空？

**适用版本**

星辰2.0

# 详细解答

## 1、 打印模板里，商品编码下有两个条形码字段，有啥区别吗？应该绑哪个？

![](https://vip.kingdee.com/download/0100a19b5174437d4baf925ae4cff6e3902a.png)

答：

- 商品编码.条形码-----对应即商品-基本信息中的，也就是基本单位的条形码；
- 条形码-----对应商品-商品条码中的条形码，即对应单位的条形码，适用于多单位商品。如果每个单位都维护了条码，开单时选了什么单位，业务单据上也是取这个单位对应的条形码。

![](https://vip.kingdee.com/download/01007a32508c797d435fa6ffec1cae8b6477.png)

![](https://vip.kingdee.com/download/0100bdd73fa768f34b9c95ea5f3ad20e6679.png)

## 2、 为啥商品资料里的条形码有值，但是单据上的条形码还是为空？

![](https://vip.kingdee.com/download/01007da965dfde7b4cd7bfb65dfab1b48e94.png)

![](https://vip.kingdee.com/download/01003c189621b5524cb4810e20e403e535d8.png)

答：一般出现这种情况都是先开单，后维护商品资料中的条形码导致的。

针对历史单据上已录入的条形码，系统**不会**取最新的商品资料上的反填到单据上，记录的都是用户最早录入的信息。

解决办法：如果需要单据上显示最新的条形码信息，需要修改单据，重新录入商品行，这样就可以带出最新的条码。

![](https://vip.kingdee.com/download/0100f961ae2fb46c4d75a68bb20c71082a21.gif)

## 3、 当商品资料里的条码更新后，打印历史做的单据时，是否可以取到最新的条码？

答：会更新。只要单据上的条形码有维护值，那么打印时，会取对应的商品资料里最新的条码。

## 4、 条码备注支持打印吗？

答：支持。但是前提是单据分录上的条形码有值，且该商品资料对应的条码备注不为空。

![](https://vip.kingdee.com/download/0100c8fbf0c63e8445b6802b186de70f4588.png)

![](https://vip.kingdee.com/download/010022d7af45783e4b2ebda2d469efcc4d92.png)

打印预览效果：

![](https://vip.kingdee.com/download/01002d6866c89e71490990be570ab61ddcf3.png)

**注意！如果条码备注对应的条形码已经发生变化了，也就是商品资料中条形码没有跟单据上的条形码一一对应，那么现在打印时，条码备注会为空。**

例如下面示例：

商品SP00093，单位【个】对应的条形码已经变成SP00093，但是单据上的条形码还是之前的SP00093CC，

![](https://vip.kingdee.com/download/0100db990246347c4230a5475cdd2ee17baa.png)

![](https://vip.kingdee.com/download/0100fde64a7a3eb24de6b7597e4726a165b4.png)

打印预览就会取不到条码备注：

![](https://vip.kingdee.com/download/01009b74c35d65bf416eb37601fb0739dc09.png)

## 5、 为啥商品资料里的条码备注有值，但是打印出来还是为空？

答：如果一开始录单时没有维护条形码，导致开单时，单据上的“条形码”为空，打印时就无法带出条码备注。后面哪怕在基础资料上维护了值，还是取不到的，一定要重新选一遍商品才行，保证商品分录的【条形码】有值就可以打印条码备注。

例如：商品SP00093一开始单位【个】没有维护条形码，但是条码备注有值。

![](https://vip.kingdee.com/download/0100415af5fc53c547e28cc656c57809e177.png)

那么开单时，单据上【个】对应的条形码也会为空：

![](https://vip.kingdee.com/download/0100dfa3ebfa521243968142ac5de59a1a03.png)

那么打印预览时，条码备注也会为空：

![](https://vip.kingdee.com/download/01002dd82e2e10264a75a78fbe435ebddca9.png)

那么后面就算重新维护上了商品资料里【个】对应的条形码，系统也不会去更新单据上有已经录入的条形码，从而导致条码备注也无法打印。

![](https://vip.kingdee.com/download/010088cc476b9a5941579880e79cd58546e3.png)

因此解决的办法就是：修改历史单据，重新录入商品行，带出最新的条形码，这样打印的时候条码备注就有值了。

注意，已审核未关闭的销售订单/采购订单可以使用订单变更的功能来修改，但如果单据已经关闭了或者无法反审核，这种情况下不支持修改。
