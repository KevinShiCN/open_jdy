# 商品列表发送商品提示换算率必须大于0怎么处理?

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：常见问题
> 作者：Schatz | 阅读：147 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/655575734764556544)

【问题描述】

商品列表发送商品提示换算率必须大于0怎么处理?

![](https://vip.kingdee.com/download/01093a48f7d1aa6e4550aa424092ec3e4c5a.png)

【解释说明】

当发送方的商品使用了带有小数位的基本单位数量,而接收方的账套中该计量单位未设置小数位或不存在此单位时,如果商品的换算率低于0.5,接收方将进行四舍五入处理,导致换算率变为0,从而无法保存商品。

例如,如下图所示,商品的基本单位为“新”,设置的多单位换算率为1个=0.25新;

![](https://vip.kingdee.com/download/0109cfebd2a827364691861134764cb41caa.png)

但接收方账套中“新”这个单位的设置没有小数位,因此换算率四舍五入后变成0,导致无法成功发送。

![](https://vip.kingdee.com/download/01097465a2bd06a640c2be964c766a8cb589.png)

若将接收方账套中“新”这个单位的小数位改为1位,即可成功发送,此时四舍五入后的换算率为0.3。

![](https://vip.kingdee.com/download/0109815a5afc45de4766b0cf0350cdaa8545.png)

为了确保两个账套的换算率一致,需要先统一两个账套中的小数位设置,然后再进行商品发送。
