# 为什么明明没有创建过部门，部门里出现了系统自动创建的本部部门？而且编辑会提示：“本部不允许修改!”有什么解决办法吗？

> 作者：进销存赵嘉丽 | 阅读：970 | 有用：15

【问题概述】

为什么明明没有创建过部门，部门里出现了系统自动创建的本部部门？

![上传图片](https://vip.kingdee.com/download/0100db09a71d80af4dac971d5f3b06f99a1d.png)

而且点击编码会提示：“本部不允许修改!”？

![上传图片](https://vip.kingdee.com/download/0100c694ba2bd74e4148bd168bb535c69e54.png)

【原因说明】

这个问题的原因是，在新增职员时，如果部门选择的是非叶子节点（就是下面还有二级节点的那个节点），系统就会自动创建一个本部，而本部是不能修改名字。

下面是重现的路径：

例如下图中，“研发专用（勿初始化）-审核时更新”这个部门，我们从左边的类别树可以看到下面还有非常多的二级部门，意味着这个不是一个叶子节点。

![上传图片](https://vip.kingdee.com/download/01006a18eb02087e48d88f130a374163f4d7.png)

如果创建职员user，部门选择“研发专用（勿初始化）-审核时更新”

![上传图片](https://vip.kingdee.com/download/0100c79a9635392b4696b1d016f1542ed422.png)

创建成功后，就会在部门中自动创建一个本部。

![上传图片](https://vip.kingdee.com/download/0100127eb00582054aeda33092de651b9a4c.png)

【解决方案】

如果这个本部部门下面挂没有任何二级部门，也就是部门分类树里，部门左侧没有折叠的三角按钮

![上传图片](https://vip.kingdee.com/download/01003f3de2e0fd8349639c5a72028c53116a.png)

打开刚刚新增的职员，“部门”清空或者挂到正确的部门里，点击“保存”

![上传图片](https://vip.kingdee.com/download/010079936541fa634ed1a8ea324cacf4ee39.png)

再到部门列表删除这个系统自动创建的本部即可成功。

![上传图片](https://vip.kingdee.com/download/010070f51ab3fa9f475b99543a85a63121a2.png)

![上传图片](https://vip.kingdee.com/download/0100868d936a462f4344a36543ea53f2edf0.png)

>
应用建议：为了避免上述情况，建议新增职员时，如果已经有明确的二级部门，就直接挂到二级部门上，不要挂上面的节点部门就不会自动创建一个本部。或者不确定部门的时候，可以先不要选择部门。

【适用版本】

星辰3.1以上所有版本
