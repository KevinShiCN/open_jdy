# 基础资料

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：功能介绍, 快速入门
> 作者：Cat_Chow | 阅读：6081 | 有用：3
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/635955561397152768)

系统基础资料是日常业务不可或缺的基础，主要包括商品资料、业务资料、财务资料和期初资料等，具体明细如下：

![](https://vip.kingdee.com/download/01098662d51af8694e2eabba25e770a5ac9d.png)

| 步骤 | 具体说明 | 学习链接 |
| --- | --- | --- |
| 1.商品资料 |  | —— |
| 1.1商品信息 | 优先维护商品信息，支持直接新增、或是批量引入 | —— |
| 1.1.1商品类别 | 若商品信息较多，可对商品进行分类管理，后续可在业务报表按类别筛选统计 | 点击学习 |
| 1.1.2普通商品 | 新增商品、商品新增相关字段说明 | 点击学习 |
| 1.1.3多单位商品 | 支持多计量单位，且多计量单位支持固定换算及浮动换算关系。注意：(1)换算率不能小于1，即以最小单位为基本单位。(2)商品设置多计量单位时目前支持设置4个单位,一个基本单位,3个副单位;只能设置一个浮动单位。  说明：专业版及旗舰版均支持浮动单位，标准版不支持浮动单位。 | 点击学习 |
| 1.1.4辅助属性商品 | 同一个商品，存在不同的规格、颜色等属性时可启用辅助属性进行管理，如衣服有不同的颜色和尺码；同时系统支持按辅助属性单独核算成本。 | 点击学习 |
| 1.1.5批次/保质期商品 | 批次、批号保质期是针对食品药品等行业,需要进行保质期、批次管理的功能。  支持同时启用批次/保质期,也支持单独启用批次或保质期；另外，系统支持按批次独立核算成本。   | 点击学习 |
| 1.1.6序列号商品 | 产品序列号管理软件广泛应用于手机、数码产品等3C行业,每个产品都有一个唯一的序列号,用于设备标识,跟踪记录产品的生产、销售、流通、售后等记录。   | 点击学习 |
| 1.1.7套装商品 | 套装商品用于处理套装或者原金蝶产品中所指的虚拟件,是指由一组具体物料(实件)组成的、以虚拟形式存在的成套件,所以套装不是一个具体物料,不进行成本核算。在销售出库时以套装子件的形式进行出库、扣减库存、计算收入。 | 点击学习 |
| 1.1.8服务商品 | 服务商品通常指的是那些不涉及实物交换，而是以服务形式提供给客户的产品，例如维修服务、咨询服务、租赁服务等。星辰维护的服务商品只影响资金往来,不影响库存，即不会产生库存。同时，服务商品不支持采购开单、采购入库即服务商品不参与成本核算。 | 点击学习 |
| 1.1.9商品条码 | 支持一码多品。注意：(1)多单位商品可以按不同的单位设置不同的商品条码(2)支持按商品属性设置商品属性条码(3)支持手工录入商品条码；或是设置条码生成规则，由系统自动生成。 | 点击学习 |
| 1.1.10商品库存预警功能 | 自定义预警规则，及时了解库存水平，防止库存过多导致的资金占用或库存不足导致的销售机会损失。 | 点击学习 |
| 1.1.11商品图片 | 支持维护商品图片，且支持批量引入商品图片信息。 | 点击学习 |
| 1.2计量单位 | 系统已预置常用单位，可以直接引用。各个单位可独立配置其数量的小数位。注意：数量小数位只能由小改大不能由大改小。 | 点击学习 |
| 1.3商品品牌 | 支持商品品牌维护，后续可以按商品品牌统计业务报表 | 点击学习 |
| 1.4标签中心 | 企业在开展经营活动时,往往会根据实际业务需要,通过一定的属性分组来对产品进行更细化的管理,可以通过标签来实现分组管理。编辑商品资料时可以选择商品标签,同一商品支持选择多个标签,后续可以在业务报表里按标签筛选。 | 点击学习 |
| 1.5辅助属性 | 商品启用辅助属性功能，可以在【辅助属性】维护属性明细。(1)系统预置5个属性组，支持修改名称，如修改为颜色、尺码等；属性组不支持新增，只能在原有的组别下是添加下级分类。(2) 辅助属性组合上限为1000个,即所有属性组可组成的属性组合总共1000个（包括启禁用状态）。   | 点击学习 |
| 1.6外部商品库 | 外部商品库是指企业在进行商品管理时，为了适应不同客户或供应商的商品编码和名称需求，而建立的一个商品编码和名称的对应关系库。这种库通常用于企业与外部合作伙伴之间的商品信息交换和对接，以确保商品信息的准确性和一致性。 | 点击学习 |
| 1.7条码生成规则 | 商品条码可手工录入，也可通过条码生成规则自动生成条码。 | 点击学习 |
| 1.8条码标签打印 | 系统支持针对某个商品/条码/入库单打印条码标签   | 点击学习 |
| 1.9商品列表常用功能 | 商品列表常用功能包括：删除、禁用、启用、筛选条件设置、批量修改及列设置等 | 点击学习 |
| 1.10商品批量引入 | 预置多种导入模板，注意普通商品、属性商品、批次及序列号商品导入的模板选择，及导入数据的编辑。注意引入选项的设置，即引入商品价格资料的规则设设置。 | 点击学习 |
| 2.业务资料 |  | —— |
| 2.1客户 |  | —— |
| 2.1.1客户类别 | 对客户进行分类管理，后续可通过销售报表按客户类别筛选统计。新增客户类别 | 点击学习 |
| 2.1.2新增客户 | 新增客户、客户新增时相关字段的说明及应用、客户列表管理功能介绍 | 点击学习 |
| 2.1.3客户信用额度管理 | 客户信用额度设置及应用指南 | 点击学习 |
| 2.1.4客户信用期限管理 | 实现应收账款到期预警功能 | 点击学习 |
| 2.2供应商 |  | —— |
| 2.2.1供应商类别 | 对供应商进行分类管理，后续可通过采购报表按供应商类别筛选统计。新增供应商类别 | 点击学习 |
| 2.2.2新增供应商 | 新增供应商、供应商新增时相关字段的说明及应用、供应商列表管理功能介绍 | 点击学习 |
| 2.2.3供应商信用额度管理 | 供应商信用额度设置及应用指南 | 点击学习 |
| 2.3仓库 | 仓库支持分类管理；支持单独设置某个仓库允许负库存；同时仓库支持仓位管理。 | 点击学习 |
| 2.4仓位 | 仓位可以指仓库中用于存放货物的特定区域或位置。每个仓位可能有特定的编号或标记，以便于货物的管理和追踪。 | 点击学习 |
| 2.5部门 | 维护部门信息，后续业务报表可按部门筛选统计，也支持按部门设置数据权限。 | 点击学习 |
| 2.6职员 | 包括采购员、业务员、仓管员等信息在职员进行维护，后续业务报表可按采购员/业务员筛选统计，也支持按职员信息设置数据权限 | 点击学习 |
| 2.7收入/支出类别 | 收入或是支出费用的类型，如快递费、油费、加盟费等等 | 点击学习 |
| 2.8辅助资料 | 系统内置以下辅助资料分类：价格等级、交货方式、退货原因、其他出库类型、其他入库类型。内置的辅助资料分类不可修改、删除，但可以根据公司业务情况新增。 | 点击学习 |
| 2.9结算期限 | 即与客户或供应商约定的货款结算规则及结算周期；除系统默认的结算期限，也可以自定义新增。 | 点击学习 |
| 2.10物流公司 | 系统预置常用的物流公司信息，在编辑销售单据时支持录入物流信息，如物流公司，顺丰快递等。 | 点击学习 |
| 3.财务资料 |  | —— |
| 3.1币别 | 进销存云支持外币业务，系统预置有多种币别，可直接启用，或是在此基础上新增币别。 | 点击学习 |
| 3.2结算方式 | 系统预置了银行转账、现金支付、微信支付、支付宝支付、银行支票五种结算方式；可以在这个基础上自定义其他结算方式。用于录入资金单据时记录结算方式。 | 点击学习 |
| 3.3账户管理 | 企业在采购销售活动中，往往会涉及支付，记录不同的结算账户，可以使得业务结算、查账对账更清晰。资金管理（出纳）涉及到的账户，也是在此新增。 | 点击学习 |
| 4.期初录入 |  | —— |
| 4.1商品期初 | 首先完成商品信息录入，方能录入商品期初数据。支持手工新增商品期初及商品期初批量导入。以月为单位，上个月底期末的商品结存数量=当月的商品期初库存数量。【示例】进销存启用日期是2024年5月1日，商品期初则按2024年4月30日的商品结存数量录入。说明：不同的商品特性导入模板不一样，注意下载对应的导入模板。 | 点击学习 |
| 4.2客户初始数据 | 进销存启用日期之前，企业与客户之间的往来余额,在启用账套时需要录入到系统中。前期的应收余额正数录的应收金额栏,应收余额负数录在预收金额栏;其他应收同理。 | 点击学习 |
| 客户初始数据支持批量导入 | 点击学习 |  |
| 4.3供应商期初数据 | 进销存启用日期之前，企业与供应商之间的往来余额,在启用账套时需要录入到系统中。前期的应付余额正数录的应付金额栏,应付余额负数录在预收金额栏;其他应付同理。 | 点击学习 |
| 供应商初始数据支持批量导入 | 点击学习 |  |
| 4.3账户初始数据 | 现金、银行账户等往来账户的初始余额，支持3种录入方式：(1)手工录入账户初始数据， (2)引入账户初始数据， (3)从科目引入账户初始数据；说明：当出纳与进销存的启用期间不同时，账户初始数据录入规则不同。 | 点击学习 |
| 4.4委托代销期初余额 | 适用于星辰专业版、星辰旗舰版系统启用之前有委托代销的数据未结算完,或是有销售暂估还未结算开票,需要登记暂估数据以便后续结算开票,可以通过录入委托代销期初登记。 | 点击学习 |
| 4.5暂估入库期初 | 适用于星辰旗舰版系统启用期间之前发生的已经采购入库、未确认应付账款的业务，需要在系统中继续跟进,可以通过录入暂估入库期初单来记录。【示例】系统的启用期间为2023年1月,2022年发生的采购业务已入库但还未确认应付账款的部分，便可以录入暂估入库期初单,后续由暂估入库期初单生成采购发票。注意：暂估入库期初不会增加库存,也不会增加应付账款,只是作为期初单据记录启用期间前的未完成的业务。当确认了应付账款时,可以通过《暂估入库期初》单下推采购发票,形成应付账款。启用系统之前的库存仍是通过【商品期初】记录。 | 点击学习 |
| 4.6采购发票期初 | 适用于星辰旗舰版比如机械设备行业，商品的销售和采购周期都比较长，采购时，需要预付一部分订金给到供应商，供应商会提前开票，所以存在已开票还未采购入库的业务数据，在软件上线启用账套时，需要记录这部分数据。 | 点击学习 |
| 4.6销售出库期初 | 适用于星辰旗舰版系统启用期间之前的发生的已经销售出库、未确认应收账款的业务需要在系统中继续跟进，可以通过录入销售出库期初单来记录。【示例】系统的启用期间为2023年1月，2022年发生的销售出库已出库但还未确认应收账款的部分，但可以录入销售出库期初单，后续由销售出库期初单生成销售发票。注意：销售出库期初不会减少库存，也不会增加应收账款，只是作为期初单据记录启用期间前的未完成的业务。当确认了应收账款时，可以通过《销售出库期初》单下推销售发票，形成应收账款。 | 点击学习 |
| 4.7销售发票期初 | 适用于星辰旗舰版比如机械设备行业，商品的销售和采购周期都比较长，销售时，为了提前收款，会提前给客户开票，存在已开票还未出库的数据，在软件上线启用账套时，需要记录这部分数据。 | 点击学习 |
| 5.价格设置 |  | —— |
| 5.1商品价格资料 | 商品采购价格及销售价格较为稳定，且所有供应商的价格统一，所有客户的销售价格统一，企业只需要维护商品信息中的采购价格和销售价格等价格即可。 | 点击学习 |
| 5.2客户价格资料 | 同一商品，不同的客户销售价格不一样【业务场景】不同客户的商品价格不同；不同客户分类的商品价格不同；不同品牌的折扣率不同；不同商品类别的折扣不同；商品进行阶梯定价，不同销售数量价格不同；商品的不同辅助属性（如颜色）价格不同。通过建立不同客户价格方案以满足多样的销售价格。 | 点击学习 |
| 5.3供应商价格资料 | 向不同的供应商采购同一商品时,可以设置不同的采购价格 【业务场景】不同供应商的商品价格不同；不同商品类别的采购价格不同；不同品牌的折扣率不同 | 点击学习 |
| 5.4价格取数优先级 | 星辰支持多种价格资料，有商品、有单据、有价格方案，针对如此多的价格来源，可通过价格取数优先级自定义设置采购类单据、销售类单据、其他入库单、盘盈单、异价调拨单中价格取数的优先级，从而满足不同的开单取价场。 | 点击学习 |
| 5.5采购销售限价 | 企业在采购过程中需要控制采购不能超过某个最高价、避免采购人员超高价采购；在销售过程中需要控制销售不能超过某个最低价或者折扣，避免出现超低价销售，导致企业出现损失。 | 点击学习 |
| 5.6批发促销 | 线下批发业务中，存在买赠等促销方案，运营提前制定促销方案，业务员开单时想要通过执行促销、选择促销方案来自动添加赠品。注意：目前批发促销只支持商品买*件赠送赠品*件（即单品满件数赠）的规则。 | 点击学习 |
