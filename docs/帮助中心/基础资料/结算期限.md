# **1 简介  **

## **1.1 应用场景**

结算期限即与客户或供应商约定的货款结算规则及结算周期。

结算期限设置后，可以在客户和供应商中关联，在录入业务单据时，系统会根据单据日期和对应的结算期限，计算填列对应往来的结算日期。设置结算期限应用后，系统会根据单据结算日期，提供对应的应收应付预警表，可以查询到往来到期和超期情况。

## **2.2 新增/修改**

系统支持新增结算期限，包括以下两种类型的结算期限。

![](https://vip.kingdee.com/download/0100b1b670ab81b549f0b81f6e0a9342c974.png)

- **标准期限**

主要应用于结算期限为固定期间的，如7天，15天，月结30天，45天等，都是标准期限，以月结45天为例，设置时选择“标准期限”，结算期限录入为“30天”，在客户中选择这个结算期限，则录入单据时，单据的结算日期会根据单据日期+30天自动计算填列。

![上传图片](https://vip.kingdee.com/download/01005c363499007841d782b767a7ebae013e.png)

**月结期限**

用于每月固定日期结算上月待结算业务，如每月5日结算上月往来款，则可以设置为每月5日，结算5天前的待结算业务。客户关联该结算方式后，如果录入2025年9月1日的业务单据，则结算日期会自动计算为2025年10月5日。

![上传图片](https://vip.kingdee.com/download/0100967bfc193ae64cefb6c41a8f85a61fb7.png)

## **2.3 参****数“设置为默认结算期限”**

勾选后新增客户和供应商均默认携带此结算期间。

![](https://vip.kingdee.com/download/010091412abe62134293b4123554fd06baa3.png)

## **2.4  删除**

已使用的结算期限不允许删除。
