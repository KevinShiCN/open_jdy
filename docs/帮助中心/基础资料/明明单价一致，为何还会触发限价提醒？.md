# 明明单价一致，为何还会触发限价提醒？

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：快速入门, 应用案例
> 作者：吊梢眉 | 阅读：26 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/794140489913451008)

### 问题现象

在开单过程中，您是否遇到过这种情况：本次开单的金额与单价，明明和历史单据（或设置的限价标准）完全一致，系统却依然弹出了“低于最低售价”或“高于最高采购价”的限价提醒，导致无法保存单据。

### 原因解析

这是因为系统进行限价判断时，使用的是**实际销售单价**（由实际不含税金额或价税合计÷数量计算得出）。由于金额和单价的小数位设置可能不同，反算出的实际销售单价可能与您录入的单价存在极其微小的差异（例如 248.4000001 与 248.4），从而导致系统判定为“低于”最近销售单价而触发提醒。

### 建议方案

建议在设置限价方案时，增加一个“容错范围”，允许极微小的计算误差。

### **所属版本**

所有版本

### **操作步骤：**

以销售限价为例：

在【基础资料>价格设置>限价方案设置】界面：修改限价公式，从“实际销售单价 < 最近销售单价”调整为 “实际销售单价 < (最近销售单价 - 容差值)”；

建议容差值： 0.01 ~ 0.1（根据公司对价格精度的严格程度设定）。

![上传图片](https://vip.kingdee.com/download/01006b5a42432245484b8a785502768bb5bd.png)

### **示例：**

在销售订单中，录入的价税合计为1964.81，税额为226，计算出的单价均为248.4。与之前开单的金额与单价一模一样，但在保存销售订单时仍提示触发了限价：

![上传图片](https://vip.kingdee.com/download/01002c96f3a4cdac4e3c964b624037643cdd.png)

设置完成后，保存销售订单时将不会再触发限价提醒。
