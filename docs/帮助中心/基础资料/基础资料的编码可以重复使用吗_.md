# 基础资料的编码可以重复使用吗?

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：常见问题
> 作者：shaoling | 阅读：152 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/705227971568885248)

【问题描述】

基础资料的编码可以重复使用吗?

【业务场景】

不小心删除了某个基础资料,还需要按原有的编码新增

既是客户又是供应商,为了方便管理,部分客户的编码需要和供应商的编码一样

【解释说明】

①基础资料的编码信息和身份证、手机号一样,具有唯一识别性,基础资料(如物料)在实际业务中可能存在 “一名多物” 或 “一物多名” 的问题,可通过编码避免混淆;

②同一类基础资料的编码不允许出现重复,对于同一类基础资料如客户,其编码是识别唯一性的标准;如果原本新增的一个客户被删除了,只要其编码还未被使用,再次新增时还可以录入原本的编码,若编码已被使用保存时会有提示

![](https://vip.kingdee.com/download/010929d0c826d1d440dba3c3d514da1c7089.png)

不同类型的基础资料,如客户和供应商,两者之间允许出现相同的编码,不影响日常业务使用

如客户中有编码为00.001的客户

![](https://vip.kingdee.com/download/0109010677bca30641a7924dcd919a1394a0.png)
供应商中新增一个编码同样为00.001的供应商可以保存成功
![](https://vip.kingdee.com/download/0109510c5dec0f104069a40bef27fbb0cb09.png)

【温馨提示】

若基础资料操作了禁用,**禁用的基础资料编码****也不允许重复使用**,禁用后系统基础资料里会保留数据
