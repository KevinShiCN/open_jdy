# 商品相关

## 1、 **星辰的辅助属性组预设只有5个，还能再添加吗？**

答：不能，目前标准产品的预设就是5个辅助属性组，结合目前目标客户的画像调研及性能考虑，不会再放开这个上限。

具体说明，详见知识：[辅助属性中是否可以再新增一级属性？](https://vip.kingdee.com/link/s/MivAN)

## 2、 **辅助属性组合上限是多少？可以再增加吗？**

答：星辰辅助属性组合上限就是**1000个**，就是所有属性组可组成的属性组合总数，1000个是不区分启用/禁用状态，也就是即使禁用了也会算进去。

![上传图片](https://vip.kingdee.com/download/0100298713a8780c4b1fb9b76cabf9822e06.png)

这个数量也是经过性能测试设计的，再多会影响系统运行，暂时不会放开上限。

## 3、 **当属性组新增了属性时，要怎么追加新的属性组合？**

答：有2种办法。

1） **在商品里直接追加新增**

如果属性组新增了属性，可以点击“生成”（就是全部生成）或“选择生成”（只选择部分组合生成）进行属性组合的追加。

![上传图片](https://vip.kingdee.com/download/0100dbb2e47f3e604e17b8d561e4a673a7cd.png)

2） **通过“覆盖新增”的引入方式来批量新增**

![上传图片](https://vip.kingdee.com/download/0100c8e7df9ca95e4a7ca44b384a57cb9c3a.png)

>
注意：引入不支持直接部分新增，必须把现在商品已经有的属性组合也一起引入，建议将辅助属性商品用“按引入模板引出”后，在商品最后添加好要增加的属性再用“覆盖新增”引入即可。

## 4、 **为什么新增的辅助属性组无法取到对应属性组的编码呢？**

答：辅助属性设置编码规则，设置的是“**辅助属性分类:编码**”，不适用于辅助属性组。换句话说，只能取一级的分类编码，取不到二级分类的编码。

![](https://vip.kingdee.com/download/01006130a7c5f254461db001156d7fbce840.png)

![](https://vip.kingdee.com/download/010056bee442c6cf4bf7ac35c31f2749dfd0.png)![](https://vip.kingdee.com/download/0100fbcda1ad39bf48a083c1b17f446762e5.png)

因此直接选择辅助属性组，新增辅助属性时，编码规则也是取一级节点，也就是“辅助属性分类”的编码，而不是取“辅助属性组”的编码。

![](https://vip.kingdee.com/download/01001b86edcb06ec4c27b100dd954d5ca8a8.png)

## **5、商品辅助属性（SKU）的编码规则中的“属性编码”是什么，能否自定义属性编码的顺序？**

答：属性编码是指商品启用辅助属性后，每个辅助属性对应的属性编码，且如果编码规则中设置了取属性编码，那么SKU编码从左到右的顺序就是根据系统内置的五个属性分类依次来取的，不能自定义属性编码的顺序，下面通过一个例子进行说明。

假设某客户的商品辅助属性（SKU编码）的编码规则如下：

![](https://vip.kingdee.com/download/01004b689035c9424f20960e4386c7e50a2a.png)

假设有辅助属性商品，启用了颜色、尺码2个辅助属性组，那么选择生成辅助属性组合时，SKU编码规则生成的编码顺序是严格取系统默认的5个属性组顺序来的（从左到右分别是1-5的顺序）；

如下图深蓝的编码为B01，大码的编码为Z01，中码的编码为Z02，则深蓝/大码的属性编码为B01Z01，深蓝/中码的属性编码为B01Z02；

![](https://vip.kingdee.com/download/010073f400b83c7541debbdc387f8d953e85.png)

如果辅助属性组是尺码在前，颜色在后，那么大码/深蓝的属性编码为Z01B01，中码/深蓝的属性编码为Z02B01：

![](https://vip.kingdee.com/download/010063450ae65d4c442e8cf78e6277a7f9fc.png)

因此如果您的业务场景对于SKU编码是有严格顺序的，建议在创建辅助属性组合时，先确认好辅助属性分类的顺序，以避免后续因为编码不符合预期而又要返工修改。

## 6、新增辅助属性商品时，可以先不添加属性组合，等开单时再添加吗？

答：可以。可以先选择辅助属性分类、属性组后保存商品，无需生成辅助属性组合。开单时再选择对应的属性组，保存单据时会自动生成辅助属性组合，并且更新到商品信息中。

![上传图片](https://vip.kingdee.com/download/01007c9da17a94824aac809d25fd5a8a7c04.png)

具体说明，详见知识：[辅助属性优化（2025年）](https://vip.kingdee.com/link/s/ZaFeT)

# 单据相关

## 7、**订单上没录辅助属性,下推出入库单后录了多行辅助属性,但是只有一行反写了源单的状态及数量,是为什么?**

答：出入库单据反写源单是根据分录行反写的,当订单上只有一行分录时,而出入库拆成了多行录入的场景下,录入方式无论是二维录入(即系统参数>基础资料参数中,开启了“开单启用二维录入”)或者直接手工新增的行,因为都只有一行是携带了源单信息,所以其他行不会反写订单。

具体说明，详见知识：[https://vip.kingdee.com/link/s/ZAMpo](https://vip.kingdee.com/link/s/ZAMpo)

## **8、辅助属性商品导出时,为什么价格分录的内容与前面的属性组是对不齐的?**

答：其实这是正常的现象,是一个应用问题。

星辰的辅助属性商品信息里的“价格设置”默认是SPU维度的,也就是不管什么属性,都是一个价格,而针对有的商品,部分属性可能由于工艺或材质不一样,成本及价格会有差异,那么需要对特殊属性设置价格,这个就是SKU维度的,如下图:

![上传图片](https://vip.kingdee.com/download/0100048da7c52acd4a9c82164d2f07c8726a.png)

如果两个维度都设置了,那么引出商品信息时,“价格分录”里:

属性组值为空的,则是SPU维度的价格;

属性组值不为空的,则是SKU维度的价格.

具体说明，详见知识：[https://vip.kingdee.com/link/s/ZAkXk](https://vip.kingdee.com/link/s/ZAkXk)

## 9、开单选择辅助属性商品，选择辅助属性时，为什么有时候会覆盖原来的行，有时会新增行？辅助属性商品批量选择辅助属性后，分录行插入的逻辑是什么样的？

答：开单录入单个辅助属性时，不管是否开启了二维录入，也不管是单个还是批量选择辅助属性后，都会根据当前行的“辅助属性”字段是否有值而做不同的插入方式（新增行或覆盖行）。具体说明见下表。

| 录入方式 | 选择辅助属性时分录的情况 | 添加新的辅助属性后 |
| --- | --- | --- |
| 单个录入 | 没有值 | 会覆盖当前行 |
| 已经有值 | 会在当前行上方插入新行，且当前行不覆盖 |  |
| 批量录入 | 当前行没有录入辅助属性 | 批量选择辅助属性弹窗中选择的辅助属性会覆盖当前行（当前行的辅助属性为弹窗中勾选的最后一个），其他行在当前行上方插入。简单点说就是下图的最后一行已经覆盖原来左列图中最后一行） |
| 当前行有录入辅助属性 | 批量选择辅助属性弹窗中选择的辅助属性均在当前行上方插入，且当前行不覆盖。简单点说就是下图最后一行还是原来左列图中最后一行 |  |

## **10、订单上可以不录入辅助属性吗？**

答：可以。系统设置>系统参数>基础资料参数>商品参数中，启用辅助属性管理后，可以设置录单时辅助属性非必录的适用单据，设置完成后去开单即可不录辅助属性。

![上传图片](https://vip.kingdee.com/download/0100d1aeb745603649c8958d69238011a6d8.png)

具体说明，详见知识：辅助属性优化（2024年）

## 11、新增商品时没有维护辅助属性，开单时先手工录入属性、单价，保存提示“辅助属性为空，但辅助属性明细不为空。”然后新增辅助属性后，单价被清空了，这是正常的吗？

![上传图片](https://vip.kingdee.com/download/010007acd149c4754bb29a68d1f6d9549753.png)

![上传图片](https://vip.kingdee.com/download/0100671789bc07214ca58c09d114b57b0353.gif)

答：辅助属性回填后，会触发重新取价，所以价格会清空，属于正常逻辑。建议先维护好新增辅助属性后，再录入价格。

![上传图片](https://vip.kingdee.com/download/01002670b25e91f4495bb4e21ec537daa6a2.gif)
