# 负库存设置及判断逻辑说明

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：功能介绍, 精华汇总, 进阶技巧
> 作者：进销存赵嘉丽 | 阅读：3984 | 有用：6

# 适用版本

星辰3.1以上所有版本

# 1** 负库存的控制及设置路径**

在“基础资料>业务资料>仓库>允许负库存”中控制：

![image.png](https://vip.kingdee.com/download/0109d913c652e5c94464be5243c88fff5de7.png)

>
注意，仓库允许负库存一旦开启后，若要关闭，必须保证该仓库下的可用库存大于0才能关闭，否则系统会校验，导致无法关闭成功。

如果仓库确实存在负库存，星辰3.1以上版本的客户可以直接看到“异常库存明细”，根据明细去检查和补足库存，完成后再关闭即可。

![image.png](https://vip.kingdee.com/download/01093d98319c7f9347859099da0db82c8cfd.png)

# 2 负库存判断的依据

- 订单类单据：可选择不控制或“可用库存”控制
- 出入库类单据：可选择“可用库存”或“即时库存与可用库存双重控制”

![上传图片](https://vip.kingdee.com/download/0100a1fe97aa124d4852b24d342da2027d63.png)

- 不支持用单据日期来控制负库存具体说明，请查看知识：[为什么不能用单据日期来控制负库存？](https://vip.kingdee.com/link/s/ZxXNu)

# 3 负库存检查的操作时点

- 库存更新参数为“保存时更新”，负库存检查在单据保存或删除时判断；
- 库存更新参数为“审核时更新”，负库存检查在单据审核或反审核时判断。

设置路径：【系统设置】-【系统参数】-【进销存参数】-【仓存参数】--“若您希望改变库存更新的方式，请选择”

![image.png](https://vip.kingdee.com/download/0109e2aea317c5234e26a1bcb54fc3810735.png)

>
注意：系统更新方式一旦开启后，再次修改时会校验单据的状态及可用库存，故在前期系统上线阶段最好先评估好业务需求，再确认好库存更新参数，以免发生业务数据后，想再次修改时会比较麻烦。

# 4 库存预警与异常检查

- 含义：库存异常检查是指浮动多单位商品若出库类单据扣减库存时，要不要检查基本单位与浮动单位库存是否异常的校验控制，具体针对以下两点进行校验检查。

1）若基本单位库存量与默认浮动单位库存量一个为正一个为负

2）若基本单位库存量与默认浮动单位库存量一个为零一个不为零

- 设置路径：系统设置>系统参数>进销存参数>库存预警与异常检查

![上传图片](https://vip.kingdee.com/download/01005acbcca027f84e569ec0e95a383f3bce.png)

参数说明：即系统是否要允许继续保存或审核单据，不予控制就是不做控制，严格控制则不允许保存或审核单据，这个参数的设置应视企业的业务管理需要而定。

>
**重点。**单据检查判断优先级：库存异常检查**优先级大于**仓库负库存检查（浮动单位库存异常会优先于负库存检查，简单来说就是，**仓库负库存参数控制基本单位的检查，如果是浮动单位负库存要看库存异常检查参数有没有控制**）
