# 仓位修改失败,“库存查询表”的字段“仓位编码”引用了此资料数据”不允许修改对应仓库,怎么处理?

> 作者：云社区用户6w624200 | 阅读：42 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/774802232259773952)

【问题描述】

仓位修改失败,“库存查询表”的字段“仓位编码”引用了此资料数据”不允许修改对应仓库,怎么处理?

【操作步骤】

一旦仓位被关联使用,可以修改其编码和名称,但对应的仓库信息则无法再进行更改。如果有仓位被使用,会提示“库存查询表”的字段“仓位编码”引用了此资料数据”不允许修改对应仓库。

![](https://vip.kingdee.com/download/0109120a71e9aea448eca492d51b366646d6.png)

1、若需修改仓位所属仓库,需先删除该仓位相关数据,建议先备份再操作删除

若相关数据已删除仍提示,可到库存查询表上校对库存后再修改试试

![](https://vip.kingdee.com/download/01090a5c8a56b2a242e3bfc76621a5d7c16f.png)
2、若历史数据不方便删除,可新建仓位后对应到正确的仓库,原仓位剩余库存做移仓转移,之后禁用该仓位
