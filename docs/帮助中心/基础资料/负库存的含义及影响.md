# 负库存的含义及影响

# **适用版本**

星辰3.1以上所有版本

# **1 负库存的概念**

简单地说，负库存就是**销售数量大于库存数量**。企业实际管理工作中受各种因素的影响，在系统的运行中，会出现实际销售出库的数量大于系统有效账面库存数量的情况，进而也就产生了负库存这种现象。

负库存是进销存管理中经常出现的一种异常现象，虽然说是异常现象，但许多负库存并不是错误的；由于产生负库存的原因复杂，往往会造成**库存成本异常**。

负库存产生的原因除人为差错外，大部分是由于**账面出入库和实际出入库在时间顺序上的不一致造成的**，如果严格遵循先入库后出库的方式，是不会产生负库存的，但这种做法在企业实际经营中往往行不通，所以在进销存管理中会在一定范围内允许出现负库存。

# **2 产生负库存的常见场景及成因、解决方案**

## **1）先卖后买**

顾客要的商品缺货，从附近商家或是批发商处暂借过来给销售，事后再补上采购入库单，这种情况下先开销售出库单，之后才录入采购入库单，在开销售出库单后，该商品的库存量就变成了负值。

## **2）人为录单差错**

**场景①：采购入库时填少了（或供应商有赠品，但是赠品没做入库）：**

当实际库存数量大于账面库存数量，这时候如果按实际库存数量进行销售就可能会出现负库存，如账面库存数量为10个，实际库存数量为15个，这时销售出库12个，账面库存会变成-2个。

**场景②：销售出库时填错了，实发数少于应发数：**

假设账面库存是10个，实发8个，但出库单填了10个，等到盘点实物时会发现账面库存与实际库存对不上，实际库存多出了2个（10-8=2）。这时再对这2个实物继续做销售，账面库存会提示库存不足，显示-2。

## **3）解决方案**

针对上面的场景，对应的解决方案是：

**先卖后买**：实际经营中确实无法避免这种情况，但是要确保事后一定要及时完成采购入库，确保入库数量和发出数量一致即可。

**人为录单错误：**

场景①：入库单录入错误，需要反审核单据再修改，如入库单已经生成下游单据，则需先录入其他入库单，先把库存补足，然后再反审核修改错误的入库单。修改完毕后要删除补录的其他入库单。

场景②：出库单填写错误导致的实发与应发差异，造成系统与实际库存对不上的，出现了盘盈，需要通过盘点单来手动校正系统库存数。

## **4）暂估入库与红冲**

采购商品如果货到但发票未到，这时无法正式入库，可以采取暂估入库的方式，即先估价入库，然后在月末结账时红冲该暂估入库单，下月再重新暂估入库，直到收到正式发票后入库为止，在暂估入库的月末红冲后，库存就有可能变成负值。

# **3 负库存的影响**

负库存可能导致**成本异常**，在负库存的情况下出库，意味着当前出库的商品是将来采购入库的商品，因此其成本只能通过推测得出。而导致成本异常的根本原因就是这个推测或参考的成本，如果它与将来实际采购入库商品的成本相差很大，就会**严重影响库存商品的成本：**

- 如果推测或参考成本大于实际成本，可能导致库存商品成本偏高；
- 如果推测或参考成本小于实际成本，可能导致库存商品成本偏低，甚至为负值。
