# **1 简介  **

## **1.1 功能介绍**

采购销售限价多项优化，细化限价策略颗粒度，满足更多场景。

## **1.2 功能特性**

- 【所有版本】采购销售限价增加限价依据
- 【所有版本】采购销售限价增加限价对象
- 【旗舰版】采购销售限价的单据范围增加采购发票、销售发票

## **1.3 适用版本**

请以功能说明中的版本为准

# **2 功能说明**

## 2.1 【所有版本】采购销售限价增加限价依据

| 类型 | 新增限价依据字段 |
| --- | --- |
| 采购限价 | 采购价、最近销售单价、批发价、零售价、最低销售价、最近销售含税单价、最高采购含税单价、历史最低采购价、最低采购含税单价、历史最高销售单价、最高销售含税单价、历史最低销售价、最低销售含税单价 |
| 销售限价 | 最高采购价、批发价、零售价、价格等级（20个）、历史最高采购价、最高采购含税单价、最低采购含税单价、历史最高销售单价、最高销售含税单价、最低销售含税单价 |

## 2.2 【所有版本】采购销售限价增加限价对象

### 1）采购销售限价增加限价对象：含税单价、单价

| 类型 | 优化前 | 优化后 |
| --- | --- | --- |
| 采购限价 | 采购限价仅可设置实际采购单价 | 采购限价增加含税单价、单价两个选项，适用于无折扣的采购业务，直接对单据中的单价或含税单价进行限价，避免因实际单价换算后存在的精度差异导致限价不通过。 |
| 销售限价 | 销售限价仅可设置实际销售单价、销售折扣。 | 销售限价增加含税单价、单价两个选项，适用于无折扣的销售业务，直接对单据中的单价或含税单价进行限价，避免因实际单价换算后存在的精度差异导致限价不通过。 |

### 2）将采购销售限价中的实际单价，拆分为实际含税单价、实际不含税单价，以便用户区分具体的限价对象

| 优化前逻辑 | 命名为实际采购单价、实际销售单价，根据系统参数“商品价格为含税单价”是否勾选，决定是用实际单价还是实际含税单价来比价（勾选参数使用实际含税单价、不勾选参数使用实际不含税单价）。 |
| --- | --- |
| 优化后逻辑 | 通过限价对象由用户指定用哪个单价来比价限价。对于历史数据（用户已经设置好的限价方案），根据系统参数“商品价格为含税单价”是否勾选自动匹配相应的限价对象更新到方案中。 |

## 2.3【旗舰版】采购销售限价的单据范围增加采购发票、销售发票

>
请注意：仅业务类型为“入库”的采购发票、业务类型为“出库”的销售发票支持限价。
