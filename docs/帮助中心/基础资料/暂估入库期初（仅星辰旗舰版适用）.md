# 暂估入库期初（仅星辰旗舰版适用）

> 作者：进销存赵嘉丽 | 阅读：3207 | 有用：4

# 1 **1 简介**

## **1.1 应用场景**

业务系统处于初始化阶段时，对于启用期间之前的发生的已经采购入库、未确认应付账款的业务需要在系统中继续跟进，可以通过录入暂估入库期初单来记录。假设系统的启用期间为2023年1月，对于启用期间之前的比如2022年发生的采购入库还未确认应付账款的部分就可以录入暂估入库期初单，后续可对其进行下推采购发票。

暂估入库期初不会增加库存，也不会增加应付账款，只是作为期初单据记录启用期间前的未完成的业务。当确认了应付账款时，可以通过暂估入库期初单下推采购发票，形成应付账款。

## **1.2 操作路径**

基础资料>期初录入>暂估入库期初

## **1.3 适用版本**

星辰3.1以上旗舰版

# 2 操作指南

## **2.1 手工新增**

启用星辰旗舰版账套后，打开基础资料>期初录入>暂估入库期初，新增暂估入库期初。单据新增时，注意事项：

• **业务类型**

区分“入库”、“退货”，在下推采购发票时，会根据业务类型生成相应的业务类型的采购发票，入库对应采购入库，入库类型的采购发票会增加应付账款，退货对应采购退货，退货类型的采购发票减少应付账款。

• **发票类型**

并不会限制采购发票的类型，相当于一个备注字段，说明前期确认的这笔业务的发票类型，在下推采购发票时，采购发票中可以修改为正式确认的发票类型。

**• 结算供应商**

若系统参数“启用第三方结算”开启，则业务单据中会增加显示“结算供应商”，区别于“供应商”。供应商用来记录提供商品的单位，结算供应商用来记录结算应付账款的单位。供应商与结算供应商可以相同，也可以不相同，都为单据的必录项。通过暂估入库期初下推生成采购发票时，采购发票中的供应商、结算供应商不允许变更。

• 录入单据时，提供了【选项】，控制修改价税合计、税额时的对其他字段的影响。

• 当系统的第一期即启用期间进行了业务期末结账，暂估入库期初不允许再录入。

## **2.2 批量引入新增**

可根据业务需求设置期初引入的模板，下载模板后按模板要求维护好对应的信息后再进行引入，系统提供了若干的预置模板，用户可按需下载，也可以自定义引入模板。
