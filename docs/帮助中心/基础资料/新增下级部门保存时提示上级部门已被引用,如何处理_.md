# 新增下级部门保存时提示上级部门已被引用,如何处理?

> 产品：金蝶云·星辰 | 分类：基础资料 | 类型：常见问题
> 作者：Cat_Chow | 阅读：696 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/553019136448539392)

【问题描述】

新增下级部门保存时提示上级部门已被引用,如何处理?

【业务场景】

新增下级部门时,提示[职员]的字段[所属部门:]引用了部门【采购一部】的上级部门【采购部】,部门【采购一部】保存失败,如图所示

![](https://vip.kingdee.com/download/0109652e84a9534f46e0bfcd9b0f9e3503b8.png)

【解释说明】

使用过的部门不能作为上级部门,未使用过的状态下才可以增加下级部门。

如图所示:

原一级部门【采购部】名下有职员信息,所以无法直接在【采购部】下面新增二级部门【采购一部】,需要先把一级部门【采购部】下的职员转移,或是删除,才能新增二级部门。即已使用的部门无法新增下级部门。

**同样,如果被单据或者凭证使用过需要删除使用记录再添加,或者通过修改部门名称变通处理一下,即原部门改成其他名称,再新建一个一级部门并且增加下级部门**

【操作步骤】

1,【基础资料】-【职员】找到一级部门【采购部】名下所有职员信息,并修改职员所在部门为其他部门
![](https://vip.kingdee.com/download/010971641242ba3a48abbca0168401aee62a.png)
或是直接删除部门信息,转到【待分配部门人员】
![](https://vip.kingdee.com/download/0109cacbf81df2b04f26bae82e30c3334668.png)
一级部门下,没有任何职员信息
![](https://vip.kingdee.com/download/0109c54f8f60095f4d3b9fee179216907217.png)
2,返回【基础资料】-【部门】,重新新增下级部门即可
![](https://vip.kingdee.com/download/01090325172b8fb34ab08c174a9d7ae8a3e3.png)
3,最后,职员重新分配部门
