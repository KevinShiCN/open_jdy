# 物料需求建议考虑安全库存、最小包装量、即时库存/预计可用库存时，计算计划订单量如何计算？

> 产品：金蝶云·星辰 | 分类：生产管理 | 类型：常见问题, 进阶技巧, 精华汇总
> 作者：移动生产白建凡 | 阅读：184 | 有用：0

# **1 问题描述**

物料需求建议考虑安全库存、最小包装量、即时库存、预计可用库存时是如何计算计划订单量的？

# 2 适用版本

星辰3.0以上专业版/旗舰版

# **3 计算逻辑**

（1）首先计算来源于单据的基础计划订单量

（2）再考虑安全库存产生的计划订单量（如有）

（3）再考虑负即时库存/负预计可用库存产生的计划订单量（如有）

（4）再考虑最小订货量产生的计划订单量

（5）再考虑最小包装量产生的计划订单量（如有）

若（1）、（2）、（3）、（4）产生的计划订单量之和未满足最小包装量，则会进一步产生来源于最小包装量的计划订单量（5），使计划订单量满足最小包装量的需求

若（4）、（5）同时产生需求建议，则合并为一条建议显示

计算过程如下：

| 需求来源 | 计划订单量 | 计划订单量合计 |
| --- | --- | --- |
| 销售订单 | 18000 | 18000 |
| 安全库存 | 1000 | 19000 |
| 预计可用库存 | 14645 | 33645 |
| 最小订货量（40000） | 6555 | 40000 |
| 最小包装量（300） | 40200 |  |

- 考虑销售订单需求、安全库存需求、预计可用库存需求后，计划订单量合计为33645
- 最小订货量是40000，最小包装量是300，所以最终的计划订单量合计需≥40000，且是300的倍数。同时满足≥40000，且是300的倍数的最小数是40200
- 最终产生的计划订单量合计为40200，此时再倒算来自于最小订货量和最小包装量的需求40200-33645=6555
