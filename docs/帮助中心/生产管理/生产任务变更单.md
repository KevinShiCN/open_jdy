# **1 简介  **

## **1.1 功能介绍**

当原生产任务因订单调整、物料短缺、工艺优化、设备故障等原因，需要变更生产数量、交期、用料清单、生产车间等内容时，可发起此单据。单据支持工作流和消息设置。

它的核心价值是：

- **规范变更流程**：支持**多级审批**，确保变更需求经过对应权限岗位审核，避免随意调整。
- **保障信息同步**：配套**消息推送**功能，自动通知生产、仓库、采购等相关协同方，确保执行口径一致。
- **留存追溯依据**：记录变更原因、内容、审批轨迹，为生产复盘提供可查凭证，避免车间盲目生产。

## **1.2 适用版本**

# 2 操作指南

新增生产任务变更单，支持变更生产任务单的商品信息、材料信息和基本信息三部分的内容。

![上传图片](https://vip.kingdee.com/download/0100be5af0b57b7f4c1e8348f5216e3abf87.png)

**一：商品信息**

- 支持对当前期间及以后期间的生产任务单添加商品。（以前期间的生产任务单不支持添加商品）
- 启用联副产品后，支持添加新的联副产品行。
- 支持修改生产任务单的现有商品行（含主产品和联副产品），支持修改数量、计划开工日期和完工日期、商品行备注等信息。修改时，可对比变更前和变更后的内容，变更前内容仅查看不可编辑。
- 支持删除商品行。

![上传图片](https://vip.kingdee.com/download/0100f23037cee2f74a7e88aa5b4700d03c43.png)

**二：材料信息**

- 支持新增子件，支持对多个任务单同时增加子件。
- 支持修改子件，支持修改用量、发料方式、仓库等信息；修改时，可对比变更前和变更后的内容，变更前内容仅查看不可编辑。
- 支持删除子件，仅支持删除没有下游关联单据的材料行，可以同时对多个任务单删除多个材料行。

![上传图片](https://vip.kingdee.com/download/0100759a14d8a20441bfafd0db5831958441.png)

备注说明：变更主料和替代料的时候，需要主料和替代料一起变更。比如删除主料时，则关联的替代料会一起返回至变更单并一起进行删除。

**三：基本信息**

- 支持对已审核未完工的生产任务单变更生产车间。
- 支持变更源单单据备注。

![上传图片](https://vip.kingdee.com/download/01002a75873526844164a5c4eefaa576e522.png)

单据保存时会根据各类修改内容进行校验是否允许修改，比如：

- 修改数量时不允许小于已执行数量；
- 对业务类型是普通的生产任务单新增子件时，不允许子件物料编码与商品编码相同；
- 删除商品或子件时，已有下游关联单据则不允许删除。

>
特别说明：生产任务变更单，审核成功之后**不支持反审核，无法删除！！！**

>
更新情况说明：生产任务变更单审核成功后会立即更新对应的生产任务单，更新的过程中会存在失败的情况，可以查询“变更记录”了解更新成功还是失败。

即：生产任务变更单审核成功，不代表一定会更新至生产任务单成功，以更新日志结果为准。

更新失败时，会给出失败原因，用户需要根据原因解决对应问题后再重做新的生产任务变更单继续变更。

![上传图片](https://vip.kingdee.com/download/010063ed70ed6ce647b4af6062c0db113982.png)

**失败原因举例：**生产任务单按可用库存控制负库存时，生产任务变更单同时往三个任务单的商品新增相同的材料A，在更新至任务单时前面两个任务单新增材料A成功，后面一个因材料A的可用库存不足，导致更新任务单失败。

此时3个任务单会有2个变更成功，1个变更失败。变更失败的任务单，需要用户解决可用库存为负的情况之后才可以继续变更。

>
业务控制说明：

当生产任务单存在已保存、未审核的生产任务变更单时，控制生产任务单不支持以下业务操作：

- 生成下级工单
- 领料、配套领料
- 质检
- 入库
- 移仓
- 预留
- BOM同步工单
- 通过【变更】按钮变更

![上传图片](https://vip.kingdee.com/download/010020dfd56d59fd48218e7b2a5f86846a6d.png)

>
生产任务变更单和【变更】按钮的区别：

**【变更】按钮**

优势：①快速处理，变更内容后点击保存即可生效。

劣势：①不支持多级审批，仅支持一张一张的进行变更。②不支持批量处理。

**生产任务变更单**

优势：

①支持设置工作流，启用多级审核。

②支持消息设置，审核成功时推送消息给到指定用户。

③支持同时对多个任务单进行变更。

④支持对本期及以后期间的生产任务单添加新商品行。

劣势：

①需要另起一张单据，变更流程更长，需审核后才生效。

②没有审核前，控制业务不能继续操作（见上方的业务控制说明）。

③不可逆向操作，审核成功后不支持反审核和删除，要修改或取消变更时需要继续走变更。

④生产任务单有关联的生产变更单之后，也不支持反审核和删除，可手动完工。

>
生产任务变更单的优势和劣势都存在，请结合实际业务进行选择何种方式进行变更。
