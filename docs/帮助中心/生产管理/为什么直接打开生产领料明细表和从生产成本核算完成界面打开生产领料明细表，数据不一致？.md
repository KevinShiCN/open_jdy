# 为什么直接打开生产领料明细表和从生产成本核算完成界面打开生产领料明细表，数据不一致？

> 产品：金蝶云·星辰 | 分类：生产管理 | 类型：常见问题, 快速入门
> 作者：Esperanza | 阅读：482 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/487286152605492992)

****

1、问题描述

直接打开生产领料明细表，过滤2023.8.01-2023.8.31的数据，跟从生产成本核算完成界面点开的生产领料明细表的数据不一致。

![上传图片](https://vip.kingdee.com/download/0100f81f557546ed4b70b3b09679024059fc.png)

![上传图片](https://vip.kingdee.com/download/0100f63897ba05d648a0ae614b358fe565f4.png)

2、原因说明

生产领料单的取数规则是：根据筛选条件中设置的单据日期范围、领料/退料部门等等条件过滤出对应的数据。

上述问题中，两次打开的报表，除了物料编码外，日期范围和其他过滤条件都一样。从生产成本核算完成界面打开的生产领料明细表过滤条件里多了很多物料编码。

在生产成本核算完成界面打开的生产领料明细表，过滤条件里默认过滤的是本期参与成本核算的任务单的领/退料数据。如果当前期间有生产任务单本期完工数量为0（在完工与在制品归集界面可以查看），则这部分生产任务单不会参与本期的生产成本核算，所以如果有材料仅被这部分任务单领用，就不会出现在生产成本核算完成界面点击打开的生产领料明细表中，所以有差异。
