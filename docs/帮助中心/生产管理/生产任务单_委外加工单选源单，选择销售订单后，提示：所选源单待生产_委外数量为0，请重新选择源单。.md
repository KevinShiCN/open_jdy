# 生产任务单/委外加工单选源单，选择销售订单后，提示：所选源单待生产/委外数量为0，请重新选择源单。

> 产品：金蝶云·星辰 | 分类：生产管理 | 类型：快速入门, 常见问题
> 作者：Esperanza | 阅读：583 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/448861630093137408)

**1、问题描述**

生产任务单/委外加工单选源单，选择销售订单后，提示：所选源单待生产/委外数量为0，请重新选择源单。

![上传图片](https://vip.kingdee.com/download/01009b308c51ad0049989a81c030d7546565.png)

**2、原因分析**

选择的销售订单可能已经销售出库、或已关联生成过委外加工单、生产任务单、产品入库单或采购单据，所以待生产/委外数为0。

可在销售订单列表勾选该单，点击【全流程跟踪】查看是否关联生成了上述单据。

![](https://vip-admin.kingdee.com/download/0109150733dce8644a8d9d261003addad55f.png)

具体的逻辑可参考这个帖子：销售订单部分关联了产品入库单/生产任务单，剩余的部分，产品入库单/生产任务单选源单选不到这个订单了，为什么？:[https://vip.kingdee.com/link/s/ZHCXk](https://vip.kingdee.com/link/s/ZHCXk)

**3、解决方案**

如果不希望做生产任务单时，待生产数量为0的单据被选择到，可以通过生产任务单列表界面的【批量新增】功能生成生产任务单。该界面默认勾选了【可生产数量为零不显示】的参数。

![](https://vip.kingdee.com/download/0109c304ba51fcad49939bc60d228b7c1b7b.png)

关联问题：参数设置的是允许生产任务单的数量大于订单的可生产的数量，但是选择销售订单关联做生产任务单还是提示待生产数量为0呢？

解答：因为这张订单在前面已经全部关联生成过任务单了，就不满足下推的条件了。一张订单只允许超额生产一次。可以找到之前下推的那张生产任务单，将数量改大或通过生产任务单变更的功能修改数量。

![](https://vip.kingdee.com/download/010939df5044eba64beba6a261f9e647cb79.png)
