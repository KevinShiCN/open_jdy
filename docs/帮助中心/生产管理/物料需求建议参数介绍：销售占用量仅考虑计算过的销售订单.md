# 物料需求建议参数介绍：销售占用量仅考虑计算过的销售订单

> 产品：金蝶云·星辰 | 分类：生产管理 | 类型：进阶技巧, 功能介绍, 新特性
> 作者：蒋章龙 | 阅读：2890 | 有用：15
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/430679831122896896)

**1、简介  **

**1.1功能介绍**

       在做物料需求建议的计算时，我们经常会考虑物料的预计可用库存，即生产在制量、采购在途量、销售占用量等等，按类型可分为预计入库量和预计出库量两类。在不考虑损耗率、成品率、安全库存等因素时，物料的净需求=毛需求-（即时库存+预计入库量-预计出库量）

       假设企业目前接到了3笔销售订单，数量如下表：

| 销售订单 | 数量 |
| --- | --- |
| S01 | 100 |
| S02 | 110 |
| S03 | 120 |
      在考虑生产在制量、不考虑销售占用量时，分别对这三张订单进行物料需求建议计算，默认计划订单全部投放生产任务单并审核，得出结果如下：

|  | 毛需求 | 预计可用库存 | 净需求 | 计划订单 |  |  |
| --- | --- | --- | --- | --- | --- | --- |
| 即时库存 | 预计入库 | 预计出库 |  |  |  |  |
| 计算S01 | 100 | 0 | 0 | 0 | 100 | 100 |
| 计算S02 | 110 | 0 | 100 | 0 | 10 | 10 |
| 计算S03 | 120 | 0 | 110 | 0 | 10 | 10 |

      可以看出，计算S01时，产生计划订单100，全部投放生产任务单并审核，100会作为预计入库（**生产在制量）**参与S02的计算，导致计算S02时，只产生了计划订单10，进而导致计算S03时，只产生了计划订单10，最终总计划订单数100+10+10=120，但是对于整体实际的需求来说100+110+120=330，应该产生330的计划订单量，对于企业来说，会因此少生产。



     承上例，在同时考虑生产在制量和销售占用量时，分别对这三张订单进行物料需求建议计算，默认计划订单全部投放生产任务单并审核，得出结果如下：

|  | 毛需求 | 预计可用库存 | 净需求 | 计划订单 |  |  |
| --- | --- | --- | --- | --- | --- | --- |
| 即时库存 | 预计入库 | 预计出库 |  |  |  |  |
| 计算S01 | 100 | 0 | 0 | 230 | 330 | 330 |
| 计算S02 | 110 | 0 | 330 | 220 | 0 | 0 |
| 计算S03 | 120 | 0 | 330 | 210 | 0 | 0 |

       可以看出，计算S01时，S02和S03会作为预计出库（**销售占用量）**参与计算，产生计划订单330，相当于全局考虑需求，后续再计算S02和S03时，不会再有需求产生。

       在这种情况下，虽然从整体角度上看，计算出的计划订单量是准确的，但是如果客户订单较多，不希望一次性全部投产，希望每次计算仅产生本单的需求，就无法实现了。

       于是增加了这个参数：**销售占用量仅考虑计算过的销售订单**
**
**

**       **承上例，在同时考虑生产在制量和销售占用量时，勾选这个参数，分别对这三张订单进行物料需求建议计算，默认计划订单全部投放生产任务单并审核，得出结果如下：

|  | 毛需求 | 预计可用库存 | 净需求 | 计划订单 |  |  |
| --- | --- | --- | --- | --- | --- | --- |
| 即时库存 | 预计入库 | 预计出库 |  |  |  |  |
| 计算S01 | 100 | 0 | 0 | 0 | 100 | 100 |
| 计算S02 | 110 | 0 | 100 | 100 | 110 | 110 |
| 计算S03 | 120 | 0 | 210 | 210 | 120 | 120 |

       可以看出，在考虑了这个参数后，计算S01时，S02和S02由于没有计算过，不会作为预计出库（销售占用量）参与计算，于是产生计划订单100；

       计算S02时，由于S01已经计算过，于是作为预计出库（销售占用量）参与计算，同时第一次计算产生的计划订单100（投放生产任务单并审核），会作为预计入库（**生产在制量**）参与计算，于是产生计划订单110；

       计算S03时，由于S01和S02已经计算过，于是作为预计出库（销售占用量）参与计算，同时第一、二次计算产生的计划订单210（投放生产任务单并审核），会作为预计入库（生产在制量）参与计算，于是产生计划订单120。

        最终，三次计算均仅产生了本单的需求，并且从整体上看，计划订单量也是准确的。

**1.2适用版本**

- 金蝶云·星辰专业版

**1.3适用场景**

- 多张销售订单需要分开进行物料需求计算，每次仅产生本单需求

**1.4参数说明**

       此参数需搭配预计可用库存参数中的销售占用量一起使用，单独勾选无效。销售订单参与过物料需求建议计算，或已关联采购订单/生产任务单/委外加工单，都视作计算过。
