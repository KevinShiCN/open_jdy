# 为什么单据上的价税合计不等于含税单价*数量？

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：4333 | 有用：16
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/358181644525434624)

# 【问题概述】

为什么单据上的价税合计不等于含税单价*数量？

# 【原因说明】

星辰的价税合计是经过许多字段运算后算出来，并不是直接等于含税单价*数量，不相等可能有以下几点原因：

### 1）计算精度问题

价税合计是经过运算后根据币别精度取值的，在和单价、税率等计算后，四舍五入后有可能存在精度差异；

### 2）存在折扣

如果商品取价取了价格策略中的折扣，又或者有做整单折扣之类的话，那么金额实际还要减掉折扣部分才能算出价税合计；

### 3）单价和金额的精度不一致

单价的精度大于金额精度，例如单价是4位小数，而金额只有2位小数；

### 4）系统设置-系统参数-基础资料参数中“商品价格为含税单价”没有开启

当单据上没有录入折扣、折扣额、整单折前价税合计、整单折扣分配额等字段的前提下，且单据上同时有录入单价、含税单价、税率，单据上的价税合计是否等于数量*含税单价，是取决于系统参数中“商品价格为含税单价”是否有开启。（如下图）。这是为什么呢？因为当单据上两个单价字段都有值时，系统取不同的值计算，如果价格是非整数，可能会存在精度差异，导致计算结果不一样，因此必须要用户指定好到底以那个价格作为计算的依据。

![上传图片](https://vip.kingdee.com/download/010077a43016264e4e40a525e0422af1bbfc.png)

关于第四点，下面通过一个简单的例子进行说明：

假设如下图，在没有任何折扣的字段值前提下，单据上的**单价、含税单据、税率**都有值：

![上传图片](https://vip.kingdee.com/download/0100582bfdd9964d4ad48add142dbc8b9896.png)

- 如果系统参数“商品价格为含税单价”**开启**，此时去录数量，那么**价税合计=含税单价*数量**，如下图

![上传图片](https://vip.kingdee.com/download/01007bc7f2c67efe4874addac6d47d1ca0bf.png)

- 如果系统参数“商品价格为含税单价”**不开启**，此时去录数量，那么**价税合计=【含税单价/（1+税率）】*数量*+税额**也就是，要先根据含税单价、税率，反算出单价，然后再去计算出最后的价税合计，如下图

![上传图片](https://vip.kingdee.com/download/0100bd71749cf1d34d90b43e235aabfc7fef.png)

### 5）单据上的“选项”（销售出库单则看“金额计算参数”）开启了“修改价税合计后反算单价”，并且录单时，手动修改过价税合计

# 【解决方案】

综上所述，价税合计并不是直接等于含税单价*数量，还有考虑商品行的折扣以及整单折扣的影响、系统参数、选项参数的影响。建议录单时把字段：实际含税单价、折扣额、整单折前价税合计、整单折扣分配额、增值税税率 (%)、税额都开启显示，看看是否有值，也检查一下单据选项（销售出库单则看“金额计算参数”），避免产生歧义。

最准确的还是看单据上的“实际含税单价”，因为实际含税单价=价税合计/数量，此时价税合计已经完成了计算，不会因为上述原因而存在一些精度的差异。
