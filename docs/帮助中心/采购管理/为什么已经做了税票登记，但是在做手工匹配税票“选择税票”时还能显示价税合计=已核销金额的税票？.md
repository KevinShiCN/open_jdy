# 为什么已经做了税票登记，但是在做手工匹配税票“选择税票”时还能显示价税合计=已核销金额的税票？

> 作者：进销存赵嘉丽 | 阅读：33 | 有用：1

# **1 问题描述**

为什么已经做了销售税票登记，但在手工匹配税票“选择税票”时仍会显示价税合计等于已核销金额的税票？正常情况下，已全部核销的税票不是应该无法被选中吗？

![](https://vip.kingdee.com/download/0100f9e3e6b4adea4f439557a915e237acb7.png)

![](https://vip.kingdee.com/download/0100aae1fc5fb1ba48e898a01231236a1893.png)

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

出现这种现象，通常是因为销项发票的分录行比销售出库单的分录行少，且销项税票中存在价税合计一正一负的行，正负抵消后价税合计为0，但总计不为0。同时，“手工匹配规则”未开启“单据商品行与税票商品行按顺序匹配”。

回到前面问题描述中的案例，恰好是下面的情况。

税票详情：第一至第四行的价税合计相加结果为0，第五、六行的价税合计相加结果为3000。

![](https://vip.kingdee.com/download/0100e75d07ce349f4a898f74d3c55b527941.png)

出库单详情：共2行分录，价税合计为3000。

| 商品行 | 价税合计 |
| --- | --- |
| 1 | 1000.00 |
| 2 | 2000.00 |
| 合计 | 3000.00 |

在此情况下，若未启用“单据商品行与税票商品行按顺序匹配”规则，由于第一至第四行的价税合计相加为0，无法匹配税票，仅第五、六行可匹配。匹配完的结果就是，在手工匹配税票弹窗中，已核销金额=出库单价税合计，但是发票管理中仍有部分商品行未开票（就是上面税票详情中的1-4行），“核销状态”显示“部分核销”，做税票登记时，就还会看到这些发票，只有当“核销状态”是“已核销”，才不会显示。

![](https://vip.kingdee.com/download/0100ddf2c59166904800a116c14db2cef4fe.png)

# **4 解决方案**

删除已存在的税票登记记录，新增税票登记，并在“手工匹配规则”中选择“单据商品行与税票商品行按顺序匹配”，然后重新登记即可。
