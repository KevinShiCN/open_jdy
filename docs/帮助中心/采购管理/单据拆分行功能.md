# 单据拆分行功能

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：功能介绍, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：4188 | 有用：19
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/383348138188923648)

## 1、功能介绍

![上传图片](https://vip.kingdee.com/download/010064d2ffb8dc584f7da6f6ec01ef464d8f.jpg)

用于关联源单的商品不同仓库、批次、保质期、单价的出入库业务拆分开单。

## 2、应用场景

1）采购订单下推入库，批次商品入库时分多个批次入库

2）订单下推出/入库，商品出/入库时以多个仓库出/入库

例如：向供应商采购商品A共20件，实际到货同时出现3个批次的情况下，采购入库单通过单据行拆分，可以实现在采购订单做一次下推后 ，直接拆分成三行分录，达到同时反写采购订单的源单状态，不同再通过多次选源单或者多次下推的方式来实现反写，从而提高批次商品的录单效率。

## 3、适用版本

星辰所有版本

## 4、功能特性

1） 适用单据：第一版迭代仅适用于采购入库单、销售出库单、委托代销单

2） 拆分后的商品行可以保持与源单关联关系，商品分录上的源单单号、源单行号、源单类型会有值

3） 拆分方式及含义：

| 拆分方式 | 含义 | 举例 | 交互页面（点击可放大查看） |
| --- | --- | --- | --- |
| 自定义拆分 | 按指定数量拆分出指定数量 | 例如：数量为10，可以指定拆成2+8，或者3+7，或者5+5等情况。 |  |
| 分摊拆分 | 将数量平均拆出，最后一行可能存在差额计算（就是有的会有除不尽的情况） | 例如：数量为5，拆分成3行，那么最后一行就会跟前面2行不太一样；如果是数量6拆分成3行，那么每行都是2 |  |
| 复制行 | 复制一行相同数量的行 | 假设商品行数量为2，复制后会新增一行数量也是2 |  |

4） 拆分方式分摊拆分和复制行时，最多新增20行！

5） 拆分位置

| 拆分位置 | 含义 | 拆分行位置示例（蓝色箭头）（点击可放大查看） |
| --- | --- | --- |
| 选中行下方插入 | 拆分出来的行紧挨被拆分的源行的下一行 |  |
| 最后一行下方插入 | 拆分出来的行直接放到最后一行分录 |  |

6） 若单据已审核，不允许执行中途拆分，会提示“单据已审核，不允许执行”

7） 若启用了工作流的单据，不允许执行中途拆分，会提示“单据正在工作流中流转，不允许执行！”

8） 拆分分录时会判断系统参数是否允许超过源单的数量，如果系统参数控制了下游单据不得超过源单数量，那么保存单据时系统会校验。

9） 套装商品目前不支持拆分

## 5、应用流程

下面以普通的批次销售出库为例讲解如何使用该功能

1） 新增销售订单01，批次商品，10个

![](https://vip.kingdee.com/download/0100a9c482fb3edd43a0b2f8f3e5d75ceda0.png)

2） 对销售订单01下推销售出库单

a. 注意，如果是手动新增的单据，录单后要先选商品，如果未选商品，录入后会提示“选中分录未填写字段【商品编码】不允许拆分

![](https://vip.kingdee.com/download/01003d14d97394d64e97b817d85651eb57d2.gif)

3） 假设要对这10个商品做拆分，下面按三种拆分方式分别演示：

| 拆分方式 | 拆分目标 | GIF演示（点击可放大查看） | 拆分后关联效果（点击可放大查看） | 注意事项 |
| --- | --- | --- | --- | --- |
| 自定义拆分 | 把数量10拆成2个批次，一个批次数量是2，另一个批次数量是8 |  |  |  |
| 分摊拆分 | 把数量10平均拆成5行 |  |  | 最多新增20行 |
| 复制行 | 复制一行数量为10 |  |  | 使用复制行时，必须填写【2-20】之间的数字，不能填写1 |

4） 无论是上面那种方式拆分，拆分完之后，保存销售出库单后，销售订单01的执行状态都会反写成“全部执行”，表示订单上的商品都通过关联源单方式完成了下推出库了：

![](https://vip.kingdee.com/download/0100eda1c5796d224bec82f9177efed71ad5.png)

在单据列表点击全流程跟踪也可以查到对应的出库单，可以看到分录行上的源单单号，源单行号、源单类型信息：

![](https://vip.kingdee.com/download/010062ff5c16a39849ef9b33056e9ff21e07.gif)
