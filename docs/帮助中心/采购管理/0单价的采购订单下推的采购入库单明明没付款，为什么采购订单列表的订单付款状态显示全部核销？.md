# 0单价的采购订单下推的采购入库单明明没付款，为什么采购订单列表的订单付款状态显示全部核销？

# 【问题概述】

0单价的采购订单下推的采购入库单明明没付款，为什么采购订单列表的订单付款状态显示全部核销？

# 【原因说明】

![上传图片](https://vip.kingdee.com/download/0100f2982a0a819c41e88110dfede15f2a8d.png)

订单付款状态是根据订单关联的采购入库单的付款核销金额与订单成交金额对比后反写的。因为订单的成交金额跟入库单上的付款金额都为0，差为0，相当于全部付款了，所以入库核销状态自动反写成了“全部核销”，属于应用问题。

# 【解决方案】

如果企业的采购业务是使用订单下推入库单的，为了避免产生理解上的歧义，建议在录订单时就录入单价。如果事后才能确认单价，可以在采购订单上使用变更功能，对订单进行补录单价。

![上传图片](https://vip.kingdee.com/download/0100b316349bf51f428aa454994c2716769b.png)

补录完价格后，采购订单列表的订单付款状态、成交金额、已执行金额、未执行金额也会被反写。

![上传图片](https://vip.kingdee.com/download/0100ca735e03b29d459aa4dbb9f17c177503.png)

当订单已全部入库单并且已审核，且货物已出库无法再反审核入库单时，如果采购入库单需要补录单价，可以使用金额变更。

![上传图片](https://vip.kingdee.com/download/0100216b86400f5c43969a3f9c13ac2a9df1.png)

销售订单下推销售出库也是一样的逻辑。本文就不再赘述。
