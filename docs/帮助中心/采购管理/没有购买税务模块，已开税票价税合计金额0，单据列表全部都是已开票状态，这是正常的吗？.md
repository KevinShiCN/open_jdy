# 没有购买税务模块，已开税票价税合计金额0，单据列表全部都是已开票状态，这是正常的吗？

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：50 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/801833710042682624)

# **1 问题描述**

没有购买税务模块，采购入库单的已开税票价税合计金额为0，单据列表的“税票状态”和“行税票状态”都是“已开票”，查询税票登记也是没有记录，这是正常的吗？

![上传图片](https://vip.kingdee.com/download/01006565b47bc0a64ab19b2ae2f8e9ed51ac.png)

![上传图片](https://vip.kingdee.com/download/01008cb016a22f3744fdae446c965c481e2a.png)

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

出现以上原因，是因为采购入库单中的商品为赠品，并且只有一行商品，赠品行默认是已开票。

![上传图片](https://vip.kingdee.com/download/0100d010586006434313a6f468502f189956.png)

## 关于赠品行的自动处理/0金额行的处理逻辑

| 商品行的状态 | 销售出库单 | 采购入库单 |  |
| --- | --- | --- | --- |
| 开启暂估 | 无暂估 |  |  |
| 赠品行 | 自动处理开票状态，自动标记为已开票： | 自动标记为已开票： | 自动标记为已开票： |
| 非赠品行但0金额行 | 不自动标记已开票，可与其他非0金额行一起通过销售税票登记、手工匹配等方式更新开票状态： | 不自动标记开票，需要进行暂估调整： | 不自动标记已开票，可与其他非0金额行一起通过采购税票登记、手工匹配等方式更新开票状态： |

## 关于含有赠品行的单据的税票状态特别说明

“税票状态”属于单据头字段，**该字段是由单据中的非赠品行的“税票状态”决定的**，如果单据中的商品行都是赠品行，则分录字段“行税票状态”全部是“已开票”，整单的表头字段税票状态也是“已开票”。
