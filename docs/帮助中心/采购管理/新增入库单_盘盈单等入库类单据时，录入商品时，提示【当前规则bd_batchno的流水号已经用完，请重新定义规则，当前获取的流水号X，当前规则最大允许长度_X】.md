# 新增入库单/盘盈单等入库类单据时，录入商品时，提示【当前规则bd_batchno的流水号已经用完，请重新定义规则，当前获取的流水号X，当前规则最大允许长度:X】需要如何处理？

【问题概述】

新增入库单/盘盈单等入库类单据时，录入商品时，提示【当前规则bd_batchno的流水号已经用完，请重新定义规则，当前获取的流水号X，当前规则最大允许长度:X】需要如何处理？

**![](https://vip.kingdee.com/download/0109e06e5594f6dc436895d09b646b73974b.png)**

【原因说明】

出现上述问题是由于批次的流水号已经被用完了，系统自动获取流水号时发现当前的流水号长度已经无法满足，导致无法新增批次。

还是上图的例子，批次编码规则设置如下，设置了流水号1位数，起始值是1，那么最大流水号就是9，可以用的流水号就是1-9之间九个流水号。

![](https://vip.kingdee.com/download/0109274fc3e8645d407ba4cd5748c163fa8d.png)

且系统参数开启了入库批次自动生成：

![](https://vip.kingdee.com/download/01098c61a038dd2b436197529be357f5321b.png)

此时单据上的最大流水号已经有9了，再新增商品行时，系统正常是会自动创建流水号10，但是由于编码规则里，批次的流水号长度只有1位，最大流水号只能是9，现在相当于冲突了，因此才会出现上述的提示。
【解决方案】

修改批次的编码规则，把流水号长度改成2，再重现选择商品即可。

**第一步：打开系统设置-编码规则-批次。**

![](https://vip.kingdee.com/download/0109e99063b62e58430f8a5100597ca5570c.png)

**第二步，****修改流水号长度为2，保存。**

![](https://vip.kingdee.com/download/01096599ba5ea0274b5a9b7bcbf0af792098.png)

**第三步，重新到单据上新增商品即可，批次会自动生成PC2022091310**

![](https://vip.kingdee.com/download/01092cb6938723794b62b37234f07a2cf0a1.png)

【适用版本】

星辰所有版本

>
实用小tips：

**为了避免上述情况出现，建议在系统实施阶段前，先评估实际业务中的一些高频场景的频次和数量，考虑存量及未来的一个平均增量，如批次数，商品数等等，提前在编码规则预留足够的流水号长度，减少上述业务操作中断带来的不便。**
