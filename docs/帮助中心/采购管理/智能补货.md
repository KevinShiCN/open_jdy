# 1、简介

![](https://vip.kingdee.com/download/01005fca1e61a0f14d74af7d571f6cee8108.png)

## 1.1 功能介绍

企业为了避免出现断货的情况，需要提前补充库存。不同行业、不同企业对于备货量的需求是不一样的。本系统提供补货方案的设置，企业可以根据自己的需要，设置不同的公式来计算建议采购量。

## 1.2 适用版本

金蝶云·星辰3.1所有版本

## 1.3 系统路径

采购管理-采购单据-智能补货

# 2、主要操作流程

## **2.1 整体流程**

### **1）打开采购管理-智能补货**

### **2）设置补货方案**

- **选择商品范围**

可以按实际业务需求设置补货范围：按特定的商品设置，或者不设置商品，作为所有商品都适用的通用方案

注意：智能补货界面,不选默认是选择全部,商品条数一次最多支持计算1000条商品,如果商品数太多，无法一次选择完毕，建议可以按商品类别设置智能补货方案。

- ** 设置补货计算公式**

 可以选择指定仓库（若指定了具体仓库，将只计算所选仓库的库存数量），也可以不设仓库，作为通用方案。

| 建议采购量 | = | 加项：备货需求库存 | + | 销售预测量： | - | 减项：现有库存 |
| --- | --- | --- | --- | --- | --- | --- |
| ○最低库存○最高库存○安全库存 | X天日均销量×商品备货天数 | ○即时库存○预计可用库存 |  |  |  |  |
| 公式字段含义 |  | 来源于商品资料中设置的数量。可以选择其中一项作为计算的依据 |  | X天日均销量=最近X天内（含今天）已审核的销售出库单数量的合计/X天（不区分仓库）系统支持3/7/15/30/90天五种类型。商品备货天数：来源于商品资料中设置的备货天数 |  | 即时库存：为系统的实时在库的库存预计可用库存：此处单独可设置预计可用库存的取值来源 |

**最低库存，最高库存，安全库存**：来源于商品资料中设置的数量。可以选择其中一项作为计算的依据

**销售预测量**：

- **X天日均销量X商品备货天数**；

举例：3天日均销量，就是最近3天内（含今天），已审核的销售出库单数量的合计/3天（不区分仓库）

- **商品备货天数**：来源于商品资料中设置的备货天数

**即时库存**：为系统的实时在库的库存

**可用库存**：此处单独设置可用库存的取值来源，不直接取预计可用库存表中的可用库存数量

**组装件计算到明细物料**：若某商品是组装件，且该商品有审核状态的组装单模板，勾选该项后，将计算该商品所需要的子件的采购量，而不是该商品本身的采购量。

### **3）建议采购量计算**

- 设置好补货公式后，点击“开始计算”，将查询出所选择的商品并根据公式计算“建议采购量”；
- 当建议采购量为正数时，本次采购量默认为建议采购量根据采购单位换算出的数量；当建议采购量为负数时，本次采购量为空，若企业仍需要采购该商品，也可以手工填写本次采购量；
- 字段取数说明：

| 字段名称 | 取数来源 |
| --- | --- |
| 采购单位 | 商品资料中的采购单位 |
| 供应商 | 商品资料中的默认供应商 |
| 采购业务员 | 需要手工维护（后续会优化） |
| 参考采购价 | 商品资料中的采购价 |
| 参考采购金额=参考采购价×本次采购量，用于预估需要的采购资金。 |  |

- 支持批量填充的字段：仓库、供应商、职员基础资料、产地、备注、交货日期

## 2.2 **拆分子件采购/生成采购订单/采购申请单**

### **1）拆分子件采购**

**应用场景：**

企业销售的商品是组合件商品时，当收到销售订单后，需要先去采购子件，再将采购回来的子件组装件组合成组件。那么可以选择点击“拆分子件采购”，将商品拆分成子件，并按子件再生成对应的采购订单/采购申请单(保存状态，可以修改)。在计算子件的采购量时，需要同步考虑子件的在途量，子件在销售订单中的需求量以及子件自己的库存等因素。

**功能特性：**

进行智能补货时，组合件商品可由用户自主决定是对组合件补货，还是对子件补货。当需要最子件补货时，在组合件的需求量基础上，计算子件的补货量。

**应用流程：**

如果只需对已下订单的商品进行补货，可直接调取销售订单中的商品

![](https://vip.kingdee.com/download/0100d99b7c90400f4a45bb050aa646525ee4.png)

若选择出来进行补货的商品为组合件商品，可以查看该组合件商品的组装模板；

![](https://vip.kingdee.com/download/01006100048a86784993b113b37c8f775637.png)

若需要对该组合件的子件进行采购，需要进行第二步补货量计算“拆分子件采购”；

第二步计算会根据组合件的需求量，即第一步中的“本次采购量”基础上进行计算

![](https://vip.kingdee.com/download/01004daecc0a19504fb1802ffdf9d1850112.png)

上图中的“子件数量”，即根据上一步组合件的待采购量和组装模板的计算公式，计算出来的需求量和智能补货设置的公式计算子件的待采购量。

### **2） ****生成采购订单/采购申请单（适用于仅专业版/旗舰版）**

**应用场景及价值：**

如果要补货的商品没有维护组装模板，属于正常采购，那么可以根据实际业务需求，填好对应的必录字段后，点击“生成采购订单”或“采购申请单”，系统会把供应商，业务员，部门，商品，采购单位，本次采购量，参考采购价，预计交货日期，备注带入采购订单/采购申请单中，生成的采购订单/采购申请单为保存状态，可以修改。
