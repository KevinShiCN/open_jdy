# 商品共享/发送商品

### **1、企业互联、外部商品库、商品共享是什么关系？对于企业的价值是什么？**

一句话说，外部商品库、商品共享都是企业互联大功能下更细分的功能，企业互联是载体，外部商品库和商品共享是依托于这个载体做的信息流的互通。

- **外部商品库**涵盖了原企业互联所对应的“供应商商品编码”、“客户商品编码”两个功能，并在原功能基础上扩展了新功能以及单据引用逻辑上的变更。关联知识：[https://vip.kingdee.com/link/s/MURSv](https://vip.kingdee.com/link/s/MURSv)
- **商品共享**将商品资料从A账套发送到B账套，实现B账套的商品资料快速创建，并在A账套中的《外部商品库》中自动创建客户商品编码匹配关系，为企业互联的基础数据搭建提供了更便捷的途径。关联知识：[https://vip.kingdee.com/link/s/MUrQH](https://vip.kingdee.com/link/s/MUrQH)

### 2、**如何设置发送商品的编码规则？是继承发送方的还是以接受方的编码规则为准？**

答：可以在互联供应商的新增页面中维护。勾选“接收该供应商的商品资料”，会出现“商品编码生成规则”，可由用户自定义。

![](https://vip.kingdee.com/download/010076327172ad5c4be781567e66a8398a59.png)

### 3、通过商品发送后，接受方还需要维护外部商品库吗？

答：需要。当账套A（互联供应商）发送商品给账套B（互联客户）时，账套B的商品资料中会创建对应商品条目，但外部商品库不会自动生成新的条目，只在A账套中生成。如果B需要再次向A发送采购订单，B需要在自己的账套内手动维护外部商品库。

### 4、发生商品支持携带价格资料吗？支持携带商品的自定义字段吗？

答：支持。2025年12月起已经支持如下功能特性：

• 企业互联发送商品，支持携带商品的价格资料、默认仓库/仓位、生产相关的最小订货量/关键件/倒冲领料等、财务的进项税率/销项税率/商品开票名称等字段。

• 企业互联发送商品，支持携带商品自定义字段（包含文本、日期、数值、引用辅助资料四种类型的字段）。

详见知识：** **[企业互联支持发送单据和商品的自定义字段及商品价格资料（2025）](https://vip.kingdee.com/link/s/Z4zjU)

# 单据发送规则

### 1、企业互联支持星辰不同版本之间互联吗？例如标准版和专业版，专业版和旗舰版。

答：企业互联支持星辰不同版本的发送，包括星辰2.0和星辰3.1版本，以及星辰专业版与星辰旗舰版之间的相互操作。也支持同一个租户ID内不同子账套之间的相互发生。

### 2、企业互联，子账号把采购订单发送给供应商，供应商账套中销售订单制单人的取值逻辑是什么呢？

答：通过企业互联发送的单据，制单人都是取得账套的**服务管理员**账号。

### 4、为什么之前可以发送企业互联单据，移交了服务管理员后，不能再发送了？

答：由于企业互联的秘钥中是包含了服务管理员的登录名的，因此，一旦服务管理员发送了改变，秘钥也会发生变化，需要重新维护。

### 5、为什么接受方收到单据的价格跟价格取数优先级取价的不一样？

答：企业互联的单据具有价格一致性，也就是发送的单据价格应与上下游保持一致，不依赖于价格优先级设置。

### 6、A账套单据价格会携带到B账套生成的单据上吗？

答：会，企业互联的单据具有价格一致性，也就是发送的单据价格应与上下游保持一致，不依赖于价格优先级设置。

### 7、多单位商品，是否支持按接受方账套商品的换算关系进行换算？

答：不支持。企业互联仅根据发送单据上的单位进行匹配，假设A是互联客户，B是互联供应商，那么A向B发送采购订单，系统将会依据A账套内的外部商品库内维护好的换算关系传递数据。无法根据接收方B账套的换算率进行换算，因为A账套没有存储B账套的单位换算信息。

### 8、已经发送给客户的销售出库单，后面又操作了销售退货，并开启了退货重发，此时还能再次发送销售出库单吗？

答：支持将出库单发送给客户后进行销售退货操作。退货重发后，是否可以重新出库并发送给客户，取决于客户的采购订单是否已关闭。若采购订单已关闭，则不允许发送。若采购订单未关闭，则可以发送。

### 9、A账套的采购订单发送到了B账套的销售订单之后，A账套的采购订单修改后可以再次发送给B账套吗？

答：不可以。已经发送成功的单据不允许再次发送。

### 10、如果单据上开启了单据体的“是否赠品”，发送后，接受方的单据上“是否赠品”会开启吗？

答：会，单据体的“是否赠品”会同步携带到接受方的账套上。

### 11、序列号商品支持发送吗？

答：支持，前提是接收方该序列号库存状态不是“在库”，因为如果在库，就违背了星辰相同商品序列号不允许重复的规则了。

### 12、当接受方的系统参数开启了单价为0、信用额度预警等参数且控制强度为“提示，允许继续操作”，当单据包含0单价，或者信用额度超过设置的值时，为什么发送单据时也会失败？

答：企业互联的单据发送时不支持这种交互式的确认模式，因此只要触发了校验，就会发送失败，哪怕控制强度是“提示，允许继续操作”。解决办法就是接受方关闭这些需要校验的参数，或则控制强度改成“不控制”，然后发送方再重新发送。

### 13、同一个商品，采购方启用了多单位品，销售方没有启用多单位，这样可以发送采购订单吗？

答：不能。多单位商品必须企业互联双方都启用多单位并且在销售方的外部商品库维护好对应的关系才能发送，否则将采购订单发送给供应商时会提示“失败原因：商品“XXXX”在对方账套中无相同名称的单位。”

### 14、金蝶云APP能推送企业互联单据吗？

答：支持。跟PC端支持的发送流程是一样的，功能操作详情请查看：[金蝶云APP能推送企业互联单据吗？](https://vip.kingdee.com/link/s/ZxELG)

### 15、单据发送支持携带单据的自定义字段吗？

答：支持。2025年12月起企业互联发送单据，支持携带以下三种类型的自定义字段：日期、数值、引用辅助资料。连同先前已支持的文本类型，目前共支持四种类型的自定义字段。

>
根据《单据新增字段》中相同字段名称和字段类型匹配不同账套的自定义字段。若接收方为公式型字段，不支持发送；若接收方的辅助资料分类和辅助资料名称不存在或不唯一，不支持发送；引用基础资料、带出已有基础资料属性的自定义字段不支持发送。

详见知识：** **[企业互联支持发送单据和商品的自定义字段及商品价格资料（2025）](https://vip.kingdee.com/link/s/Z4zjU)

发送规则设置入口：打开互联客户列表、互联供应商列表，单击对应的互联客户编码或互联供应商编码即可看到。

# 发送限制（不支持发送的场景）

### **1、企业互联有什么是不支持发送的**

答：不支持携带商品标签；不支持发送附件。

### 2、发送单据时，如果接受方的商品已经禁用，可以发送成功吗？

答：不可以，发送时会校验接受方账套的商品状态，禁用商品会发送失败。

# 常见提示语

### 1、企业互联，发送采购订单给对方提示：“商品分录-仓库没填写“，实际采购订单上是有录入仓库的，这是为什么？

**![](https://vip-admin.kingdee.com/download/01095c29b74417ca4047b2e8ce6d78018455.png)**

**答**：出现上述原因一般是因为**接受方**的销售订单设置了单据的商品分录.仓库必录（在单据字段配置中设置），所以发送时会有校验。

**解决办法**：让接受方（即销售方)关闭【单据字段配置-销售订单】的商品分录.仓库必录，如果接受方不允许关闭此参数，则必须要维护发送方商品信息的“默认仓库”后，重新在单据上选商品后，再重新保存单据后发送即可。（注意，更新商品资料是不会更新历史单据上的信息的，因此务必一定要重新选择商品后再发送！）

### **2、企业互联，客户向供应商发送采购订单提示：“对方无法识别订单中的商品“是为什么？**

**![](https://vip-admin.kingdee.com/download/0109095ea97b11d34e1295ac36d9a10fe48a.png)**

**答**：出现上述原因一般是因为销售方（接受方）的外部商品库配置不对。

**解决办法**：需要销售方（接受方）将采购订单上的商品配置到自己账套的**外部商品库-外部商品编码**中。

![](https://vip-admin.kingdee.com/download/010936cbfb812bde4dfea3d610eed18eafca.png)

### 3、**企业互联，供应商发送销售出库单，提示：“XXX商品的客户商品编码，是接收方商品的主商品编码。解决办法：请在《外部商品库》中将外部商品编码变更为辅助属性编码维护绑定关系”是什么意思？该如何解决？**

**![](https://vip-admin.kingdee.com/download/0109cd334ce04b17478cab9fe00c9bd973d5.png)**

**答：**如果对方商品启用了辅助属性管理，需要将对方商品的“SKU编码”维护到“外部商品编码”中才行，因为此时如果只维护商品编码，辅助属性组合有很多的时候，是一对多的关系，系统是不知道如何识别的，所以必现靠SKU编码进行唯一识别，

**解决办法：**见下图，要把商品资料里的SKU编码填入到外部商品库的“外部商品编码”中。

![](https://vip-admin.kingdee.com/download/01099128a751af4249d0b52ed8baeded2669.png)
