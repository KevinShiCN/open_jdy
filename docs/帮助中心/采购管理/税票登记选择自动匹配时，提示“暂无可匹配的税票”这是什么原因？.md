# 税票登记选择自动匹配时，提示“暂无可匹配的税票”这是什么原因？

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：52 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/800036177108833024)

# **1 问题描述**

- 旗舰版的发票列表选择税票登记自动匹配时，提示“暂无可匹配的税票”。
- 标准版/专业版销售出库单列表选择“税票登记自动匹配”时提示“暂无可匹配的税票”，但实际存在税票，这是什么原因？

![](https://vip.kingdee.com/download/01007be05acf9aed4cf082d3b9c0cb451556.png)

# 2 适用版本

星辰3.0以上所有版本

# **3 原因解析**

出现“暂无可匹配的税票”通常是因为没有符合自动匹配规则的单据。

# **4 解决方案**

检查自动匹配规则是否开启，勾选的单据与预期要匹配的税票是否符合开启的匹配规则。若不符合，需要开启对应符合的规则。关于匹配规则的说明如下：

| 自动匹配规则 | 什么情况下会自动匹配成功 |
| --- | --- |
| 单据商品名称及税收分类编码=税票货物名称及税收分类编码，且单据商品行未开税票价税合计=税票商品行未核销金额。 | 单据上商品名称和税收分类编码与税票上的一致，且单据商品行的未开税票价税合计=税票商品行未核销金额。简单说：单据上是A，税票上也必须是A，且单据上A的未开税票价税合计等于税票上A的未核销金额。 |
| 单据商品行未开税票价税合计=税票商品行未核销金额 | 无论商品是什么，只要单据商品行未开税票价税合计等于税票商品行未核销金额即可。 |
| 单据总未开税票价税合计=税票总未核销金额 | 无论商品是什么，只要单据总的未开税票价税合计等于税票总的未核销金额即可。 |
| 客户商品名称及商品税收分类编码=税票货物名称及商品税收分类编码，且单据商品行未开税票价税合计=税票商品行未核销金额。 | 与规则1类似，只是对比的是客户商品信息与税票上的商品信息。单据上商品对应的客户商品名称和税收分类编码与税票上的一致，且单据商品行的未开税票价税合计等于税票商品行未核销金额。简单说：外部商品库中维护了商品A对应客户商品B，那么税票上商品必须是B，且单据上A的未开税票价税合计等于税票上B的未核销金额。 |
