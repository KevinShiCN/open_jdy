# 引入单据提示单价为0,价税合计不为0是什么原因,如何处理

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题
> 作者：云社区用户0531183 | 阅读：159 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/703054805291494400)

【问题描述】

引入单据提示单价为0,价税合计不为0是什么原因,如何处理?

【解释说明】

当没有开启参数“修改价税合计反算单价”,或者引入时选择“如实引入"由于操作失误,可能会产生有单价无价税合计,或者有价税合计无单价的数据。为避免这种情况,系统对此类异常数据的输入做了校验。当单价为0,价税合计不为0,或者单价不为0且无折扣,价税合计为0时,不允许保存和引入单据。

![](https://vip.kingdee.com/download/01090db4b31983194c458051d2f7c9d1eb69.png)

【解决方案】

1.开启参数修改价税合计反算单价。在单据新增界面点击“更多”-“选项”打开单据选项界面。![](https://vip.kingdee.com/download/010907a53e7969524def98b06c7c9b62f1c6.png)

勾选“修改价税合计后反算单价”并点击确认。![](https://vip.kingdee.com/download/01090110793673144b7191c83f3af3c3490b.png)

2. 点击引入参数设置,检查是否选择的是“单价、价税合计按模板填写如实引入”。修改为其他方式。
