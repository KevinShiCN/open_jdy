# 采购申请单上的“采购”和“合并采购”有什么不一样？当企业存在一个和多个供应商时，如何做采购申请单？

# 应用说明

## 1、采购申请单上表头和表体的“建议供应商”应用场景分别是什么？

![上传图片](https://vip.kingdee.com/download/01000c56c01114174cecb036eaea69d1e52d.png)

回答：

- 表头“建议供应商”：适用于只有一个供应商的情况
- 表体“建议供应商”：如果申请单中商品涉及多供应商采购，则可以不填表头的“建议供应商”，直接填商品分录（即表体）上的“建议供应商”即可。

## 2、采购申请单已审核后“采购”和“合并采购”有什么不一样？它们各自下推的采购订单上的“供应商”是怎么取值的？

![上传图片](https://vip.kingdee.com/download/010026b37f4913b74413a755eb618094f8ce.png)

回答：

| 功能名称 | 应用场景 | 采购申请单下推的采购订单上的“供应商”取值逻辑 |
| --- | --- | --- |
| 采购 | 适用于只有一个供应商采购的场景，直接点“采购”下推生成对应的一张采购订单 | 采购订单单据头的“供应商”按申请单单据头的“建议供应商”取数。如果申请单上的表头“建议供应商”没填，则下推时采购订单上的“供应商”为空。 |
| 合并采购 | 适用于：1）同时涉及多个不同的供应商采购的场景。如一张单上的同一个商品是由多个不同的供应商提供的，或者一张单上包含了多个不同的商品，且该商品都是由不同的供应商提供的2）列表支持选中多单后，把多单的相同商品的批准数量合并汇总集中采购3）一张单上包含多行相同的商品，可按合并规则进行汇总采购功能详情请点击：采购申请单合并采购 | 采购订单单据头的“供应商”按采购申请单商品分录的“建议供应商”取数，一个供应商对应一张采购订单，如果有多个不同的供应商，则生成多张采购订单如果申请上单据分录上 “建议供应商”全部为空，不管单据头的“建议供应商”有没有填，点击“合并采购”后只会生成一张采购订单，且“供应商”为空如果申请上单据分录上 “建议供应商”部分为空，部分有值，不管单据头的“建议供应商”有没有填，点击“合并采购”后，所有为空的商品分录会生成一张采购订单，且“供应商”为空，有值的商品分录会生成另外的采购订单，且订单上的供应商就是取的申请单商品分录上的建议供应商。 |
