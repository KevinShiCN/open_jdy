# 高频常见问题

### 1. 引出时，多少数据会直接下载文件，超过多少数据时，才需要到引出结果查询里下载？

**答**：当一次引出数据超过 500 行，就会异步下载，也就是需要到引出结果查询里下载。

### 2. 用户修改了单据列设置的显示名称后，引出后还是取系统的列名，这是为什么？

答：按引入模板引出，还是取系统的列名的，如果希望取修改后的列表，需要按列表引出，则是按显示名称来引入。

### 3. 按列表引出时，数值精度是按列表显示的导出吗？

答：列表引出单据都是所见即所得的，列表精度显示多少，导出显示的格式化就是一致的。而点击excel里的单元格后可以看到实际存储的精度，这个则跟鼠标在系统里放到数值看到的悬浮值是一致的。

### 4. 支持只引出某张单的其中一行或者多行的商品行吗？

答：不支持。单据的打印、引出都是没有按行处理的，都是整单打印、整单引出。

### 5. 引入结果查询、引出结果查询里的内容是账套内每个用户都可以查看和下载的吗？

答：只有服务管理员、账套管理员才能查看所有的结果，普通用户只能查看自己操作的结果。这里做的控制是避免普通用户下载了自己没有权限的数据，所以不能看到其他人的操作记录。

### 6. 引入失败文件里说第x行数据有问题,但是明明该行的数据是正常的,为什么无法引入?

答：引入错误文件里的“错误原因”,具体的报错都是定位到具体的行的,但是要注意,由于引入模板的前3行都是有内置的信息的,**引出实际的错误原因的第一行是从第四行开始,**不能直接去对比文件左侧的序号,**报错实际行号=3+N(N就是报错信息里的行数)。**

**知识详细说明：**[引入失败文件里说第x行数据有问题,但是明明数据是正常的,为什么无法引入?](https://vip.kingdee.com/link/s/Zpqgo)

# **商品引入相关**

### 1. 想引入商品的安全库存、最低库存、最高库存，但无法引入是什么原因？/如何在商品引入模板里设置仓库的管理方式？  如何引入商品的最低库存数量、安全库存数量、最高库存数量？

**原因说明**：引入失败通常是未正确设置 “仓库管理方式” 导致的。

**解决方案**：[如何在商品引入模板里设置仓库的管理方式？ 如何引入商品的最低库存数量、安全库存数量、最高库存数量？](https://vip.kingdee.com/links/MCFvg)

### 2. 错误原因：商品编码 XXX开启属性条码维护，条码信息请维护条码的辅助属性信息！开启属性条码维护，信息请维护条码的辅助属性信息！

**原因说明**：系统里的资料已经维护了 “辅助属性条码” 信息，而商品的覆盖新增引入的文件里没有维护对应的列，导致校验失败。

**解决方案**：在商品引入模板里开启 “辅助属性条码”，并维护好对应的值后重新覆盖引入。

![](https://vip.kingdee.com/download/01098d086f20635940a5b4384570853a5293.png)

### 3. 错误商品编码 XXXXXX：商品的套装信息中的属性组合与该商品不匹配

**原因说明**：因为引入模板中，套装商品未填写所有已启用的辅助属性组。

**解决方案**：在自定义套装商品引入模板中，把所有已启用的属性组给打开，填写后重新引入即可。

![](https://vip.kingdee.com/download/01006c3131ef70214c18952084e7b5451c1e.png)

### 4. 商品资料引入，模板没有设置条形码是必填，但是导入仍然提示填写条形码，应如何设置？

**原因说明**：出现这个提示是因为单据字段配置中的商品资料开启了条形码必录，关闭后即可引入。

**解决方案**：[商品资料引入，模板没有设置条码是必填，但是导入仍然提示请填写条形码，应该如何设置？](https://vip.kingdee.com/links/MCTPG)

### 5. 引入辅助属性商品时，引入失败，提示 “第一行有值，其他行也必须填值”

**原因说明**：引入辅助属性商品时，同一个商品编码下的 “辅助属性分录” 的属性组必须都有值，不能出现空白行。

**解决方案**：把引入文件里，同一个辅助属性商品下的 “辅助属性分录” 对应的所有属性组和辅助属性（如果引入文件有这一列字段）需补充完整，如果没有这一列则引入后系统会自动生成或补充完整后再重新引入即可。详见知识：[引入辅助属性商品时,引入失败,提示“第一行有值,其他行也必须填值!”是什么原因?要如何解决?](https://vip.kingdee.com/link/s/ZF54q)

### 6. 多单位商品引入时，开启了多单位自动换算，但还是引入失败，提示“价格信息，启用了多单位换算，只能引入任意一个单位的价格。价格信息，启用了多单位换算，只能引入任意一个单位的价格。”是什么原因？

**原因说明**：引入选项如果勾选了“多单位商品价格引入时，引入其中一个单位的价格，自动换算其他单位的价格”，那么商品引入模板的价格分录部分，只需填写其中任意一个单位即可，不需要填上所有单位，引入后系统会自动换算其他单位的价格。

**解决方案**：需要把多余的单位行删除后再重新引入，参考下方图片，即删除黄色的两行。

![](https://vip-admin.kingdee.com/download/0109f5adffd0554b4e18911e8f85882c7f50.png)

### 7. 引入时，如果商品名称没有重复的，可以只按商品名称引入，不引入商品编码吗？

**答**：可以，在单据的引入模板的商品分录下，把 “商品编码” 的引入属性改成【名称】即可。详见右侧链接：[引入单据时，如果商品名称没有重复的，只想按商品名称引入，不引入商品编码可以吗？](https://vip.kingdee.com/links/MCT0k)

# 单据相关

### 1. 单据可以不填写单据编号，然后引入时自动生成吗？

**答：**可以，但是只支持按 “引入模板” 填写的 Excel。用户需要在引入模板中，关闭单据编号字段，然后下载引入模板，填写好单据的其他字段后，上传引入模板引入即可。注意，不能直接通过列表引出后再导入。

### 2. 单据列表的引入参数选了“单价、价税合计按模板填写如实引入”，单价和价税合计也都填了，为什么引入时提示“单据“XXXX”：第X行的价税合计计算异常，请将数量修改为其他值再改回原值，手动触发重新计算。”？

**答**：就算引入参数开启如实引入，也必须要符合计算的等式，即单价*（1+税率）=价税合计，又或者实际录入的价税合计不能跟恒等式的结果偏差太大，如果偏差太大，也会校验不通过，无法引入。

这是系统为了避免人工引入时填写有误，造成不必要的金额损失所增加的校验。

>
例如：数量2，单价5

无税率的情况下：价税合计=2*5=10，如果录入的价税合计等于11或者12，就会引入失败。

**解决方案**：需要把价税合计改成等于或者接近于公式：单价*（1+税率）的计算结果。

### 3. 如果想要引入文件的每一行都有对应单据编号，而不是像引入模板那样同一张单的只显示第一行的单据头信息，该怎么设置？

**答：**使用“引入> 引出分析数据”即可实现。“引出分析数据”默认不合并表头字段，如需合并，可以在引出模版中进行配置

![](https://vip.kingdee.com/download/010023a120bffe5741e3948102402d7d839d.png)

引出模板配置字段：

![](https://vip.kingdee.com/download/010025fdb337ec9842a795ef4793a2663725.png)

具体操作可以知识：[如果想要引入文件的每一行都有对应单据编号，而不是像引入模板那样同一张单的只显示第一行的单据头信息，改怎么设置？](https://vip.kingdee.com/link/s/ZG6qp)

### 4. 组装拆卸模板引入，提示：模板编号XXX： 第[X]行商品[XXX]的辅助属性信息输入有误。

**原因说明**：引入模板设置问题。组装拆卸模板中含有辅助属性商品，但是引入模板没有开启辅助属性字段。

**解决方案**：开启引入模板中的辅助属性字段。

### 5.引入出入库类单据时，提示：第X行商品“XXXX”的辅助属性信息输入有误。是什么原因，要如何处理？

![上传图片](https://vip.kingdee.com/download/01003ac1ad8ea75c42f19d454d61f67e759c.png)

**原因说明**：由于出入库类单据会影响库存，当引入辅助属性商品时，对应的引入模板也必须录到对应维度的字段，也就是属性组，不能只引入“辅助属性”字段。

**解决方案**：开启引入模板中的“属性组”字段，重新在引入模板上填写好属性组后再重新引入即可。

![上传图片](https://vip.kingdee.com/download/01009b7ed4955dea4289a9bb52e71afd73b2.png)

# 常见错误原因（单据/基础资料都适用）

### 1. 错误原因：单据编号XXXXXX：不允许操作以前期间的单据，请先反结账！

**原因说明**：系统已经结账，引入了比当前期间更早的单据。

**解决方案**：反结账到需要引入单据的日期所在的期间，再重新引入。

### 2. 引入进度弹窗里显示成功：0，失败：0。点击“查看详情”提示 “导入文件格式错误或者没有业务数据”是什么原因？

![](https://vip.kingdee.com/download/01093299871d12a744978cb66a22c9821cf0.png)

**原因说明**：系统对引入文件是有格式校验的，系统正常的引入模板的前三行必须要有，如果在维护数据时，不小心删除模板的前三行任意一行，导致缺失。

**解决方案**：详见知识： [引入时没有反应，没有成功也没有失败，点击查看详情，提示“导入文件格式错误或者没有业务数据!”要怎么解决？](https://vip.kingdee.com/link/s/ZFCrc)

### 3. 错误原因：XXX不存在或者被禁用

**原因说明**：存在以下几种原因

原因1：基础资料没有创建，或者创建后被删除了，或者被禁用了；

原因2：基础资料跟引入文件上的不一致，一般可能是字段前后存在一些空格，导致对不上。

原因3：商品都没有启用特性（如：辅助属性），但是引入模板上开启填入了相关的特性字段，如：属性组，导致校验报错。

**解决方案**：

1、检查基础资料的状态，必须是启用状态的，如果搜索不到，则需要先在系统里创建基础资料

2、把基础资料里导出来，然后跟需要引入的文件做一个匹配，看一下提示错误的字段在两边是否相等，如果不等，说明引入资料填写不对，需要按照系统里的字段来填写到引入模板上

3、检查引入模板，关闭不需要引入的相关字段

### 4. 错误原因：数据行或者分录行存在必录项未录入

**原因说明**：必录项为空。

![](https://vip.kingdee.com/download/0100fb0d1b158f19424b9c7cd26e10efa5c3.png)

**解决方案**：补充必录项信息。

### 5. 报错文件里没有看到报错信息，但是引入失败

![](https://vip.kingdee.com/download/01000c88319e9b844a11a86fa82fe76db655.png)**原因说明**：这种情况下，一般是失败信息不在前面的行里，而是在下方的数据行里，需要对“错误原因”列使用筛选器过滤出来查看，不能只看第一行数据没有内容就认为没有错误原因。

**解决方案**：用筛选器找到报错的数据行，并进行调整。

![](https://vip.kingdee.com/download/0100eba27fccb0484d08bc5de63627e9eca6.png)

### 6. 引入基础资料/单据时，第5000行以后没有具体错误原因，只写了“引入失败，请检查行数据 ”是什么原因？

**原因说明：**当引入数据超过5000行后，如果前面5000行都引入失败，则超过5000的行数就不再提示具体的报错信息，统一显示“引入失败，请检查行数据”。

**解决方案：**针对前面5000行都失败的大数据引入，建议减少数据行后，分批引入.另外，进销存普通单据的引入上限是 3000 行。超过3000行，请分批引入。

### 7.**单据列表数据不多，但是引出文件中没有其他数据，只有一行提示：引出数据缓存已经被清除或者本次引出数据量过大，请稍后重试（或分批引出），是什么原因？**

**原因说明：**通常是因为浏览器存在以下情况：

1、有使用迅雷下载（或者是下载技术由迅雷提供的）；

2、浏览器安装了第三方插件（如：鲁大师）。

**解决方案：**

1、不要用迅雷下载，或者使用谷歌chrome浏览器来登录星辰和下载文件；

2、浏览器卸载第三方插件；。
