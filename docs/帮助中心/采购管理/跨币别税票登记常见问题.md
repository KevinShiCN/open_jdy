# 跨币别税票登记常见问题

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：29 | 有用：2
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/816242689246492160)

# **1 问题描述**

- 税票是本位币开票，业务单据是外币，支持做税票登记吗？
- 税票是本位币开票，业务单据是外币，且外币金额和汇率换算后等于本位币金额，但税票登记后，核销状态不是全部核销，是为什么？

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

首先，星辰的税票登记支持跨币别登记，税票登记列表点击新增，选择“币别”后，单据信息中展示的是对应币别的业务单据。

![上传图片](https://vip.kingdee.com/download/01007ea507cc2b29418380d1c08ca8f7220b.png)

![上传图片](https://vip.kingdee.com/download/0100a38a9e1bbb6347d0a31c4345b9ebe23e.png)

其次，税票登记中并没有处理外币与本位币的换算，仅提供登记的作用。换句话说，选择的外币金额是多少，对应的“本次开票价税合计”，也是取的外币的，不会换算成本位币。

示例说明：

以销售税票登记为例，假设税票的币别是人民币，价税合计是18000元。业务单据的币别是美元，未开票价税合计是2500美元，汇率是7.2，换算成人民币即18000元。但税票登记时，“本次开票价税合计”实际亦然是2500美元，并不会根据汇率和币别进行换算。

![上传图片](https://vip.kingdee.com/download/010035f1c7cab31d4a429e9303f763306db3.png)

这样完成税票登记后，发票管理中查询该税票，会发现“核销状态”为“部分核销”，这是正常的。因为税票登记仅作纯数字的记录，不做换算，所以核销的金额并不相等，不能全部核销。

![上传图片](https://vip.kingdee.com/download/0100260a5db19dc34f3483d862c8cca58c09.png)

采购税票登记也是同理，本文不再赘述。

# **4 解决方案**

- 若“商品未开税票价税合计”大于税票金额，可修改单据信息中的“本次开票价税合计”，让其等于税票信息的中的未核销金额。这样可保证税票登记完成后，“核销状态”为“全部核销”
- 若“商品未开税票价税合计”小于税票金额，由于目前税票登记暂不支持跨币别的换算，只能做登记，“核销状态”只能为“部分核销”。
