# 采购销售订单上填写了预收订金不生成预收预付款单有什么影响?

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题
> 作者：shaoling | 阅读：46 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/724066809753802240)

【问题描述】

采购销售订单上填写了预收订金不生成预收预付款单有什么影响?

【解释说明】

若采购 / 销售订单填写了预收 / 预付订金但未生成预收款单 / 预付款单,“本次订金” 字段会显示填写的金额,但 “本次订金状态” 仍显示 “空白”,即系统默认该款项**尚未实际收付**。后续**需生成预收款单 / 预付款单并审核生效**,状态才会更新为 “已收”

【效果展示】

若在销售订单中填写预收订金后未生成预收款单,或在采购订单中填写预付订金后未生成预付款单,“本次订金状态” 会显示为 “空白”,即系统默认该订金尚未完成收款流程

![](https://vip.kingdee.com/download/010930574a683ac042ddb75ece16600edefd.png)
若订单生成预收款单 / 预付款单但未审核,订单中 “本次订金状态” 会显示为 “未收”(销售场景)或 “未付”(采购场景)

![](https://vip.kingdee.com/download/0109758f3f4671894bf3a075ab6a5bfcb61b.png)
只有当生成的预收款单或预付款单审核生效后,“本次订金状态” 才会显示为 “已收”或“已付”

![](https://vip.kingdee.com/download/0109e06e49b27cd948c58a2850f0db45ab74.png)
