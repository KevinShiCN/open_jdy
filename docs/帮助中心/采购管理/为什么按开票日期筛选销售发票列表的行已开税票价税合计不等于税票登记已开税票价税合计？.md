# 为什么按开票日期筛选销售发票列表的行已开税票价税合计不等于税票登记已开税票价税合计？

# **1 问题描述**

为什么按开票日期筛选销售发票列表的“已开税票价税合计”不等于税票登记的“已开税票价税合计”？

![](https://vip.kingdee.com/download/010018b6c0ebe602403793175135bd574de4.png)

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

销售发票多次进行税票登记，且存在跨期（与税票登记筛选期间不一致）时，销售发票列表会关联显示所有已做税票登记的单据，而税票登记列表仅显示对应查询期间的记录，因此产生差异。

例如：一张销售发票分两次进行税票登记，一次在八月，一次九月。税票登记完成后，都会反写对应单据信息的“已开税票价税合计”。

在销售发票列表查询开票日期在九月的单据，由于前面完成税票登记后，已经反写了“已开税票价税合计”，因此九月份查询出来的是同时**包含了九月及以前的数据**，是一个**累计值**。

当税票登记列表查询开票日期在九月的单据，只能展示在当月登记的记录，结果就是：已开税票价税合计**不等于**税票登记的开票价税合计。

# **4 解决方案**

目前系统没有自动匹配的功能，如果需要查出差异值，需要导出数据后用VLOOKUP函数做匹配进行查找。若超出可导出限制，建议分批导出操作。
