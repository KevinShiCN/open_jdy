# **一、更新内容及应用价值**

1）本次更新完善了【单据新增字段】中“带出已有基础资料属性”的数据源；

2）本次更新完善了“自定义公式”中数据源。

**二、应用场景**

- 场景一：商品添加自定义字段叫做“标准售价”，销售订单带出“标准售价”字段在单据体，再自定义个销售订单的自定义字段叫做差额，差额=(单价-标准售价)*数量，主要用于计算业务员提成。
- 场景二：在客户资料中新增自定义字段”物流运输单价“，在销售出库单中新增自定义公式字段等于 物流运输单价*重量，以计算本单的总物流费用。
- 场景三： 在打印销售出库单时，需要打印出销售单的本单应收账款-抵扣预收款的金额，可以通过增加自定义字段来实现打印。

**三、功能特性及说明**

**1）“带出已有基础资料属性”的选项中，针对****商品、客户、供应商****增加字段**。包括以下明细：

- 当选择引用“商品”的属性时，增加字段：描述、长、宽、高、体积、商品标签、商品类型、注册证号、生产许可证、毛重、净重、重量单位、体积单位、备货天数、交货周期。
- 当选择引用“供应商”的属性时，增加字段：公司地址、开票名称、开票税号、银行账号、信用额度、结算期限。
- 当选择引用“客户”的属性时，增加字段：价格等级、客户标签、信用额度、结算期限、开票税号、发票类型

**2）在单据中进行自定义公式时，****可以将基础资料中的数值型字段作为公式的一部份参与运算**。

- 销售类单据的表头提供客户的数值型字段的展示和引用；
- 采购类单据的表头提供供应商的数值型字段展示和引用；
- 销售、采购、库存类单据的表体字段提供商品的数值型字段展示和引用，以便用户在单据中实现公式计算的需求。
