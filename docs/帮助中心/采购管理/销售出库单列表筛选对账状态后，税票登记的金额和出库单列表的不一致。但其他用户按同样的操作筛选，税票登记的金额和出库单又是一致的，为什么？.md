# 销售出库单列表筛选对账状态后，税票登记的金额和出库单列表的不一致。但其他用户按同样的操作筛选，税票登记的金额和出库单又是一致的，为什么？

# **1 问题描述**

销售出库单列表筛选对账状态后，税票登记的金额和出库单列表的不一致。但其他用户按同样的操作筛选，税票登记的金额和出库单又是一致的，为什么？

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

通常是由于列表列设置差异导致的。

例如：某账号出库单列表未开启“同一单据拆分显示”，则按整单携带，意味着这个单所有的未开票的分录都携带过去了。

![上传图片](https://vip.kingdee.com/download/0100b411b5c3226f4927bd65ba5f1abb2503.png)

而其他用户的出库单列表开启“同一单据拆分显示”，则按分录携带；同时列表过滤了对账状态。若同一张单据的分录对账状态不一致，不同携带方式带出的分录数不同，金额自然也不同。

![上传图片](https://vip.kingdee.com/download/01006b2827475b8d4b638dd04b84198e5415.png)

# **4 解决方案**

为了避免查询差异引发歧义，建议同一个账套下的用户在进行税票登记操作时，要保证列设置的查看方式一致。
