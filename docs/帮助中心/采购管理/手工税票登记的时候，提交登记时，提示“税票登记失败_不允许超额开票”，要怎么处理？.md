# 手工税票登记的时候，提交登记时，提示“税票登记失败:不允许超额开票”，要怎么处理？

> 产品：金蝶云·星辰 | 分类：采购管理 | 类型：常见问题, 快速入门, 精华汇总
> 作者：进销存赵嘉丽 | 阅读：33 | 有用：1

# **1 问题描述**

销售出库单进行手工税票登记的时候，明明税票信息中的“开票价税合计”等于单据信息中“本次开票价税合计”的合计行，但提交登记时，提示“税票登记失败：不允许超额开票”，要怎么处理？

![上传图片](https://vip.kingdee.com/download/01009840264425d64cfda0565e89429e2ae4.png)

# 2 适用版本

星辰3.0及以上所有版本

# **3 原因解析**

出现以上提示通常是，单据信息“自动填充本次开票价税合计”后，手工修改了部分行的“本次开票价税合计”，并且“本次开票价税合计”大于“商品未开税票价税合计”。

![上传图片](https://vip.kingdee.com/download/0100ce7a417efa2341f3bd5f4e0b5d488b42.png)

>
税票登记时，需要满足以下两个数值校验条件：

①税票信息中的“开票价税合计”**等于**单据信息中“本次开票价税合计”之和

②单据信息中，每一行的“本次开票价税合计”**小于等于**“商品未开税票价税合计”

# **4 解决方案**

把单据信息复制到Excel中，比较“本次开票价税合计”、“商品未开税票价税合计”两列的数值，看看是否存在“本次开票价税合计”大于“商品未开税票价税合计”的分录。如果有，那就是问题所在。需要把“本次开票价税合计”改成≤ “商品未开税票价税合计”后再重新登记。
