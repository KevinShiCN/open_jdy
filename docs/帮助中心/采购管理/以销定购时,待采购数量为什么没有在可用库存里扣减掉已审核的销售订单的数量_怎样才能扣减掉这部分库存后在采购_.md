# 以销定购时,待采购数量为什么没有在可用库存里扣减掉已审核的销售订单的数量?怎样才能扣减掉这部分库存后在采购?

> 作者：进销存赵嘉丽 | 阅读：495 | 有用：0

**【问题描述】**

以销定购时,待采购数量为什么没有在可用库存里扣减掉已审核的销售订单的数量?怎样才能扣减掉这部分库存后在采购?

例如下图: 广州仓还有库存150盒,销售订单数量共计300盒,那么待采购数量为什么不是150盒,而是300盒?

![](https://vip.kingdee.com/download/01094c7d791f29614ffd8e4c35a9263615d2.png)**【原因说明】**

目前以销定购页面没有扣减销售订单的可用库存是因为销售订单是一对多的关系,在库存数量小于订单数量的前提下,系统是不知道该把可用库存优先分配给哪个订单的,所以以销定购的待采购数量都没有扣减掉这部分。

**【解决方案】****
**

**如果希望以销定购时可以扣除掉有货的库存的话,有两种方案:**

**方案一:使用智能补货**

还是以问题描述的订单为例进行讲解,具体步骤如下:**
**

1)先把商品上的“库存预警”数量清空,如下图(因为设置为0才是以销定购的场景,不清除的话则会按预警设置的库存水位补货):

![](https://vip.kingdee.com/download/0109acfd9b28ef1c4864bbfd871effd0dfc2.png)

2)打开智能补货,减项勾选“预计可用库存”:

![](https://vip.kingdee.com/download/0109cf7af65cd1d24f5ba715895871b9aa3f.png)

3)打开“设置预计可用库存公式”,勾选“已审核的销售订单”、“包含仓库为空的销售订单”2项;

![](https://vip.kingdee.com/download/01091da4dbb536f54fd498a3cc0b9a5e6350.png)4)选择要补货的订单,选择要补货的仓库,点击开始计算即可。

![](https://vip.kingdee.com/download/01097693a53b13904985813d66592dde8a8b.png)
可以看到计算出来的采购数量就是150盒了,这样就实现了我们的最初目的。

**方案二:****先在销售订单列表操作预留,再做以销定购**

![](https://vip.kingdee.com/download/0109370bc390c94b45bc808d37783f7c753e.png)
