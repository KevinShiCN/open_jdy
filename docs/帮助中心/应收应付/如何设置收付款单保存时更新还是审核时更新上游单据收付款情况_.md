# 如何设置收付款单保存时更新还是审核时更新上游单据收付款情况?

> 产品：金蝶云·星辰 | 分类：应收应付 | 类型：常见问题
> 作者：Simone邓 | 阅读：14 | 有用：0
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/805242137200857600)

【问题描述】

如何设置收付款单保存时更新还是审核时更新上游单据收付款情况?

【适用版本】

星辰3.0以上标准版/专业版/旗舰版/所有版本

【业务场景】

保存时更新适用于业务流程简单、收付款单录入错误率低,需要实时掌握出入库单收付款动态,快速响应业务的场景;

审核时更新适用于业务流程规范、需严格管控收付款准确性;收付款单需多级审核,避免录入错误等场景

【操作步骤】

【系统设置】-【系统参数】-【应收应付参数】,资金单据更新上游单据的方式可选择按保存或审核后更新

**保存时更新:**应收应付单保存时更新上游源单收付款状态及已收付款金额;

**审核时更新:**应收应付单审核时更新上游源单的收付款状态及已收付款审核金额;

![](https://vip.kingdee.com/download/010904a9d99bf4db44f48b92ec402047f069.png)

**更新逻辑说明**

以专业版销售出库单为例,收款信息更新规则的处理逻辑如下:

**1)当勾选“保存时更新”,系统处理逻辑与当前保持一致:**

●收款单 / 核销单保存时,实时更新销售出库单原有字段(“已收款金额”“未收款金额”“已核销金额”“未核销金额”);

●收款状态根据 “已收款金额” 与 “应收金额” 的对比结果自动判断。

**2)当勾选“审核时更新”―系统新增字段并区分更新时机:**

●新增字段:

表头增加 “已收款审核金额”“已收款审核金额本位币”;表体增加 “已核销审核金额”“已核销审核金额本位币”。

●更新规则:

收款单 / 核销单保存时:仅更新原有字段(“已收款金额”“未收款金额”“已核销金额”“未核销金额”);

收款单 / 核销单审核时:同步更新上述新增字段;

收款状态根据 “已收款审核金额” 与 “应收金额” 的对比结果自动判断。

【温馨提示】

不存在未审核的资金单据情况下支持由保存时更新切换到审核时更新;由审核时更新切换到保存时更新。

**校验的单据包括:**收款单,收款退款单,预收单,预收退款单,付款单,付款退款单,预付单,预付退款单,核销单、同单抵扣预收预付的采购和销售单据(标准版专业版:采购入库单、销售出库单 ,旗舰版:采购发票、销售发票;同单收付款的单据)
