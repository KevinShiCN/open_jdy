# 反审核其他支出单,删除时提示“已被xxxx单同单分摊,不允许直接删除”该如何处理?

> 产品：金蝶云·星辰 | 分类：应收应付 | 类型：常见问题
> 作者：进销存赵嘉丽 | 阅读：651 | 有用：1
> 来源：[金蝶云社区](https://vip.kingdee.com/knowledge/716457276995002880)

【问题概述】

当系统参数开启强制分摊同单费用后,在已审核的单据详情中,打开费用弹窗,删除费用分录时,系统会提示“请删除关联单据:XXX,然后重试。”

![](https://vip.kingdee.com/download/0109aa9215e0ac144723abfe7847d95962d0.png)

但是反审核关联的其他支出单后,点击删除时又提示“已被XXXX单同单分摊,不允许直接删除”,两边都不能删除,应该如何处理?

![](https://vip.kingdee.com/download/0109ffb67356f3b649c596bf5e2092c80ade.png)

【原因说明】

由于系统参数开启了强制分摊同单费用的参数,费用在生成的同时已经分摊到单据中了,可以在其他支出单列表查看到单据的“分摊状态”是“已分摊”:
![](https://vip.kingdee.com/download/010991816b0afbab463aa230f89c2e2d1126.png)
这种情况下,不管是其他支出单单据详情还是单据列表,都不允许直接删除,只能在源单的业务单据上进行处理。

【解决方法】

具体的处理方式还取决于系统参数中的“删除单据时,必须同步删除同单费用生成的单据”是否勾选了,下面就2种不同的设置参数的删除操作进行说明。

| 系统参数 | 删除方法 | 具体操作步骤 | 说明 |  |
| --- | --- | --- | --- | --- |
| 勾选了“删除单据时,必须同步删除同单费用生成的单据” | 直接删除所勾选的单据类型 | 1、先反审核单据,然后点击“删除”2、出现了操作确认弹窗,点击“同时删除”,这样就会把业务单据和费用关联的单据一起删除: | 注意,这种场景下,业务单据和费用单据都会同时删除,不能只删除费用相关的单据,而不删除业务单据。 |  |
| 没勾选“删除单据时,必须同步删除同单费用生成的单据” | 在所勾选的单据类型的单据详情中的费用明细行进行删除 | 1、打开业务单据的费用弹窗,点击关联生成的其他支出单单号2、如果其他支出单还没审核,则直接看第3步,如果已经审核了,则先反审核其他支出单3、点击“删除”,并确认4、其他支出单删除成功后,会看到业务单据的费用弹窗中,其他支出单已经为空了,接下来用户可根据实际需求进行修改:场景1:如果是要修改费用,则重新修改金额字段后,点击“保存并分摊”;场景2:如果是不需要分摊任何费用了,需要清空费用,则点击费用分录前的减号“-”,删除该分录,然后点击“保存并分摊”;5、操作完毕会看到“费用已分摊”的提示,单据详情上的费用字段也会根据上一步修改的结果进行更新;6、最后“保存”单据即可。注意,这一步一定不能省略,因为只有通过单据上“保存”才能触发更新数据库的结果。 |  |  |

【适用版本】

星辰3.1以上所有版本
