# **1 简介  **

## **1.1 功能介绍**

      分期付款功能主要应用于企业的大额采购或项目合作场景，通过将一次性的大额支出转化为可管理的小额分期，有效缓解企业的短期资金周转压力，从而帮助用户维持健康的现金流，并能够把握原本因预算限制而无法达成的采购与投资机会。

## **1.2 适用版本**

星辰3.1以上旗舰版

## **1.3 操作指引**

## **1.3.1 参数设置**

路径：系统设置-基础设置-系统参数-应收应付参数，参数维护页面如下：

![上传图片](https://vip.kingdee.com/download/01007680b048fb464994b0204f4e924ef453.png)

可勾选【启用分期付款】参数，该参数默认不勾选，勾选后，可对采购发票、其他支出单维护分期付款计划；

## **1.3.2 基础数据维护**

路径：基础数据-业务资料-收付款条件，点击【新增】按钮，弹窗如下：

![上传图片](https://vip.kingdee.com/download/01003c4460c5ac084c088bd7c80433a63591.png)

维护收付款条件名称、金额比例、结算期限（计算每一期付款计划的结算日期的依据）；

## **1.3.3 分期付款维护**

路径：采购管理-采购单据-采购发票或应收应付-应付单据-其他支出，以下以采购发票单为例说明

在新增采购发票页面，通过列设置显示付款条件字段，选择付款条件，带出分期付款计划表体，如下图：

![上传图片](https://vip.kingdee.com/download/0100fd0da27456b74860a5623aa21a796e96.png)

系统会根据发票的成交金额、单据日期分别计算分期付款计划；

举例说明：a）维护的付款条件基础设置如下：

| 金额比例 | 结算期限 |
| --- | --- |
| 40% | 结算期限:30天 |
| 50% | 结算期限:60天 |
| 10% | 结算期限:90天 |

b)采购发票成交金额为8000.00元，单据日期为2025-7-1，采购发票的分期付款计划计算过程如下表：

| 单据日期 | 2025-7-1 | 成交金额（单位：元） | 8000.00 |
| --- | --- | --- | --- |
| 付款计划 |  |  |  |
| 应付比例 | 计算说明 | 应付日期 | 应付金额 |
| 40% | 单据日期+30，成交金额*40% | 2025-7-31 | 3200.00 |
| 50% | 单据日期+60，成交金额*50% | 2025-8-30 | 4000.00 |
| 10% | 单据日期+90，成交金额*10% | 2025-9-29 | 800.00 |

## **1.3.4 分期付款核销**

1）付款、手工核销等选源单时，带有付款计划的单据交互页面保持不变；

2）针对分期付款的采购发票、其他支出单，在做同单付款、付款单付款、自动核销、手工核销后，系统会自动根据已核销金额，按照分期付款的应付日期依次往后核销分期付款计划；

承1.3.3的例子：

c)针对采购发票付款：做付款单时源单选择采购发票，付款4000元，如下图：

d)付款计划核销：采购发票付款计划是以付款金额按应付日期核销从前往后依次核销，核销结果如下图：

## **1.3.5 账龄统计与预警**

对存在分期付款的采购发票与其他支出单，在统计账龄分析表与应付账款预警表时，按分期付款计划统计；

1)账龄统计表：按付款计划行的应付日期计算账龄分组；

2)应付账款预警表：超期预警按付款计划行的应付日期预警；
